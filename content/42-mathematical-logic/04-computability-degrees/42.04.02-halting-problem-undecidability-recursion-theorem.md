## Intuition [Beginner]

Some questions about computer programs sound like they should have a mechanical answer, yet none exists. The most famous one is the halting question: given a program and an input, will the program eventually stop, or will it loop forever? You might hope for a master checker — a program that reads any other program, looks at its input, and prints "stops" or "loops" without ever being wrong and without itself running forever. The surprising fact of this unit is that no such master checker can exist, and the reason is a single, very short argument.

The argument is a kind of self-reference trap. Suppose the master checker existed. Then you could build a mischief program that first asks the checker, "what will I do when I am run on my own code?" — and then deliberately does the opposite. If the checker says this program stops, the mischief program loops; if the checker says it loops, the mischief program stops. Run the mischief program on its own code and the checker is wrong no matter what it says. Since a correct checker cannot be wrong, the checker cannot exist.

That one trick — feed a thing its own description and flip the answer — is the seed of a whole landscape. Once you know the halting question has no mechanical answer, you can show that many other questions inherit the same fate by quietly smuggling the halting question inside them. A rough rule emerges: almost any interesting yes-or-no question about what a program *computes* (not how it is written) has no general decision procedure.

There is a hopeful twin to all this. The same self-reference that traps the checker can be used on purpose. You can always write a program that builds and prints its own source code, and more generally a program that is allowed to refer to itself while it runs. Self-reference, handled carefully, is not a paradox but a tool.

## Visual [Beginner]

The diagonal trap, drawn as a table. Imagine listing every program down the side and every input across the top, and in each cell writing whether that program stops on that input. The mischief program is built to disagree with the diagonal of this table — the cells where a program is run on its own number — so it cannot appear anywhere in the list.

```
            input = program number
            P0    P1    P2    P3   ...
        +-----------------------------
   P0   | STOP  loop  STOP  loop      <- on the diagonal: STOP
   P1   | loop  loop  STOP  STOP      <- on the diagonal: loop
   P2   | STOP  STOP  loop  loop      <- on the diagonal: loop
   P3   | STOP  loop  loop  STOP      <- on the diagonal: STOP
   ...  |
        diagonal reads:  STOP loop loop STOP ...

   mischief program D, run on input n, does the OPPOSITE of cell (Pn, n):
        diagonal:        STOP  loop  loop  STOP ...
        D does instead:  loop  STOP  STOP  loop ...

   Could D be some row Pk in the table?
        At column k, D must differ from Pk's diagonal cell.
        So D's row disagrees with every row at its own column.
        D is not in the list. Contradiction.
```

Read it as a staring contest between `D` and the table. Whatever row you guess `D` is, look at that row's diagonal cell: `D` was defined to do the opposite there. So `D` matches no row. Yet if a halting checker existed, `D` would be a perfectly good program and would have a row. The table cannot hold `D`, so the checker cannot exist.

| program run on its own number | diagonal says | `D` is built to do |
|---|---|---|
| `P0` on `0` | STOP | loop |
| `P1` on `1` | loop | STOP |
| `P2` on `2` | loop | STOP |
| `P3` on `3` | STOP | loop |

The flip down the diagonal is the whole proof: `D` is everywhere different from the list, so it is not on the list.

## Worked example [Beginner]

We pin down the contradiction with one concrete program, using only addition and "ask the checker." Pretend a perfect halting checker $H$ exists: given a program $P$ and an input $n$, $H(P, n)$ returns $1$ if $P$ stops on $n$ and $0$ if $P$ loops on $n$, always correctly and always quickly.

Step 1. Build a new program $D$. On input $n$ (a program number), $D$ runs $H$ on the pair "program number $n$, input $n$" — that is, it asks whether program $n$ stops when fed its own number.

Step 2. Make $D$ contrary. If $H$ answers $1$ (program $n$ stops on $n$), then $D$ enters a loop and never stops. If $H$ answers $0$ (program $n$ loops on $n$), then $D$ stops immediately.

Step 3. Give $D$ its own number. Say $D$ is program number $d$. Run $D$ on input $d$. Inside, $D$ asks $H$ whether program $d$ stops on $d$ — but program $d$ *is* $D$, so this asks whether $D$ stops on $d$, the very thing we are running.

Step 4. Watch both answers fail. If $H$ says $D$ stops on $d$, then by Step 2 $D$ loops on $d$ — so $H$ was wrong. If $H$ says $D$ loops on $d$, then by Step 2 $D$ stops on $d$ — so $H$ was wrong again. Either way the perfect checker is wrong on this one input.

What this tells us: a checker that is right on every program-and-input pair contradicts itself the moment you feed it $D$ on $d$. There is no escape value for $H$ to return. So a perfect, always-halting halting checker is impossible — the halting question has no general mechanical answer.

## Formal definition [Intermediate+]

Fix the standard enumeration $\{\varphi_e\}_{e \in \mathbb{N}}$ of unary partial computable functions, the universal function $U(e,x) \simeq \varphi_e(x)$, and the s-m-n functions $s^m_n$ from [42.04.01]. Write $\varphi_e(x)\!\downarrow$ when the computation halts (the value is defined) and $\varphi_e(x)\!\uparrow$ when it diverges; $\simeq$ denotes Kleene equality (both sides defined and equal, or both undefined). The **halting set** (the diagonal halting set) is
$$
K = \{\, e : \varphi_e(e)\!\downarrow \,\},
$$
and the **general halting set** is $K_0 = \{\, \langle e, x\rangle : \varphi_e(x)\!\downarrow \,\}$, using a computable pairing $\langle\cdot,\cdot\rangle$. A set $A \subseteq \mathbb{N}$ is **computable** if its characteristic function $\chi_A$ is computable, and **computably enumerable** (c.e.) if it is the domain of a partial computable function [ref: TODO_REF Soare Ch. 2].

For sets $A, B \subseteq \mathbb{N}$, $A$ is **many-one reducible** to $B$, written $A \le_m B$, if there is a total computable $f$ with
$$
x \in A \iff f(x) \in B \qquad \text{for all } x.
$$
The relation $\le_m$ is reflexive and transitive; $A \equiv_m B$ means $A \le_m B \le_m A$. A c.e. set $C$ is **$m$-complete** if $A \le_m C$ for every c.e. $A$. Many-one reducibility transports decidability downward: if $A \le_m B$ and $B$ is computable then $A$ is computable, and if $A \le_m B$ and $B$ is c.e. then $A$ is c.e. The contrapositive is the working tool — to show $A$ undecidable, reduce a known-undecidable set (typically $K$) to it.

An **index set** is a set $I \subseteq \mathbb{N}$ that respects the functions computed, not the programs: $\varphi_a = \varphi_b \implies (a \in I \iff b \in I)$. Equivalently $I$ is the set of indices of the c.e. sets (or partial functions) lying in some class $\mathcal{C}$ of c.e. sets, $I = \{\, e : W_e \in \mathcal{C}\,\}$ where $W_e = \mathrm{dom}(\varphi_e)$. The two index sets $\emptyset$ and $\mathbb{N}$ correspond to the empty class and the class of all c.e. sets; every other index set encodes a property held by some but not all c.e. sets. A **fixed point** of a total computable $f$ is an index $n$ with $\varphi_n = \varphi_{f(n)}$.

### Counterexamples to common slips [Intermediate+]

- *"$K$ is uncomputable because it is infinite or complicated."* Size and apparent complexity are irrelevant: $K$ is c.e., so it is the range of a computable function and can be listed. What fails is *deciding non-membership* — its complement $\overline K$ is not even c.e. Uncomputability is the asymmetry between $K$ and $\overline K$, not bulk.

- *"$A \le_m B$ means $A$ is a subset of $B$."* It means a computable $f$ pulls membership in $A$ back from membership in $B$; $f$ need not be injective, monotone, or onto, and $A, B$ may be disjoint. The reduction is a translation of questions, not an inclusion of sets.

- *"Rice's theorem says every property of programs is undecidable."* It governs *extensional* properties — properties of the computed function, i.e. index sets. Properties of the *syntax* ("does the program have more than ten states?", "does it ever print symbol $7$ in its first step?") can be perfectly decidable. The hypothesis "$I$ is an index set" is doing real work.

- *"The recursion theorem gives a fixed value, a number $n$ with $f(n) = n$."* It gives a fixed point *up to the computed function*: $\varphi_n = \varphi_{f(n)}$, an equality of programs' behaviour, not $f(n) = n$. The transformation $f$ may move every index; the theorem only guarantees one index whose program is unchanged in meaning.

## Key theorem with proof [Intermediate+]

The signature result is that $K$ is the simplest possible undecidable set: c.e., and maximally so under $\le_m$. Everything downstream — Rice's theorem, the undecidability of totality and emptiness, Church's theorem — is a reduction from $K$.

**Theorem (the halting set is c.e. and $m$-complete).** *The set $K = \{e : \varphi_e(e)\!\downarrow\}$ is computably enumerable but not computable. Moreover $K$ is $m$-complete: for every c.e. set $A$, $A \le_m K$.*

*Proof.* $K$ is c.e. because $K = \mathrm{dom}(\psi)$ for the partial computable $\psi(e) \simeq U(e,e)$: running the universal machine on $(e,e)$ halts exactly when $\varphi_e(e)\!\downarrow$.

$K$ is not computable. Suppose $\chi_K$ were computable. Define
$$
g(e) \simeq \begin{cases} \varphi_e(e) + 1 & \text{if } e \in K, \\ 0 & \text{if } e \notin K. \end{cases}
$$
Since $\chi_K$ is computable and $U$ supplies $\varphi_e(e)$ on $K$, $g$ is total computable, so $g = \varphi_{e_0}$ for some index $e_0$. Evaluate at $e_0$. If $e_0 \in K$ then $g(e_0) = \varphi_{e_0}(e_0) + 1 = g(e_0) + 1$, impossible. If $e_0 \notin K$ then $\varphi_{e_0}(e_0)\!\uparrow$, yet $g(e_0) = 0$ is defined and $g = \varphi_{e_0}$ forces $\varphi_{e_0}(e_0)\!\downarrow$, a contradiction. Hence $\chi_K$ is not computable.

For $m$-completeness, let $A$ be c.e., say $A = \mathrm{dom}(\varphi_a)$. Consider the partial computable function of two arguments $\theta(x, y) \simeq \varphi_a(x)$ — it ignores $y$ and runs $\varphi_a$ on $x$. Let $c$ be an index with $\varphi^{(2)}_c = \theta$. By the s-m-n theorem there is a total computable $f(x) = s^1_1(c, x)$ with $\varphi_{f(x)}(y) \simeq \varphi^{(2)}_c(x, y) \simeq \varphi_a(x)$ for all $y$. Thus $\varphi_{f(x)}$ is the everywhere-defined-constant or everywhere-undefined function according as $\varphi_a(x)\!\downarrow$ or $\uparrow$. In particular $\varphi_{f(x)}(f(x))\!\downarrow \iff \varphi_a(x)\!\downarrow \iff x \in A$, so $x \in A \iff f(x) \in K$. Hence $A \le_m K$. $\square$

**Bridge.** This theorem is the foundational reason undecidability is not a scattered collection of accidents but a single phenomenon: $K$ sits at the top of the c.e. sets under $\le_m$, so any problem to which $K$ reduces is undecidable, and any c.e. problem reduces to $K$. The diagonal that defeats $\chi_K$ is exactly the self-application $\varphi_e(e)$ the universal machine of [42.04.01] makes available — this is exactly the construction the recursion theorem will run *forwards* instead of for contradiction. The reduction technique builds toward Rice's theorem, where $K$ is reduced into an arbitrary extensional property, and it appears again in the undecidability of first-order validity, where $K$ is reduced into provability (Church's theorem, cross-ref [42.01.10]). The central insight is that $m$-completeness packages "as hard as the halting problem" into a transitive relation, so putting these together one reduction from $K$ certifies an entire family of undecidable questions at once.

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Show that the general halting set $K_0 = \{\langle e,x\rangle : \varphi_e(x)\!\downarrow\}$ satisfies $K \le_m K_0$ and $K_0 \le_m K$, so $K \equiv_m K_0$.

<details class="exercise__hint"><summary>Hint</summary>

For $K \le_m K_0$ map $e \mapsto \langle e, e\rangle$. For $K_0 \le_m K$ use s-m-n to turn a pair into a single self-halting index.

</details>

<details class="exercise__answer"><summary>Answer</summary>

$K \le_m K_0$: the map $e \mapsto \langle e, e\rangle$ is computable and $e \in K \iff \varphi_e(e)\!\downarrow \iff \langle e,e\rangle \in K_0$. For $K_0 \le_m K$: define $\theta(\langle e,x\rangle, y) \simeq \varphi_e(x)$ (ignoring $y$), with index $c$, and set $f(\langle e,x\rangle) = s^1_1(c, \langle e,x\rangle)$ by s-m-n, so $\varphi_{f(\langle e,x\rangle)}(y) \simeq \varphi_e(x)$ for all $y$. Then $\varphi_{f(\langle e,x\rangle)}(f(\langle e,x\rangle))\!\downarrow \iff \varphi_e(x)\!\downarrow \iff \langle e,x\rangle \in K_0$, so $\langle e,x\rangle \in K_0 \iff f(\langle e,x\rangle) \in K$. Both maps are total computable, giving $K \equiv_m K_0$. Rubric: full credit for the diagonal embedding one way and the s-m-n constant-function construction the other.

</details>

</aside>

Prove the **halting problem for inputs** $H_e = \{x : \varphi_e(x)\!\downarrow\}$ can be the empty set, all of $\mathbb{N}$, or a properly intermediate c.e. set, and explain why "is $H_e = \mathbb{N}$?" (the totality question) is therefore not settled by examining $e$ syntactically.

</details>

**Exercise 6 (hard, symbolic).**

Prove Rice's theorem: if $I$ is an index set with $I \ne \emptyset$ and $I \ne \mathbb{N}$, then $K \le_m I$ or $K \le_m \overline I$; conclude $I$ is undecidable.

<details class="exercise__hint"><summary>Hint</summary>

Let $\emptyset$-program have index $e_\emptyset$. Arrange the reduction so that the target function is a fixed "good" function when $x \in K$ and the empty function when $x \notin K$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Let $e_\emptyset$ compute the everywhere-undefined function $\varnothing$. Since $I$ is a non-empty, non-full index set, either $e_\emptyset \notin I$ or $e_\emptyset \in I$; treat $e_\emptyset \notin I$ (else replace $I$ by $\overline I$, also an index set, and prove $K \le_m \overline I$). As $I \ne \emptyset$ pick $b \in I$, computing some $\varphi_b$. Define $\theta(x, y) \simeq \varphi_b(y)$ if $x \in K$, undefined otherwise; concretely $\theta(x,y) \simeq (\text{run } \varphi_x(x)\text{; then output } \varphi_b(y))$, partial computable. Let $c$ index $\theta$ and put $f(x) = s^1_1(c, x)$, so $\varphi_{f(x)} = \varphi_b$ if $x \in K$ and $\varphi_{f(x)} = \varnothing$ if $x \notin K$. Since $I$ is an index set, $x \in K \implies f(x) \in I$ (as $\varphi_{f(x)} = \varphi_b$, $b \in I$) and $x \notin K \implies f(x) \notin I$ (as $\varphi_{f(x)} = \varnothing$, $e_\emptyset \notin I$). Thus $x \in K \iff f(x) \in I$, i.e. $K \le_m I$. As $K$ is undecidable and decidability transfers down $\le_m$, $I$ is undecidable. Rubric: full credit for the $\varnothing$-vs-good-function reduction, the index-set invariance step, and the reduction to $\overline I$ in the complementary case.

</details>

</aside>

**Exercise 7 (hard, symbolic).**

Using the recursion theorem (every total computable $f$ has an $n$ with $\varphi_n = \varphi_{f(n)}$), prove there is an index $q$ with $\varphi_q(x) = q$ for all $x$ — a program that, on every input, outputs its own index (a quine).

<details class="exercise__hint"><summary>Hint</summary>

Apply s-m-n to the function $(e,x) \mapsto e$ to get a total computable $f$ with $\varphi_{f(e)}(x) = e$, then take a fixed point.

</details>

<details class="exercise__answer"><summary>Answer</summary>

The two-argument function $\pi(e, x) = e$ is computable; by s-m-n there is a total computable $f$ with $\varphi_{f(e)}(x) = \pi(e,x) = e$ for all $x$ — program $f(e)$ ignores its input and prints the constant $e$. By the recursion theorem $f$ has a fixed point $q$: $\varphi_q = \varphi_{f(q)}$. But $\varphi_{f(q)}(x) = q$ for all $x$, so $\varphi_q(x) = q$ for all $x$. Thus $q$ is a program that outputs its own index on every input. Rubric: full credit for building the constant-printer family by s-m-n and applying the fixed-point theorem to obtain the self-reproducing index.

</details>

</aside>

## Advanced results [Master]

The recursion theorem is the positive form of the diagonal: the same self-application that makes $K$ undecidable, run constructively, gives every effective transformation a fixed point. The theorems below develop the fixed-point machinery, sharpen Rice's theorem to the c.e. case, and map the undecidable problems that lie strictly above $K$ in logical complexity.

**Theorem 1 (Kleene recursion / fixed-point theorem).** For every total computable $f$ there is an index $n$ with $\varphi_n = \varphi_{f(n)}$ [ref: TODO_REF Rogers Ch. 11]. The proof is a single diagonal use of s-m-n: from the partial computable $(v, y) \mapsto \varphi_{f(s^1_1(v,v))}(y)$ take an index $w$ and set $n = s^1_1(w, w)$, whereupon $\varphi_n(y) \simeq \varphi^{(2)}_w(w, y) \simeq \varphi_{f(s^1_1(w,w))}(y) = \varphi_{f(n)}(y)$. The fixed point is obtained uniformly in an index for $f$, and the construction is the mirror image of the halting diagonal — there the self-application produced a contradiction, here it produces an invariant index.

**Theorem 2 (second recursion theorem; programs that read their own code).** For every partial computable $\psi(e, x)$ there is an index $n$ with $\varphi_n(x) \simeq \psi(n, x)$ for all $x$ [ref: TODO_REF Rogers Ch. 11]. Thus any program may be written as though it has access to its own index $n$: define the desired behaviour as $\psi(n, x)$ treating $n$ as a known constant, and the theorem supplies a genuine $n$ realising it. This legitimises definition by effective self-reference — recursive definitions that quote the program being defined, self-modifying schemes, and the formal construction of quines — and re-proves Rice's theorem in one line: if an index set $I$ were decidable, the program "compute $\varphi_b$ if $n \notin I$, else diverge," self-referential in $n$, would land on the wrong side of $I$, contradicting the index-set invariance.

**Theorem 3 (Rice-Shapiro).** Let $\mathcal{C}$ be a class of c.e. sets (partial functions) and $I_{\mathcal C} = \{e : W_e \in \mathcal C\}$ its index set. If $I_{\mathcal C}$ is c.e., then for every c.e. set $A$: $A \in \mathcal C$ iff some finite subset $D \subseteq A$ has $D \in \mathcal C$ [ref: TODO_REF Odifreddi Ch. III]. Membership in a c.e. index set is therefore *determined by finite approximations*: a c.e. property of the computed set cannot depend on the whole infinite behaviour, only on finitely much of it. The totality class $\{\mathbb N\}$ violates this (no finite set is total), reproving that $\mathrm{Tot}$ is not c.e.; the non-emptiness class violates it in the other direction, confirming $\overline E$ is c.e. while $E$ is not.

**Theorem 4 (undecidable problems beyond halting).** Several natural index sets sit strictly above $K$ in the arithmetical hierarchy [42.04.05]. The **totality problem** $\mathrm{Tot} = \{e : \varphi_e \text{ total}\}$ is $\Pi_2$-complete; the **emptiness problem** $\mathrm{Emp} = \{e : W_e = \emptyset\}$ is $\Pi_1$-complete (so $\overline{\mathrm{Emp}}$ is $m$-equivalent to $K$); the **finiteness problem** $\mathrm{Fin} = \{e : W_e \text{ finite}\}$ is $\Sigma_2$-complete [ref: TODO_REF Soare Ch. 4]. Outside the index sets lie undecidable problems of different texture: the **Post correspondence problem** (PCP) — given finite lists of word pairs, is there a matching concatenation? — is undecidable by reduction from $K$ via Turing-machine computation histories; the **busy beaver function** $\Sigma(n)$ — the maximum number of $1$s a halting $n$-state machine prints — is total but not computable, growing faster than any computable function, since computing it would decide halting for $n$-state machines; and the **word problem for finitely presented groups** and **Hilbert's tenth problem** (Diophantine solvability) are undecidable, the latter because every c.e. set is Diophantine (cross-ref co-produced [42.04.07]).

**Theorem 5 (the bridge to logic: Church's theorem and the undecidability of arithmetic).** Because computation is arithmetisable, $K \le_m \mathrm{Th}(\mathcal N)$ and $K \le_m \{\ulcorner\sigma\urcorner : \mathrm{PA} \vdash \sigma\}$, so the theory of the standard model and the provability set of Peano arithmetic are undecidable [ref: TODO_REF Odifreddi Ch. III]. Likewise $K$ reduces to first-order validity: from a machine and input one builds a sentence valid iff the machine halts, so the set of valid first-order sentences is undecidable — **Church's theorem** on the Entscheidungsproblem, cross-ref [42.01.10]. The recursion theorem reappears inside logic as the diagonal lemma behind Gödel's first incompleteness theorem [42.01.09]: the construction of a sentence asserting its own unprovability is the syntactic twin of the self-referential index of Theorem 2.

**Synthesis.** The single diagonal at $K$ is the foundational reason the undecidability landscape is connected rather than a list of separate impossibilities: $m$-completeness makes $K$ a universal source, so every undecidability proof in the chapter is a reduction $K \le_m (\,\cdot\,)$, and this is exactly Rice's theorem read as "reduce $K$ into any extensional class other than the empty and full ones." The recursion theorem is dual to this diagonal — the self-application $\varphi_e(e)$ that defeats the halting checker, run forwards, gives every effective transformation a fixed point, and the second recursion theorem generalises that into programs with access to their own code. Putting these together, Rice-Shapiro stratifies the undecidable index sets by whether the property is captured by finite approximations, which is precisely what lifts $\mathrm{Tot}$, $\mathrm{Fin}$, and $\mathrm{Emp}$ off the c.e. floor into $\Pi_2$, $\Sigma_2$, and $\Pi_1$ of the arithmetical hierarchy [42.04.05]. The bridge from this purely computational diagonal to logic is the arithmetisation that sends $K$ into provability and validity: the central insight is that Gödel's self-referential sentence and Kleene's self-referential program are one construction in two languages, so the incompleteness of arithmetic [42.01.09] and the unsolvability of the Entscheidungsproblem [42.01.10] are the logical shadow of the halting problem proved here.

## Full proof set [Master]

**Proposition 1 ($K$ is c.e. but not computable).** *$K = \{e : \varphi_e(e)\!\downarrow\}$ is computably enumerable and not computable.*

*Proof.* $K = \mathrm{dom}(\psi)$ for $\psi(e) \simeq U(e,e)$, partial computable, so $K$ is c.e. If $\chi_K$ were computable, then $g(e) = \varphi_e(e) + 1$ for $e \in K$ and $g(e) = 0$ for $e \notin K$ is total computable; let $g = \varphi_{e_0}$. If $e_0 \in K$ then $g(e_0) = \varphi_{e_0}(e_0) + 1 = g(e_0) + 1$, impossible; if $e_0 \notin K$ then $\varphi_{e_0}(e_0)\!\uparrow$ while $g(e_0) = 0$ is defined, contradicting $g = \varphi_{e_0}$. So $\chi_K$ is not computable. $\square$

**Proposition 2 ($K$ is $m$-complete).** *For every c.e. $A$, $A \le_m K$.*

*Proof.* Write $A = \mathrm{dom}(\varphi_a)$. Let $\theta(x,y) \simeq \varphi_a(x)$, with index $c$, and $f(x) = s^1_1(c,x)$ by s-m-n, so $\varphi_{f(x)}(y) \simeq \varphi_a(x)$ for all $y$. Then $\varphi_{f(x)}(f(x))\!\downarrow \iff \varphi_a(x)\!\downarrow \iff x \in A$, whence $x \in A \iff f(x) \in K$. As $f$ is total computable, $A \le_m K$. $\square$

**Proposition 3 (recursion theorem).** *Every total computable $f$ has $n$ with $\varphi_n = \varphi_{f(n)}$.*

*Proof.* The function $\psi(v, y) \simeq \varphi_{f(s^1_1(v,v))}(y)$ is partial computable, since $s^1_1$ and $f$ are total computable and $U$ supplies the outer evaluation. Take $w$ with $\varphi^{(2)}_w = \psi$ and set $n = s^1_1(w, w)$. For all $y$,
$$
\varphi_n(y) = \varphi_{s^1_1(w,w)}(y) \simeq \varphi^{(2)}_w(w, y) = \psi(w, y) \simeq \varphi_{f(s^1_1(w,w))}(y) = \varphi_{f(n)}(y),
$$
using s-m-n at the first step and the definition of $\psi$ at the last. Hence $\varphi_n = \varphi_{f(n)}$. $\square$

**Proposition 4 (second recursion theorem).** *For every partial computable $\psi(e,x)$ there is $n$ with $\varphi_n(x) \simeq \psi(n, x)$ for all $x$.*

*Proof.* By s-m-n there is a total computable $d$ with $\varphi_{d(e)}(x) \simeq \psi(e, x)$ for all $e, x$ (freeze $e$ as a parameter in $\psi$). Apply Proposition 3 to $d$: there is $n$ with $\varphi_n = \varphi_{d(n)}$. Then $\varphi_n(x) \simeq \varphi_{d(n)}(x) \simeq \psi(n, x)$ for all $x$. $\square$

**Proposition 5 (Rice's theorem).** *If $I$ is an index set with $\emptyset \ne I \ne \mathbb{N}$, then $I$ is undecidable.*

*Proof.* Let $e_\emptyset$ compute the everywhere-undefined function $\varnothing$. Without loss of generality $e_\emptyset \notin I$ (otherwise apply the argument to the index set $\overline I$, whose undecidability is equivalent). Since $I \ne \emptyset$, fix $b \in I$. Define $\theta(x, y) \simeq \varphi_b(y)$ if $\varphi_x(x)\!\downarrow$, and $\uparrow$ otherwise — operationally, run $\varphi_x(x)$ first, then simulate $\varphi_b(y)$. This is partial computable; let $c$ be an index for it and $f(x) = s^1_1(c, x)$. Then $\varphi_{f(x)} = \varphi_b$ when $x \in K$ and $\varphi_{f(x)} = \varnothing$ when $x \notin K$. As $I$ is an index set, $x \in K \iff f(x) \in I$ (using $b \in I$ and $e_\emptyset \notin I$). Thus $K \le_m I$; were $I$ computable, $K$ would be, contradicting Proposition 1. $\square$

**Proposition 6 (Rice-Shapiro, the finite-approximation direction).** *Let $I_{\mathcal C} = \{e : W_e \in \mathcal C\}$ be c.e. If a c.e. set $A \in \mathcal C$, then some finite $D \subseteq A$ has $D \in \mathcal C$.*

*Proof.* Suppose $A \in \mathcal C$ but no finite subset of $A$ lies in $\mathcal C$. Build, using the recursion theorem, an index $n$ for a c.e. set $W_n$ defined by dovetailing: enumerate $A$, but at each stage also simulate $\varphi_n(n)$ (legitimate since the second recursion theorem hands the construction its own index $n$); enumerate elements of $A$ into $W_n$ only until the moment, if ever, that $n \in I_{\mathcal C}$ is witnessed by the c.e. enumeration of $I_{\mathcal C}$, then freeze $W_n$ at its current finite value. If $n$ is never enumerated into $I_{\mathcal C}$, then $W_n = A \in \mathcal C$, so $n \in I_{\mathcal C}$ — contradiction, so $n$ *is* enumerated, at which point $W_n$ is a finite subset $D \subseteq A$ with $n \in I_{\mathcal C}$, i.e. $D = W_n \in \mathcal C$, contradicting the assumption that no finite subset of $A$ is in $\mathcal C$. Hence some finite $D \subseteq A$ lies in $\mathcal C$. $\square$

**Proposition 7 (the busy beaver function is not computable).** *Let $\Sigma(n)$ be the largest number of $1$s printed by a halting Turing machine with $n$ states (on blank input, fixed alphabet). Then $\Sigma$ is total but not computable.*

*Proof.* $\Sigma$ is total because for each $n$ there are finitely many $n$-state machines and the maximum over the halting ones is a well-defined natural number. Suppose $\Sigma$ were computable. Then so is any total $B(n) \ge \Sigma(n)$, e.g. $B = \Sigma$. Given a machine $M$ with $n$ states started on blank tape, run it for a bounded number of steps determined by $B(n)$ together with the finite number of distinct configurations expressible with $B(n)$ tape cells: if $M$ has not halted within that bound it has entered a configuration loop and never halts. This decides whether $n$-state machines halt on blank tape, and a padding argument reduces the general blank-tape halting problem (hence $K$ after the standard encoding) to it, contradicting Proposition 1. So $\Sigma$ is not computable, and in fact dominates every computable function eventually. $\square$

## Connections [Master]

- Models of computation [42.04.01] supplies everything this unit diagonalises against: the enumeration $\{\varphi_e\}$, the universal function $U$ that makes $\varphi_e(e)$ a legitimate partial computable object, and the s-m-n theorem that every reduction and the recursion theorem invoke. That unit owns the fixed model and its acceptable-numbering structure; this unit owns the undecidability and fixed-point consequences read off that model — Proposition 4 of [42.04.01] previews the present $K$ result, which is developed here in full with $m$-completeness.

- Computably enumerable sets and the priority method [42.04.03] continue the reducibility theory begun here: $\le_m$ is refined to Turing reducibility $\le_T$, $K$ becomes the canonical $\Sigma_1$ object, and the existence of c.e. sets strictly between computable and $m$-complete (Post's problem, solved by Friedberg-Muchnik) is the next structural question. The co-produced unit [42.04.03] builds the c.e. degree structure on top of the $K$-and-reductions apparatus established here; this unit owns the halting set and the many-one theory it sits atop.

- The arithmetical hierarchy [42.04.05] classifies the undecidable problems of Theorem 4 by quantifier complexity: $K$ is $\Sigma_1$-complete, $\mathrm{Tot}$ is $\Pi_2$-complete, $\mathrm{Fin}$ is $\Sigma_2$-complete, and Rice-Shapiro is the tool that lifts a property off the $\Sigma_1$ floor by showing it is not finitely approximable. That unit relativises $K$ to oracles to build the jump hierarchy; this unit provides the un-relativised base case and the index sets that populate the first levels.

- Gödel incompleteness [42.01.09] and Church's theorem [42.01.10] are the logical projection of the halting diagonal: arithmetisation gives $K \le_m \{\ulcorner\sigma\urcorner : \mathrm{PA}\vdash\sigma\}$ and $K \le_m \{\text{valid first-order sentences}\}$, so undecidability of provability and of validity follow from Proposition 1, and the recursion theorem's self-referential index (Proposition 4) is the computational form of the diagonal lemma that builds the Gödel sentence. The word problem and Hilbert's tenth problem reductions are co-produced in [42.04.07], where the MRDP theorem makes every c.e. set Diophantine.

## Historical & philosophical context [Master]

The halting problem and the recursion theorem are twin consequences of the 1936 analysis of computation. Alan Turing's 1936 paper proved the unsolvability of the *Entscheidungsproblem* by exhibiting an undecidable problem about his machines — essentially the printing/halting problem — through a diagonal argument modelled on Cantor's proof that the reals are uncountable and on the self-reference in Gödel's 1931 incompleteness theorem [ref: TODO_REF Soare Ch. 2]. Alonzo Church had reached the undecidability of first-order validity the same year by the $\lambda$-calculus route, and the reduction of an undecidable arithmetical problem to validity is now called Church's theorem. Emil Post, in unpublished work of the 1920s–30s and in his 1944 paper on c.e. sets, introduced reducibility and the structure of degrees, posing the problem — whether there is a c.e. set neither computable nor $m$-complete — that organised the next two decades of the subject.

Stephen Kleene proved the recursion theorem and isolated the s-m-n theorem in the 1930s and systematised them in *Introduction to Metamathematics* (1952); the fixed-point theorem and its second form gave self-reference a rigorous, paradox-free standing, turning the device that defeats the halting checker into a construction principle. Henry Gordon Rice's 1953 paper established that every extensional property of the c.e. sets — every property of what a program computes rather than how it is written — is undecidable, drawing the boundary the chapter's index-set theory formalises [ref: TODO_REF Rogers Ch. 9]. John Myhill identified the $m$-complete c.e. sets with the creative sets and showed them all recursively isomorphic to $K$, fixing the halting set as the canonical representative of its degree.
