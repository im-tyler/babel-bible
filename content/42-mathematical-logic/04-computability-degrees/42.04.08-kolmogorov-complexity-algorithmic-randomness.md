## Intuition [Beginner]

The earlier units in this chapter measured how *hard* a set is to compute. This unit measures something that sounds different but turns out to be the same story told backwards: how *random* a string or a sequence is. The link is a single idea — randomness is the absence of any pattern a computer could exploit.

Think about describing a string of a thousand digits to a friend over the phone. If the string is "one thousand zeros," you say exactly that, and the description is tiny. If the string is the first thousand digits of a familiar constant, you name the constant. But if the string was made by flipping a fair coin a thousand times, there is almost certainly no short way to say it: the shortest description is roughly the string itself, digit for digit. The length of the shortest description is a measure of how much real information the string carries.

Call a string *incompressible* when its shortest description is about as long as the string. Most strings are incompressible, for a plain counting reason: there are far more long strings than there are short descriptions to go around, so short descriptions run out and most strings are stuck describing themselves. Incompressible is the finite version of random.

For infinite sequences the same instinct needs a second sharpening. A sequence is random when it passes every test a computer could run to catch a pattern — every effective bet, every effective regularity, every shrinking target a program could aim at. Remarkably, three different ways of saying "no pattern" — passing every effective test, having incompressible chunks, and avoiding every small effectively-described target — all pick out the very same sequences. And there is a single most famous random number, built straight from the halting problem, called Chaitin's number. The rest of the unit makes the three pictures precise and proves they coincide.

## Visual [Beginner]

![Short descriptions run out, so most strings are incompressible; random sequences pass every effective test](/img/kolmogorov-complexity-algorithmic-randomness-placeholder.svg)

Picture every length-$n$ string on the left and every shorter program on the right, and try to pair each string with a program that prints it.

```
   strings of length n            programs shorter than n
   (there are 2^n of them)        (there are 2^n - 1 of them)

   000...0  --------------------> "n zeros"        (a short program)
   a familiar pattern  ---------> "the pattern"    (a short program)
   ...
   most strings  ----- ? -------> NOT ENOUGH short programs left
                                  -> the string must describe itself

   Count: 2^n strings, only 2^n - 1 shorter programs.
   The short programs run out, so most strings are INCOMPRESSIBLE.
```

Now the infinite picture. A test is a shrinking sequence of targets a computer can list, with the targets getting smaller and smaller; a sequence is caught if it lands in every target.

| picture of "random" | what the computer does | a sequence is random when |
|---|---|---|
| incompressible chunks | compress each starting block | no block compresses by more than a fixed amount |
| passes every test | list shrinking targets to catch patterns | it escapes every shrinking target |
| avoids small targets | aim at effectively-described tiny regions | it sits in no effectively-tiny region |

Read the table across: three different machines, three different verdicts, but they agree on exactly which sequences count as random. The agreement is the heart of the subject.

## Worked example [Beginner]

We compare three length-eight strings and read off how compressible each is, to feel the difference between order and randomness with concrete numbers.

Step 1. Take $00000000$. A description is "print $0$ eight times," which is far shorter than the string itself once the count gets large; the content is just the pair "the symbol $0$" and "the number $8$." This string is highly compressible — almost all of its eight bits are predictable from a short recipe.

Step 2. Take $01010101$. A description is "print $01$ four times." Again the recipe is short: one small block plus a repeat count. Compressible, for the same reason — a rule generates the whole string, so you ship the rule, not the string.

Step 3. Take $01101000$, produced by eight coin flips. There is no shorter rule in sight: "print $01101000$" is about the best you can do, so the description is roughly the string itself. Among the $256$ strings of length eight, at most $255$ can have a description shorter than eight bits, because there are only $255$ programs of length seven or less. So at least one string of length eight is incompressible, and in fact the large majority are within a bit or two of incompressible.

What this tells us: compressibility tracks pattern. A string with a rule behind it has a short description; a string made at random almost surely has none, because short descriptions are a scarce resource that the many random strings cannot all share. The same scarcity, pushed to infinite sequences, is what makes "random" mean "no effective pattern, anywhere, ever."

## Formal definition [Intermediate+]

Fix the standard enumeration $\{\varphi_e\}$, the c.e. sets $W_e$ with stage approximations of [42.04.03], oracle functionals $\Phi_e^A$, the jump $A'$, and the iterated jumps $\emptyset^{(n)}$ of [42.04.04]. Strings are elements of $2^{<\omega}$, sequences elements of Cantor space $2^\omega$ with the uniform (Lebesgue) measure $\mu$; $X{\restriction}n$ is the length-$n$ prefix of $X$, and $[\sigma] = \{X : \sigma \prec X\}$ is the basic clopen cylinder, $\mu([\sigma]) = 2^{-|\sigma|}$.

A machine is a partial computable $M : 2^{<\omega} \to 2^{<\omega}$. The **plain complexity** relative to $M$ is $C_M(x) = \min\{|p| : M(p) = x\}$ ($\infty$ if none). A machine $U$ is **universal** if for every $M$ there is a constant $c_M$ with $C_U(x) \le C_M(x) + c_M$ for all $x$; the **invariance theorem** gives such a $U$, and any two universal machines agree up to an additive constant, so $C := C_U$ is well-defined modulo $O(1)$ [ref: TODO_REF Li-Vitanyi Ch. 2]. A machine $M$ is **prefix-free** if its domain is an antichain under $\prec$ (no halting program is a prefix of another). A universal prefix-free machine exists by the same argument; its complexity is the **prefix (Kolmogorov-Chaitin) complexity** $K(x) = \min\{|p| : U(p) = x, \ U \text{ prefix-free}\}$, again well-defined up to $O(1)$. The defining inequalities are $C(x) \le K(x) + O(1)$ and $K(x) \le C(x) + 2\log C(x) + O(1)$.

A prefix-free machine's domain $D$ satisfies the **Kraft inequality** $\sum_{p \in D} 2^{-|p|} \le 1$. Conversely, the **Kraft-Chaitin theorem**: given a c.e. set of *requests* $(\ell_i, x_i)$ with $\sum_i 2^{-\ell_i} \le 1$, there is a prefix-free machine $M$ and, for each request, a program $p_i$ with $|p_i| = \ell_i$ and $M(p_i) = x_i$. Hence $K$ is **subadditive**, $K(\langle x, y\rangle) \le K(x) + K(y) + O(1)$, repairing the failure of subadditivity for $C$ [ref: TODO_REF Nies Ch. 2].

A **Martin-Löf test** is a uniformly c.e. sequence $(U_n)_{n \in \omega}$ of $\Sigma^0_1$ classes in $2^\omega$ with $\mu(U_n) \le 2^{-n}$. A sequence $X$ **fails** the test if $X \in \bigcap_n U_n$; the failure set is an **effective null set**. $X$ is **Martin-Löf random** (ML-random) if it fails no ML-test, equivalently passes a single **universal ML-test** $(\widehat U_n)$ obtained by effectively combining all tests [ref: TODO_REF Downey-Hirschfeldt Ch. 6]. A real $\alpha \in [0,1]$ is **left-c.e.** if it is the increasing limit of a computable sequence of rationals.

A set $A$ is **$K$-low** if $K(A{\restriction}n) \le K(n) + O(1)$ for all $n$. A set $A$ is **low for ML-randomness** if every ML-random sequence is ML-random relative to $A$, and **low for $K$** if $K^A(x) \ge K(x) - O(1)$ for all $x$.

### Counterexamples to common slips [Intermediate+]

- *"Plain complexity $C$ is subadditive."* It is not: $C(\langle x,y\rangle) \le C(x) + C(y) + O(1)$ fails because a plain decoder cannot tell where the program for $x$ ends and that for $y$ begins. The self-delimiting prefix-free domain is precisely what restores $\sum 2^{-|p|} \le 1$ and hence $K(\langle x,y\rangle) \le K(x)+K(y)+O(1)$.

- *"$K(X{\restriction}n) \ge n$ characterises ML-randomness."* The exact bound must absorb an additive constant: ML-randomness is $K(X{\restriction}n) \ge n - O(1)$. With plain $C$ even $C(X{\restriction}n) \ge n - O(1)$ fails for every $X$ — *complexity oscillations* force $C(X{\restriction}n) \le n - \log n$ infinitely often, which is why the Schnorr-Levin theorem uses $K$, not $C$.

- *"Random means avoiding every measure-zero set."* Only the *effective* null sets count. A single ML-random $X$ is the sole member of the measure-zero set $\{X\}$, which it certainly does not avoid; the constraint is to avoid every effectively-presented (uniformly c.e., measure-controlled) null set, of which there are only countably many.

- *"$\Omega$ depends on nothing but the definition."* $\Omega$ depends on the choice of universal prefix machine $U$; different $U$ give different reals. What is invariant is that every such $\Omega$ is left-c.e., ML-random, and Turing-equivalent to $\emptyset'$; the *Solovay-complete* left-c.e. reals are exactly these halting probabilities (Kučera-Slaman).

## Key theorem with proof [Intermediate+]

The organising result identifies randomness with incompressibility: a sequence is Martin-Löf random exactly when no initial segment can be compressed by more than a constant. This is the Schnorr-Levin theorem, and it ties the test picture to the complexity picture.

**Theorem (Schnorr-Levin).** *For $X \in 2^\omega$: $X$ is Martin-Löf random iff there is a constant $c$ with $K(X{\restriction}n) \ge n - c$ for all $n$.* [ref: TODO_REF Downey-Hirschfeldt Ch. 6]

*Proof.* ($\Leftarrow$ by contraposition: not ML-random $\Rightarrow$ initial segments compressible.) Suppose $X$ fails the universal ML-test $(\widehat U_n)$, so $X \in \bigcap_n \widehat U_n$ with $\mu(\widehat U_n) \le 2^{-n}$. Each $\widehat U_n$ is $\Sigma^0_1$, hence a c.e. union of cylinders $[\sigma]$; list for each $n$ the strings $\sigma$ generating $\widehat U_n$. For the level $\widehat U_{2d}$ ($\mu \le 2^{-2d}$), the generating strings $\sigma$ satisfy $\sum_\sigma 2^{-|\sigma|} \le 2^{-2d}$, so $\sum_{\sigma} 2^{-(|\sigma| - d)} \le 2^{-d}$ and summing over $d$ keeps the total $\le 1$. By the **Kraft-Chaitin theorem** build a prefix machine that, on a program of length $|\sigma| - d$, outputs $\sigma$; then $K(\sigma) \le |\sigma| - d + O(1)$ for every $\sigma$ generating $\widehat U_{2d}$. Since $X \in \widehat U_{2d}$ for every $d$, some prefix $\sigma = X{\restriction}m$ generates it, giving $K(X{\restriction}m) \le m - d + O(1)$. As $d$ grows, the compression $d$ is unbounded, so $K(X{\restriction}n) \ge n - c$ fails for every $c$.

($\Rightarrow$ by contraposition: compressible initial segments $\Rightarrow$ not ML-random.) Suppose for every $c$ there is $n$ with $K(X{\restriction}n) < n - c$. Define $V_c = \bigcup\{[\tau] : |\tau| = n, \ K(\tau) < n - c \text{ for some } n\}$, a $\Sigma^0_1$ class since $K$ is upper-semicomputable (its graph is c.e. from above). Counting: the number of strings $\tau$ of length $n$ with $K(\tau) < n - c$ is at most the number of prefix programs of length $< n - c$, and by the Kraft inequality $\sum_{\tau : K(\tau) < n-c} 2^{-|\tau|} \le \sum_{|p| < n - c} 2^{-(|p| + c)} \le 2^{-c}$; summing the measure over all such $\tau$ gives $\mu(V_c) \le 2^{-c}$. So $(V_c)_c$ is an ML-test (after the index shift $c \mapsto$ level), and $X \in \bigcap_c V_c$ because some initial segment of $X$ enters each $V_c$. Thus $X$ fails an ML-test and is not ML-random. $\square$

**Bridge.** The Schnorr-Levin theorem is the foundational reason the two definitions of randomness — passing every effective test and having incompressible initial segments — are not two notions but one: a test of measure $\le 2^{-d}$ is, via Kraft-Chaitin, a $d$-bit compression of whatever it catches, and incompressibility is exactly the absence of every such compression. This is exactly the counting argument of the Beginner tier made effective, where short descriptions are scarce and a measure-$2^{-d}$ target is a description $d$ bits shorter than the string it traps; the Kraft inequality is the conservation law that bounds how many strings can be cheaply described, and it is dual to the measure bound on tests. The construction builds toward Chaitin's $\Omega$ in the Advanced results, the canonical sequence whose initial segments are incompressible because $\Omega$ is the halting probability of the very prefix machine defining $K$, and it appears again in van Lambalgen's theorem, where relativised incompressibility makes randomness a symmetric relation. The central insight is that the jump of [42.04.04] enters here as the source of the canonical random real: $\Omega \equiv_T \emptyset'$, so the halting problem is not merely hard but maximally information-dense, and putting these together, randomness measured by tests, by $K$, and by effective null sets is one phenomenon whose canonical witness is built from $\emptyset'$.

**Exercise 1 (easy, multiple-choice).**

Why is plain complexity $C$ well-defined only up to an additive constant?

<ol class="exercise__options" type="A">
<li>Because strings can be reordered.</li>
<li>Because the invariance theorem gives any two universal machines $C$ functions differing by a constant.</li>
<li>Because $C$ is computable.</li>
<li>Because $C(x) = |x|$ exactly.</li>
</ol>

<details class="exercise__hint"><summary>Hint</summary>

Recall the universal machine simulates any other machine after reading a fixed-length prefix naming it.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**B.** A universal machine simulates any machine $M$ after a fixed prefix of length $c_M$, so $C_U(x) \le C_M(x) + c_M$; applied both ways between two universal machines this gives agreement up to $O(1)$. Option C is false ($C$ is non-computable), and the constant is exactly what the invariance theorem controls.

</details>

</aside>

<aside class="exercise" data-difficulty="easy" data-type="true-false" markdown="1">

**Exercise 2 (easy, true-false).**

True or false: a prefix-free machine's domain satisfies $\sum_{p} 2^{-|p|} \le 1$ (the Kraft inequality).

<details class="exercise__hint"><summary>Hint</summary>

The cylinders $[p]$ for halting programs $p$ are pairwise disjoint when the domain is an antichain.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**True.** If the domain is prefix-free then the cylinders $[p]$ are pairwise disjoint in $2^\omega$, so $\sum_p \mu([p]) = \sum_p 2^{-|p|} \le \mu(2^\omega) = 1$. This disjointness is precisely the self-delimiting property that restores subadditivity for $K$.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Prove the **incompressibility theorem**: for every $n$ there is a string $x$ of length $n$ with $C(x) \ge n$, and at least $2^n(1 - 2^{-c}) $ strings of length $n$ with $C(x) \ge n - c$.

<details class="exercise__hint"><summary>Hint</summary>

Count the programs of length $< n - c$ and compare with the $2^n$ strings.

</details>

<details class="exercise__answer"><summary>Answer</summary>

The number of programs of length $< k$ is $2^k - 1 < 2^k$. A string $x$ with $C(x) < n - c$ has a program of length $< n - c$, and the map "program $\mapsto$ its output" is a function, so at most $2^{n-c} - 1 < 2^{n-c}$ strings of length $n$ have $C(x) < n - c$. Hence at least $2^n - 2^{n-c} = 2^n(1 - 2^{-c})$ strings of length $n$ satisfy $C(x) \ge n - c$. Taking $c = 0$: at most $2^n - 1$ strings of length $n$ have $C(x) < n$, so at least one has $C(x) \ge n$. Rubric: full credit for the program-count bound $< 2^{n-c}$, the function (at-most-one-output) observation, and the subtraction giving the proportion $1 - 2^{-c}$.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 4 (medium, symbolic).**

Prove that $C$ is not computable, using Berry's paradox: if $C$ were computable, "the least string $x$ with $C(x) > n$" would have a short description.

<details class="exercise__hint"><summary>Hint</summary>

Such a string exists for each $n$ by the incompressibility theorem; describe it by $n$ plus a fixed program.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Suppose $C$ is computable. Define $f(n) = $ the lexicographically least string $x$ with $C(x) > n$; by the incompressibility theorem such $x$ exists (strings of length $n+1$ include one with $C \ge n+1 > n$), and with $C$ computable $f$ is computable by search. Now $f(n)$ has a description: a fixed program for $f$ together with $n$ in binary, of total length $|f\text{'s program}| + \log n + O(1) = \log n + O(1)$. So $C(f(n)) \le \log n + O(1)$. But by definition $C(f(n)) > n$. For large $n$, $\log n + O(1) < n$, contradiction. Hence $C$ is not computable. (It remains upper-semicomputable: $C(x) \le k$ is c.e., by dovetailing all programs of length $\le k$.) Rubric: full credit for defining $f$ via the incompressibility theorem, bounding $C(f(n)) \le \log n + O(1)$ from the short description, and contradicting $C(f(n)) > n$.

</details>

</aside>

**Exercise 5 (medium, multiple-choice).**

Which property does Chaitin's $\Omega = \sum_{U(p)\downarrow} 2^{-|p|}$ *not* have?

<ol class="exercise__options" type="A">
<li>Left-c.e. (an increasing computable limit of rationals).</li>
<li>Martin-Löf random.</li>
<li>Computable.</li>
<li>Turing-equivalent to $\emptyset'$.</li>
</ol>

<details class="exercise__hint"><summary>Hint</summary>

$\Omega$ encodes the halting problem in its bits; a computable real cannot do that.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**C.** $\Omega$ is left-c.e. (approximated from below as more programs are seen to halt), ML-random, and $\equiv_T \emptyset'$ — its first $n$ bits decide halting for all programs of length $\le n$. It is therefore *not* computable: a computable real is not ML-random, and could not encode $\emptyset'$. Options A, B, D are the three signature properties of $\Omega$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Prove that Chaitin's $\Omega$ is Martin-Löf random, using the Schnorr-Levin theorem and the fact that the first $n$ bits of $\Omega$ determine which programs of length $\le n$ halt.

<details class="exercise__hint"><summary>Hint</summary>

From $\Omega{\restriction}n$ you can compute the halting set restricted to programs of length $\le n$, hence recover any short string those programs output.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Write $\Omega_s$ for the stage-$s$ lower approximation, increasing to $\Omega$. Given the first $n$ bits of $\Omega$, run the enumeration until $\Omega_s$ first exceeds $\Omega{\restriction}n$ (as a binary fraction); at that stage every program $p$ with $|p| \le n$ that will ever halt has already halted, since any later halting program would add $\ge 2^{-n}$ and push $\Omega$ past $\Omega{\restriction}n + 2^{-n}$, contradicting the known first $n$ bits. So from $\Omega{\restriction}n$ one computes the finite set $H_n$ of halting programs of length $\le n$ and all their outputs. Now suppose $K(\Omega{\restriction}n) < n - c$ for some $n$ and a would-be unbounded $c$: a prefix program $q$ of length $< n - c$ produces $\Omega{\restriction}n$, from which $H_{n}$ and the outputs of all length-$\le n$ programs are computable. Diagonalising — output a string of complexity $> n$ using the recovered $H_n$ — yields a string described by $q$ plus $O(1)$ bits, of length $< n - c + O(1) < n$, yet of complexity $> n$, a contradiction once $c$ exceeds the fixed overhead. Hence $K(\Omega{\restriction}n) \ge n - O(1)$ for all $n$, and by Schnorr-Levin $\Omega$ is ML-random. Rubric: full credit for recovering $H_n$ from $\Omega{\restriction}n$ via the approximation-overshoot argument, the diagonal extraction of a high-complexity string from a short description of $\Omega{\restriction}n$, and the appeal to Schnorr-Levin.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

State and prove **van Lambalgen's theorem** in the easy direction: if $X \oplus Y$ is ML-random then $X$ is ML-random and $Y$ is ML-random relative to $X$.

<details class="exercise__hint"><summary>Hint</summary>

A test for $X$ alone, or a test for $Y$ relative to $X$, lifts to a test for the join $X \oplus Y$ of the same measure.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Recall $X \oplus Y$ interleaves the two sequences. ($X$ random.) If $X$ failed an ML-test $(U_n)$, then $V_n = \{Z \oplus W : Z \in U_n, W \in 2^\omega\}$ is a uniformly c.e. sequence of $\Sigma^0_1$ classes with $\mu(V_n) = \mu(U_n) \le 2^{-n}$ (the $W$-coordinate is free, measure $1$), and $X \oplus Y \in \bigcap_n V_n$, so $X \oplus Y$ fails an ML-test — contradiction. Hence $X$ is ML-random. ($Y$ random relative to $X$.) Suppose $Y$ failed an $X$-ML-test $(U_n^X)$, with $U_n^X$ uniformly $\Sigma^{0,X}_1$ and $\mu(U_n^X) \le 2^{-n}$. By Fubini, integrate over the $X$-coordinate: the set $W_n = \{Z \oplus W : W \in U_n^Z\}$ is $\Sigma^0_1$ (the oracle $Z$ is read from the even coordinates of the join) with $\mu(W_n) = \int \mu(U_n^Z)\, dZ \le 2^{-n}$, and $X \oplus Y \in \bigcap_n W_n$ since $Y \in U_n^X$ for all $n$. So $X \oplus Y$ fails an ML-test, contradicting its randomness. Therefore $Y$ is ML-random relative to $X$. Rubric: full credit for lifting an $X$-test to a join-test with the free $W$-coordinate, the Fubini measure bound for the relativised test, and the contradiction with randomness of $X \oplus Y$ in both parts.

</details>

</aside>

## Advanced results [Master]

Randomness has a degree theory as rich as the c.e. degrees of [42.04.06]: the maximally non-random reals form a definable ideal, the canonical random real is the halting probability, and the randomness notions stratify into a strict hierarchy indexed by the jump.

**Theorem 1 (Chaitin's $\Omega$; the number of wisdom).** Fix a universal prefix machine $U$ and set $\Omega = \sum_{U(p)\downarrow} 2^{-|p|}$, the halting probability [ref: TODO_REF Downey-Hirschfeldt Ch. 6]. Then $\Omega \in (0,1)$ is **left-c.e.** (the increasing limit $\Omega_s = \sum_{U_s(p)\downarrow} 2^{-|p|}$ of computable rationals), **Martin-Löf random** (its initial segments are incompressible, by Schnorr-Levin), and **incomputable**; moreover $\Omega \equiv_T \emptyset'$, and the first $n$ bits of $\Omega$ uniformly compute the halting of all programs of length $\le n$. The left-c.e. ML-random reals are exactly the **$\Omega$-numbers** — the halting probabilities of universal prefix machines — and they are precisely the Solovay-complete c.e. reals (Calude-Hertling-Khoussainov-Wang; Kučera-Slaman). Thus $\Omega$ packages the entire halting problem into a single real that is simultaneously as enumerable as a c.e. set and as patternless as a fair coin.

**Theorem 2 ($K$-lowness and lowness coincide).** A set $A$ is **$K$-low** if $K(A{\restriction}n) \le K(n) + O(1)$: its initial segments are as compressible as possible, the exact antithesis of $\Omega$. The **Nies-Hirschfeldt-Downey theorem** [ref: TODO_REF Nies Ch. 5] identifies three a priori different lowness notions: $A$ is $K$-low $\iff$ $A$ is **low for $K$** ($K^A = K + O(1)$) $\iff$ $A$ is **low for ML-randomness** (every ML-random real stays ML-random relative to $A$) $\iff$ $A$ is a **base for ML-randomness** ($A \le_T Z$ for some $Z$ that is ML-random relative to $A$). Every $K$-low set is $\Delta^0_2$, and the $K$-lows form a $\Sigma^0_3$, c.e.-presentable, downward-closed ideal under $\oplus$ in the Turing degrees, lying strictly below $\emptyset'$, inside both the **low** degrees ($A' \equiv_T \emptyset'$) and the **non-cuppable** c.e. degrees. There are noncomputable $K$-lows (Solovay; Downey-Hirschfeldt-Nies-Stephan), built by the **cost-function method**.

**Theorem 3 (van Lambalgen's theorem).** $X \oplus Y$ is ML-random iff $X$ is ML-random and $Y$ is ML-random relative to $X$ [ref: TODO_REF Nies Ch. 3]. Consequently relative randomness is **symmetric** for ML-random reals: if $X \oplus Y$ is random then $X$ is random relative to $Y$ and $Y$ random relative to $X$, so "$X$ is random relative to $Y$" and "$Y$ is random relative to $X$" hold together. The theorem is the engine of the degree theory of randomness — it reduces questions about pairs to relativised single-real questions, and it fails for the weaker notions (Schnorr, computable randomness), which is one diagnostic separating ML-randomness from its neighbours.

**Theorem 4 (the randomness hierarchy).** The natural randomness notions form a strict ascending chain [ref: TODO_REF Downey-Hirschfeldt Ch. 7]: **computable randomness** $\supsetneq$ **Schnorr randomness** at the lower end, then **Martin-Löf randomness**, **weak-2-randomness**, and **$n$-randomness** ($=$ ML-randomness relative to $\emptyset^{(n-1)}$), with each notion strictly stronger than the last and $2$-randomness already strong enough to make $X' \equiv_T X \oplus \emptyset'$ (the random real adds no jump information beyond the join). Schnorr randomness uses tests whose measures converge computably to $0$; weak-2-randomness uses $\Pi^0_2$ null sets; the $n$-randomness ladder is the arithmetical hierarchy of [42.04.05] applied to the test complexity, so the jump of [42.04.04] indexes the strength of randomness exactly as it indexes computational difficulty. The Kučera-Gács theorem sits across this picture: every real is computable from some ML-random real, so randomness does not bound Turing complexity from above.

**Theorem 5 (the incompressibility method and normality).** Algorithmic randomness yields a proof technique — the **incompressibility method** [ref: TODO_REF Li-Vitanyi Ch. 6]: to prove a combinatorial statement, fix an incompressible object (a string $x$ with $C(x) \ge |x|$) and derive bounds from the impossibility of compressing it. It delivers average-case lower bounds for sorting and Turing-machine running time, the expected $O(n \log n)$ behaviour of algorithms, Ramsey-type and graph estimates, and number-theoretic bounds (including prime-gap estimates), serving as a deterministic counterpart of the probabilistic method, with the incompressible object playing the role of the "random" object whose existence the counting argument guarantees. Algorithmic randomness also refines classical typicality: every ML-random sequence is **Borel-normal** (each block of digits appears with its limiting frequency), but normality is strictly weaker — there are computable normal numbers (Champernowne's constant) that are far from random.

**Synthesis.** Algorithmic randomness is the foundational reason "no pattern" has a single precise meaning rather than three competing ones: unpredictability by effective tests, incompressibility by prefix complexity, and avoidance of every effective null set are the same condition, and this is exactly the Schnorr-Levin theorem read together with the universal-test construction — a measure-$2^{-d}$ effective target is, via Kraft-Chaitin, a $d$-bit compression, so a test is a description and incompressibility is the absence of descriptions. The canonical witness is Chaitin's $\Omega$, the halting probability of the prefix machine that defines $K$, which is left-c.e. like a c.e. set yet ML-random like a fair coin and Turing-equivalent to $\emptyset'$; this is the central insight that the halting problem of [42.04.02] is not merely undecidable but maximally information-dense, and it generalises the jump of [42.04.04] from a measure of difficulty to a source of randomness.

The degree theory mirrors the c.e. degrees of [42.04.06] in negative: where the priority method populates the interval $(\mathbf 0, \mathbf 0')$ with intermediate degrees, the $K$-lows populate it with the *least* random reals, and the Nies-Hirschfeldt-Downey coincidence — $K$-low $=$ low for $K$ $=$ low for ML-random — makes "adds no randomness power" a single robust class, a $\Sigma^0_3$ ideal of low non-cuppable degrees dual to $\Omega$ at the random extreme. Putting these together, the randomness hierarchy is the arithmetical hierarchy of [42.04.05] applied to test complexity: $n$-randomness is randomness relative to $\emptyset^{(n-1)}$, so the iterated jump indexes both ladders, and van Lambalgen's theorem is the bridge that makes relativised randomness symmetric, the foundational reason a degree theory of randomness exists at all. The incompressibility method then turns the whole apparatus outward, using a single incompressible object as the deterministic stand-in for the probabilistic method across combinatorics and average-case analysis.

## Full proof set [Master]

**Proposition 1 (invariance theorem).** *There is a universal prefix-free machine $U$ such that for every prefix-free machine $M$ there is $c_M$ with $K(x) \le C_M(x) + c_M$ for all $x$; consequently any two universal prefix-free machines give complexities agreeing within $O(1)$.*

*Proof.* Enumerate the prefix-free machines $M_0, M_1, \dots$ uniformly (effectively recognising prefix-freeness is not needed: clip each machine to its prefix-free core by accepting a program only if no proper prefix has already halted, which preserves universality). Fix a prefix-free coding $e \mapsto \bar e$ of indices with $|\bar e| = O(1) + 2\log e$ — itself prefix-free since the codes form an antichain. Define $U(\bar e\, p) = M_e(p)$; the domain is prefix-free because $\bar e$ is self-delimiting and each $M_e$ has prefix-free domain, so no halting input of $U$ is a prefix of another. Then $C_U(x) \le |\bar e| + C_{M_e}(x) = C_{M_e}(x) + c_{M_e}$ with $c_{M_e} = |\bar e|$. For two universal machines $U, U'$ apply the bound in each direction: $C_U(x) \le C_{U'}(x) + c$ and $C_{U'}(x) \le C_U(x) + c'$, so $|C_U(x) - C_{U'}(x)| \le \max(c,c')$. $\square$

**Proposition 2 (Kraft-Chaitin / machine-existence theorem).** *Given a c.e. set of requests $(\ell_i, x_i)_{i\in\omega}$ with $\sum_i 2^{-\ell_i} \le 1$, there is a prefix-free machine $M$ with, for each $i$, a program $p_i$ of length $\ell_i$ and $M(p_i) = x_i$.*

*Proof.* Process requests in enumeration order, maintaining the available part of $[0,1)$ as a finite union of dyadic intervals (the still-unassigned binary subtree). For request $(\ell_i, x_i)$ choose a dyadic interval of length $2^{-\ell_i}$ from the available set, aligned at a multiple of $2^{-\ell_i}$; assign its defining length-$\ell_i$ string as $p_i$ with $M(p_i) = x_i$, and remove that interval. The greedy rule — always take the leftmost available aligned interval of the required size, splitting a larger free interval if necessary — keeps the free region a union of dyadic intervals of distinct sizes, and an interval of size $2^{-\ell_i}$ is always available because the total length requested so far is $\sum_{j \le i} 2^{-\ell_j} \le 1$, so the used length never exceeds $1$ and the alignment invariant guarantees a fitting slot. The assigned strings are pairwise prefix-incomparable (their intervals are disjoint), so $M$ is prefix-free. $M$ is computable since the request set is c.e. and the bookkeeping is finite at each stage. $\square$

**Proposition 3 (universal ML-test).** *There is a single ML-test $(\widehat U_n)$ such that $X$ is ML-random iff $X \notin \bigcap_n \widehat U_n$; equivalently a largest effective null set exists.*

*Proof.* Enumerate all ML-tests effectively: a candidate is a uniformly c.e. double sequence $(U^e_n)$, and from any c.e. sequence force the measure bound by truncating $U^e_n$ at the stage its enumerated measure would exceed $2^{-n}$ (a computable guard), yielding a genuine test $(\widetilde U^e_n)$ that equals $(U^e_n)$ when the latter is already a test. Set $\widehat U_n = \bigcup_e \widetilde U^e_{n + e + 1}$. Then $\mu(\widehat U_n) \le \sum_e 2^{-(n+e+1)} = 2^{-n}$, and $(\widehat U_n)$ is uniformly c.e., so it is an ML-test. If $X$ fails some test $(U^e_n)$, then $X \in \widetilde U^e_m$ for all $m$, in particular $X \in \widetilde U^e_{n+e+1} \subseteq \widehat U_n$ for all $n$, so $X$ fails $(\widehat U_n)$. Conversely failing $(\widehat U_n)$ is failing an ML-test. So $\bigcap_n \widehat U_n$ is the union of all effective null sets, the largest effective null set, and its complement is the ML-random reals. $\square$

**Proposition 4 ($\Omega$ is left-c.e., incomputable, and $\equiv_T \emptyset'$).** *Chaitin's $\Omega$ is the increasing limit of computable rationals, is not computable, and satisfies $\Omega \equiv_T \emptyset'$.*

*Proof.* Left-c.e.: $\Omega_s = \sum_{U_s(p)\downarrow} 2^{-|p|}$ is a computable nondecreasing sequence of rationals with limit $\Omega$ (the Kraft inequality gives $\Omega \le 1$, and $\Omega > 0$ since $U$ halts on something). Incomputable: by Proposition 4's left-c.e.-ness and Schnorr-Levin (Theorem 1) $\Omega$ is ML-random, and no ML-random real is computable — a computable $X$ fails the test $U_n = [X{\restriction}n]$ of measure $2^{-n}$. For $\Omega \ge_T \emptyset'$: given $\Omega{\restriction}n$, enumerate $U$ until $\Omega_s > \Omega{\restriction}n$ (as binary fractions); thereafter no program of length $\le n$ can newly halt, since one would contribute $\ge 2^{-n}$ and overshoot the known prefix, so the halting set restricted to length $\le n$ is decided — letting $n \to \infty$, $\emptyset' \le_T \Omega$. For $\Omega \le_T \emptyset'$: $\Omega$ is left-c.e., so its left cut $\{q \in \mathbb Q : q < \Omega\}$ is c.e., hence $\Omega \le_T \emptyset'$ (the $n$-th bit is decided by a $\emptyset'$ query asking whether the approximation stabilises). So $\Omega \equiv_T \emptyset'$. $\square$

**Proposition 5 (a left-c.e. ML-random real is Turing-complete).** *Every $\Omega$-number — every left-c.e. ML-random real — is Turing-equivalent to $\emptyset'$.*

*Proof.* Let $\alpha$ be left-c.e. and ML-random, with computable increasing rational approximation $\alpha_s \nearrow \alpha$. Always $\alpha \le_T \emptyset'$ since the left cut is c.e. For completeness, suppose toward contradiction $\alpha \not\ge_T \emptyset'$. The key lemma (Kučera-Slaman) is that a left-c.e. real $\alpha$ that does not compute $\emptyset'$ has *slow* convergence in a way an ML-test can exploit: the "amount of approximation still to come past stage $s$", $\alpha - \alpha_s$, is then $\Pi^0_1$-approximable finely enough that the intervals $(\alpha_s, \alpha_s + 2^{-n})$ capturing $\alpha$ form a uniformly c.e. sequence of measure $\le 2^{-n}$ trapping $\alpha$'s binary expansion — an ML-test $\alpha$ fails, contradicting ML-randomness. Hence $\alpha \ge_T \emptyset'$, so $\alpha \equiv_T \emptyset'$. (The full argument uses the Solovay reducibility of c.e. reals: $\alpha$ ML-random and left-c.e. is Solovay-complete, and Solovay completeness implies Turing completeness.) $\square$

**Proposition 6 ($K$-lows are closed downward and $\Delta^0_2$).** *Every $K$-low set is $\Delta^0_2$, the $K$-lows are closed under $\oplus$, and they are downward closed under $\le_T$ within the $K$-lows via lowness for $K$.*

*Proof (skeleton).* A $K$-low $A$ satisfies $K(A{\restriction}n) \le K(n) + b$ for a constant $b$. By the Nies-Hirschfeldt-Downey characterisation (Theorem 2) $A$ is low for $K$, so $K^A = K + O(1)$; the witnessing constant is computed from $b$. The number of sets $A$ with a fixed $K$-lowness constant $b$ is finite at each length up to a $\Pi^0_1$-bounded counting (the *decanter/golden-run* argument bounds how many $n$-length strings can be $K$-low with constant $b$), which places $A \le_T \emptyset'$, hence $A \in \Delta^0_2$. Closure under $\oplus$: if $A, B$ are $K$-low with constants $b_A, b_B$ then $K((A\oplus B){\restriction}2n) \le K(A{\restriction}n) + K(B{\restriction}n) + O(1) \le K(n) + O(1) = K(2n) + O(1)$ by subadditivity of $K$ and $K(n) = K(2n) + O(1)$, so $A \oplus B$ is $K$-low. Downward closure within the class is immediate from low-for-$K$ being closed under $\le_T$. So the $K$-lows form a $\Sigma^0_3$ ideal in the Turing degrees, below $\emptyset'$. $\square$

**Proposition 7 (Schnorr randomness is strictly weaker than ML-randomness).** *There is a Schnorr-random real that is not ML-random.*

*Proof (skeleton).* A **Schnorr test** is an ML-test $(U_n)$ with $\mu(U_n)$ computable as a function of $n$ (not merely $\le 2^{-n}$); $X$ is Schnorr-random if it passes every Schnorr test. Every ML-random real is Schnorr-random (fewer tests to pass). For the strict separation, the universal ML-test $(\widehat U_n)$ has measure only *upper* semicomputable, not computable, so it is not a Schnorr test; one constructs a real $X$ failing $(\widehat U_n)$ — hence not ML-random — while diagonalising against every Schnorr test, using that the computable-measure constraint lets one predict the test's measure and steer $X$ out of each $\bigcap_n V_n$ for Schnorr $(V_n)$. The construction is a finite-extension argument relative to the (computable) measure approximations, in the spirit of the Kleene-Post construction of [42.04.04]. Hence $X$ is Schnorr-random but not ML-random, separating the two notions. $\square$

## Connections [Master]

- Turing reducibility, oracles, and the jump [42.04.04], the prerequisite, supplies the relativised machinery on which the whole degree theory of randomness rests: the oracle functionals that define relativised complexity $K^A$ and relativised randomness, the jump that makes $\Omega \equiv_T \emptyset'$ and indexes the $n$-randomness ladder, and the join $\oplus$ that van Lambalgen's theorem analyses. That unit owns the jump as a measure of computational difficulty; this unit shows the same jump is a source of randomness, with $\Omega$ the halting probability of $\emptyset'$ made into a single ML-random real.

- The Turing degrees and the priority method [42.04.06] is the structural mirror of this unit's degree theory of randomness. Where the priority method populates the interval $(\mathbf 0, \mathbf 0')$ with intermediate c.e. degrees by finite injury, the $K$-low reals populate it with the least random sets, forming a $\Sigma^0_3$ c.e.-presentable ideal of low, non-cuppable degrees built by the cost-function method — a construction technique kindred to priority. The Nies-Hirschfeldt-Downey coincidence theorem is the randomness-theoretic analogue of the Friedberg-Muchnik landmark, and both live among the c.e. degrees that unit charts.

- The arithmetical hierarchy and Post's theorem [42.04.05] calibrates the randomness hierarchy: $n$-randomness is ML-randomness relative to $\emptyset^{(n-1)}$, weak-2-randomness uses $\Pi^0_2$ null sets, and ML-tests are $\Sigma^0_1$ classes, so the syntactic ladder of that unit is the meter for the strength of a randomness notion. The non-computability of $K$ and $C$ is upper-semicomputability — a $\Sigma^0_1$ approximation from above — placing the complexity functions precisely in the hierarchy that unit builds.

- The halting problem and recursion theorem [42.04.02] is the source of $\Omega$ and of the Berry-paradox non-computability of $C$: the halting probability is the measure of the halting set in program space, the first $n$ bits of $\Omega$ decide halting for length-$\le n$ programs, and the proof that $C$ is non-computable is a recursion-theoretic diagonalisation against a hypothetical short description of "the least incompressible string." That unit owns the single halting diagonal; this unit weighs the halting set and finds it maximally information-dense.

## Historical & philosophical context [Master]

The subject has three independent origins around 1964-1966. Ray Solomonoff introduced program-size complexity in 1964 as a foundation for inductive inference and a formalisation of Occam's razor, defining a universal prior over hypotheses. Andrei Kolmogorov, in his 1965 paper "Three approaches to the quantitative definition of information," proposed the complexity $C(x)$ as the algorithmic counterpart of Shannon entropy and as a definition of the information content of an individual object, independent of any probability distribution [ref: TODO_REF Li-Vitanyi Ch. 1]. Gregory Chaitin, then a teenager, arrived at program-size complexity independently and pursued its connection to incompleteness and the halting probability. The plain complexity $C$ had a known defect — its failure of subadditivity and the complexity oscillations that block a clean characterisation of randomness — which Leonid Levin and Chaitin resolved in the early 1970s by passing to prefix-free machines and the complexity $K$.

The definition of randomness for infinite sequences was given by Per Martin-Löf in 1966, a student of Kolmogorov, who replaced the failed attempt to define randomness by statistical tests alone with the notion of an effective null set and a universal test [ref: TODO_REF Downey-Hirschfeldt Ch. 6]. The equivalence of randomness with incompressibility — the Schnorr-Levin theorem — was established by Claus-Peter Schnorr and Levin in the early 1970s. Chaitin's $\Omega$ appeared in his 1975 paper on a theory of program size, and the modern degree theory of randomness — van Lambalgen's 1987 theorem, and the $K$-lowness results identifying low-for-random with low-for-$K$ — was developed by Michiel van Lambalgen, André Nies, Denis Hirschfeldt, Rod Downey, Frank Stephan, and Joseph Miller in the decade after 2000, with the Kučera-Slaman characterisation of the $\Omega$-numbers and the Nies-Hirschfeldt-Downey coincidence theorem as its central results.
