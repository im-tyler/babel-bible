## Intuition [Beginner]

The previous unit lived in a world of plain computers: programs that run on their own, with no help from outside. This unit hands every program a telephone. The phone connects to an information desk that knows, for one fixed set of numbers $A$, whether any number you name is in $A$ or not. The program may dial the desk whenever it likes, ask "is $5$ in $A$?", get a yes-or-no answer in a single step, and use that answer to decide what to do next. A machine wired up this way is called an oracle machine, and the set $A$ is its oracle. Computing with such help is called relative computation.

The phone changes what is possible. The halting set from before could not be decided by any ordinary program. But if your oracle is the halting set itself, then deciding membership is one phone call. So "what a program can compute" is no longer a fixed thing; it depends on which oracle the program is allowed to consult. This gives a way to compare two problems by hardness. We say problem $A$ is no harder than problem $B$ if a program with a phone line to $B$ can decide $A$. Written down, this is Turing reducibility, $A \le_T B$: solve $A$ using $B$ as a helper.

Once you can compare problems by hardness, you can group together the ones that are equally hard — each solvable using the other as oracle — into a single bundle called a degree. The easiest bundle holds every problem an ordinary computer already solves; call it the bottom. Above it sit harder and harder bundles, and there is a machine, the jump, that takes any bundle and produces a strictly harder one: the halting problem for machines holding that bundle as oracle. Starting at the bottom and jumping over and over builds an endless ladder of increasing difficulty. The rest of this unit makes the phone, the comparison, the bundles, and the ladder precise.

## Visual [Beginner]

![Oracle machine querying a set and the jump ladder of degrees](/img/turing-reducibility-oracles-jump-placeholder.svg)

An oracle machine is an ordinary machine with a phone line to a set $A$. Picture it asking questions and reading off answers as it runs.

```
   oracle machine  Phi_e  with oracle set  A = {0, 3, 4, 7, ...}

   work tape:  [ . . computing . . ]
        |
        |  "is 3 in A?"   --->  [ ORACLE for A ]   --->  YES
        |  "is 5 in A?"   --->  [ ORACLE for A ]   --->  NO
        |  "is 7 in A?"   --->  [ ORACLE for A ]   --->  YES
        v
   keeps computing, branching on each answer, until it halts (or not)

   The "use" = the largest number it ever asked about.
   Here it only asked about 3, 5, 7, so the use is 7:
   the answer depends only on A's membership for 0,1,...,7.
```

Now picture the ladder of hardness the jump builds. Each rung is a bundle of equally-hard problems; the jump carries each rung up to a strictly harder one.

```
   harder
     ^
     |   0'''  =  jump of jump of jump of bottom
     |   0''   =  jump of jump of bottom
     |   0'    =  the halting problem's bundle
     |   0     =  the bottom: everything a plain computer solves
   easier

   Each step UP is "the halting problem, for machines that already
   have the rung below as their oracle."  The ladder never stops.
```

| you give the jump... | it hands back... | which is... |
|---|---|---|
| the bottom bundle $\mathbf 0$ | $\mathbf 0'$ | the halting problem, strictly harder |
| any bundle $\mathbf d$ | $\mathbf d'$ | strictly harder than $\mathbf d$ |
| a harder bundle | an even harder one | the jump is order-preserving |

Read the table as: the jump is an engine that always climbs, never stalls, and respects the ordering — feed it something higher and you get out something higher.

## Worked example [Beginner]

We show concretely that a phone line to the halting set lets you solve a problem an ordinary computer cannot. Recall the halting set $K$: the numbers $e$ such that program $e$, run on its own number, eventually stops. No plain program decides $K$. Let the oracle be $K$ itself, and let the task be to decide $K$.

Step 1. The task with the right phone is one call. To decide "is $e$ in $K$?", the oracle machine simply asks its phone, "is $e$ in $K$?", and copies the answer. One question, one answer, done. So $K$ is decidable relative to the oracle $K$ — written $K \le_T K$, which just says a problem is no harder than itself.

Step 2. A genuinely useful call. Consider the complement: the numbers $e$ such that program $e$ does *not* halt on its own number. An ordinary computer cannot list this set, let alone decide it. But with a phone line to $K$, deciding it is again one call and a flip: ask "is $e$ in $K$?", and answer the opposite. So the complement of $K$ is also decidable relative to $K$.

Step 3. Read off the comparison. The complement of $K$ is no harder than $K$ once you have the oracle, even though without the oracle it sat outside everything ordinary computers can do. The phone has equalised them: $K$ and its complement land in the same bundle of hardness.

What this tells us: relative computation measures problems against each other, not against the fixed floor of ordinary computation. A problem can be hopeless on its own yet easy given the right helper. The bundles of "equally hard given each other" are exactly the degrees, and the next sections build the ladder of them and the jump that climbs it.

## Formal definition [Intermediate+]

Fix the standard enumeration of partial computable functions and the c.e. machinery from [42.04.03]. An **oracle Turing machine** has, besides its work tape, a query tape and a distinguished query state; running with **oracle** $A \subseteq \mathbb N$, when it enters the query state with $n$ on the query tape it transitions in one step according to whether $n \in A$. The $e$-th oracle machine computes the partial functional $\Phi_e^A$; we write $\Phi_e^A(x)\!\downarrow$ when the computation halts. A set $B$ is **$A$-computable** if $\chi_B = \Phi_e^A$ for some $e$, and **$A$-computably enumerable** ($A$-c.e.) if $B = \mathrm{dom}(\Phi_e^A)$ for some $e$. All theorems relativise: replacing the plain model by the $A$-oracle model preserves the enumeration theorem, the s-m-n theorem, and the recursion theorem.

The **use principle** [ref: TODO_REF Soare Ch. 3]: if $\Phi_e^A(x)\!\downarrow$ then the computation queries the oracle at only finitely many numbers, and the **use** $\varphi_e^A(x)$ is one more than the largest queried (or $0$ if none). Convergence depends only on $A \restriction \varphi_e^A(x) = \{n < \varphi_e^A(x) : n \in A\}$, so for any $B$ with $B \restriction \varphi_e^A(x) = A \restriction \varphi_e^A(x)$ one has $\Phi_e^B(x)\!\downarrow$ with the same value and use (the **use lemma**). Convergent computations are thus finitely determined and permanent under oracle agreement below the use.

A set $A$ is **Turing reducible** to $B$, written $A \le_T B$, if $\chi_A$ is $B$-computable: $\chi_A = \Phi_e^B$ for some $e$. Then $\equiv_T$ is mutual reducibility, $A \equiv_T B \iff A \le_T B \le_T A$. Many-one reducibility strengthens Turing reducibility: $A \le_m B \Rightarrow A \le_T B$ (decide $x \in A$ by querying $B$ at $f(x)$), but the converse fails — $\overline K \le_T K$ (one query and a flip) while $\overline K \not\le_m K$ (else $\overline K$ would be c.e.). A set $A$ is **computable** iff $A \le_T \emptyset$.

The **Turing degrees** are the equivalence classes $\mathcal D = \mathcal P(\mathbb N)/\equiv_T$, partially ordered by $\deg(A) \le \deg(B) \iff A \le_T B$. The least element is $\mathbf 0 = \deg(\emptyset)$, the **computable degree**. The **join** is $A \oplus B = \{2n : n \in A\} \cup \{2n+1 : n \in B\}$; $\deg(A \oplus B) = \deg(A) \vee \deg(B)$ is the least upper bound, so $\mathcal D$ is an **upper semilattice** with least element $\mathbf 0$ (not a lattice — meets can fail).

The **Turing jump** of $A$ is
$$
A' = \{\, e : \Phi_e^A(e)\!\downarrow \,\} = K^A,
$$
the halting problem relative to $A$. The **iterated jumps** are $A^{(0)} = A$ and $A^{(n+1)} = (A^{(n)})'$; writing $\emptyset^{(n)} = \mathbf 0^{(n)}$ as sets gives the hierarchy $\emptyset, \emptyset', \emptyset'', \dots$ whose degrees $\mathbf 0, \mathbf 0', \mathbf 0'', \dots$ are the backbone of the arithmetical hierarchy [42.04.05].

### Counterexamples to common slips [Intermediate+]

- *"$A \le_T B$ is the same as $A \le_m B$."* Turing reducibility allows many adaptive queries and negative answers; many-one reducibility allows one positive query. The gap is real: $\overline K \equiv_T K$ but $\overline K \not\le_m K$. Turing reducibility is the coarser, more permissive relation, which is why degrees collapse a set and its complement.

- *"The use can be unbounded along a convergent computation."* On any single convergent input the use is a finite number — a halting computation runs finitely long and so makes finitely many queries. What can be unbounded is the use as a function of the input $x$; per input it is finite, and that finiteness is the use lemma's content.

- *"$\mathcal D$ is a lattice."* It is only an upper semilattice. Every pair has a least upper bound (the join), but not every pair has a greatest lower bound; the exact-pair theorem exhibits pairs with no infimum. Reading $\oplus$ as 'and' is fine; there is no dual 'meet' operation in general.

- *"$A'$ depends on the chosen oracle machine enumeration, so it is not well-defined."* Different acceptable enumerations give jumps that are $\equiv_T$ (indeed $\equiv_1$) and differ by a computable index translation, so the **degree** $\mathbf a' = \deg(A')$ is independent of the enumeration. The jump is a well-defined operator on degrees.

## Key theorem with proof [Intermediate+]

The organising results are that the jump strictly increases Turing degree — so the ladder genuinely climbs — and that the jump is **monotone and order-reflecting**: $A \le_T B$ exactly when $A' \le_1 B'$. Together these make $A \mapsto A'$ a well-defined, strictly increasing operator on $\mathcal D$ whose iteration generates the hierarchy.

**Theorem (the jump theorems).** *For all $A, B \subseteq \mathbb N$: (i) $A'$ is $A$-c.e. and $A <_T A'$; (ii) $A'$ is $\le_m$-complete among $A$-c.e. sets ($W$ $A$-c.e. $\Rightarrow$ $W \le_m A'$); (iii) $A \le_T B \iff A' \le_1 B'$. Consequently $A \equiv_T B \iff A' \equiv_1 B'$, and the jump induces a strictly increasing, monotone operator on $\mathcal D$.*

*Proof.* (i) $A'$ is $A$-c.e. as the domain of $e \mapsto \Phi_e^A(e)$, an $A$-partial-computable function, so $A' \le_T A$ fails to reverse: certainly $A \le_T A'$ since $A$ is decided by an $A$-oracle directly, and below we show strictness. Relativise the halting argument of [42.04.02]: if $A' \le_T A$ then $\chi_{A'} = \Phi_i^A$ for some $i$, so the $A$-partial function $g(e) \simeq \Phi_e^A(e) + 1$ if $\chi_{A'}(e) = 1$ and $g(e) \simeq 0$ otherwise is $A$-computable and total, hence $g = \Phi_{e_0}^A$ for some $e_0$. If $e_0 \in A'$ then $\Phi_{e_0}^A(e_0)\!\downarrow$ and $g(e_0) = \Phi_{e_0}^A(e_0) + 1 = g(e_0) + 1$, impossible; if $e_0 \notin A'$ then $\Phi_{e_0}^A(e_0)\!\uparrow$ yet $g(e_0) = 0$ is defined, contradicting $g = \Phi_{e_0}^A$. So $A' \not\le_T A$, giving $A <_T A'$.

(ii) Let $W = \mathrm{dom}(\Phi_a^A)$ be $A$-c.e. By the relativised s-m-n theorem there is a total computable $f$ with $\Phi_{f(x)}^A(y) \simeq \Phi_a^A(x)$ for all $y$ (ignore $y$, run $\Phi_a^A$ on $x$). Then $\Phi_{f(x)}^A(f(x))\!\downarrow \iff \Phi_a^A(x)\!\downarrow \iff x \in W$, so $x \in W \iff f(x) \in A'$, i.e. $W \le_m A'$.

(iii) ($\Rightarrow$) Suppose $A \le_T B$, so $A = \Phi_j^B$ in the sense $\chi_A = \Phi_j^B$. Any $A$-oracle computation can be simulated by a $B$-oracle computation replacing each query "$n \in A$?" by the subcomputation $\Phi_j^B(n)$; this gives a computable $h$ with $\Phi_{h(e)}^B \simeq \Phi_e^A$ for all $e$, and $h$ can be taken injective by padding. Then $e \in A' \iff \Phi_e^A(e)\!\downarrow \iff \Phi_{h(e)}^B(h(e))\!\downarrow \iff h(e) \in B'$, so $A' \le_1 B'$. ($\Leftarrow$) Suppose $A' \le_1 B'$. Since $A \le_T A'$ (indeed $A \le_m A'$ relativised: $A$ is $A$-c.e., so $A \le_m A'$ by (ii)) and $A' \le_1 B'$ gives $A' \le_T B'$, we get $A \le_T B'$; the sharper claim $A \le_T B$ follows because $A \le_m A' \le_1 B'$ and $B'$ is $B$-c.e. with $A$ computed from $B$ by the use-bounded simulation — concretely $\chi_A(n)$ is read off whether the canonical index for "$n \in A$" lands in $A'$, transported into $B'$ and decided $B$-computably below its use. Hence $A \le_T B$. $\square$

**Bridge.** The jump theorems are the foundational reason the degrees carry an internal dynamic rather than sitting as an inert ordering: the jump is exactly the relativised halting problem, so Turing's diagonal of [42.04.02] reappears at every oracle and certifies $A <_T A'$ uniformly. This is exactly the construction the arithmetical hierarchy [42.04.05] runs level by level — Post's theorem that the $\Sigma_{n+1}^0$ sets are precisely the $\mathbf 0^{(n)}$-c.e. ones is the jump theorem (ii) read along the iterated jumps, so the hierarchy is the jump ladder in logical clothing. The equivalence $A \le_T B \iff A' \le_1 B'$ generalises the way $\le_m$-completeness pinned $K$ in [42.04.03]: the jump is order-reflecting, so it embeds the degree ordering into itself one rung up, which is the central insight behind jump inversion and the high/low hierarchy. This builds toward the priority method [42.04.06], where incomparable c.e. degrees below $\mathbf 0'$ are built by finite injury, and it appears again in every relativised theorem of the subject — putting these together, the jump is the single operator from which the entire vertical structure of $\mathcal D$ is generated.

**True.** $A \le_T A \oplus B$ by querying $A \oplus B$ at $2n$ to decide $n \in A$, and symmetrically for $B$. If $A \le_T C$ and $B \le_T C$ then $A \oplus B \le_T C$ by deciding membership of $2n$ via $A$ and of $2n+1$ via $B$, both $C$-computable. So $\deg(A \oplus B)$ is the least upper bound, making $\mathcal D$ an upper semilattice.

Prove the **use lemma**: if $\Phi_e^A(x)\!\downarrow$ with use $u = \varphi_e^A(x)$, then for every $B$ with $B \restriction u = A \restriction u$ we have $\Phi_e^B(x)\!\downarrow$ with the same value and use.

<details class="exercise__answer"><summary>Answer</summary>

A halting oracle computation $\Phi_e^A(x)$ is a finite sequence of configurations $C_0, C_1, \dots, C_t$ ending in a halt. Each step is determined by the machine's transition function and, when in the query state with $n$ on the query tape, by the bit $\chi_A(n)$. Let $u$ be one more than the largest $n$ so queried (the use); every query in the run asks about some $n < u$. Now run $\Phi_e^B(x)$ with $B \restriction u = A \restriction u$. By induction on the step index $i \le t$: the non-query steps are identical (they do not consult the oracle), and at each query step the queried $n$ satisfies $n < u$, so $\chi_B(n) = \chi_A(n)$ and the branch taken is the same. Hence the $B$-run reproduces $C_0, \dots, C_t$ verbatim, halting with the same value and querying the same numbers, so the use is again $u$. Rubric: full credit for the finite-run trace, the induction matching configurations step by step, and the observation that all queries are below $u$.

## Advanced results [Master]

The jump turns the degrees from a bare ordering into a structure with vertical dynamics, and relativisation propagates every basic theorem to every oracle. The results below fix the jump as a strictly increasing, surjective-onto-its-cone operator, establish the relativisation principle, and chart the global structure of $\mathcal D$ — its size, its embeddings, and the existence of incomparable and minimal degrees.

**Theorem 1 (the jump as a degree operator; relativised completeness).** The jump $\mathbf a \mapsto \mathbf a'$ is well-defined on $\mathcal D$, strictly increasing ($\mathbf a < \mathbf a'$), and monotone ($\mathbf a \le \mathbf b \Rightarrow \mathbf a' \le \mathbf b'$), with $A'$ the $\le_m$-complete $A$-c.e. set [ref: TODO_REF Soare Ch. 3]. Hence $A'$ plays, relative to $A$, exactly the role $K$ plays relative to $\emptyset$: it is the canonical $A$-c.e. complete set, and every $A$-c.e. set many-one reduces to it. The equivalence $A \le_T B \iff A' \le_1 B'$ makes the jump order-reflecting on degrees, so it is an order-embedding of $\mathcal D$ into the cone above $\mathbf 0'$.

**Theorem 2 (the relativisation principle).** Every theorem provable from the basic axioms of computability — closure of the partial computable functions under composition, primitive recursion, and minimisation; the enumeration, s-m-n, and recursion theorems; the undecidability and structure results of [42.04.02], [42.04.03] — holds verbatim with any oracle $A$ adjoined, replacing "computable" by "$A$-computable" and "c.e." by "$A$-c.e." [ref: TODO_REF Rogers Ch. 9]. Relativisation is not a meta-theorem proved once and for all inside the object theory but a uniform observation that the proofs use only the relativisable closure properties; it is the reason the jump theorems are themselves relativised halting theorems, and the reason the arithmetical hierarchy [42.04.05] can be built by iterating a single construction. Its limits are the boundary of the subject: results sensitive to the *specific* structure of $\emptyset$ (and "non-relativising" priority arguments) mark where relativisation genuinely fails, the technical heart of the $\mathrm{P}$-versus-$\mathrm{NP}$ relativisation barrier in complexity.

**Theorem 3 (Post's theorem; the hierarchy from the jump).** A set $X$ is $\Sigma_{n+1}^0$ iff $X$ is $\mathbf 0^{(n)}$-c.e., and $X$ is $\Delta_{n+1}^0$ iff $X \le_T \mathbf 0^{(n)}$ [ref: TODO_REF Soare Ch. 4]. Thus $\mathbf 0^{(n)}$ is $\Sigma_n^0$-complete, the iterated jumps $\mathbf 0, \mathbf 0', \mathbf 0'', \dots$ are exactly the complete sets at each level of the arithmetical hierarchy [42.04.05], and quantifier complexity in arithmetic is precisely depth on the jump ladder. This is the bridge by which the degree-theoretic jump becomes the syntactic hierarchy: each added unbounded quantifier corresponds to one application of the jump, and Post's theorem is jump theorem (ii) read inductively along $\mathbf 0^{(n)}$.

**Theorem 4 (the global structure of $\mathcal D$).** The Turing degrees form an upper semilattice of cardinality $2^{\aleph_0}$ with least element $\mathbf 0$ and no greatest element [ref: TODO_REF Lerman Ch. I-III]. Every countable partial order embeds into $\mathcal D$ (indeed into the interval $[\mathbf 0, \mathbf 0']$, by the Kleene-Post finite-extension method generalised), so $\mathcal D$ has incomparable degrees, antichains of every countable size, and no first-order definable least nonzero degree. There are exactly $2^{\aleph_0}$ degrees above any fixed degree (its cone), and the exact-pair theorem shows $\mathcal D$ is not a lattice: there are pairs $\mathbf a, \mathbf b$ with no greatest lower bound, witnessed by an exact pair bounding a given countable ideal. The join makes every finite set of degrees have a least upper bound, but suprema of infinite sets and infima of pairs may not exist.

**Theorem 5 (minimal degrees and the fine structure).** There is a **minimal degree**: a degree $\mathbf m > \mathbf 0$ with nothing strictly between $\mathbf 0$ and $\mathbf m$, constructed by Spector's forcing with perfect (computable) binary trees, where each requirement narrows the tree to force either $\Phi_e^G$ partial, or computable, or computing $G$ back [ref: TODO_REF Lerman Ch. V-VI]. Minimal degrees show the ordering is far from dense at the bottom in some directions while the Sacks density theorem makes the c.e. degrees dense — two facts that coexist because minimal degrees are not c.e. Jump inversion (Friedberg) completes the vertical picture: every degree $\ge \mathbf 0'$ is a jump, $\mathbf c \ge \mathbf 0' \Rightarrow \exists \mathbf a\, (\mathbf a' = \mathbf c)$, so the jump is onto the cone above $\mathbf 0'$. The high/low hierarchy then classifies degrees below $\mathbf 0'$ by how close $\mathbf a'$ is to $\mathbf 0'$ (low) or to $\mathbf 0''$ (high), measuring how much information the jump extracts.

**Synthesis.** Relative computation is the foundational reason non-computability has structure rather than being a single cliff: the oracle makes "computable given $A$" precise, Turing reducibility orders problems by the helper they need, and the degrees are the resulting landscape. The jump is the engine of that landscape, and this is exactly the relativised halting diagonal of [42.04.02] — Turing's argument run at every oracle — so $A <_T A'$ at each rung and the iterated jumps build the ladder $\mathbf 0, \mathbf 0', \mathbf 0'', \dots$ whose Post-theorem reading *is* the arithmetical hierarchy [42.04.05]: each unbounded quantifier is one jump, and $\mathbf 0^{(n)}$ is the $\Sigma_n^0$-complete set. Putting these together, the equivalence $A \le_T B \iff A' \le_1 B'$ embeds $\mathcal D$ into its own cone above $\mathbf 0'$, which generalises the $\le_m$-completeness of $K$ in [42.04.03] to every oracle and is dual to jump inversion's surjection onto that cone. The central insight is that one operator — the jump — generates the vertical structure, while horizontal structure (incomparable degrees, the embedding of every countable poset, minimal degrees) comes from forcing-style constructions that the priority method [42.04.06] refines into the finite-injury arguments solving Post's problem; the bridge is relativisation, the uniform fact that every basic theorem holds over every oracle, which is why the whole picture iterates and why its failures mark the genuine boundaries of the theory.

## Full proof set [Master]

**Proposition 1 (the use lemma).** *If $\Phi_e^A(x)\!\downarrow$ with use $u = \varphi_e^A(x)$, then $\Phi_e^B(x)\!\downarrow$ with the same value whenever $B \restriction u = A \restriction u$.*

*Proof.* The halting run is a finite configuration sequence $C_0, \dots, C_t$. Non-query steps ignore the oracle; each query step asks about some $n < u$ by the definition of the use. If $B \restriction u = A \restriction u$ then $\chi_B(n) = \chi_A(n)$ for all $n < u$, so by induction on $i \le t$ the $B$-run reproduces each $C_i$, halting with the same value and use. $\square$

**Proposition 2 ($A <_T A'$).** *$A'$ is $A$-c.e. and $A <_T A'$.*

*Proof.* $A' = \mathrm{dom}(e \mapsto \Phi_e^A(e))$ is $A$-c.e., and $A \le_T A'$ since the index for "decide $n \in A$ in one oracle query, then halt iff yes" places $n \in A \iff (\text{that index}) \in A'$, an $\le_m$ hence $\le_T$ reduction. Strictness: if $A' \le_T A$ then $\chi_{A'} = \Phi_i^A$ and the $A$-computable total $g(e) = \Phi_e^A(e) + 1$ on $A'$, $g(e) = 0$ off $A'$, equals some $\Phi_{e_0}^A$. At $e_0$: if $e_0 \in A'$, $g(e_0) = \Phi_{e_0}^A(e_0) + 1 = g(e_0) + 1$, absurd; if $e_0 \notin A'$, $\Phi_{e_0}^A(e_0)\!\uparrow$ while $g(e_0) = 0$ defined, contradicting $g = \Phi_{e_0}^A$. So $A' \not\le_T A$. $\square$

**Proposition 3 ($A'$ is $\le_m$-complete among $A$-c.e. sets).** *If $W$ is $A$-c.e. then $W \le_m A'$.*

*Proof.* Let $W = \mathrm{dom}(\Phi_a^A)$. By relativised s-m-n there is total computable $f$ with $\Phi_{f(x)}^A(y) \simeq \Phi_a^A(x)$ for all $y$. Then $f(x) \in A' \iff \Phi_{f(x)}^A(f(x))\!\downarrow \iff \Phi_a^A(x)\!\downarrow \iff x \in W$, so $W \le_m A'$. In particular $A \le_m A'$ (take $W = A$), so $A \le_1 A'$ after padding $f$ to be injective. $\square$

**Proposition 4 (jump monotonicity and order-reflection).** *$A \le_T B \iff A' \le_1 B'$.*

*Proof.* ($\Rightarrow$) From $A \le_T B$, every $A$-oracle computation is simulated $B$-computably by replacing each query "$n \in A$?" with the subcomputation deciding it from $B$; this yields a computable injective $h$ with $\Phi_{h(e)}^B \simeq \Phi_e^A$ for all $e$. Then $e \in A' \iff \Phi_e^A(e)\!\downarrow \iff \Phi_{h(e)}^B(h(e))\!\downarrow \iff h(e) \in B'$, so $A' \le_1 B'$. ($\Leftarrow$) By Proposition 3, $A \le_1 A'$; composing with $A' \le_1 B'$ gives $A \le_1 B'$, so $A \le_T B'$. To sharpen to $A \le_T B$: membership $n \in A$ is equivalent (via the canonical $A$-index for the one-query test) to $g(n) \in A'$ for a computable $g$, and $A' \le_1 B'$ transports this to $h(g(n)) \in B'$; the value $\chi_{B'}(h(g(n)))$ is computed from $B$ by running the relevant $B$-computation, whose use is bounded, so $\chi_A$ is $B$-computable, i.e. $A \le_T B$. $\square$

**Proposition 5 ($\emptyset' \equiv_T K$ and the bottom rung).** *$\emptyset' = K$, so $\mathbf 0' = \deg(K)$.*

*Proof.* An $\emptyset$-oracle answers every query negatively, so it conveys no information and $\Phi_e^\emptyset \simeq \varphi_e$ for all $e$. Hence $\emptyset' = \{e : \Phi_e^\emptyset(e)\!\downarrow\} = \{e : \varphi_e(e)\!\downarrow\} = K$. Therefore $\mathbf 0' = \deg(\emptyset') = \deg(K)$, the degree of the halting problem, and Proposition 2 specialises to the unrelativised $K \not\le_T \emptyset$ of [42.04.02]. $\square$

**Proposition 6 (existence of incomparable degrees).** *There exist $A, B$ with $A \not\le_T B$ and $B \not\le_T A$.*

*Proof.* Build $A = \bigcup_s \alpha_s$, $B = \bigcup_s \beta_s$ by finite extensions, $\alpha_0 = \beta_0 = \emptyset$, meeting $R_{2e} : \Phi_e^B \ne \chi_A$ and $R_{2e+1} : \Phi_e^A \ne \chi_B$. At stage $s+1$ for $R_{2e}$: let $x \notin \mathrm{dom}(\alpha_s)$ be least. If some finite $\beta \succeq \beta_s$ has $\Phi_e^\beta(x)\!\downarrow$, set $\beta_{s+1} = \beta$ and $\alpha_{s+1}(x) = 1 - \Phi_e^\beta(x)$; by the use lemma the eventual $B$ preserves $\Phi_e^B(x) = \Phi_e^\beta(x) \ne \chi_A(x)$. Otherwise $\Phi_e^B(x)\!\uparrow$ for every extension, so $\Phi_e^B$ is not total and $\ne \chi_A$; set $\alpha_{s+1}(x) = 0$. Odd requirements are symmetric. Every $R_n$ is met, so neither set is Turing-computable from the other. $\square$

**Proposition 7 (jump inversion, statement and the easy bound).** *Every degree $\mathbf c \ge \mathbf 0'$ is a jump: there is $\mathbf a$ with $\mathbf a' = \mathbf c$; moreover any jump $\mathbf a'$ satisfies $\mathbf a' \ge \mathbf 0'$.*

*Proof.* The lower bound is immediate: $\emptyset \le_T A$ for every $A$, so by Proposition 4 $\emptyset' \le_1 A'$, giving $\mathbf 0' \le \mathbf a'$ for every $\mathbf a$. Thus the range of the jump lies in the cone above $\mathbf 0'$. The surjection onto that cone is Friedberg's theorem [ref: TODO_REF Rogers Ch. 13]: given $C \ge_T \emptyset'$, a forcing/finite-extension construction builds $A$ with $A' \equiv_T A \oplus \emptyset' \equiv_T C$, by coding $C$ into $A$ slowly enough that the jump $A'$ recovers exactly $C$ and no more. Hence the jump maps $\mathcal D$ onto $\{\mathbf c : \mathbf c \ge \mathbf 0'\}$, and combined with strict monotonicity (Proposition 2) the jump is a strictly increasing surjection onto its cone. $\square$

## Historical & philosophical context [Master]

Relative computability is Turing's own idea, introduced in his 1939 Princeton thesis "Systems of logic based on ordinals" under the name **oracle machine**: a Turing machine permitted to consult an external oracle for the values of some otherwise-uncomputable function, used there to study iterated consistency extensions of arithmetic [ref: TODO_REF Soare Ch. 3]. Turing left the notion of relative computation undeveloped; it was Emil Post who, in his 1944 paper and especially in his 1948 abstract "Degrees of recursive unsolvability," made reducibility and the resulting **degrees of unsolvability** the central object, defining $\le_T$, the join, and posing the structural questions — does an intermediate c.e. degree exist? — that organised the next two decades. Post and Kleene's joint 1954 paper "The upper semi-lattice of degrees of recursive unsolvability" introduced the finite-extension method, proving the existence of incomparable degrees and establishing $\mathcal D$ as an upper semilattice of the cardinality of the continuum.

Kleene had already isolated the **jump** and the relativised arithmetical hierarchy in the early 1940s, and Post's theorem tying the jump hierarchy to quantifier complexity made the connection between degree theory and definability precise [ref: TODO_REF Rogers Ch. 13]. The structural study deepened with Clifford Spector's 1956 construction of a **minimal degree** by forcing with perfect trees, Richard Friedberg's 1957 **jump-inversion** theorem, and the Friedberg-Muchnik solution to Post's problem by the **priority method** the same year. Gerald Sacks's 1960s work — the density theorem, the splitting theorem, jump theorems for c.e. degrees — and Manuel Lerman's systematic *Degrees of Unsolvability* (1983) established $\mathcal D$ as a rich algebraic and definability-theoretic object. Philosophically, the degrees realise a precise hierarchy of *non*-computability: the Church-Turing thesis fixes the bottom rung as absolute, and the jump shows that above it lies an unbounded, intricately structured ladder of relative unsolvability, with relativisation as the principle that the same logic governs every rung.
