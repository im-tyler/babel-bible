## Intuition [Beginner]

Imagine you are packing a suitcase. You use small containers for things you bring rarely and large containers for things you bring often. The Huffman algorithm does this for data: it assigns short codewords to frequent symbols and long codewords to rare ones.

Start with every symbol as its own node, each labelled with its probability. Find the two nodes with the smallest probabilities and merge them into a parent node whose probability is their sum. Repeat until one node remains. The resulting binary tree is the Huffman tree. Reading the left/right branches as 0/1 gives the codewords.

Why is this good? Because the algorithm is greedy but optimal. At every step, the two least probable symbols are siblings in the tree. This guarantees that no other prefix-free code can have a shorter expected length. The proof works by an exchange argument: if any optimal code had those two symbols far apart, you could swap them with deeper nodes and shorten the code, contradicting optimality.

The redundancy bound says the expected length of a Huffman code exceeds the entropy by less than one bit, and in fact by less than $p_{\max} + 0.086$ bits, where $p_{\max}$ is the largest symbol probability. For a uniform source with many symbols, $p_{\max}$ is small and the code is nearly optimal.

|---|---|---|---|---|

## Worked example [Beginner]

Consider English letter frequencies (approximate, top 8 letters plus a catch-all). The probabilities are:

- E: 0.13, T: 0.09, A: 0.08, O: 0.08, I: 0.07, N: 0.07, S: 0.06, H: 0.06, other: 0.36.

**Step 1.** Place all nine symbols in a list with their probabilities.

**Step 2.** Merge the two smallest: S (0.06) and H (0.06) become a node of weight 0.12.

**Step 3.** Merge the next two smallest: I (0.07) and N (0.07) become weight 0.14.

**Step 4.** Now the list is: E (0.13), {S,H} (0.12), T (0.09), A (0.08), O (0.08), {I,N} (0.14), other (0.36). Merge A and O: weight 0.16.

**Step 5.** Merge T (0.09) and {S,H} (0.12): weight 0.21.

**Step 6.** Merge E (0.13) and {I,N} (0.14): weight 0.27.

Continue until one node remains. The expected codeword length is approximately 2.84 bits per letter. The entropy of this distribution is approximately 2.79 bits. The redundancy is about 0.05 bits, well within the bound.

Compare this with a fixed-length code. With 9 symbols, a fixed-length code needs 4 bits per symbol (since $2^3 = 8 < 9 \leq 16 = 2^4$). The Huffman code saves about 1.16 bits per symbol, a 29% reduction. Over a million characters of English text, the Huffman code would save roughly 1.16 million bits, or about 145 kilobytes. This is why Huffman coding is the backbone of file compression.

## Formal definition [Intermediate+]

Let $\mathcal{X} = \{x_1, \dots, x_m\}$ be a finite source alphabet with probability distribution $p = (p_1, \dots, p_m)$ where $p_i \geq 0$ and $\sum p_i = 1$.

**Definition (Prefix-free code).** A code $C: \mathcal{X} \to \{0,1\}^*$ is *prefix-free* if no codeword is a prefix of another. The set of codewords $\{C(x) : x \in \mathcal{X}\}$ forms a prefix-free set.

**Definition (Expected length).** The expected length of a code $C$ for source $p$ is

$$
L(C) = \sum_{i=1}^{m} p_i \, l_i,
$$

where $l_i = |C(x_i)|$ is the length of the codeword for $x_i$.

**Definition (Huffman code).** The Huffman code is constructed by the following algorithm:

1. Create a leaf node for each symbol $x_i$ with weight $p_i$.
2. While more than one node remains:
   a. Select the two nodes with the smallest weights.
   b. Create a new internal node with weight equal to the sum of the two selected nodes' weights.
   c. Make the two selected nodes children of the new node (assign 0 to one branch, 1 to the other).
3. The remaining node is the root of the Huffman tree.

The codeword for each symbol is the sequence of branch labels on the path from root to leaf.

### Counterexamples to common slips

- **The Huffman code is not unique.** Different tie-breaking rules produce different trees with different codeword assignments. All such codes have the same expected length (which is optimal).

- **Optimality is within the class of prefix-free codes.** The Huffman code minimises $L(C)$ over all prefix-free codes. It does not minimise over all uniquely decodable codes, but by Kraft's inequality the minimum expected length is the same for both classes.

- **The bound $L - H < 1$ holds for any distribution, but the tighter bound involves $p_{\max}$.** For a highly skewed distribution with $p_{\max} \approx 1$, the redundancy can be close to zero; for a near-uniform distribution, it is also close to zero. The worst case is in between.

### Key theorem with proof [Intermediate+]

**Theorem (Optimality of Huffman codes).** For any discrete source with distribution $p$, the Huffman code achieves the minimum expected codeword length among all prefix-free codes.

*Proof.* The proof proceeds by induction on $m = |\mathcal{X}|$ using an exchange argument.

**Base case ($m = 2$).** With two symbols, the Huffman code assigns 0 and 1, giving expected length 1. Any prefix-free code for two symbols must have at least one codeword of length 1, and the only such code has expected length 1. So the Huffman code is optimal.

**Inductive step.** Assume optimality holds for all sources with $m - 1$ symbols. Let $p$ be a distribution on $m$ symbols, and assume (by relabelling) that $p_1 \leq p_2 \leq \dots \leq p_m$. Let $C^*$ be any optimal prefix-free code for $p$ with codeword lengths $l_1^* \geq l_2^* \geq \dots \geq l_m^*$.

*Claim:* In $C^*$, the two symbols with the smallest probabilities ($x_1$ and $x_2$) can be assumed to be siblings (i.e., their codewords differ only in the last bit, and $l_1^* = l_2^*$).

*Proof of claim:* If $l_1^* > l_2^*$, swap the codeword of $x_1$ with that of any symbol at depth $l_2^*$. Since $p_1 \leq p_2$ and the swap shortens the codeword of $x_1$ while lengthening the codeword of a higher-probability symbol, the expected length decreases or stays the same. Similarly, if $x_1$ is not a sibling of $x_2$ but has some other symbol $x_j$ as a sibling at the same depth, swap $x_2$ and $x_j$ (which does not change expected length since $p_2 \leq p_j$ and they are at the same depth). Hence we may assume $x_1$ and $x_2$ are siblings.

Now merge $x_1$ and $x_2$ into a single meta-symbol $x'$ with probability $p' = p_1 + p_2$. This creates a source with $m - 1$ symbols. By the induction hypothesis, the Huffman code for this reduced source is optimal. The Huffman code for the original source is obtained by expanding $x'$ back into $x_1$ and $x_2$ (appending 0 and 1 respectively), which adds exactly $p' \cdot 1 = p_1 + p_2$ to the expected length. Any optimal code for the original source must also have $x_1$ and $x_2$ as siblings (by the claim), so it similarly corresponds to a code for the reduced source. By the induction hypothesis, no code for the reduced source has shorter expected length than the Huffman code. Therefore no code for the original source has shorter expected length than the Huffman code. $\square$

**Bridge.** The optimality of Huffman codes builds toward the method of types in [46.02.06], where universal codes operate without knowing the source distribution and the redundancy bound here quantifies the price of ignorance. This is exactly the starting point for universal source coding, where the redundancy over entropy becomes the overhead of learning the distribution from data. The exchange argument appears again in [46.03.03] as the structural backbone of Fano's inequality, and the foundational reason entropy governs compression is that the Huffman redundancy vanishes as the alphabet grows. The central insight is that greedy local merging produces a globally optimal code, generalises to $D$-ary alphabets for non-binary channels, and is dual to the sphere-packing argument in [46.03.04] where the geometric structure of codewords determines capacity.

**Exercise (medium, short answer).**

Prove that the Huffman code for a dyadic distribution (one where every $p_i$ is of the form $2^{-k_i}$ for integers $k_i$) achieves expected length exactly equal to the entropy $H(X)$.

<details class="exercise__hint"><summary>Hint</summary>

For a dyadic distribution, the Kraft sum $\sum 2^{-l_i}$ equals 1 with $l_i = -\log_2 p_i$, so setting codeword lengths equal to $-\log_2 p_i$ gives a valid prefix-free code.

</details>

<details class="exercise__answer"><summary>Answer</summary>

For a dyadic distribution, $p_i = 2^{-k_i}$ for positive integers $k_i$. The Kraft sum with $l_i = k_i = -\log_2 p_i$ is $\sum 2^{-k_i} = \sum p_i = 1$. By Kraft's theorem, there exists a prefix-free code with these lengths. The expected length is $\sum p_i k_i = \sum p_i (-\log_2 p_i) = H(X)$. Since $L \geq H(X)$ for any prefix-free code, this code is optimal, and the Huffman code must achieve the same expected length.

</details>

</aside>

**Exercise (medium, numeric).**

A source has probabilities $(0.6, 0.1, 0.1, 0.1, 0.1)$. Compute the Huffman expected length $L$, the entropy $H$, and verify the redundancy bound $L - H < p_{\max}$.

<details class="exercise__answer"><summary>Answer</summary>

Huffman: merge any two of the four 0.1 symbols (weight 0.2). Merge another two (weight 0.2). Merge the two 0.2 nodes (weight 0.4). Merge 0.6 and 0.4.

Codewords: 0.6 $\to$ 0 (length 1), the four 0.1 symbols get lengths 3, 3, 3, 3 (or 3, 3, 4, 4 depending on tie-breaking).

Best expected length: $L = 0.6 \times 1 + 0.4 \times 3 = 1.8$ bits.

$H = -0.6 \log_2 0.6 - 4 \times 0.1 \log_2 0.1 = 0.442 + 0.4 \times 3.322 = 0.442 + 1.329 = 1.771$ bits.

Redundancy: $L - H = 1.8 - 1.771 = 0.029$ bits. Bound: $p_{\max} = 0.6$. Indeed $0.029 < 0.6$.

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

Prove the redundancy bound $L^* - H(X) < p_{\max}$ where $L^*$ is the expected length of the Huffman code and $p_{\max} = \max_i p_i$.

<details class="exercise__hint"><summary>Hint</summary>

Use the Shannon code with lengths $l_i = \lceil -\log_2 p_i \rceil$, which satisfies $L_S - H < 1$. Then show that the Huffman code is at least as good as the Shannon code. Strengthen the bound using the structure of the Huffman tree.

</details>

<details class="exercise__answer"><summary>Answer</summary>

The Shannon code has lengths $l_i = \lceil -\log_2 p_i \rceil$ and satisfies $H(X) \leq L_S < H(X) + 1$. Since the Huffman code is optimal among prefix-free codes, $L^* \leq L_S$, so $L^* - H(X) < 1$.

For the tighter bound: let $p_{\max} = p_1$. Consider the reduced source obtained by merging the two least-probable symbols. By induction on $m$, let $C'$ be the Huffman code for the reduced source with expected length $L'$. Then $L^* = L' + p_1 + p_2$ where $p_1, p_2$ are the merged probabilities. By the induction hypothesis, $L' - H' < p_{\max}'$ where $p_{\max}' = \max(p_1 + p_2, p_3, \dots, p_m)$. Since $H = H' + (p_1 + p_2) H_b(p_1/(p_1+p_2))$ where $H_b \leq 1$, we get $L^* - H = L' - H' + (p_1+p_2)(1 - H_b(\cdot)) \leq p_{\max}'$. Direct analysis of the Shannon code lengths shows the sharper bound $L^* - H < p_{\max}$ (Gallager 1978).

</details>

</aside>

**Exercise (hard, short answer).**

Construct a source distribution for which the Huffman redundancy $L^* - H$ is arbitrarily close to 1. Explain why the bound cannot be improved below 1 without additional assumptions.

<details class="exercise__answer"><summary>Answer</summary>

Consider a source with two symbols of probabilities $(p, 1-p)$ where $p$ is close to $1/2$ but not dyadic. The Huffman code assigns lengths $(1, 1)$ with expected length $L = 1$. A more precise construction: let $m = 2^k + 1$ symbols with probabilities $p_i = 1/m$ for all $i$. The Huffman code has expected length $\lceil \log_2 m \rceil = k+1$ while $H = \log_2 m \approx k$. The redundancy is approximately $1 - \log_2(1 + 2^{-k}) \to 1$ as $k \to \infty$.

</details>

**Exercise (hard, short answer).**

Describe the $D$-ary Huffman algorithm for constructing optimal prefix-free codes over a $D$-symbol alphabet. What modification is needed when $|\mathcal{X}|$ does not have the right residue modulo $D - 1$?

<details class="exercise__answer"><summary>Answer</summary>

The $D$-ary Huffman algorithm merges the $D$ nodes with the smallest probabilities at each step (rather than 2). The tree has branching factor $D$.

The modification: a complete $D$-ary tree on $m$ leaves requires $m \equiv 1 \pmod{D-1}$. When $m \not\equiv 1 \pmod{D-1}$, add $D - 1 - ((m-1) \bmod (D-1))$ dummy symbols with probability 0 before running the algorithm. These dummy symbols will be merged first and produce no codewords, ensuring the algorithm terminates with a valid $D$-ary prefix-free code.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Show that the Huffman code for the geometric distribution $p_i = (1-p)p^{i-1}$ for $i = 1, 2, 3, \dots$ (truncated to $m$ symbols) has codeword lengths that are non-decreasing in $i$.

<details class="exercise__answer"><summary>Answer</summary>

The geometric distribution has $p_1 \geq p_2 \geq p_3 \geq \dots$. At each step of the Huffman algorithm, the two smallest-probability symbols are merged. These are always the symbols with the largest indices, so they are assigned the longest codewords. The merge creates a meta-symbol with probability $p_{m-1} + p_m$, which is still less than $p_{m-2}$ for the geometric distribution (since $p_m < p_{m-1}$). Therefore, at each subsequent step, the algorithm continues to merge from the tail, and the codeword lengths are non-decreasing: $l_1 \leq l_2 \leq \dots \leq l_m$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

Prove that the Huffman code is the unique optimal prefix-free code (up to relabelling of branch labels) if and only if all probabilities are distinct.

<details class="exercise__answer"><summary>Answer</summary>

If all probabilities are distinct, the two smallest probabilities are unique at each step of the algorithm, so there is no tie-breaking ambiguity. The resulting tree structure is unique (up to swapping 0/1 labels on branches).

If some probabilities are equal, there are at least two symbols with the same probability. At some step, the algorithm must choose among tied probabilities. Different choices produce different tree structures with different codeword assignments (though the same optimal expected length). For example, with probabilities $(1/3, 1/3, 1/3)$, the first merge can pair any two of the three symbols, giving three structurally different trees.

</details>

## Advanced results [Master]

### The sibling property and code characterisation

Gallager (1978) introduced the *sibling property*: a binary prefix code on $m$ symbols is a Huffman code if and only if the corresponding code tree has the property that its $2m - 1$ nodes (internal nodes and leaves combined) can be listed in order of non-increasing probability so that each pair of siblings is adjacent in the list.

This characterisation provides an efficient test for whether a given prefix-free code is a Huffman code without re-running the algorithm: compute the node probabilities and check the sibling property. It also gives a clean proof that the Huffman code is optimal, since any non-Huffman code violates the sibling property, and the violation identifies a local swap that reduces expected length.

### The Shannon code and its competitive optimality

The Shannon code assigns length $l_i = \lceil -\log_2 p_i \rceil$ to symbol $x_i$. By construction, $H(X) \leq L_S < H(X) + 1$. The Huffman code $L^*$ satisfies $L^* \leq L_S$, so $L^* - H(X) < 1$.

Cover and Thomas (2006, §5.5) prove a stronger result: the Shannon code is *competitively optimal* in the sense that for any other prefix-free code $C'$ with lengths $l_i'$,

$$
\Pr[l_S(X) < l'(X)] \geq \Pr[l_S(X) > l'(X)].
$$

The Huffman code inherits this property. This means the Huffman code is not just optimal on average but also dominates any competitor symbol-by-symbol in a probabilistic sense.

### Tight redundancy bounds

The general redundancy bound is $L^* - H(X) < p_{\max} + \log_2(e/(e+1)) \approx p_{\max} + 0.0861$, due to Gallager (1978). A sharper result by Capocelli, De Santis, and Gargano (1991) shows:

$$
L^* - H(X) \leq \sigma - \frac{H(X) + \sigma}{1 + \ln 2 \cdot p_{\max}^{-1}},
$$

where $\sigma = 1 - H_b(p_{\max})$ and $H_b$ is the binary entropy. For the special case $p_{\max} \geq 1/2$, the bound simplifies to $L^* - H(X) < p_{\max}$.

The redundancy is zero exactly when the distribution is dyadic (all probabilities are negative powers of 2). This is the only case where the Shannon code lengths $-\log_2 p_i$ are all integers, and the Huffman code achieves exact entropy.

### Adaptive and canonical Huffman codes

In practice, the source distribution is often unknown. Adaptive Huffman coding (the FGK algorithm of Faller, Gallager, and Knuth) updates the Huffman tree incrementally as symbols are observed, maintaining the sibling property at each step. The redundancy of adaptive Huffman coding over a stationary source converges to the static redundancy as $n \to \infty$.

Canonical Huffman codes assign codewords in a fixed order (e.g., the first codeword of length $l$ is the binary increment of the last codeword of length $l-1$), which allows compact representation of the codebook: only the codeword lengths need to be transmitted, and the decoder reconstructs the canonical code from these lengths.

**Synthesis.** The Huffman algorithm is the unique greedy procedure that produces an optimal prefix-free code; the exchange argument at its core builds toward the method of types in [46.02.06] where universal codes achieve entropy without knowing the distribution. The redundancy bound $L^* - H < p_{\max} + 0.086$ quantifies the gap between practice and theory, and the sibling property characterises Huffman codes structurally. The central insight is that local greedy merging guarantees global optimality, generalises to $D$-ary alphabets and adaptive settings, and is dual to the Kraft inequality that constrains all prefix-free codes. Putting these together, the Huffman code is not merely a good heuristic but an exact solution to the optimal coding problem, with the redundancy bound characterising how far any practical code can deviate from the entropy limit.

## Full proof set [Master]

**Proposition (Redundancy bound via Shannon code).** Let $C^*$ be the Huffman code and $C_S$ be the Shannon code for distribution $p$. Then $L(C^*) \leq L(C_S) < H(X) + 1$.

*Proof.* The Shannon code assigns lengths $l_i = \lceil -\log_2 p_i \rceil$. Since $-\log_2 p_i \leq \lceil -\log_2 p_i \rceil < -\log_2 p_i + 1$, we have $p_i l_i < p_i(-\log_2 p_i + 1)$ for each $i$. Summing: $L(C_S) = \sum p_i l_i < \sum p_i(-\log_2 p_i) + \sum p_i = H(X) + 1$.

Since the Huffman code minimises expected length over all prefix-free codes, $L(C^*) \leq L(C_S) < H(X) + 1$. $\square$

**Proposition (Huffman redundancy for dyadic distributions).** If the distribution $p$ is dyadic (each $p_i = 2^{-k_i}$ for positive integers $k_i$), then $L(C^*) = H(X)$.

*Proof.* For a dyadic distribution, the Kraft sum with $l_i = k_i = -\log_2 p_i$ is $\sum 2^{-k_i} = \sum p_i = 1$, so by Kraft's theorem there exists a prefix-free code with these lengths. The expected length is $\sum p_i k_i = H(X)$. Since $L \geq H(X)$ for any code (by the noiseless coding theorem, which follows from $L \geq H$ via Kraft's inequality and the log-sum inequality), this code achieves the lower bound. The Huffman code, being optimal, must also achieve $L(C^*) = H(X)$. $\square$

## Historical & philosophical context [Master]

David Huffman was a graduate student at MIT when he invented the algorithm in 1951. His information theory professor, Robert Fano, assigned the class a term paper on optimal prefix-free codes. Huffman, unable to prove anything about the existing top-down approach (Shannon-Fano coding), discarded it and invented the bottom-up merge algorithm in a flash of insight. The result was published in the Proceedings of the IRE in 1952 as "A Method for the Construction of Minimum-Redundancy Codes" (40(9), pp. 1098-1101). Fano is said to have remarked that Huffman had solved the problem that Fano and Shannon had both failed to crack.

The algorithm became one of the most widely deployed compression techniques in computing. It forms the backbone of JPEG image compression (after DCT and quantisation), MP3 audio compression, and the DEFLATE algorithm used in gzip and PNG. Canonical Huffman codes were adopted by the deflate format to minimise header overhead.

Robert Gallager's 1978 paper "Variations on a Theme by Huffman" (IEEE Trans. Info. Theory 24(6), pp. 668-674) deepened the theoretical understanding by introducing the sibling property and proving the tighter redundancy bounds. The adaptive FGK algorithm (Faller 1973, Gallager 1978, Knuth 1985) extended Huffman coding to the online setting where the source distribution is learned on the fly.
