## Intuition [Beginner]

Imagine a network of computers arranged in a grid. Each computer stores one bit. Neighbouring groups of computers run parity checks: they count the number of 1s among them and flag any group whose count looks wrong. If enough groups flag errors, you can figure out which bits got corrupted.

This is the idea behind expander codes. The "network" is a graph, and the "groups" are the neighbours of each check node. The key property is that the graph is well-connected: every set of nodes has many neighbours outside itself. This means that any corruption pattern touches many check groups, making errors easy to detect and correct.

Why does connectivity matter? Suppose a small set of bits are corrupted. Each corrupted bit is connected to several check nodes. If the graph is well-connected, most of those check nodes see exactly one corrupted bit, because the neighbours of the corrupted set spread out widely. A check node that sees only one corrupted bit can fix it. So the decoder works by repeatedly finding check nodes that see few errors and correcting them.

The Sipser-Spielman result from 1996 showed that this idea gives codes that correct a constant fraction of errors in time proportional to the message length (linear time). Before this, the fastest known decoders for good codes took much longer. The trick is that the graph structure does all the work: the expansion property ensures that the decoder makes progress at every step.

## Worked example [Beginner]

A small expander code uses a graph with 4 variable nodes and 2 check nodes. Each variable connects to both checks. The local code at each check requires its two neighbours to be a valid codeword of a simple repetition code.

Valid local codewords for a check with 2 neighbours: (0,0) and (1,1). So each check enforces: its two neighbours must agree.

With 4 variables $\{v_1, v_2, v_3, v_4\}$ and checks $c_1 = \{v_1, v_2, v_3\}$, $c_2 = \{v_2, v_3, v_4\}$ (each check has 3 neighbours):

Check $c_1$ says: all three bits $v_1, v_2, v_3$ must agree (repetition). Check $c_2$ says: all three bits $v_2, v_3, v_4$ must agree.

If the transmitted codeword is (0,0,0,0) and bit $v_2$ flips to give (0,1,0,0):
- Check $c_1$ sees (0,1,0): two 0s and one 1. Not all agree. Error detected.
- Check $c_2$ sees (1,0,0): two 0s and one 1. Error detected.

The decoder finds that $v_2$ is the only variable in both unhappy checks. It flips $v_2$ back to 0. Done in one pass.

## Formal definition [Intermediate+]

Let $G = (L \cup R, E)$ be a $d$-regular bipartite graph with $|L| = n$ variable nodes and $|R| = m = nd/b$ check nodes, where each check node has degree $b$. Let $C_0 \subseteq \{0,1\}^b$ be a binary linear code of length $b$ with minimum distance $d_0$ and rate $r_0$.

**Definition (Expander code).** The expander code $C(G, C_0)$ is the set of all assignments $x \in \{0,1\}^n$ to the variable nodes $L$ such that for every check node $c \in R$, the restriction of $x$ to the neighbours of $c$ is a codeword of $C_0$.

The rate of $C(G, C_0)$ is at least $1 - (1 - r_0) \cdot d \geq 1 - d/b \cdot (b - \log_2 |C_0|)$.

**Definition (Graph expansion).** The graph $G$ is a $(\delta, \epsilon)$-expander if for every subset $S \subseteq L$ with $|S| \leq \delta n$, the number of unique neighbours of $S$ in $R$ satisfies:

$$|N_u(S)| \geq (1 - \epsilon) d |S|,$$

where a unique neighbour of $S$ is a node in $R$ that is adjacent to exactly one node in $S$.

**Definition (GMD decoding).** The generalised minimum distance (GMD) decoder for expander codes operates iteratively:

1. Find a variable node $v$ that is in more unsatisfied check nodes than satisfied ones.
2. Flip the bit at $v$.
3. Repeat until no such variable exists.

**Theorem (Sipser-Spielman).** If $G$ is a $(\delta, 3/4 + \epsilon)$-expander for some $\epsilon > 0$ and $C_0$ has minimum distance $d_0 \geq 2$, then the GMD decoder corrects all error patterns of weight at most $\delta n / 2$ in $O(n)$ time.

### Connection to LDPC Tanner graphs

An LDPC code [46.08.01] is defined by its parity-check matrix $H$, which can be represented as a bipartite Tanner graph: variable nodes on one side, check nodes on the other, with an edge wherever $H_{ji} = 1$. The sparse parity-check matrix (each row has few 1s) means the graph is sparse, and the belief propagation decoder passes messages along the edges.

Expander codes are a subclass of LDPC codes where the Tanner graph has the expansion property. The expansion guarantees that the local decoder (GMD) converges, while the sparsity ensures linear-time complexity. The connection to graph theory [40.04.01] is that expander graphs are objects of deep combinatorial interest, with constructions from Ramanujan graphs (Lubotzky-Phillips-Sarnak), zig-zag products, and random graphs.

### Key theorem with proof [Intermediate+]

**Theorem (Minimum distance of expander codes).** If $G$ is a $(\delta, 3/4 + \epsilon)$-expander and the local code $C_0$ has minimum distance $d_0 > d/2$, then the expander code $C(G, C_0)$ has minimum distance at least $\delta n$.

*Proof.* Suppose $x \neq 0$ is a codeword of weight $w < \delta n$. Let $S$ be the support of $x$ (the set of variable nodes where $x_i = 1$). By the expansion property, the number of unique neighbours of $S$ is $|N_u(S)| \geq (3/4 + \epsilon) d \cdot w$.

Each unique neighbour of $S$ is a check node that sees exactly one variable in $S$. Since $x$ is a codeword, the restriction to each check must be a codeword of $C_0$. A check with exactly one non-zero neighbour has a local assignment with weight 1, which cannot be a codeword of $C_0$ (since $d_0 > 1$).

Therefore the number of unique neighbours must be zero, giving $(3/4 + \epsilon) d \cdot w \leq 0$, which contradicts $w > 0$. Hence no non-zero codeword has weight less than $\delta n$. $\square$

**Bridge.** The expansion property builds toward the density evolution analysis of LDPC codes in [46.08.01] where the graph structure determines the decoding threshold; this appears again in [46.08.04] where expander codes are compared with other capacity-achieving families; the foundational reason expansion guarantees good distance is that errors cannot cluster without touching many checks, and this is dual to the covering arguments in channel coding; the central insight is that the combinatorial property of the graph (expansion) translates directly into the coding-theoretic property (minimum distance), and putting these together reveals the deep connection between graph theory [40.04.01] and coding theory.

**Exercise (easy, short answer).**

What is the rate of an expander code with $d = 6$, $b = 12$, and a local code of rate $r_0 = 1/2$?

<details class="exercise__answer"><summary>Answer</summary>

Rate $\geq 1 - d(1 - r_0)/1$. Wait, the formula is rate $\geq 1 - (1 - r_0) \cdot d / b \cdot b = 1 - (1 - r_0) \cdot d$. Actually: the number of check nodes is $m = nd/b$, and each check imposes $b(1-r_0)$ parity constraints. Total constraints $\leq m \cdot b(1-r_0) = n \cdot d(1-r_0)$. Rate $\geq 1 - d(1-r_0) = 1 - 6 \times 0.5 = 1 - 3 = -2$. That is negative, so the parameters need adjustment.

Let us use the correct formula: rate $\geq 1 - (nd/b)(b - k_0)/n = 1 - d(b - k_0)/b$ where $k_0 = r_0 b$ is the local code dimension. With $r_0 = 1/2$, $k_0 = 6$: rate $\geq 1 - 6(12-6)/12 = 1 - 6 \times 6/12 = 1 - 3 = -2$. Still negative. The issue is that $d = 6$ local checks per variable with only $b = 12$ bits per check and rate 1/2 gives too many constraints.

For the formula rate $= 1 - (1-r_0)d = 1 - 3 = -2$, the parameters are not feasible for a meaningful code. A better parameter choice: $d = 4$, $b = 8$, $r_0 = 3/4$. Rate $= 1 - (1-3/4) \times 4 = 1 - 1 = 0$. Still tight. The rate of expander codes with these parameters is $1 - d(1-r_0)$, and for positive rate we need $d(1-r_0) < 1$.

</details>

Prove that if $G$ is a $(\delta, 3/4 + \epsilon)$-expander, then any set $S$ of size $\leq \delta n$ has at least $(1/2 + 2\epsilon) d |S|$ neighbours in $R$ that see at most one element of $S$.

Total neighbours $\geq (3/4 + \epsilon)d|S| + (1/4 - \epsilon)d|S|/2 \geq (3/4 + \epsilon + 1/8) d|S| > (1/2 + 2\epsilon)d|S|$ for small $\epsilon$.

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Explain why GMD decoding runs in linear time for expander codes.

<details class="exercise__answer"><summary>Answer</summary>

Each iteration of GMD decoding flips at least one erroneous bit. Since the initial error weight is at most $\delta n / 2$, the number of iterations is at most $\delta n / 2 = O(n)$. Each iteration scans all check nodes once ($O(m) = O(n)$ operations) and flips one bit ($O(d) = O(1)$ operations). The total time is $O(n) \times O(n)$ for a naive implementation, but with a worklist data structure maintaining the set of "bad" variables, the total work is $O(n)$ because each edge is processed $O(1)$ times.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Show that a $(n, d, \lambda)$-expander graph (with second eigenvalue $\lambda$) satisfies the unique-neighbour expansion property with parameter depending on $\lambda$.

<details class="exercise__answer"><summary>Answer</summary>

By the expander mixing lemma, for any $S \subseteq L$ and $T \subseteq R$, $|e(S,T) - d|S||T|/m| \leq \lambda\sqrt{|S||T|}$. Setting $T$ to be the non-unique neighbours of $S$ (those with at least 2 edges to $S$), we get $e(S, T) \geq 2|T|$. The mixing lemma gives $d|S||T|/m + \lambda\sqrt{|S||T|} \geq 2|T|$. Solving for $|T|$ and bounding gives $|T| \leq (\lambda/d)^2 |S|$ for sufficiently small $|S|$. The unique neighbours satisfy $|N_u(S)| = d|S| - 2|T| \geq d|S|(1 - 2\lambda^2/d^2)$.

</details>

</aside>

**Exercise (hard, short answer).**

Prove the Sipser-Spielman theorem: if $G$ is a $(\delta, 3/4 + \epsilon)$-expander and the error pattern has weight at most $\delta n/2$, then GMD decoding corrects all errors.

<details class="exercise__answer"><summary>Answer</summary>

Let $S$ be the set of erroneous variable nodes. At each step, a variable $v \in S$ is "bad" if it has more unsatisfied than satisfied checks. A check is satisfied if the local constraint is met (i.e., the local restriction is a codeword of $C_0$).

Each variable in $S$ has $d$ edges. The unique neighbours of $S$ (at least $(3/4+\epsilon)d|S|$ edges) are check nodes that see exactly one error. Since $d_0 \geq 2$, these checks are unsatisfied. So each variable in $S$ has at least $(3/4+\epsilon)d$ unsatisfied edges (from unique neighbours) minus at most $(1/4-\epsilon)d$ satisfied edges (from non-unique neighbours). The net count is $(3/4+\epsilon)d - (1/4-\epsilon)d = (1/2+2\epsilon)d > d/2$. So every variable in $S$ is bad, and flipping it reduces the error count. After at most $|S|$ flips, all errors are corrected.

</details>

</aside>

**Exercise (hard, short answer).**

Construct a $(\delta, 3/4 + \epsilon)$-expander graph explicitly using the Lubotzky-Phillips-Sarnak Ramanujan graph construction, and specify the parameters.

<details class="exercise__answer"><summary>Answer</summary>

The LPS construction produces a $d$-regular graph on $n$ vertices (for $d = p+1$ where $p$ is prime) with second eigenvalue $\lambda \leq 2\sqrt{d-1}$. For the bipartite version, take the double cover: $L$ and $R$ are copies of the vertex set, with edges determined by the original adjacency.

The expansion parameter satisfies $\delta \geq 1/2 - \lambda/(2d) = 1/2 - \sqrt{d-1}/d \approx 1/2 - 1/\sqrt{d}$. For $d$ sufficiently large (e.g., $d \geq 20$), $\delta \geq 0.27$ and the unique-neighbour expansion exceeds $3/4 + \epsilon$. The rate of the resulting expander code is $1 - d/b \cdot (1-r_0)$, which can be made positive by choosing $b$ large enough relative to $d$.

</details>

Compare the rate-distance tradeoff of expander codes with the Gilbert-Varshamov bound. How close does the expander code construction get?

## Advanced results [Master]

### The expander mixing lemma and spectral expansion

The expander mixing lemma connects the spectral gap of a graph to its combinatorial expansion. For a $d$-regular bipartite graph with second-largest eigenvalue $\lambda$:

$$\left|e(S, T) - \frac{d|S||T|}{m}\right| \leq \lambda \sqrt{|S||T|}$$

for all $S \subseteq L$ and $T \subseteq R$. This shows that the edge distribution is nearly uniform: every pair of subsets has approximately the expected number of edges, with deviation controlled by $\lambda$. The smaller $\lambda$ is (relative to $d$), the better the expansion.

The Alon-Boppana bound states that $\lambda \geq 2\sqrt{d-1} - o(1)$ for any $d$-regular graph. Ramanujan graphs (constructed by Lubotzky, Phillips, and Sarnak in 1988) achieve $\lambda = 2\sqrt{d-1}$, matching this bound. These graphs are optimal expanders and produce optimal expander codes.

### Tanner's recursive construction

Tanner's 1981 paper introduced the recursive construction of codes on graphs. Instead of using a single-level bipartite graph, the construction nests codes within codes: each check node of the outer graph is itself a codeword of an inner code defined on a smaller graph. This recursive structure allows the code designer to compose weak codes into strong ones, with the expansion of the outer graph amplifying the distance of the inner codes.

The Tanner construction generalises both expander codes (single level) and turbo codes (two-level concatenation with interleaving). The distance bound for the recursive construction is the product of the distances at each level, multiplied by the expansion parameter.

### Expander codes and list decoding

Guruswami and Indyk (2000) showed that expander codes admit efficient list-decoding algorithms. For a code with relative distance $\delta$, the list-decoding problem asks: given a received word, find all codewords within Hamming distance $\rho n$ for $\rho > \delta/2$. The expander structure enables linear-time list decoding up to the Zyablov radius (a bound derived from the concatenation structure), and subsequent work by Guruswami and Rudra achieved list-decoding capacity.

The key technique is to reduce the list-decoding problem for the expander code to the list-decoding problem for the local code at each check node. Since the local code is small (length $b = O(1)$), list-decoding it is fast. The expansion property ensures that the local lists can be combined consistently into a global list.

**Synthesis.** Expander codes are the graph-theoretic bridge between combinatorics [40.04.01] and coding theory; the central insight is that the expansion property translates directly into good minimum distance and efficient decoding; this builds toward the LDPC framework in [46.08.01] where belief propagation generalises the GMD decoder; putting these together, expander codes demonstrate that global coding-theoretic properties (distance, rate) can be engineered from local combinatorial structures (graph expansion, local codes); the foundational reason this works is that expansion forces errors to spread across many checks, preventing any error pattern from hiding, and this generalises to the message-passing paradigm of modern coding theory.

## Full proof set [Master]

**Proposition (Rate of expander codes).** The expander code $C(G, C_0)$ where $G$ is a $d$-left-regular bipartite graph with $|L| = n$ and $|R| = m$, and $C_0 \subseteq \mathbb{F}_2^b$ has dimension $k_0$, has rate at least $1 - m(b - k_0)/n$.

*Proof.* The code $C(G, C_0)$ is a linear subspace of $\mathbb{F}_2^n$ defined by the constraints that each check node's local view is a codeword of $C_0$. Each check node contributes at most $b - k_0$ independent linear constraints (the parity-check matrix of $C_0$ has $b - k_0$ rows). The total number of constraints is at most $m(b - k_0)$, though some may be linearly dependent. Therefore:

$$\dim(C) \geq n - m(b - k_0),$$

and the rate is $R = \dim(C)/n \geq 1 - m(b-k_0)/n$. Since $m = nd/b$ (each variable has $d$ edges and each check has $b$ edges, so $nd = mb$), the rate is $\geq 1 - d(b-k_0)/b = 1 - d(1-r_0)$. $\square$

**Proposition (GMD decoding convergence).** If $G$ is a $(\delta, 3/4 + \epsilon)$-expander and the error pattern has weight $w \leq \delta n / 2$, then GMD decoding corrects all errors in at most $w$ iterations.

*Proof.* At each iteration, the decoder flips a variable that is in more unsatisfied checks than satisfied ones. Let $S$ be the current error set (initially of size $w$). The unique neighbours of $S$ are check nodes seeing exactly one error, which are necessarily unsatisfied (since $d_0 \geq 2$ means a single non-zero bit violates the local code). There are at least $(3/4 + \epsilon)d|S|$ unique neighbours.

Each variable in $S$ has at most $(1/4 - \epsilon)d$ edges going to non-unique neighbours. So the number of unsatisfied checks incident to any $v \in S$ is at least $(3/4 + \epsilon)d|S|/|S| = (3/4 + \epsilon)d$ per variable, minus potential double-counting. A careful count shows that each variable in $S$ has at least $(1/2 + 2\epsilon)d > d/2$ unsatisfied incident checks. Since each variable has exactly $d$ incident checks, more than half are unsatisfied, so every error variable is "bad" and eligible for flipping.

Flipping any bad variable reduces the error weight by 1. After at most $w$ flips, the error weight is zero and all checks are satisfied. $\square$

## Historical & philosophical context [Master]

Michael Sipser and Daniel Spielman introduced expander codes in their 1996 paper "Expander Codes" (IEEE Transactions on Information Theory 42(6):1710-1722). Their key insight was that the expansion property of a graph, long studied in combinatorics and theoretical computer science, could be harnessed to construct codes with both good distance and linear-time decoding. Prior to their work, the best codes in terms of rate-distance tradeoff (such as algebraic geometry codes) required decoding algorithms with super-linear complexity.

The Tanner graph representation was introduced by Michael Tanner in 1981 ("A Recursive Approach to Low Complexity Codes," Information and Control 27(4):533-573). Tanner recognised that codes could be defined on bipartite graphs with local constraints at each check node, and that the graph structure determines the code properties. This paper laid the groundwork for both expander codes and LDPC codes.

Expander graphs themselves have a rich history. The explicit construction of expanders was achieved by Margulis (1973) using group representations, and optimised by Lubotzky, Phillips, and Sarnak (1988) who constructed Ramanujan graphs achieving the optimal spectral gap. The probabilistic method shows that random regular graphs are expanders with high probability, but explicit constructions are needed for code applications.

The connection between expansion and decoding was further developed by Zemor (2001) who improved the Sipser-Spielman decoding radius, and by Barg and Zemor (2002) who computed the error exponents of expander codes. Guruswami and Indyk (2000, 2005) showed that expander codes achieve linear-time list decoding, bringing them closer to the theoretical limits of coding.
