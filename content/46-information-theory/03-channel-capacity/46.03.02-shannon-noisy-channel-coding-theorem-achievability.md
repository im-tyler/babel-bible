## Intuition [Beginner]

You have a noisy channel. You want to send messages reliably. How fast can you go? Shannon answered this in 1948: the maximum reliable rate is the channel capacity $C$, a single number determined entirely by the channel's noise statistics. Any rate below $C$ is achievable. Any rate above $C$ is not.

The surprise is that the capacity is nonzero at all. A channel that flips 10% of your bits seems like it should corrupt any message beyond recovery. But if you use the channel many times and encode cleverly, the errors average out. You can push through almost $C$ bits of information per use, with errors that vanish as you use the channel more and more.

How do you prove such a thing exists? Shannon's trick was random coding. Instead of designing one clever code, imagine picking codewords at random from a giant hat. Each codeword is a sequence of symbols drawn independently from a fixed distribution. You pick $2^{nR}$ such codewords, each of length $n$, for a rate of $R$ bits per channel use.

Then you transmit one codeword through the noisy channel. The receiver gets a corrupted version. To decode, the receiver looks for the codeword that is "jointly typical" with the received sequence — meaning the pair looks like it was generated by the joint input-output distribution of the channel. If the rate is below capacity, there is almost surely only one such codeword, and the decoder recovers the message.

The argument works on average over the random codebook. Most random codebooks are good. The probability that a randomly chosen codebook causes a decoding error goes to zero as the block length grows. Therefore, at least one good codebook exists. You do not know which one, but existence is enough.

## Worked example [Beginner]

Consider a binary symmetric channel (BSC) with crossover probability $p = 0.1$. Its capacity is $C = 1 - H(0.1)$, where $H$ is the binary entropy function.

First compute $H(0.1)$. This is $-0.1 \log_2 0.1 - 0.9 \log_2 0.9$. Working it out: $0.1 \times 3.322 + 0.9 \times 0.152 = 0.332 + 0.137 = 0.469$ bits. So $C = 1 - 0.469 = 0.531$ bits per channel use.

The theorem says: for any rate $R < 0.531$, you can find codes that achieve error probability as small as you like, provided you use the channel enough times. Let us check $R = 0.5$, which is safely below capacity.

With $R = 0.5$ and block length $n = 100$, you have $2^{50}$ codewords. That is about $10^{15}$ codewords — far too many to write down, but the random coding argument says they exist. Each codeword has 100 bits. The channel flips about 10 of them.

The decoder receives a 100-bit sequence with roughly 10 errors. It checks which codeword is jointly typical with the received sequence. A jointly typical pair $(X^n, Y^n)$ satisfies two conditions: $X^n$ is typical for the input distribution, $Y^n$ is typical for the output distribution, and the pair is typical for the joint distribution. For $R = 0.5 < C = 0.531$, there are few enough codewords that the wrong ones are unlikely to be jointly typical with the received word. The error probability vanishes as $n$ grows.

## Formal definition [Intermediate+]

Let $p(y|x)$ be a discrete memoryless channel with input alphabet $\mathcal{X}$ and output alphabet $\mathcal{Y}$. Let $C = \max_{p(x)} I(X;Y)$ be the channel capacity, and let $p^*(x)$ denote a capacity-achieving input distribution.

**Definition (Codebook).** An $(n, M)$ code for the channel consists of:
1. An index set $\mathcal{W} = \{1, 2, \dots, M\}$ (the message set).
2. An encoding function $f: \mathcal{W} \to \mathcal{X}^n$ mapping each message to a codeword $\mathbf{x}^n(w) = (x_1(w), \dots, x_n(w))$.
3. A decoding function $g: \mathcal{Y}^n \to \mathcal{W}$ mapping each received sequence to a message estimate.

The rate of the code is $R = (\log_2 M) / n$ bits per channel use.

**Definition (Probability of error).** The maximal probability of error for the code is:

$$
P_e^{(n)} = \max_{w \in \mathcal{W}} \Pr\bigl[g(Y^n) \neq w \mid X^n = f(w)\bigr].
$$

**Definition (Achievable rate).** A rate $R$ is achievable if there exists a sequence of $(n, 2^{nR})$ codes with $P_e^{(n)} \to 0$ as $n \to \infty$.

**Definition (Channel capacity).** The channel capacity is the supremum of all achievable rates:

$$
C = \sup\{R : R \text{ is achievable}\}.
$$

### Counterexamples to common slips

- **The theorem does not say the error is zero.** It says the error *probability* goes to zero as the block length increases. For any finite $n$, the error is positive. The asymptotic statement is that you can make the error as small as you like by using a long enough code.

- **Achievability and converse are two separate proofs.** The achievability part shows $R < C$ implies reliable codes exist. The converse shows $R > C$ implies reliable codes do not exist. Together they pin down $C$ as the sharp threshold.

- **Random coding does not produce a usable code.** The random coding argument shows existence. It does not give you a specific codebook or a practical decoder. The gap between existence and construction is where coding theory lives.

### Key theorem with proof [Intermediate+]

**Theorem (Shannon's noisy channel coding theorem — achievability).** For a discrete memoryless channel with capacity $C$, every rate $R < C$ is achievable. That is, for any $R < C$, there exists a sequence of $(n, 2^{nR})$ codes with $P_e^{(n)} \to 0$ as $n \to \infty$.

*Proof.* The proof has four steps: random codebook generation, joint typicality decoding, error analysis, and the existence argument.

**Step 1: Random codebook generation.** Fix a rate $R < C$. Fix the capacity-achieving input distribution $p^*(x)$. Generate a codebook $\mathcal{C}$ with $M = 2^{nR}$ codewords by drawing each codeword independently: for each message $w \in \{1, \dots, M\}$, the codeword $\mathbf{X}^n(w) = (X_1(w), \dots, X_n(w))$ is drawn i.i.d. from $p^*(x)$. The $M$ codewords are independent of each other.

**Step 2: Joint typicality decoder.** After receiving $\mathbf{Y}^n$, the decoder declares message $\hat{w}$ if there is a unique $w$ such that $(\mathbf{X}^n(w), \mathbf{Y}^n)$ is jointly $\epsilon$-typical with respect to the joint distribution $p^*(x)p(y|x)$. If no such $w$ exists or more than one exists, the decoder declares an error.

Recall from [46.02.01] that $(\mathbf{x}^n, \mathbf{y}^n)$ is jointly $\epsilon$-typical if:

$$
\left|-\frac{1}{n}\log p^*(\mathbf{x}^n) - H(X)\right| < \epsilon, \quad \left|-\frac{1}{n}\log p(\mathbf{y}^n) - H(Y)\right| < \epsilon,
$$

$$
\left|-\frac{1}{n}\log p(\mathbf{x}^n, \mathbf{y}^n) - H(X,Y)\right| < \epsilon.
$$

**Step 3: Error analysis.** Let $W$ be uniform on $\{1, \dots, M\}$ and independent of the codebook. By symmetry of the random coding construction, the average error probability is the same for all messages, so:

$$
\Pr(\text{error}) = \Pr(\hat{W} \neq 1 \mid W = 1).
$$

An error occurs if either the transmitted codeword is not jointly typical with the received sequence (event $E_1$), or some other codeword is jointly typical with the received sequence (event $E_{2,w}$ for wrong codeword $w$).

By the joint AEP, $\Pr(E_1) \to 0$ as $n \to \infty$.

For the wrong codewords: codeword $\mathbf{X}^n(w)$ for $w \neq 1$ is generated independently of $\mathbf{Y}^n$ (which depends only on $\mathbf{X}^n(1)$). By the joint AEP, the probability that $\mathbf{X}^n(w)$ is jointly typical with $\mathbf{Y}^n$ is at most $2^{-n(I(X;Y) - 3\epsilon)}$. By the union bound:

$$
\Pr\!\left(\bigcup_{w=2}^{M} E_{2,w}\right) \leq (M - 1) \cdot 2^{-n(I(X;Y) - 3\epsilon)} = 2^{nR} \cdot 2^{-n(I(X;Y) - 3\epsilon)} = 2^{-n(I(X;Y) - R - 3\epsilon)}.
$$

Since $R < C = I(X;Y)$ at the capacity-achieving distribution, we can choose $\epsilon$ small enough that $I(X;Y) - R - 3\epsilon > 0$. Then the union bound probability vanishes exponentially in $n$.

**Step 4: Existence.** The average error probability over the random codebook vanishes. Therefore, there exists at least one codebook for which the error probability is at most the average. This codebook gives an $(n, 2^{nR})$ code with $P_e^{(n)} \to 0$. $\square$

**Bridge.** The achievability proof builds toward the converse in [46.03.03], where Fano's inequality shows that rates above $C$ cannot be achieved, completing the full coding theorem. The joint typicality decoder appears again in rate-distortion theory [46.04.01] as the same typicality test used to match source sequences to reproduction codewords. The foundational reason random coding works is that independently generated codewords spread out in the output space so that the probability of two codewords landing in the same typicality region is exponentially small — this is exactly the packing interpretation of capacity. The random coding exponent $E_r(R) = \max_s [E_0(s, p^*) - sR]$ governing the speed of convergence generalises to the reliability function of the channel. Putting these together, the achievability proof reveals that capacity is not merely an upper bound but a sharp threshold: the boundary between the achievable and the impossible.

**Exercise (medium, short answer).**

Explain why the codewords in the random codebook are generated independently rather than, say, as rows of a structured matrix.

<details class="exercise__hint"><summary>Hint</summary>

The analysis relies on the independence of wrong codewords from the received sequence.
</details>

<details class="exercise__answer"><summary>Answer</summary>

Independence is needed for the error analysis. When a wrong codeword $\mathbf{X}^n(w)$ is generated independently of the transmitted codeword and the received sequence, the probability of joint typicality is bounded by $2^{-nI(X;Y)}$ by the joint AEP. If codewords were correlated (e.g., sharing structure), the wrong-codeword probability would not factor this way, and the union bound would not give the exponential decay. Independence makes the analysis tractable.

</details>

**Exercise (medium, short answer).**

In the error analysis, the union bound over $M - 1$ wrong codewords gives a probability of at most $(M-1) \cdot 2^{-n(I(X;Y) - 3\epsilon)}$. Why is this bound tight enough to show vanishing error?

<details class="exercise__answer"><summary>Answer</summary>

Substituting $M = 2^{nR}$, the bound becomes $2^{nR} \cdot 2^{-n(I(X;Y) - 3\epsilon)} = 2^{-n(I(X;Y) - R - 3\epsilon)}$. Since $R < I(X;Y) = C$ and $\epsilon$ can be chosen so that $I(X;Y) - R - 3\epsilon > 0$, the exponent is positive and the bound vanishes exponentially. The factor $M - 1 \approx 2^{nR}$ grows exponentially, but the typicality probability $2^{-nI(X;Y)}$ decays faster because $I(X;Y) > R$.

</details>

**Exercise (hard, short answer).**

Prove that the average error probability over the random codebook is the same for all messages (i.e., $\Pr(\hat{W} \neq w \mid W = w)$ does not depend on $w$).

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

Show that the probability of the joint typicality event $\Pr\{(\mathbf{X}^n, \mathbf{Y}^n) \in A_\epsilon^{(n)}\}$ converges to 1 as $n \to \infty$, where $A_\epsilon^{(n)}$ is the jointly $\epsilon$-typical set and $(\mathbf{X}^n, \mathbf{Y}^n)$ is drawn from the joint distribution $p^*(x)^n p(y|x)^n$.

<details class="exercise__hint"><summary>Hint</summary>

Use the law of large numbers on the empirical entropies.
</details>

<details class="exercise__answer"><summary>Answer</summary>

The pair $(\mathbf{X}^n, \mathbf{Y}^n)$ consists of $n$ i.i.d. draws from $p^*(x, y) = p^*(x)p(y|x)$. The joint typicality conditions require the empirical averages $-\frac{1}{n}\log p^*(\mathbf{X}^n)$, $-\frac{1}{n}\log p(\mathbf{Y}^n)$, and $-\frac{1}{n}\log p(\mathbf{X}^n, \mathbf{Y}^n)$ to be within $\epsilon$ of $H(X)$, $H(Y)$, and $H(X,Y)$ respectively. Each of these is an average of i.i.d. random variables: $-\frac{1}{n}\log p^*(\mathbf{X}^n) = \frac{1}{n}\sum_{i=1}^n (-\log p^*(X_i))$, and similarly for the others. By the weak law of large numbers, each average converges in probability to its expected value. The intersection of three high-probability events is also high-probability (each event has probability $\geq 1 - \delta$ for large $n$, so the intersection has probability $\geq 1 - 3\delta$). Therefore the probability of being jointly typical converges to 1.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

For the binary erasure channel with erasure probability $e$, capacity is $C = 1 - e$. The random coding achievability proof uses a uniform input distribution. Sketch the joint typicality decoder for the BEC: what does joint typicality mean when the received word contains erasure symbols?

<details class="exercise__answer"><summary>Answer</summary>

On the BEC, the output alphabet is $\{0, 1, ?\}$. A jointly typical pair $(\mathbf{x}^n, \mathbf{y}^n)$ satisfies: (1) the empirical distribution of $\mathbf{x}^n$ is close to uniform, (2) the empirical distribution of $\mathbf{y}^n$ is close to the output distribution, and (3) the joint empirical distribution is close to $p(x,y)$. On the BEC, joint typicality has a simple interpretation: on unerased positions, $y_i = x_i$; on erased positions, $y_i = ?$ and $x_i$ can be anything. For a wrong codeword $\mathbf{X}^n(w)$ generated independently of $\mathbf{Y}^n$, the unerased positions of $\mathbf{Y}^n$ are random with respect to $\mathbf{X}^n(w)$, so the probability of matching all unerased positions is roughly $2^{-n(1-e)}$. The decoder effectively checks agreement on the unerased positions; a wrong codeword matches with probability $\approx 2^{-n(1-e)}$, and with $2^{nR}$ codewords the union bound gives $2^{-n(1-e-R)}$, which vanishes when $R < 1 - e = C$.

</details>

</aside>

**Exercise (hard, short answer).**

The random coding exponent (Gallager's $E_r(R)$) is defined as $E_r(R) = \max_{0 \leq s \leq 1} \max_{p(x)} [E_0(s, p) - sR]$ where $E_0(s, p) = -\log \sum_y (\sum_x p(x) p(y|x)^{1-s})^{1/(1-s)}$. Show that $E_r(R) > 0$ for $R < C$ and $E_r(C) = 0$.

<details class="exercise__hint"><summary>Hint</summary>

Evaluate $E_0$ at $s \to 0$ and use $dE_0/ds|_{s=0}$ to connect to mutual information.
</details>

<details class="exercise__answer"><summary>Answer</summary>

At $s = 0$: $E_0(0, p) = -\log \sum_y 1 = -\log |\mathcal{Y}|$... wait, more carefully: $E_0(0, p) = -\log \sum_y (\sum_x p(x))^1 = -\log \sum_y 1 = 0$ since $|\mathcal{Y}|$ terms each equal 1. So $E_r(R) \geq E_0(0, p) - 0 = 0$.

Compute $\frac{\partial E_0}{\partial s}\big|_{s=0}$. By the envelope theorem, $\frac{\partial E_0}{\partial s}\big|_{s=0, p = p^*} = I(X;Y) = C$ (this is the mutual information at the capacity-achieving distribution). Therefore for small $s > 0$: $E_0(s, p^*) \approx sC + O(s^2)$, so $E_0(s, p^*) - sR \approx s(C - R) + O(s^2)$. For $R < C$, choosing small positive $s$ gives $E_r(R) > 0$. At $R = C$, $s(C - R) = 0$ and the higher-order terms are non-positive (since $E_0$ is concave in $s$), giving $E_r(C) = 0$.

</details>

## Lean formalization [Intermediate+]

Mathlib has the `PMF` type for discrete distributions and basic measure-theoretic machinery, but it does not define a channel coding problem. There is no codebook type (a finite collection of codewords), no encoding/decoding function, no probability of error computation, no joint typicality set, and no achievability theorem. The random codebook generation requires product distributions on $\mathcal{X}^n$, which Mathlib can express via `PMF.bind` and `Fin` vectors, but the typicality condition and its convergence properties are absent. A `Codex.InformationTheory.ChannelCoding` module would need to define codebooks, decoding functions, error probabilities, and prove the achievability bound via the joint AEP and the union bound. This unit ships without formalization.

## Advanced results [Master]

### The random coding exponent

The achievability proof shows that $P_e^{(n)} \to 0$ for $R < C$, but it does not specify the speed. Gallager (1965) sharpened the analysis to show that the error probability decays exponentially:

$$
P_e^{(n)} \leq 2^{-n E_r(R)},
$$

where the random coding exponent (or Gallager exponent) is:

$$
E_r(R) = \max_{0 \leq s \leq 1} \max_{p(x)} \left[E_0(s, p) - sR\right],
$$

with

$$
E_0(s, p) = -\log_2 \sum_{y \in \mathcal{Y}} \left(\sum_{x \in \mathcal{X}} p(x) p(y|x)^{1-s}\right)^{1/(1-s)}.
$$

The function $E_r(R)$ is positive for $R < C$, zero at $R = C$, and decreasing in $R$. It gives a quantitative bound on how fast the error decays as a function of the gap $C - R$ and the channel structure. For the BSC with $p = 0.1$ and $R = 0.5$, the exponent is approximately $E_r(0.5) \approx 0.02$, giving $P_e^{(n)} \leq 2^{-0.02n}$. At $n = 1000$, this is about $2^{-20} \approx 10^{-6}$.

### The method of types achievability

An alternative proof of achievability uses the method of types rather than joint typicality. A type (or empirical distribution) of a sequence $\mathbf{x}^n$ is the distribution $P_{\mathbf{x}^n}$ that assigns frequency $n_i/n$ to symbol $i$. The type class $T(P)$ is the set of all sequences with empirical distribution $P$.

The key properties are: (1) the number of types is polynomial in $n$ (at most $(n+1)^{|\mathcal{X}|-1}$), (2) each type class has size $|T(P)| = 2^{nH(P)}(1 + o(1))$, and (3) the probability of a type class under distribution $Q$ is $Q^n(T(P)) = 2^{-nD(P\|Q)}(1 + o(1))$.

For the achievability proof, the encoder sends codewords drawn uniformly from type classes close to the capacity-achieving type $P^*$. The decoder finds the unique codeword whose joint type with the received sequence is close to the joint distribution $P^* \times p(y|x)$. The error analysis counts the number of type classes that could cause confusion and bounds the probability using the divergence between the true joint type and the confounding type. This gives a stronger bound on the pre-factor (polynomial instead of exponential in $n$) at the cost of a slightly more involved counting argument.

### Strong converse and the strong coding theorem

The standard achievability proof shows that rates below $C$ are achievable with error probability tending to zero. The strong converse (Wolfowitz, 1957) shows something stronger: for rates above $C$, the error probability tends to 1 (not merely stays bounded away from zero). Together:

**Theorem (Strong converse).** For any sequence of $(n, 2^{nR})$ codes with $R > C$:

$$
P_e^{(n)} \to 1 \quad \text{as } n \to \infty.
$$

The strong converse is proved using the strong typicality version of the joint AEP and a refinement of Fano's inequality. The key insight is that when $R > C$, there are more codewords than the typicality regions can accommodate, and most codewords collide, forcing the decoder into errors with overwhelming probability.

### The reliability function

The random coding exponent $E_r(R)$ is a lower bound on the best achievable error exponent. The reliability function $E(R)$ is the best achievable exponent:

$$
E(R) = -\lim_{n \to \infty} \frac{1}{n} \log P_e^{*,(n)}(R),
$$

where $P_e^{*,(n)}(R)$ is the minimum error probability over all $(n, 2^{nR})$ codes. The reliability function satisfies $E_r(R) \leq E(R) \leq E_{sp}(R)$, where $E_{sp}(R)$ is the sphere-packing exponent (the best known upper bound on $E(R)$). For $R$ close to $C$, $E_r(R) = E(R) = E_{sp}(R)$, so the reliability function is known exactly near capacity. For low rates, a gap between $E_r$ and $E_{sp}$ persists, and the exact reliability function is one of the major open problems in information theory.

**Synthesis.** Shannon's achievability proof is the foundational result that gives capacity its operational meaning: the maximum $C = \max_{p(x)} I(X;Y)$ is the sharp threshold between reliable and unreliable communication. The random coding argument is the central insight — generating codewords independently from the capacity-achieving distribution ensures that wrong codewords are exponentially unlikely to masquerade as the transmitted one, and this is exactly the packing interpretation of the typicality regions. The Gallager exponent refines this to a quantitative bound on the error decay, building toward the reliability function $E(R)$ which remains partially open. The method of types provides an alternative achievability route that appears again in the strong converse and in rate-distortion theory. The achievability proof generalises to continuous channels (AWGN), to channels with memory, and to multiuser settings, putting these together into the information-theoretic framework that underpins all of modern communications.

## Full proof set [Master]

**Proposition (Size of the jointly typical set).** Let $A_\epsilon^{(n)}$ be the jointly $\epsilon$-typical set for the distribution $p(x,y) = p^*(x)p(y|x)$. Then:

$$
|A_\epsilon^{(n)}| \leq 2^{n(H(X,Y) + \epsilon)}.
$$

*Proof.* By definition, for any $(\mathbf{x}^n, \mathbf{y}^n) \in A_\epsilon^{(n)}$:

$$
p(\mathbf{x}^n, \mathbf{y}^n) \geq 2^{-n(H(X,Y) + \epsilon)}.
$$

Since $p$ is a probability distribution:

$$
1 \geq \sum_{(\mathbf{x}^n, \mathbf{y}^n) \in A_\epsilon^{(n)}} p(\mathbf{x}^n, \mathbf{y}^n) \geq |A_\epsilon^{(n)}| \cdot 2^{-n(H(X,Y) + \epsilon)}.
$$

Rearranging: $|A_\epsilon^{(n)}| \leq 2^{n(H(X,Y) + \epsilon)}$. $\square$

**Proposition (Probability that a wrong codeword is jointly typical).** Let $\mathbf{X}^n(w)$ be drawn from $(p^*)^n$ independently of $\mathbf{Y}^n$ (which was generated by transmitting $\mathbf{X}^n(1)$ through the channel). Then for $n$ sufficiently large:

$$
\Pr\{(\mathbf{X}^n(w), \mathbf{Y}^n) \in A_\epsilon^{(n)}\} \leq 2^{-n(I(X;Y) - 3\epsilon)}.
$$

*Proof.* The joint distribution of $(\mathbf{X}^n(w), \mathbf{Y}^n)$ factors as $(p^*)^n \times p_{Y|X}^n$ when conditioned on $\mathbf{X}^n(w)$, but since $\mathbf{X}^n(w)$ is independent of the transmitted codeword, the pair $(\mathbf{X}^n(w), \mathbf{Y}^n)$ has the product distribution $p^*(x)^n \cdot p_Y(y)^n$ (not the joint distribution $p^*(x)^n p(y|x)^n$). 

For $(\mathbf{x}^n, \mathbf{y}^n) \in A_\epsilon^{(n)}$:

$$
p^*(\mathbf{x}^n) \cdot p_Y(\mathbf{y}^n) = p(\mathbf{x}^n, \mathbf{y}^n) \cdot \frac{p^*(\mathbf{x}^n) p_Y(\mathbf{y}^n)}{p(\mathbf{x}^n, \mathbf{y}^n)} = p(\mathbf{x}^n, \mathbf{y}^n) \cdot 2^{-n\left[\frac{1}{n}\log\frac{p(\mathbf{x}^n, \mathbf{y}^n)}{p^*(\mathbf{x}^n)p_Y(\mathbf{y}^n)}\right]}.
$$

On the typical set, $\frac{1}{n}\log\frac{p(\mathbf{x}^n, \mathbf{y}^n)}{p^*(\mathbf{x}^n)p_Y(\mathbf{y}^n)}$ is within $\epsilon$ of $I(X;Y)$ (since $H(X,Y) - H(X) - H(Y) = -I(X;Y)$, and each term is within $\epsilon$ of its entropy). Therefore:

$$
p^*(\mathbf{x}^n) \cdot p_Y(\mathbf{y}^n) \leq p(\mathbf{x}^n, \mathbf{y}^n) \cdot 2^{-n(I(X;Y) - \epsilon)} \leq 2^{-n(H(X,Y) - \epsilon)} \cdot 2^{-n(I(X;Y) - \epsilon)}.
$$

The probability under the product measure is:

$$
\sum_{A_\epsilon^{(n)}} p^*(\mathbf{x}^n) p_Y(\mathbf{y}^n) \leq |A_\epsilon^{(n)}| \cdot 2^{-n(H(X,Y) + I(X;Y) - 2\epsilon)}.
$$

Using the size bound $|A_\epsilon^{(n)}| \leq 2^{n(H(X,Y) + \epsilon)}$:

$$
\Pr\{(\mathbf{X}^n(w), \mathbf{Y}^n) \in A_\epsilon^{(n)}\} \leq 2^{n(H(X,Y) + \epsilon)} \cdot 2^{-n(H(X,Y) + I(X;Y) - 2\epsilon)} = 2^{-n(I(X;Y) - 3\epsilon)}. \qquad \square
$$

## Historical & philosophical context [Master]

Claude Shannon proved the noisy channel coding theorem in Part III of his 1948 paper "A Mathematical Theory of Communication" (Bell System Technical Journal 27, 379-423 and 623-656). The achievability proof used a random coding argument that was entirely novel. Shannon generated codewords at random and showed that the average error probability over all possible codes vanishes for rates below capacity, establishing the existence of good codes without constructing any specific one.

The random coding argument was controversial when first presented. Many mathematicians and engineers found it unsatisfying that the proof shows existence without construction. John von Neumann reportedly defended Shannon's approach, pointing out that probabilistic existence proofs are standard in mathematics. The argument is analogous to the probabilistic method in combinatorics (Erdos, 1947): showing that a random structure has a desired property with positive probability proves that such a structure exists.

Robert Gallager refined Shannon's error analysis in his 1965 paper "Simple Derivation of the Coding Theorem and Some Applications" (IEEE Transactions on Information Theory 11, 3-18). Gallager introduced the random coding exponent $E_r(R)$ and showed that the error probability decays as $2^{-nE_r(R)}$, giving a quantitative handle on the speed of convergence. Gallager's bound was the foundation for the reliability function analysis developed by the Soviet school (Pinsker, Dobrushin) and by Csiszar and Korner.

The strong converse was proved by Jacob Wolfowitz in 1957 in "The Coding of Messages Subject to Chance Errors" (Illinois Journal of Mathematics 1, 591-606). The strong converse upgraded the converse from "rates above $C$ have bounded error" to "rates above $C$ have error tending to 1," making the coding theorem a sharp threshold.

The gap between existence and construction defined coding theory for the next seven decades. Shannon's proof showed that good codes exist but provided no guidance on how to find them. Constructing explicit codes that approach capacity required the development of algebraic coding theory (Hamming, Reed-Solomon, BCH codes in the 1950s-1960s), concatenated codes (Forney, 1966), and finally iterative decoding (LDPC and turbo codes in the 1990s-2000s). Each of these constructions is an attempt to close the gap between Shannon's existence proof and practical engineering.
