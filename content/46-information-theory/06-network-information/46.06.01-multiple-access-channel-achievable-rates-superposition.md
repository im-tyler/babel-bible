## Intuition [Beginner]

Two people want to send messages to a single receiver over a shared noisy channel. Think of two cell phones in the same cell tower coverage area, both trying to upload data at the same time. The tower hears the superposition of both signals, corrupted by noise. How fast can each user send?

The answer is a region, not a single number. User 1 can send at rate $R_1$ and user 2 at rate $R_2$, but the rates are coupled. If user 1 sends faster, user 2 must slow down, and vice versa. The set of all simultaneously achievable rate pairs $(R_1, R_2)$ is the capacity region.

For the simplest model — two independent transmitters and one receiver — the capacity region is a pentagon in the $(R_1, R_2)$ plane. The pentagon has five sides: two individual rate constraints ($R_1$ and $R_2$ each bounded above), one sum-rate constraint ($R_1 + R_2$ bounded above), and two non-negativity constraints.

The sum-rate bound is the total information the channel can carry. No matter how you split it, the combined rate cannot exceed $I(X_1, X_2; Y)$, the mutual information between both inputs and the output. This is the channel's total capacity when both senders are considered as one.

The individual bounds $R_1 \leq I(X_1; Y | X_2)$ say: even if user 2's message were known perfectly, user 1 could not exceed this rate. The conditioning on $X_2$ means "treating user 2's signal as known rather than interference."

Each corner of the pentagon represents a different operating point. At one corner, user 1 gets maximum rate while user 2 gets the minimum. At another corner, the roles reverse. The sum-rate corner gives the fairest split. Time-sharing between corners fills in the rest.

## Visual [Beginner]

*Figure: the MAC capacity region as a pentagon in the $(R_1, R_2)$ plane. The vertices are: origin, $(I(X_1;Y|X_2), 0)$, $(I(X_1;Y|X_2), I(X_2;Y|X_1))$ on the sum-rate line, $(I(X_1;Y|X_2), I(X_2;Y|X_1))$ is not a vertex — the vertices are $(I(X_1;Y|X_2), I(X_2;Y) - I(X_1;Y|X_2))$ and $(I(X_1;Y) - I(X_2;Y|X_1), I(X_2;Y|X_1))$. The diagonal line $R_1 + R_2 = I(X_1, X_2; Y)$ is the sum-rate bound.*

| Constraint | Formula | Meaning |
|---|---|---|
| User 1 individual | $R_1 \leq I(X_1; Y | X_2)$ | Rate 1 with signal 2 known |
| User 2 individual | $R_2 \leq I(X_2; Y | X_1)$ | Rate 2 with signal 1 known |
| Sum-rate | $R_1 + R_2 \leq I(X_1, X_2; Y)$ | Total channel capacity |
| Non-negativity | $R_1 \geq 0$, $R_2 \geq 0$ | Rates must be non-negative |

## Worked example [Beginner]

Two users transmit binary symbols over a binary adder channel. User 1 sends $X_1 \in \{0, 1\}$ and user 2 sends $X_2 \in \{0, 1\}$. The receiver observes $Y = X_1 + X_2 \in \{0, 1, 2\}$ (no noise).

With uniform inputs: $I(X_1, X_2; Y) = H(Y) - H(Y|X_1,X_2) = H(Y) = H(1/4, 1/2, 1/4) = 1.5$ bits. $I(X_1; Y|X_2) = H(X_1) = 1$ bit (knowing $X_2$, the output determines $X_1$). Similarly $I(X_2; Y|X_1) = 1$ bit.

The capacity region is: $R_1 \leq 1$, $R_2 \leq 1$, $R_1 + R_2 \leq 1.5$. This is a pentagon with vertices $(0, 0)$, $(1, 0)$, $(1, 0.5)$, $(0.5, 1)$, $(0, 1)$.

Each user alone could send 1 bit. Together they can send 1.5 bits total — more than either alone, but less than the sum of their individual capacities (2 bits). The channel is a shared resource, and the sum-rate reflects the total information-carrying capacity.

**Exercise (easy, true/false).**

In a MAC, user 1 can always achieve rate $R_1 = I(X_1; Y)$ regardless of what user 2 does.

<details class="exercise__answer"><summary>Answer</summary>

**False.** User 1's maximum rate depends on user 2's signal. If user 2 is also transmitting, user 1's signal faces interference. The individual rate bound is $R_1 \leq I(X_1; Y|X_2)$, which conditions on user 2's input, not the unconditional $I(X_1; Y)$.

</details>

</aside>

<aside class="exercise" data-difficulty="easy" data-type="multiple-choice" markdown="1">

**Exercise (easy, multiple choice).**

What does the sum-rate constraint $R_1 + R_2 \leq I(X_1, X_2; Y)$ represent?

A. Each user's individual capacity

B. The total information-carrying capacity of the channel

C. The rate user 1 can achieve when user 2 is silent

D. The capacity of a point-to-point channel

<details class="exercise__answer"><summary>Answer</summary>

**B.** The sum-rate bound is the total information the channel output carries about both inputs combined. No coding scheme can exceed this total regardless of how the rate is split between users.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Explain why treating the other user's signal as noise is suboptimal for the MAC.

<details class="exercise__hint"><summary>Hint</summary>

What does joint decoding do that single-user decoding cannot?

</details>

<details class="exercise__answer"><summary>Answer</summary>

Treating user 2's signal as noise means user 1 sees an effective channel with reduced capacity (since user 2's signal acts as interference). Joint decoding exploits the structure of both users' codebooks to simultaneously decode both messages. The joint decoder can separate the two signals because each user's codewords are drawn from an independent random codebook, and the joint typicality of the received signal with both codewords is a much stronger constraint than individual typicality with either codeword alone. This is why the sum-rate bound $I(X_1, X_2; Y)$ exceeds the sum of the "treat as noise" rates.

</details>

</aside>

## Formal definition [Intermediate+]

A two-user discrete memoryless multiple-access channel (MAC) has input alphabets $\mathcal{X}_1, \mathcal{X}_2$, output alphabet $\mathcal{Y}$, and transition probabilities $p(y|x_1, x_2)$.

**Definition (MAC code).** A $(2^{nR_1}, 2^{nR_2}, n)$ MAC code consists of:

1. Two encoders: $f_1: \{1, \dots, 2^{nR_1}\} \to \mathcal{X}_1^n$ and $f_2: \{1, \dots, 2^{nR_2}\} \to \mathcal{X}_2^n$, mapping messages to codewords.
2. A decoder $g: \mathcal{Y}^n \to \{1, \dots, 2^{nR_1}\} \times \{1, \dots, 2^{nR_2}\}$ mapping the received sequence to message estimates.

The average probability of error is $P_e^{(n)} = \Pr[(\hat{W}_1, \hat{W}_2) \neq (W_1, W_2)]$.

**Definition (MAC capacity region).** The capacity region $\mathcal{C}$ is the closure of the set of achievable rate pairs $(R_1, R_2)$.

**Theorem (Ahlswede 1971, Liao 1972).** The capacity region of the two-user discrete memoryless MAC is:

$$\mathcal{C} = \text{conv} \bigcup_{p(x_1)p(x_2)} \{(R_1, R_2) : R_1 \leq I(X_1; Y|X_2),\ R_2 \leq I(X_2; Y|X_1),\ R_1 + R_2 \leq I(X_1, X_2; Y)\},$$

where the union is over all product distributions $p(x_1)p(x_2)$ and conv denotes convex closure.

### Key theorem with proof [Intermediate+]

**Theorem (MAC achievability).** Fix a product distribution $p(x_1)p(x_2)$. The rate region:

$$\mathcal{R}(p_1, p_2) = \{(R_1, R_2) : R_1 \leq I(X_1; Y|X_2),\ R_2 \leq I(X_2; Y|X_1),\ R_1 + R_2 \leq I(X_1, X_2; Y)\}$$

is achievable.

*Proof (random coding with joint typicality decoding).*

**Codebook generation.** Generate $2^{nR_1}$ codewords $X_1^n(w_1)$ for user 1 by drawing each symbol i.i.d. from $p(x_1)$. Independently generate $2^{nR_2}$ codewords $X_2^n(w_2)$ for user 2 by drawing each symbol i.i.d. from $p(x_2)$.

**Encoding.** User 1 sends $X_1^n(W_1)$, user 2 sends $X_2^n(W_2)$.

**Decoding.** The receiver looks for the unique pair $(\tilde{w}_1, \tilde{w}_2)$ such that $(X_1^n(\tilde{w}_1), X_2^n(\tilde{w}_2), Y^n)$ is jointly typical. If a unique pair exists, output it. Otherwise, declare error.

**Error analysis.** Let $(W_1, W_2) = (1, 1)$ without loss of generality. An error occurs if:
1. $(X_1^n(1), X_2^n(1), Y^n) \notin \mathcal{T}_\epsilon^{(n)}$: probability $\to 0$ by the joint AEP.
2. Some impostor pair $(\tilde{w}_1, \tilde{w}_2) \neq (1, 1)$ is jointly typical with $Y^n$.

For a pair with $\tilde{w}_1 \neq 1, \tilde{w}_2 \neq 1$: $X_1^n(\tilde{w}_1)$ and $X_2^n(\tilde{w}_2)$ are independent of $Y^n$, so the probability of joint typicality is approximately $2^{-nI(X_1, X_2; Y)}$. There are $2^{nR_1} \cdot 2^{nR_2}$ such pairs, so the expected number of impostors is $2^{n(R_1 + R_2 - I(X_1, X_2; Y))} \to 0$ when $R_1 + R_2 < I(X_1, X_2; Y)$.

For $\tilde{w}_1 \neq 1, \tilde{w}_2 = 1$: $X_1^n(\tilde{w}_1)$ is independent of $(X_2^n(1), Y^n)$ but $X_2^n(1)$ is given. The probability is $2^{-nI(X_1; Y|X_2)}$, and there are $2^{nR_1}$ such pairs. The constraint $R_1 < I(X_1; Y|X_2)$ ensures these vanish.

By symmetry, $R_2 < I(X_2; Y|X_1)$ controls the $\tilde{w}_2 \neq 1, \tilde{w}_1 = 1$ case. By the union bound, $P_e^{(n)} \to 0$. $\square$

**Bridge.** The MAC capacity region builds toward every multi-user network information theory result by introducing the pentagonal achievable region as the fundamental object. This is exactly the structure that reappears in the broadcast channel [46.06.02] (with reversed roles) and the interference channel [46.06.04] (with two such pentagons interacting). The result appears again in the relay channel [46.06.03] as one face of the cut-set bound: the MAC from relay-and-source to destination constrains the overall achievable rate. The foundational reason the MAC region is a pentagon (not a rectangle) is that the sum-rate constraint $R_1 + R_2 \leq I(X_1, X_2; Y)$ couples the users: the channel output carries joint information about both inputs that cannot be decomposed into independent parts. Putting these together, the MAC is the simplest multi-user network beyond point-to-point and its capacity region is the prototype for all multi-user rate regions; the central insight is that joint decoding extracts more capacity than treating interference as noise, and the gap between these two strategies grows with the channel's joint information content. The result generalises to more than two users by the same technique, and the duality with Slepian-Wolf [46.05.01] reveals the deep symmetry between source and channel coding in the multi-user setting.

## Exercises [Intermediate+]

<aside class="exercise" data-difficulty="easy" data-type="true-false" markdown="1">

**Exercise (easy, true/false).**

In a MAC, the encoders for the two users must coordinate their codebook design.

<details class="exercise__answer"><summary>Answer</summary>

**False.** Each user independently generates its own codebook from the product distribution $p(x_1)p(x_2)$. The codebooks are independent; coordination happens only at the decoder via joint typicality decoding.

</details>

</aside>

<aside class="exercise" data-difficulty="easy" data-type="numeric" markdown="1">

**Exercise (easy, numeric).**

A two-user binary adder MAC ($Y = X_1 + X_2$, no noise) has uniform binary inputs. What is the maximum sum rate $R_1 + R_2$?

<details class="exercise__hint"><summary>Hint</summary>

Compute $I(X_1, X_2; Y)$ for uniform $X_1, X_2$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

With uniform inputs, $Y$ takes values 0, 1, 2 with probabilities 1/4, 1/2, 1/4. $H(Y) = -2(1/4)\log_2(1/4) - (1/2)\log_2(1/2) = 1 + 1 + 0.5 = 1.5$ bits. Since $Y$ is a function of $(X_1, X_2)$, $H(Y|X_1,X_2) = 0$. So $I(X_1,X_2;Y) = H(Y) = 1.5$ bits. Maximum sum rate: 1.5 bits per channel use.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Explain the difference between successive cancellation decoding and joint typicality decoding for the MAC.

<details class="exercise__hint"><summary>Hint</summary>

Successive cancellation decodes one user first, then uses that decoded message to decode the second user.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Joint typicality decoding searches for the unique pair of codewords (one from each user) that is jointly typical with the received sequence. Successive cancellation first decodes one user (say user 1) treating user 2 as noise, then subtracts user 1's decoded codeword from the received signal and decodes user 2 from the residual. The advantage of successive cancellation is that it achieves the corner points of the MAC pentagon (the rate pairs on the sum-rate boundary) with simpler decoding, while joint typicality achieves the entire pentagon but requires searching over all codeword pairs.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="numeric" markdown="1">

**Exercise (medium, numeric).**

A Gaussian MAC has $Y = X_1 + X_2 + Z$ where $Z \sim \mathcal{N}(0, 1)$, $X_1$ has power constraint $P_1 = 3$, and $X_2$ has power constraint $P_2 = 7$. Compute the MAC capacity region boundaries.

<details class="exercise__hint"><summary>Hint</summary>

Use the Gaussian MAC capacity: $R_1 \leq \frac{1}{2}\log(1 + P_1)$, $R_2 \leq \frac{1}{2}\log(1 + P_2)$, $R_1 + R_2 \leq \frac{1}{2}\log(1 + P_1 + P_2)$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

$R_1 \leq \frac{1}{2}\log_2(1 + 3) = \frac{1}{2}\log_2 4 = 1$ bit/use (treating user 2 as noise is not optimal; the conditional bound with Gaussian signaling is $R_1 \leq \frac{1}{2}\log_2(1 + P_1) = 1$ when $X_2$ is known; but the correct individual bound is $R_1 \leq \frac{1}{2}\log_2(1 + P_1/(1+0))$ — actually for the Gaussian MAC with independent inputs the bounds are: $R_1 \leq \frac{1}{2}\log(1 + P_1/N)$ when user 2 is decoded first and subtracted, giving $R_1 \leq \frac{1}{2}\log_2(1 + 3/1) = 1$ bit. $R_2 \leq \frac{1}{2}\log_2(1 + P_2/N) = \frac{1}{2}\log_2(1 + 7) \approx 1.5$ bits. Sum-rate: $R_1 + R_2 \leq \frac{1}{2}\log_2(1 + (P_1 + P_2)/N) = \frac{1}{2}\log_2(1 + 10) \approx 1.73$ bits/use.

Wait — the standard Gaussian MAC bounds with successive cancellation are: when decoding user 2 first (treating user 1 as noise), user 2 gets $R_2 \leq \frac{1}{2}\log_2(1 + P_2/(P_1 + N))$. After subtracting, user 1 gets $R_1 \leq \frac{1}{2}\log_2(1 + P_1/N)$. The pentagon vertices depend on decoding order.

The full region: $R_1 \leq \frac{1}{2}\log_2(1 + P_1/N) = 1$ bit. $R_2 \leq \frac{1}{2}\log_2(1 + P_2/N) = 1.5$ bits. Sum-rate: $R_1 + R_2 \leq \frac{1}{2}\log_2(1 + (P_1+P_2)/N) = 1.73$ bits. And $R_1 \leq \frac{1}{2}\log_2(1 + P_1/(P_2+N)) = \frac{1}{2}\log_2(1 + 3/8) \approx 0.26$ bits (user 1 treating user 2 as noise), $R_2 \leq \frac{1}{2}\log_2(1 + P_2/(P_1+N)) = \frac{1}{2}\log_2(1 + 7/4) \approx 0.97$ bits.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise (medium, short answer).**

Prove that the MAC capacity region is convex. Why does time-sharing work?

<details class="exercise__hint"><summary>Hint</summary>

Use a time-sharing random variable to switch between two operating points.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Given two achievable rate pairs $(R_1^{(a)}, R_2^{(a)})$ and $(R_1^{(b)}, R_2^{(b)})$ achieved by codes of length $n$, construct a new code of length $2n$: use code (a) for the first $n$ symbols and code (b) for the second $n$ symbols. The resulting rate pair is the average $(R_1^{(a)} + R_1^{(b)})/2, (R_2^{(a)} + R_2^{(b)})/2)$. More generally, introduce a time-sharing random variable $Q$ and use input distributions $p(x_1|Q)p(x_2|Q)$. The convex closure in the capacity region formula accounts for this time-sharing. Time-sharing works because the encoders can agree on a schedule (using a shared clock) without communicating their messages.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

Prove the MAC converse: any achievable rate pair must satisfy $R_1 \leq I(X_1; Y|X_2, Q)$, $R_2 \leq I(X_2; Y|X_1, Q)$, $R_1 + R_2 \leq I(X_1, X_2; Y|Q)$ for some $p(q)p(x_1|q)p(x_2|q)$.

<details class="exercise__hint"><summary>Hint</summary>

Apply Fano's inequality and use the chain rule for mutual information with the time-sharing variable $Q = (W_2, Y^{i-1})$ or similar.

</details>

<details class="exercise__answer"><summary>Answer</summary>

By Fano's inequality, $H(W_1, W_2 | Y^n) \leq n\epsilon_n$ where $\epsilon_n \to 0$. Then:

$n(R_1 + R_2) = H(W_1, W_2) = I(W_1, W_2; Y^n) + H(W_1, W_2|Y^n) \leq I(W_1, W_2; Y^n) + n\epsilon_n.$

By the data-processing inequality and the memoryless channel property:

$I(W_1, W_2; Y^n) \leq I(X_1^n, X_2^n; Y^n) = \sum_{i=1}^n I(X_{1,i}, X_{2,i}; Y_i | Y^{i-1}).$

Since $X_{1,i}$ is a function of $W_1$ only and $X_{2,i}$ is a function of $W_2$ only (independent encoders), setting $Q_i = (Y^{i-1})$ gives $I(X_{1,i}, X_{2,i}; Y_i | Q_i) \leq I(X_{1,i}, X_{2,i}; Y_i | Q_i)$ and the single-letter form follows by introducing the auxiliary variable $Q$ uniform on $\{1, \dots, n\}$.

For the individual bound: $nR_1 \leq I(W_1; Y^n, W_2) + n\epsilon_n \leq I(W_1; Y^n | W_2) + n\epsilon_n$. By the chain rule and the memoryless property with the encoder structure, this reduces to $\sum_i I(X_{1,i}; Y_i | X_{2,i}, Q_i)$, giving $R_1 \leq I(X_1; Y|X_2, Q)$ in single-letter form.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

Show that for the Gaussian MAC with power constraints $P_1, P_2$ and noise variance $N$, the sum-rate capacity is $\frac{1}{2}\log_2(1 + (P_1 + P_2)/N)$ and is achieved by independent Gaussian inputs.

<details class="exercise__hint"><summary>Hint</summary>

Maximize $I(X_1, X_2; Y)$ subject to $E[X_1^2] \leq P_1$, $E[X_2^2] \leq P_2$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

$I(X_1, X_2; Y) = H(Y) - H(Y|X_1, X_2) = H(Y) - H(Z)$. Since $Z \sim \mathcal{N}(0, N)$, $H(Z) = \frac{1}{2}\log(2\pi e N)$. To maximize $H(Y)$, note that $Y = X_1 + X_2 + Z$ has variance $\text{Var}(X_1) + \text{Var}(X_2) + N \leq P_1 + P_2 + N$. The Gaussian distribution maximizes entropy for a given variance, so $H(Y) \leq \frac{1}{2}\log(2\pi e(P_1 + P_2 + N))$. Equality is achieved when $X_1 \sim \mathcal{N}(0, P_1)$ and $X_2 \sim \mathcal{N}(0, P_2)$ independently. The sum-rate capacity is $\frac{1}{2}\log((P_1 + P_2 + N)/N) = \frac{1}{2}\log(1 + (P_1 + P_2)/N)$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise (hard, short answer).**

The MAC with feedback: show that feedback can enlarge the MAC capacity region by allowing users to cooperate through the feedback signal.

<details class="exercise__hint"><summary>Hint</summary>

Contrast with the single-user case where feedback does not increase capacity. In the MAC, feedback lets users learn each other's messages and cooperate.

</details>

<details class="exercise__answer"><summary>Answer</summary>

In the single-user DMC, feedback does not increase capacity because the transmitter already knows the message and the channel is memoryless. In the MAC, feedback from the receiver lets each user observe the channel output, which depends on both users' inputs. Through the output, user 1 can infer information about user 2's message (and vice versa). This enables cooperation: after learning part of user 2's message via feedback, user 1 can help relay it, increasing the sum rate. Gaarder and Wolf (1975) showed that for the binary adder MAC with feedback, the capacity region is strictly larger than without feedback. The enlarged region comes from the users' ability to cooperatively resolve each other's messages through the feedback channel, breaking the independent-encoder constraint of the non-feedback MAC.

</details>

</aside>

## Lean formalization [Intermediate+]

Mathlib does not define multiple-access channels, MAC codes, or the capacity region as a subset of rate space. The achievability proof requires independently generated codebooks for each user and a joint typicality decoder that searches over all codeword pairs. The converse uses Fano's inequality with a multi-message version of the data-processing inequality. The convexity of the capacity region (via time-sharing) requires defining a time-sharing random variable and showing that convex combinations of achievable rate pairs are achievable. None of these structures exist in Mathlib. This unit ships without formalization.

## Advanced results [Master]

### The Gaussian MAC and successive cancellation

The Gaussian MAC $Y = X_1 + X_2 + Z$ with $Z \sim \mathcal{N}(0, N)$ and power constraints $E[X_i^2] \leq P_i$ has the capacity region:

$$R_1 \leq \frac{1}{2}\log\left(1 + \frac{P_1}{N}\right), \quad R_2 \leq \frac{1}{2}\log\left(1 + \frac{P_2}{N}\right),$$
$$R_1 + R_2 \leq \frac{1}{2}\log\left(1 + \frac{P_1 + P_2}{N}\right).$$

The sum-rate bound is achieved by independent Gaussian inputs. The individual bounds are achieved by successive cancellation: decode user 2 first (treating user 1 as noise), then subtract user 2's contribution and decode user 1 with no interference.

The two decoding orders give two corner points of the pentagon:
- Decode user 2 first: $R_2 \leq \frac{1}{2}\log(1 + P_2/(P_1 + N))$, $R_1 \leq \frac{1}{2}\log(1 + P_1/N)$.
- Decode user 1 first: $R_1 \leq \frac{1}{2}\log(1 + P_1/(P_2 + N))$, $R_2 \leq \frac{1}{2}\log(1 + P_2/N)$.

### The MAC with more than two users

For $k$ users with inputs $X_1, \dots, X_k$ and output $Y$, the capacity region generalizes to:

$$\sum_{i \in S} R_i \leq I(X_S; Y | X_{S^c})$$

for every subset $S \subseteq \{1, \dots, k\}$. There are $2^k - 1$ such constraints. The achievability proof uses independently generated codebooks for each user and joint typicality decoding over all $k$ users. The converse follows the same Fano argument.

The number of constraints grows exponentially with $k$, making the full characterization complex for large numbers of users. However, the sum-rate constraint (with $S = \{1, \dots, k\}$) is often the binding constraint in practice, and it is simply $I(X_1, \dots, X_k; Y)$ maximized over product input distributions.

### The MAC duality with Slepian-Wolf

The MAC capacity region and the Slepian-Wolf achievable region are information-theoretic duals:

| | Slepian-Wolf (source coding) | MAC (channel coding) |
|---|---|---|
| Individual bound | $R_1 \geq H(X_1\|X_2)$ | $R_1 \leq I(X_1; Y\|X_2)$ |
| Sum bound | $R_1 + R_2 \geq H(X_1, X_2)$ | $R_1 + R_2 \leq I(X_1, X_2; Y)$ |
| Inequality direction | Lower bound (at least) | Upper bound (at most) |
| Quantity | Entropy | Mutual information |

The duality reflects the complementary nature of compression (rates above entropy) and communication (rates below capacity).

**Synthesis.** The MAC capacity region is the foundational object of network information theory: a pentagonal region in rate space defined by individual conditional-mutual-information bounds and a sum-rate mutual-information bound. The central insight is that joint decoding extracts the full channel capacity $I(X_1, X_2; Y)$ by exploiting the structure of both users' codebooks simultaneously, rather than treating either user's signal as noise. The result builds toward the broadcast channel [46.06.02], where one sender distributes information to multiple receivers, and the relay channel [46.06.03], where intermediate nodes create a cascade of MAC and broadcast sub-channels. The foundational reason the MAC region is a pentagon is that the three constraints — two individual bounds and one sum-rate bound — are independent: each captures a different aspect of the channel's information-carrying capacity. The duality with Slepian-Wolf [46.05.01] reveals the central insight that source coding and channel coding are mirror images in the multi-user setting, with entropy and mutual information playing symmetric roles. Putting these together, the MAC is the prototype for all multi-user rate regions, and its pentagonal structure is the load-bearing template for network capacity analysis.

## Full proof set [Master]

**Proposition (MAC converse).** Any achievable rate pair $(R_1, R_2)$ satisfies $R_1 + R_2 \leq I(X_1, X_2; Y|Q)$ for some $p(q)p(x_1|q)p(x_2|q)$.

*Proof.* By Fano's inequality:

$$H(W_1, W_2 | Y^n) \leq 1 + P_e^{(n)} \cdot 2nR \leq n\epsilon_n$$

where $\epsilon_n \to 0$ as $P_e^{(n)} \to 0$. Then:

$$n(R_1 + R_2) = H(W_1) + H(W_2) = H(W_1, W_2) = I(W_1, W_2; Y^n) + H(W_1, W_2 | Y^n).$$

Since $W_1 \to X_1^n \to Y^n$ and $W_2 \to X_2^n \to Y^n$ with the independent encoder constraint:

$$I(W_1, W_2; Y^n) \leq I(X_1^n, X_2^n; Y^n) = \sum_{i=1}^n I(X_{1,i}, X_{2,i}; Y_i | Y^{i-1}).$$

Introduce $Q \sim \text{Uniform}\{1, \dots, n\}$ independent of everything, and define $X_1 = X_{1,Q}$, $X_2 = X_{2,Q}$, $Y = Y_Q$. Then:

$$\frac{1}{n}\sum_{i=1}^n I(X_{1,i}, X_{2,i}; Y_i | Y^{i-1}) = I(X_1, X_2; Y | Q).$$

The product structure $p(x_1|q)p(x_2|q)$ follows from the independent encoder constraint: $X_{1,i}$ is a function of $W_1$ only, $X_{2,i}$ is a function of $W_2$ only, and $W_1 \perp W_2$. $\square$

## Connections [Master]

- [46.03.02] — The single-user channel coding theorem is the MAC with one user; the MAC individual bounds reduce to $I(X;Y)$ when the other user is absent.
- [46.01.02] — Mutual information $I(X_1; Y|X_2)$ and $I(X_1, X_2; Y)$ define the MAC capacity region boundaries; the chain rule for mutual information decomposes the sum-rate.
- [46.05.01] — The Slepian-Wolf region is the source-coding dual of the MAC capacity region, with entropy replacing mutual information and inequality directions reversed.
- [46.06.02] — The broadcast channel is the role-reversal of the MAC: one sender, multiple receivers, with superposition coding as the dual of joint decoding.
- [46.06.03] — The relay channel contains a MAC sub-problem (relay + source to destination), and the MAC capacity region provides one face of the cut-set bound.
- [46.03.06] — Feedback does not increase single-user capacity but can enlarge the MAC capacity region, because feedback enables cooperation between users.

## Historical & philosophical context [Master]

Rudolf Ahlswede established the MAC capacity region in "Multi-way Communication Channels" (Proc. 2nd Int. Symp. Information Theory, Tsahkadsor, Armenia, 1971). H. H. J. Liao independently derived the same result in his 1972 dissertation at the University of Hawaii ("Multiple Access Channels"). The Ahlswede-Liao theorem was the first complete characterization of a multi-user channel's capacity region and established the pentagonal structure that has since become the template for all multi-user rate regions.

The proof technique — independently generated codebooks with joint typicality decoding — introduced the paradigm for multi-user achievability proofs. The converse, using Fano's inequality with the independent-encoder constraint, established the method for multi-user converses.

The Gaussian MAC with successive cancellation decoding has become the theoretical foundation for modern multiple-access technologies including CDMA (Code Division Multiple Access), OFDMA (Orthogonal Frequency Division Multiple Access), and non-orthogonal multiple access (NOMA) in 5G cellular systems. The successive cancellation decoder, where one user's signal is decoded first and then subtracted before decoding the next user, is the standard receiver architecture for these systems.

Cover, El Gamal, and Salehi (1980) showed that the MAC capacity region interacts with correlated source coding in a way that breaks the separation theorem, establishing that joint source-channel coding can outperform separate coding in multi-user settings.
