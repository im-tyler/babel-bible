## Intuition [Beginner]

Some problems are hard to solve exactly. We know this from NP-completeness: finding the smallest vertex cover in a graph, or the maximum number of satisfiable clauses in a 3SAT formula, is believed to require exponential time. But what if we only need an approximate answer? Could we find a vertex cover that is at most twice the optimal size? Or satisfy at least 90% of clauses?

The answer depends on the problem, and it is sometimes a definitive no. The theory of hardness of approximation tells us not just that finding the exact optimum is hard, but that even getting close to it is hard. For some problems, there is a hard wall: no efficient algorithm can guarantee a solution within a certain ratio of the optimum, unless P equals NP.

The key tool is the PCP theorem. It says that proofs of NP statements can be checked by reading only a constant number of bits of the proof. This sounds like a statement about verification, but it has a striking consequence for optimisation: it creates a gap in the objective function. The gap makes it NP-hard to tell whether the optimum is large or small, which means no approximation algorithm can cross the gap.

## Visual [Beginner]

Consider MAX-3SAT on a formula with 8 clauses. Each clause has exactly 3 literals.

```
Satisfiable formula:     All 8/8 clauses satisfied (optimal)
Random assignment:       ~7/8 of clauses satisfied on expectation
Hard gap instance:       Either 8/8 OR at most 7/8 + eps satisfied
                          (telling which is NP-hard)
```

| Scenario | Fraction satisfiable | Approximable? |
|---|---|---|
| Satisfiable instance | 1.0 | Find any satisfying assignment (NP-hard) |
| General instance | Between 7/8 and 1.0 | Ratio 8/7 is best possible (P) |
| Gap instance (PCP) | Either 1.0 or at most 7/8 + eps | Telling apart is NP-hard |

The gap is the difference between the two thresholds in the gap instance. The PCP theorem converts any NP-hard decision problem into a gap-3SAT instance, making the gap version itself NP-hard.

## Worked example [Beginner]

We have a 3CNF formula with 6 clauses:

$(x_1 \vee x_2 \vee x_3) \wedge (\bar{x}_1 \vee x_2 \vee \bar{x}_3) \wedge (x_1 \vee \bar{x}_2 \vee x_3) \wedge (\bar{x}_1 \vee \bar{x}_2 \vee \bar{x}_3) \wedge (x_1 \vee x_2 \vee \bar{x}_3) \wedge (\bar{x}_1 \vee x_2 \vee x_3)$

A random truth assignment satisfies each clause with probability 7/8 (the only way a clause fails is if all three literals are false). So a random assignment satisfies $6 \times 7/8 = 5.25$ clauses on expectation.

Can we do better? For this small instance, we can try all 8 truth assignments and find the best one. The assignment $x_1 = T, x_2 = T, x_3 = F$ satisfies all 6 clauses.

But for large formulas with thousands of variables, exhaustive search is impossible. And the PCP theorem says something stronger: there exist formulas where no efficient algorithm can reliably tell whether all clauses are satisfiable or only 7/8 + eps can be satisfied. The gap between 1.0 and 7/8 + eps is the inapproximability gap.

## Formal definition [Intermediate+]

**Definition (PCP verifier).** A $(r, q)$-PCP verifier for a language $L$ is a probabilistic polynomial-time Turing machine $V$ that, given input $x$ and oracle access to a proof string $\pi$, uses at most $r(|x|)$ random bits and queries at most $q(|x|)$ bits of $\pi$, such that:

- *(Completeness)* If $x \in L$, there exists $\pi$ such that $\Pr[V^{\pi}(x) \text{ accepts}] = 1$.
- *(Soundness)* If $x \notin L$, then for every $\pi$, $\Pr[V^{\pi}(x) \text{ accepts}] \leq 1/2$.

**Theorem (PCP theorem, Dinur 2007 / Arora-Safra 1998).** $NP = PCP(\log n, 1)$.

**Definition (gap problem).** For a maximisation problem with optimum $OPT$, the $(c, s)$-gap problem asks: given an instance, output YES if $OPT \geq c$ and NO if $OPT \leq s$, with the promise that one of these holds.

**Definition (approximation ratio).** An algorithm $A$ is an $\alpha$-approximation for a maximisation problem if for every instance, $A$ produces a solution with value at least $\alpha \cdot OPT$. For minimisation, $A$ produces a solution with value at most $\alpha \cdot OPT$.

### Counterexamples to common slips

- *"The PCP theorem says NP proofs can be checked quickly."* The verification was already polynomial; the novelty is that only O(1) bits of the proof are read, yet soundness is maintained with constant probability.

- *"Inapproximability means no algorithm works at all."* It means no algorithm achieves a ratio better than a specific threshold. MAX-3SAT has a 7/8-approximation (random assignment), but no $(7/8 + \varepsilon)$-approximation unless P = NP.

- *"Hardness of approximation is separate from the PCP theorem."* The PCP theorem is the engine of all modern inapproximability results. Every hardness-of-approximation result for an NP-hard optimisation problem ultimately reduces to it.

### Key theorem with proof [Intermediate+]

**Theorem (Håstad's optimal inapproximability for MAX-3SAT).** *For every $\varepsilon > 0$, it is NP-hard to distinguish satisfiable 3CNF formulas from 3CNF formulas where at most a $7/8 + \varepsilon$ fraction of clauses are simultaneously satisfiable. Consequently, MAX-3SAT is NP-hard to approximate within factor $7/8 + \varepsilon$ for any $\varepsilon > 0$.*

*Proof sketch.* Start from the PCP theorem NP = PCP(log n, 1). The verifier $V$ for SAT uses $O(\log n)$ random bits $r$ and reads $q = O(1)$ bits of the proof $\pi$. Håstad's 3-bit PCP ensures $q = 3$: the verifier reads exactly 3 bits of $\pi$ and accepts iff the XOR of those 3 bits equals 0 (or 1, depending on the test).

Construct a 3CNF formula from this verifier. For each random string $r$, the verifier's test on bits $i_r, j_r, k_r$ of $\pi$ becomes a clause on variables $y_{i_r}, y_{j_r}, y_{k_r}$ (the proof bits). The clause is satisfied iff the test passes. This gives a 3CNF formula $\varphi$ with one clause per random string.

*Completeness.* If the original formula is satisfiable, there exists a proof $\pi$ that makes $V$ accept with probability 1. The assignment $y_i = \pi_i$ satisfies all clauses of $\varphi$.

*Soundness.* If the original formula is unsatisfiable, every proof $\pi$ makes $V$ accept with probability at most $\alpha$ for some constant $\alpha < 1$. Håstad's analysis improves this to $\alpha = 1/2 + \delta$ for some $\delta > 0$, and Fourier analysis of the long code shows the rejection probability can be boosted to make $\alpha \leq 7/8 + \varepsilon$.

The random assignment satisfies $7/8$ of clauses on expectation (each 3-variable clause is satisfied by 7 of 8 assignments). So $OPT \geq 7/8 \cdot m$ where $m$ is the number of clauses. If we could approximate within $7/8 + \varepsilon$, we could distinguish $OPT = m$ from $OPT \leq (7/8 + \varepsilon)m$, which is NP-hard by the gap above. $\square$

**Bridge.** The PCP theorem is the foundational reason that approximation resistance exists: by encoding NP-hardness into a constant-query verifier, it creates an objective-function gap that no polynomial-time algorithm can cross. This builds toward the approximation algorithms of [47.04.08] where we see that some problems (like Vertex Cover with ratio 2) are far from their inapproximability bound, while MAX-3SAT sits exactly at the limit. The central insight is that the PCP theorem converts a decision problem (is the formula satisfiable?) into a structural property of the solution landscape (either a perfect solution exists or every solution is far from perfect), and this is exactly the pattern that appears again in [47.03.04] where interactive proofs achieve their power through a similar gap between completeness and soundness. The bridge is that hardness of approximation is not a separate phenomenon from computational hardness but a restatement of it at the level of solution quality.

<aside class="exercise" data-difficulty="easy" data-type="short-answer" markdown="1">

**Exercise 2 (easy, short answer).**

Explain why a random truth assignment satisfies at least 7/8 of the clauses of any 3CNF formula on expectation.

<details class="exercise__hint"><summary>Hint</summary>

Each clause has 3 literals. The only assignment to those 3 variables that fails to satisfy the clause is the one that makes all 3 literals false.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Each 3-literal clause is satisfied by 7 of the 8 possible assignments to its variables. A uniformly random assignment satisfies each clause with probability 7/8. By linearity of expectation, the expected number of satisfied clauses is $7/8 \cdot m$ where $m$ is the total number of clauses.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Prove that if MAX-E3SAT (exact 3SAT, where each clause has exactly 3 distinct variables) can be approximated within factor $7/8 + \varepsilon$ for some $\varepsilon > 0$, then P = NP.

<details class="exercise__hint"><summary>Hint</summary>

Use Håstad's result and the fact that SAT is NP-complete.

</details>

<details class="exercise__answer"><summary>Answer</summary>

By Håstad's theorem, it is NP-hard to distinguish satisfiable E3CNF formulas from those where at most a $7/8 + \varepsilon$ fraction of clauses are satisfiable. If an algorithm $A$ achieves ratio $7/8 + \varepsilon$, then on a satisfiable formula, $A$ returns an assignment satisfying at least $(7/8 + \varepsilon) \cdot OPT = (7/8 + \varepsilon) \cdot m$ clauses. But if the formula is satisfiable, $OPT = m$, so $A$ satisfies at least $(7/8 + \varepsilon) \cdot m$ clauses. If the formula is not satisfiable, then at most $(7/8 + \varepsilon) \cdot m$ clauses can be satisfied. Wait—this gives a contradiction only at the boundary. More precisely: given any SAT instance $\varphi$, reduce it to a gap-E3SAT instance via the PCP theorem. If $\varphi$ is satisfiable, the gap instance has $OPT = m$. If not, $OPT \leq (7/8 + \varepsilon/2)m$. An algorithm with ratio $7/8 + \varepsilon$ would return at least $(7/8 + \varepsilon) \cdot OPT$ clauses. In the YES case, that is at least $(7/8 + \varepsilon)m$. In the NO case, at most $(7/8 + \varepsilon/2)m < (7/8 + \varepsilon)m$. So the algorithm distinguishes the two cases, solving SAT in polynomial time. This implies P = NP.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 4 (medium, symbolic).**

A gap-producing reduction from 3SAT to Vertex Cover produces a graph $G$ from a 3CNF formula $\varphi$ such that: if $\varphi$ is satisfiable, then $G$ has a vertex cover of size at most $k$; if $\varphi$ is unsatisfiable, then every vertex cover of $G$ has size at least $\alpha \cdot k$ for some constant $\alpha > 1$. Show that this implies Vertex Cover is NP-hard to approximate within factor $\alpha$.

<details class="exercise__answer"><summary>Answer</summary>

Suppose there exists a polynomial-time $\alpha'$-approximation algorithm $A$ for Vertex Cover with $\alpha' < \alpha$. Given a 3CNF formula $\varphi$, apply the gap-producing reduction to obtain $G$ and $k$. Run $A$ on $G$ to get a vertex cover $C$ with $|C| \leq \alpha' \cdot OPT(G)$. If $\varphi$ is satisfiable, $OPT(G) \leq k$, so $|C| \leq \alpha' \cdot k < \alpha \cdot k$. If $\varphi$ is unsatisfiable, $OPT(G) \geq \alpha \cdot k$, so $|C| \geq OPT(G) \geq \alpha \cdot k > \alpha' \cdot k$. Therefore, $|C| < \alpha \cdot k$ iff $\varphi$ is satisfiable, solving SAT in polynomial time. So Vertex Cover cannot be approximated within $\alpha$ unless P = NP.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 5 (medium, short answer).**

Why does the PCP theorem give a hardness result for MAX-3SAT but not directly for MAX-2SAT? What additional step is needed?

<details class="exercise__answer"><summary>Answer</summary>

The PCP verifier reads 3 bits of the proof, producing 3-variable clauses. To get a hardness result for MAX-2SAT, one must reduce 3SAT to 2SAT using a gap-preserving reduction, for instance by replacing each 3-clause $(x \vee y \vee z)$ with a collection of 2-clauses and auxiliary variables. Håstad showed this is possible, yielding that MAX-2SAT is hard to approximate beyond factor $21/22 + \varepsilon \approx 0.954$.

</details>

</aside>

**Exercise 6 (hard, symbolic).**

Let MAX-E3SAT-Ek denote the constraint satisfaction problem where each clause involves exactly 3 variables from a Boolean domain and each variable appears in exactly $k$ clauses. Prove that for sufficiently large $k$, MAX-E3SAT-Ek is NP-hard to approximate within $7/8 + \varepsilon$ for any $\varepsilon > 0$.

<details class="exercise__hint"><summary>Hint</summary>

Use the fact that the PCP theorem can be instantiated on regular constraint graphs (each variable has bounded degree) and apply expander graph techniques to regularise the instance.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Starting from a general MAX-E3SAT instance $\varphi$, construct an expander graph $H$ on the variable set. Replace each occurrence of variable $x_i$ in clause $C_j$ with a new variable $x_{i,j}$, adding consistency constraints $x_{i,j} = x_{i,j'}$ using 3-clauses over the edges of $H$. Since $H$ is an expander, the soundness analysis transfers: if the original instance has OPT less than $7/8 + \varepsilon$, then after the consistency constraints, any assignment violates at least a constant fraction of either the original clauses or the consistency constraints. The number of constraints per variable becomes $O(k)$ for some $k = O(1)$. The resulting instance is an E3SAT-Ek instance (after adjusting for parity), and the gap $(1, 7/8 + \varepsilon)$ is preserved up to constant factors. For sufficiently large $k$, the soundness error can be absorbed, and Håstad's hardness carries through.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

The Unique Games Conjecture (UGC) of Khot (2002) states that for every $\varepsilon, \delta > 0$, there exists an alphabet size $k$ such that it is NP-hard to distinguish whether a Unique Label Cover instance has value at least $1 - \varepsilon$ or at most $\delta$. Assuming the UGC, prove that Vertex Cover is hard to approximate within $2 - \varepsilon$ for any $\varepsilon > 0$.

<details class="exercise__hint"><summary>Hint</summary>

Use the reduction of Khot and Regev (2008) from Unique Label Cover to Vertex Cover.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Given a Unique Label Cover instance $\Lambda$ on graph $G = (V, E)$ with alphabet $\Sigma$, construct a new graph $G'$ as follows. The vertex set of $G'$ consists of pairs $(v, a)$ for $v \in V$ and $a \in \Sigma$. For each edge $(u, v) \in E$ with permutation $\pi_{uv}$, add an edge between $(u, a)$ and $(v, \pi_{uv}(a))$ for each $a \in \Sigma$. A vertex cover in $G'$ selects vertices $(v, a)$; the complement is an independent set. If $\Lambda$ has value $1 - \varepsilon$, then the labelling $\ell$ gives an independent set $\{(v, \ell(v)) : v \in V\}$ of size $|V|$ in $G'$, so the vertex cover has size $|V|(|\Sigma| - 1)$. If $\Lambda$ has value at most $\delta$, Khot and Regev show that every independent set has size at most $(\delta + o(1))|V|$, giving a vertex cover of size at least $|V||\Sigma|(1 - \delta - o(1))$. The ratio is $\frac{|\Sigma|(1-\delta)}{|\Sigma|-1} \to 2$ as $|\Sigma| \to \infty$ and $\delta \to 0$, proving hardness within $2 - \varepsilon$. Rubric: full credit for constructing the label-cover to vertex-cover graph and bounding the independent set sizes in both cases.

</details>

</aside>

## Advanced results [Master]

Three families of results sharpen the boundary between what is approximable and what is not.

**Result 1 (optimal inapproximability via the long code).** The Håstad 3-bit PCP achieves optimal hardness for several CSPs simultaneously. The technique encodes proof bits using the long code of a function $f : \{0,1\}^k \to \{0,1\}$, defined by $\text{Long}(f) = (\bigoplus_{i : S_i = 1} f(i))_{S \subseteq [k]}$. The Fourier analysis of this encoding shows that any accepted proof must correlate with a dictator function, and the acceptance probability is bounded by the maximum Fourier coefficient of the proof function plus $1/2 + \varepsilon$. This gives hardness $7/8 + \varepsilon$ for MAX-3SAT, $1/2 + \varepsilon$ for MAX-LIN-2, and $n^{1-\varepsilon}$ for Max-Clique [ref: TODO_REF Håstad 2001].

**Result 2 (the UGC and its consequences).** Khot's Unique Games Conjecture (2002) posits that Unique Label Cover is NP-hard to approximate within any constant. If true, it implies a dramatic sharpening of inapproximability: Vertex Cover becomes hard within $2 - \varepsilon$ (matching the simple 2-approximation), Max-Cut becomes hard within the Goemans-Williamson constant $\alpha_{GW} \approx 0.878$ (matching the SDP algorithm), and every CSP has its approximation threshold determined by semidefinite programming [ref: TODO_REF Arora-Barak Ch. 11]. The UGC remains open; a disproof would reshape the landscape.

**Result 3 (the sliding-scale conjecture).** Bellare, Goldwasser, Lund and Russell (1993) conjectured that for every constant $q$, there exists $c_q > 0$ such that for all constants $s < 1$, $(1, s)$-gap $q$-CSP is NP-hard. This would mean that the soundness can be made arbitrarily small while keeping the query complexity constant. Dinur's proof of the PCP theorem via gap amplification (2007) achieves some soundness reduction but not the full sliding scale; achieving it remains a central open problem in complexity theory [ref: TODO_REF Arora-Barak Ch. 11].

**Synthesis.** The PCP theorem is the foundational reason the landscape of approximability is structured rather than chaotic: it creates a universal mechanism for turning computational hardness into approximation hardness, and this builds toward the algorithmic upper bounds of [47.04.08] that complement the lower bounds proven here. The central insight is that the gap between completeness and soundness in a PCP verifier translates directly into the gap between what an approximation algorithm can guarantee and what is information-theoretically possible, and this is exactly the pattern that generalises from MAX-3SAT to every NP optimisation problem via gap-preserving reductions. Putting these together with the interactive proof framework of [47.03.04] and the derandomisation techniques of [47.05.05] yields the full picture: the PCP theorem is the structural backbone connecting computational complexity, proof systems, and approximation algorithms, and its extensions (UGC, sliding scale) push toward the exact boundary of efficient computation.

## Full proof set [Master]

**Proposition 1 (gap-3SAT from PCP).** *If NP = PCP(log n, 1), then for every $\varepsilon > 0$, the $(1, 7/8 + \varepsilon)$-gap-3SAT problem is NP-hard.*

*Proof.* Let $L \in NP$. By the PCP theorem, $L$ has a $(\log n, q)$-PCP verifier $V$ for some constant $q$. Håstad's transformation produces a $(\log n, 3)$-PCP $V'$ for $L$ that reads exactly 3 proof bits and checks their XOR. For each random string $r \in \{0,1\}^{c \log n}$, define a clause $C_r$ on variables $y_{i_r}, y_{j_r}, y_{k_r}$ (the three queried positions): $C_r = (y_{i_r} \oplus y_{j_r} \oplus y_{k_r} = b_r)$ for the appropriate bit $b_r$. The formula $\varphi = \bigwedge_r C_r$ has $m = n^c$ clauses.

*Completeness:* If $x \in L$, there exists a proof $\pi$ accepted with probability 1. The assignment $y_i = \pi_i$ satisfies every clause $C_r$, so $OPT(\varphi) = m$.

*Soundness:* If $x \notin L$, every proof $\pi$ is accepted by $V'$ with probability at most $s$ for some constant $s < 1$. The Håstad analysis shows $s$ can be made at most $7/8 + \varepsilon$ for any desired $\varepsilon > 0$ (by choosing the long-code test parameter appropriately). So at most $(7/8 + \varepsilon)m$ clauses can be simultaneously satisfied. $\square$

**Proposition 2 (Vertex Cover inapproximability).** *There exists a constant $\alpha > 1$ such that Vertex Cover is NP-hard to approximate within factor $\alpha$.*

*Proof.* By Proposition 1, $(1, 7/8 + \varepsilon)$-gap-3SAT is NP-hard for any $\varepsilon > 0$. Given a 3CNF formula $\varphi$ with $m$ clauses and $n$ variables, construct a graph $G$: create a triangle for each clause $C_j = (\ell_1 \vee \ell_2 \vee \ell_3)$ with vertices $v_{j,1}, v_{j,2}, v_{j,3}$, and add an edge between $v_{j,k}$ and $v_{j',k'}$ whenever they correspond to opposite literals (one is $x_i$ and the other is $\bar{x}_i$). If $\varphi$ is satisfiable, selecting one true literal per clause gives a vertex cover of size $m$ (pick the two non-true-literal vertices from each triangle). If at most $(7/8 + \varepsilon)m$ clauses are satisfiable, then any vertex cover must include at least $\alpha m$ vertices for some $\alpha > 1$ depending on $\varepsilon$, because the contradiction edges force the cover to be large. An $\alpha$-approximation for Vertex Cover would distinguish the two cases, solving the gap problem. $\square$

## Historical & philosophical context [Master]

The PCP theorem was proved in two stages. Arora and Safra (1998) established NP = PCP(log n, polylog n), and Arora, Lund, Motwani, Sudan and Szegedy (1998) strengthened this to NP = PCP(log n, 1), earning the Gödel Prize in 2001. Dinur's alternative proof (2007) uses gap amplification via graph powering, providing a combinatorial route to the same result. Håstad (2001) refined the PCP to achieve optimal inapproximability for MAX-3SAT using Fourier analysis of the long code.

The philosophical significance is that approximation is not a universal escape from NP-hardness: some problems have hard walls beyond which no efficient algorithm can reach. This resolved a decades-old question about whether NP-hard optimisation problems could always be tamed by accepting approximate solutions. The answer is nuanced—some can (knapsack admits an FPTAS), some cannot (MAX-3SAT at 7/8, Set Cover at ln n)—and the PCP theorem is the tool that draws the line [ref: TODO_REF Arora-Barak Ch. 11].
