## Intuition [Beginner]

Imagine you are sorting a deck of cards. One strategy: pick a card at random, split the deck into cards smaller and cards larger, then sort each half. This is randomized quicksort. The random choice means you never get stuck with a bad split — on average, the splits are good enough to sort quickly.

Randomized algorithms use coin flips (or more generally, random number generation) to make decisions during execution. There are two flavours. Las Vegas algorithms always produce the correct answer, but their running time varies. Monte Carlo algorithms run in a fixed time but might produce a wrong answer with small probability.

Randomized quicksort is Las Vegas: it always correctly sorts the array, and its expected running time is $O(n \log n)$ regardless of the input. The randomness is in the algorithm, not the input. No adversary can craft a worst-case input because the pivot choice is random.

Skip lists extend this idea to dictionaries. Instead of maintaining a complex balance invariant (like red-black trees), you build multiple levels of linked lists where each node is promoted to a higher level with probability 1/2. Search starts at the top level and drops down, achieving $O(\log n)$ expected time with a simple implementation.

Treaps combine binary search trees with random priorities. Each key gets a random priority, and the tree is a heap on priorities. The randomness of priorities ensures the tree is balanced in expectation, giving $O(\log n)$ expected time for all operations.

## Visual [Beginner]

Skip list for keys {3, 7, 12, 21, 35, 42}:

```
Level 3:  HEAD ─────────────────────────→ 42
Level 2:  HEAD ──────→ 12 ──────────────→ 42
Level 1:  HEAD ──→ 7 ──→ 12 ──→ 21 ────→ 42
Level 0:  HEAD → 3 → 7 → 12 → 21 → 35 → 42
```

Each node was promoted to higher levels with probability 1/2. To search for 21: start at level 3, move right (42 is too large), drop to level 2, move right (12), move right (42 too large), drop to level 1, move right (21 found).

| Structure | Expected search time | Worst case | Code complexity |
|---|---|---|---|
| Sorted array | $O(\log n)$ | $O(\log n)$ | Simple but insert is $O(n)$ |
| Red-black tree | $O(\log n)$ | $O(\log n)$ | Complex |
| Skip list | $O(\log n)$ | $O(n)$ unlikely | Simple |
| Treap | $O(\log n)$ | $O(n)$ unlikely | Moderate |

## Worked example [Beginner]

We sort the array [4, 1, 7, 3, 6] using randomized quicksort.

**Step 1.** Choose a random pivot from [4, 1, 7, 3, 6]. Suppose we pick 4 (index 0).

Partition: elements less than 4 go left, elements greater go right.

Left: [1, 3]. Right: [7, 6]. Pivot: [4].

**Step 2a.** Sort left [1, 3]. Choose random pivot: suppose 1.

Partition: left [], right [3]. Pivot [1].

Result: [1, 3].

**Step 2b.** Sort right [7, 6]. Choose random pivot: suppose 6.

Partition: left [], right [7]. Pivot [6].

Result: [6, 7].

**Combine.** [1, 3] + [4] + [6, 7] = [1, 3, 4, 6, 7].

The pivot choices led to a balanced partition (2 elements on each side of 4). In the worst case, if we always picked the smallest element, each partition would give 0 and $n-1$ elements, leading to $O(n^2)$ time. But the probability of consistently bad choices is exponentially small.

## Formal definition [Intermediate+]

**Definition (Las Vegas algorithm).** A *Las Vegas algorithm* is a randomized algorithm that always produces a correct output but whose running time $T$ is a random variable. The *expected running time* is $E[T]$.

**Definition (Monte Carlo algorithm).** A *Monte Carlo algorithm* is a randomized algorithm that runs in deterministic time but may produce an incorrect output with probability at most $\delta$ (the error probability).

**Definition (randomized quicksort).** Randomized quicksort selects a pivot uniformly at random from the subarray $A[p..r]$ and partitions around it:

```
RANDOMIZED-QUICKSORT(A, p, r):
  if p < r:
    q = RANDOMIZED-PARTITION(A, p, r)
    RANDOMIZED-QUICKSORT(A, p, q - 1)
    RANDOMIZED-QUICKSORT(A, q + 1, r)
```

**Theorem (expected running time of randomized quicksort).** *The expected running time of randomized quicksort on an array of $n$ elements is $O(n \lg n)$.*

*Proof sketch.* Let $z_1, \ldots, z_n$ be the elements in sorted order. Define indicator $X_{ij} = 1$ if $z_i$ and $z_j$ are compared during the algorithm. The total number of comparisons is $X = \sum_{i < j} X_{ij}$. By linearity of expectation, $E[X] = \sum_{i < j} \Pr[X_{ij} = 1]$.

Two elements $z_i$ and $z_j$ are compared if and only if one of them is chosen as a pivot before any element between them (i.e., $z_{i+1}, \ldots, z_{j-1}$) is chosen. Among the set $\{z_i, z_{i+1}, \ldots, z_j\}$, each element is equally likely to be chosen as pivot first. So $\Pr[X_{ij} = 1] = 2 / (j - i + 1)$.

Therefore $E[X] = \sum_{i < j} \frac{2}{j-i+1} = \sum_{\ell=2}^{n} \frac{2(n-\ell+1)}{\ell} \le 2n \sum_{\ell=2}^{n} \frac{1}{\ell} \le 2n \cdot H_n = O(n \lg n)$. $\square$

**Definition (skip list).** A *skip list* for a sorted set $S$ of $n$ elements is a collection of linked lists $L_0, L_1, \ldots, L_h$ where $L_0$ contains all elements of $S$ in sorted order, and each element is promoted from $L_i$ to $L_{i+1}$ independently with probability $1/2$. The maximum level $h$ is $O(\log n)$ with high probability.

**Theorem (skip list search cost).** *The expected cost of a search in a skip list with $n$ elements is $O(\lg n)$.*

**Definition (treap).** A *treap* for a set of $n$ key-priority pairs $(k_i, p_i)$ is a binary search tree on the keys that is simultaneously a max-heap on the priorities. When priorities are chosen independently and uniformly at random, the expected height of the treap is $O(\lg n)$.

### Counterexamples to common slips

- *"Randomized quicksort has $O(n^2)$ worst case so it is bad."* The worst case is exponentially unlikely. For any fixed input, the probability of $O(n^2)$ time is at most $1/n^c$ for any constant $c$. In practice, randomized quicksort is faster than mergesort due to better cache behaviour.

- *"Skip lists are probabilistically balanced but unreliable."* The failure probability is exponentially small. The probability that a skip list search takes more than $c \log n$ steps is at most $1/n^{c-1}$ for appropriate constants.

- *"Treaps can degenerate into linked lists."* In expectation, the height is $O(\log n)$. The probability of height exceeding $c \log n$ is exponentially small. Treaps are "balanced with high probability."

### Key theorem with proof [Intermediate+]

**Theorem (treap expected height).** *The expected height of a treap on $n$ keys with random priorities is $O(\lg n)$.*

*Proof.* Let $X_{ij}$ be the indicator that node $i$ is an ancestor of node $j$ (or vice versa). The depth of node $j$ is $D_j = \sum_{i \ne j} X_{ij}$ and the height is $H = \max_j D_j$.

Node $i$ is an ancestor of node $j$ iff among the nodes on the path from $i$ to $j$ in the key ordering, node $i$ has the highest priority. Since priorities are random, this probability is $1 / |i - j + 1|$ for each pair.

By linearity of expectation, $E[D_j] = \sum_{i \ne j} 1/|i-j|+1 \le 2H_n = O(\lg n)$ where $H_n$ is the $n$-th harmonic number.

For the height, $E[H] = E[\max_j D_j]$. While the maximum of expectations can exceed the expectation of the maximum, a more careful analysis using the fact that $D_j$ is the sum of independent indicators (they are not independent, but weakly correlated) gives $E[H] = O(\lg n)$. More precisely, by a result of Devroye (1986), $E[H] \le 4.311 \lg n$ for large $n$. $\square$

**Bridge.** Randomized data structures are the foundational reason simple algorithms can match the performance guarantees of deterministic structures: randomisation replaces complex invariant maintenance with probabilistic balance. This builds toward the hashing algorithms of [47.05.01], where universal hashing achieves $O(1)$ expected performance with simpler code than deterministic alternatives, and it appears again in [47.05.03] where primality testing uses randomness to achieve polynomial time with high probability. The central insight is that random choices are "good on average" — this is exactly the principle that makes randomized quicksort, skip lists, and treaps work. Putting these together yields a design philosophy: when a deterministic invariant is expensive to maintain, replace it with a random choice that achieves the same effect in expectation.

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Prove that the probability that two elements $z_i$ and $z_j$ (with $i < j$) are compared in randomized quicksort is exactly $2 / (j - i + 1)$.

<details class="exercise__hint"><summary>Hint</summary>

$z_i$ and $z_j$ are compared iff one of them is the first pivot chosen from $\{z_i, z_{i+1}, \ldots, z_j\}$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

The elements $z_i, z_{i+1}, \ldots, z_j$ form a contiguous group in the sorted order. When quicksort processes a subarray containing this group, the first pivot chosen from this group determines which pairs are compared. If $z_i$ or $z_j$ is chosen first (probability $2 / (j - i + 1)$), then $z_i$ and $z_j$ are compared during the partition step. If any of $z_{i+1}, \ldots, z_{j-1}$ is chosen first, then $z_i$ and $z_j$ are placed in different subarrays and will never be compared. So $\Pr[X_{ij} = 1] = 2 / (j - i + 1)$. Rubric: full credit for the "first pivot in the interval" argument.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 4 (medium, symbolic).**

Prove that the expected height of a skip list with $n$ elements is $O(\log n)$.

<details class="exercise__hint"><summary>Hint</summary>

The maximum level is $\max_i L_i$ where $L_i$ is the number of promotions for element $i$. Each $L_i$ is geometric(1/2). Bound the maximum of $n$ geometric random variables.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Each element is promoted independently with probability 1/2 at each level. The number of levels for element $i$ is a geometric random variable $L_i \sim \text{Geom}(1/2)$ (number of successes before first failure). The skip list height is $H = \max_i L_i$.

$\Pr[H \ge k] = \Pr[\exists i : L_i \ge k] \le n \cdot \Pr[L_1 \ge k] = n \cdot 2^{-k}$ (union bound). For $k = c \lg n$: $\Pr[H \ge c \lg n] \le n \cdot n^{-c} = n^{1-c}$. For $c \ge 2$, this is at most $1/n$. So $H = O(\log n)$ with high probability, and $E[H] = O(\log n)$. Rubric: full credit for the union bound argument.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 5 (medium, short answer).**

Describe the insertion procedure for a treap. What is its expected running time?

<details class="exercise__answer"><summary>Answer</summary>

To insert a key $k$ with random priority $p$: first, insert $k$ as a leaf using standard BST insertion (following the BST ordering). Then, rotate $k$ upward until the heap property on priorities is restored (the parent's priority is $\ge p$). This is called a "bubble-up" or "heapify-up." Each rotation takes $O(1)$, and the expected number of rotations is $O(\log n)$ because the expected depth of a random node in a random BST is $O(\log n)$. Expected running time: $O(\log n)$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Prove that the expected depth of a node in a treap with $n$ keys and random priorities is at most $2H_n - 1 = O(\lg n)$ where $H_n$ is the $n$-th harmonic number.

<details class="exercise__hint"><summary>Hint</summary>

Let $D_j$ be the depth of node $j$. Show that $\Pr[j \text{ is ancestor of } i] = 1/(|i-j|+1)$ and sum over all $i$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Node $j$ is an ancestor of node $i$ in the treap iff $j$ has the highest priority among all keys in the range $[\min(i,j), \max(i,j)]$. Since priorities are random, this probability is $1 / (\max(i,j) - \min(i,j) + 1) = 1 / (|i-j| + 1)$.

The depth of node $j$ is the number of ancestors of $j$ (excluding $j$ itself). Node $i \ne j$ is an ancestor of $j$ with probability $1 / (|i-j| + 1)$. By linearity of expectation:

$E[D_j] = \sum_{i \ne j} \frac{1}{|i-j|+1} = \sum_{k=1}^{n-j} \frac{1}{k+1} + \sum_{k=1}^{j-1} \frac{1}{k+1} \le 2 \sum_{k=1}^{n} \frac{1}{k} = 2H_n - 2 < 2 \ln n$.

Rubric: full credit for the ancestor probability and the harmonic sum.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

Show that randomized quicksort makes at most $2n \ln n$ comparisons in expectation.

<details class="exercise__answer"><summary>Answer</summary>

From the analysis: $E[X] = \sum_{i < j} \Pr[X_{ij} = 1] = \sum_{i < j} \frac{2}{j-i+1}$. Let $\ell = j - i$. For each $\ell$ from 1 to $n-1$, there are $n - \ell$ pairs $(i, j)$ with $j - i = \ell$. So:

$E[X] = \sum_{\ell=1}^{n-1} \frac{2(n - \ell)}{\ell + 1} \le 2n \sum_{\ell=1}^{n-1} \frac{1}{\ell + 1} = 2n \sum_{k=2}^{n} \frac{1}{k} \le 2n \int_1^n \frac{1}{x} dx = 2n \ln n$.

Rubric: full credit for the summation bound.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 8 (medium, symbolic).**

A skip list has $n$ elements. What is the expected total number of nodes across all levels?

<details class="exercise__answer"><summary>Answer</summary>

Each of the $n$ elements appears at level 0 with probability 1, at level 1 with probability 1/2, at level 2 with probability 1/4, etc. The expected number of appearances of a single element is $1 + 1/2 + 1/4 + \cdots = 2$. Total expected nodes: $2n$. So the space is $O(n)$ expected.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 9 (hard, short answer).**

Prove that the expected search cost in a skip list is $O(\log n)$. Analyse the backward search path from the target node to the head.

<details class="exercise__answer"><summary>Answer</summary>

Search in a skip list starts at the topmost level and moves right, dropping down when the next node exceeds the target. Consider the backward analysis: starting from the target node, trace the search path backwards to the head. At each step, either the previous node is at the same level (moved right) or at a higher level (dropped down). Each backward step either moves left on the same level or moves up one level. Moving up happens with probability 1/2 (because the node at the next level up was promoted). The expected number of steps to reach level 0 and the head is bounded by the number of level-ups ($O(\log n)$ expected levels) plus the number of horizontal moves per level ($O(1)$ expected, because each sequence of horizontal moves at a given level is bounded by a geometric random variable with parameter 1/2). Total expected cost: $O(\log n)$.

Rubric: full credit for the backward analysis.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="true-false" markdown="1">

**Exercise 10 (medium, true-false).**

True or false: deterministic quicksort (always choosing the first element as pivot) has $O(n \log n)$ expected running time when the input is a random permutation.

<details class="exercise__answer"><summary>Answer</summary>

**True.** If the input is a uniformly random permutation, then the first element is a uniformly random pivot. The analysis is identical to randomized quicksort: $E[T(n)] = O(n \log n)$. The difference is that deterministic quicksort has a worst case for adversarial inputs, while randomized quicksort does not.

## Advanced results [Master]

Three results sharpen the theory of randomized algorithms for sorting and searching.

**Theorem 1 (high-probability bound for quicksort).** *With probability at least $1 - 1/n^c$ (for any constant $c > 0$), randomized quicksort runs in $O(n \log n)$ time.*

This follows from a Chernoff-bound analysis of the number of comparisons. The tail decays exponentially, so even moderate deviations from the expected $2n \ln n$ comparisons are unlikely. This means randomized quicksort is not just good on average — it is good with overwhelming probability.

**Theorem 2 (skip list with high probability).** *All operations (search, insert, delete) in a skip list with $n$ elements run in $O(\lg n)$ time with probability at least $1 - 1/n^c$ for any constant $c$.*

The proof uses the fact that the skip list height is $O(\log n)$ with high probability (union bound on geometric random variables) and that the expected number of horizontal moves per level is $O(1)$ with exponentially small tail. This matches the guarantees of balanced search trees without the complex rebalancing logic [ref: TODO_REF Motwani & Raghavan Ch. 2].

**Theorem 3 (treap split and merge).** *Treaps support split and merge operations in $O(\lg n)$ expected time.*

Split($T$, $k$) divides treap $T$ into two treaps at key $k$. Merge($T_1$, $T_2$) combines two treaps (all keys in $T_1$ are less than all keys in $T_2$). Both operations are fundamental building blocks for more advanced data structures like interval trees and link-cut trees.

**Synthesis.** Randomized data structures are the foundational reason simple algorithms can achieve logarithmic-time performance: randomisation replaces deterministic invariants with probabilistic ones that hold with high probability. The central insight is that random choices are "good on average" — this is exactly the principle that unifies randomized quicksort, skip lists, and treaps, despite their very different structures. This builds toward the broader theory of randomized algorithms, where randomness serves as a resource that can simplify algorithm design, improve performance, and enable computations that are difficult or impossible deterministically. Putting these together with the universal hashing of [47.05.01] and the primality testing of [47.05.03] yields a unified framework for using randomness as an algorithmic tool.

## Full proof set [Master]

**Proposition 1 (randomized quicksort expected comparisons).** *The expected number of comparisons in randomized quicksort on $n$ distinct elements is at most $2n \ln n$.*

*Proof.* Let $z_1 < z_2 < \cdots < z_n$ be the elements in sorted order. Define $X_{ij} = [z_i \text{ and } z_j \text{ are compared}]$. Total comparisons: $X = \sum_{1 \le i < j \le n} X_{ij}$.

Claim: $\Pr[X_{ij} = 1] = \frac{2}{j - i + 1}$.

The elements $\{z_i, z_{i+1}, \ldots, z_j\}$ form a contiguous range in sorted order. When quicksort first selects a pivot from this range, $z_i$ and $z_j$ are compared iff the pivot is $z_i$ or $z_j$ (then one is compared to all others in the range). If any $z_k$ with $i < k < j$ is the pivot, then $z_i$ and $z_j$ are placed in different subarrays and never compared. Since the pivot is chosen uniformly at random, the first pivot from this range is equally likely to be any of the $j - i + 1$ elements. So $\Pr[X_{ij} = 1] = 2/(j-i+1)$.

$$E[X] = \sum_{i < j} \frac{2}{j-i+1} = \sum_{\ell=1}^{n-1} \frac{2(n-\ell)}{\ell+1} \le 2n \sum_{k=2}^{n} \frac{1}{k} \le 2n \int_1^n \frac{dx}{x} = 2n \ln n. \quad \square$$

**Proposition 2 (skip list height bound).** *The height of a skip list with $n$ elements exceeds $c \lg n$ with probability at most $n^{1-c}$.*

*Proof.* Each element is promoted to level $k$ with probability $2^{-k}$. The height $H = \max_i L_i$ where $L_i \sim \text{Geom}(1/2)$. By union bound:

$$\Pr[H \ge c \lg n] = \Pr[\exists i : L_i \ge c \lg n] \le n \cdot \Pr[L_1 \ge c \lg n] = n \cdot 2^{-c \lg n} = n \cdot n^{-c} = n^{1-c}. \quad \square$$

**Proposition 3 (treap expected depth).** *The expected depth of any node in a treap with $n$ keys and random priorities is at most $2H_n - 1$.*

*Proof.* The depth of node $j$ is $D_j = \sum_{i \ne j} [i \text{ is ancestor of } j]$. Node $i$ is an ancestor of node $j$ iff $i$ has the highest priority among $\{z_{\min(i,j)}, \ldots, z_{\max(i,j)}\}$. Since priorities are uniform and independent, this probability is $1/(|i-j|+1)$. So:

$$E[D_j] = \sum_{i \ne j} \frac{1}{|i-j|+1} = \sum_{k=1}^{n-j} \frac{1}{k+1} + \sum_{k=1}^{j-1} \frac{1}{k+1} \le 2(H_n - 1) = 2H_n - 2 < 2 \ln n. \quad \square$$

## Historical & philosophical context [Master]

Randomized quicksort was analysed by Hoare (1962) in his original paper on quicksort, where he noted that random pivot selection avoids worst-case behaviour. The formal analysis using indicator random variables is due to the textbook treatment by CLRS [ref: TODO_REF CLRS §7.4].

Skip lists were invented by William Pugh in 1990. His motivation was to create a probabilistic alternative to balanced trees that is simpler to implement and has comparable performance. The key insight was that multiple levels of linked lists with geometrically decreasing density mimic the structure of a balanced tree without requiring explicit balancing [ref: TODO_REF Motwani & Raghavan Ch. 2].

Treaps were introduced by Aragon and Seidel in 1989 (published 1996). The name "treap" was coined by Cecilia Aragon, who was also a championship-level rock climber. The treap's simplicity — a BST with random priorities — demonstrates a philosophical point: randomness can replace algorithmic cleverness. Instead of a complex rebalancing algorithm, the treap relies on the fact that random priorities produce balanced trees in expectation.

The broader philosophical point is that randomness is a computational resource. Just as time and space constrain what algorithms can do, randomness expands what can be done efficiently. The question of whether randomness genuinely helps (the BPP vs P question of [47.03.01]) remains one of the central open problems in complexity theory.
