## Intuition [Beginner]

A hash table maps keys to array positions using a hash function. A good hash function spreads keys evenly so that few keys land in the same slot (a collision). A bad hash function concentrates keys, causing many collisions and slow lookups.

The problem is: how do you guarantee a good hash function? If an adversary knows your hash function, they can choose keys that all collide. Universal hashing solves this by choosing the hash function at random from a carefully designed family. No matter what keys the adversary picks, a randomly chosen function from a universal family has low collision probability.

Think of it like seating guests at a dinner party. If you always seat people by last name, a family reunion would cluster everyone at one table. But if you assign seats by a random formula (say, "birthday mod number of tables"), then no group can guarantee they all sit together. Universal hashing is that random formula, with a mathematical guarantee that collisions are rare.

Perfect hashing goes further: for a fixed, known set of keys, it constructs a hash table with zero collisions and constant-time lookup. The FKS construction (named after Fredman, Komlos, and Szemeredi) achieves this in $O(n)$ space for $n$ keys.

## Visual [Beginner]

Universal hashing: choosing a random function from a family:

```
Hash family H = {h_1, h_2, h_3, ...}
Keys: {Alice, Bob, Carol, Dave}

         h_1    h_2    h_3    h_4
Alice:    0      2      0      1
Bob:      1      0      2      3
Carol:    0      2      1      1      ← h_1: Alice & Carol collide
Dave:     2      1      0      0      ← h_3: Carol & Dave don't
```

Pick $h$ at random from $\mathcal{H}$. For any two distinct keys, the probability that they collide is at most $1/m$ where $m$ is the table size.

| Property | Universal hash | Fixed hash |
|---|---|---|
| Collision probability | $\le 1/m$ for any pair | Depends on input |
| Adversary resistance | No input causes many collisions | Adversary can force collisions |
| Choice | Random from family | Deterministic |
| Expected chain length | $\le n/m$ (load factor) | Unbounded in worst case |

## Worked example [Beginner]

A universal hash family for keys that are integers in $\{0, 1, \ldots, p - 1\}$ (where $p$ is a prime larger than any key) and a hash table of size $m$:

$$h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$$

where $a \in \{1, 2, \ldots, p - 1\}$ and $b \in \{0, 1, \ldots, p - 1\}$ are chosen uniformly at random.

Let $p = 17$ and $m = 6$. Pick $a = 3, b = 7$. Compute $h_{3,7}(5) = ((3 \cdot 5 + 7) \bmod 17) \bmod 6 = 22 \bmod 17 \bmod 6 = 5 \bmod 6 = 5$. Compute $h_{3,7}(11) = ((3 \cdot 11 + 7) \bmod 17) \bmod 6 = 40 \bmod 17 \bmod 6 = 6 \bmod 6 = 0$. No collision.

Pick different parameters: $a = 5, b = 2$. $h_{5,2}(5) = ((27) \bmod 17) \bmod 6 = 10 \bmod 6 = 4$. $h_{5,2}(11) = ((57) \bmod 17) \bmod 6 = 6 \bmod 6 = 0$. Still no collision.

The collision probability for any pair of distinct keys is at most $1/m = 1/6$, regardless of which keys are chosen.

## Formal definition [Intermediate+]

**Definition (universal hash family).** Let $U$ be a universe of keys and $m$ a positive integer. A family $\mathcal{H}$ of hash functions from $U$ to $\{0, 1, \ldots, m - 1\}$ is *universal* if for every pair of distinct keys $x, y \in U$ with $x \ne y$:

$$\frac{|\{h \in \mathcal{H} : h(x) = h(y)\}|}{|\mathcal{H}|} \le \frac{1}{m}.$$

Equivalently, if $h$ is chosen uniformly at random from $\mathcal{H}$:

$$\Pr_{h \in \mathcal{H}}[h(x) = h(y)] \le \frac{1}{m}.$$

The family is *strongly universal* (or pairwise independent) if for all $x \ne y$ and all $i, j \in \{0, \ldots, m-1\}$:

$$\Pr_{h \in \mathcal{H}}[h(x) = i \wedge h(y) = j] = \frac{1}{m^2}.$$

**Definition ($k$-universal).** A family $\mathcal{H}$ is *$k$-universal* if for any $k$ distinct keys $x_1, \ldots, x_k$ and any values $v_1, \ldots, v_k$:

$$\Pr_{h \in \mathcal{H}}[h(x_1) = v_1 \wedge \cdots \wedge h(x_k) = v_k] = \frac{1}{m^k}.$$

**Theorem (collision bound for universal hashing).** If $h$ is chosen from a universal family $\mathcal{H}$ and $n$ keys are stored in a table of size $m$, then for any key $x$, the expected number of other keys colliding with $x$ is at most $n / m$ (the load factor $\alpha$).

*Proof.* Let $S$ be the set of $n$ stored keys. For a query key $x \notin S$, the number of collisions is $\sum_{y \in S} \mathbf{1}[h(x) = h(y)]$. By linearity of expectation: $\mathbb{E}[\text{collisions}] = \sum_{y \in S} \Pr[h(x) = h(y)] \le n \cdot (1/m) = n/m$. $\square$

### Counterexamples to common slips

- *"Universal hashing prevents all collisions."* No. It bounds the expected number of collisions; individual pairs can still collide. Perfect hashing (FKS) is needed for zero collisions.

- *"A universal hash family must be large."* The family $h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$ has only $p(p-1)$ functions, which is polynomial in the key range. Universality does not require an exponentially large family.

- *"Strongly universal implies universal."* It does: strong universality gives $\Pr[h(x) = h(y)] = \sum_i \Pr[h(x) = i \wedge h(y) = i] = m \cdot (1/m^2) = 1/m$, which satisfies the universal bound with equality.

### Key theorem with proof [Intermediate+]

**Theorem (FKS construction).** *Given a static set $S$ of $n$ keys, the FKS scheme constructs a hash table using $O(n)$ space with $O(1)$ worst-case lookup time. The construction succeeds with high probability.*

*Proof.* Top level: choose a universal hash function $h$ mapping keys to $m = n$ slots. Let $n_i$ be the number of keys landing in slot $i$. By universality, $\mathbb{E}[n_i] = 1$ for each $i$ (if the load is balanced). The total space for second-level tables is $\sum_i n_i^2$. By linearity of expectation:

$$\mathbb{E}\left[\sum_i n_i^2\right] = \mathbb{E}\left[\sum_i n_i\right] + 2 \mathbb{E}\left[\sum_{i} \binom{n_i}{2}\right] = n + 2 \cdot \text{(total collisions)}.$$

The total collisions are $\sum_{x \ne y \in S} \Pr[h(x) = h(y)] \le \binom{n}{2} / m = n(n-1)/(2n) < n/2$. So $\mathbb{E}[\sum n_i^2] < n + n = 2n = O(n)$.

By Markov's inequality, $\Pr[\sum n_i^2 > 4n] < 1/2$. So a random $h$ gives total second-level space at most $4n$ with probability at least $1/2$. Repeat until this holds.

Second level: for each slot $i$ with $n_i$ keys, choose a universal hash function $h_i$ mapping to $m_i = n_i^2$ slots. By the birthday paradox, the collision probability for any pair is $\le 1/n_i^2$, so the expected number of collisions is $\binom{n_i}{2}/n_i^2 < 1/2$. Markov gives: $\Pr[\text{any collision}] < 1/2$. Repeat until collision-free.

Lookup: compute $j = h(x)$, then $h_j(x)$. Two hash computations, $O(1)$. $\square$

**Bridge.** Universal hashing is the foundational reason hash tables achieve $O(1)$ expected performance without assumptions about the input distribution: random choice from a structured family ensures that no adversary can force pathological behaviour. This builds toward the randomised algorithms of [47.05.02] where randomisation provides similar adversarial robustness for sorting and searching, and it appears again in [47.05.03] where Miller-Rabin primality testing uses random witnesses with the same collision-probability argument. The central insight is that the pairwise independence property of universal hash families gives strong concentration bounds on the number of collisions, and putting these together with the two-level FKS construction yields $O(1)$ worst-case lookups in $O(n)$ space — the static equivalent of an ideal hash table.

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 4 (medium, short answer).**

Prove that the family $\mathcal{H} = \{h_{a,b} : h_{a,b}(x) = ((ax + b) \bmod p) \bmod m, a \in \{1, \ldots, p-1\}, b \in \{0, \ldots, p-1\}\}$ is universal, where $p$ is prime and $p > m$.

<details class="exercise__hint"><summary>Hint</summary>

For $x \ne y$, count the pairs $(a, b)$ with $h_{a,b}(x) = h_{a,b}(y)$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

For $x \ne y$ and $a \in \{1, \ldots, p-1\}$: the values $r_x = (ax + b) \bmod p$ and $r_y = (ay + b) \bmod p$ are distinct (since $a(x-y) \not\equiv 0 \pmod p$). The pair $(r_x, r_y)$ is uniformly distributed over all pairs of distinct values in $\{0, \ldots, p-1\}$ as $(a, b)$ varies. The condition $h(x) = h(y)$ requires $r_x \equiv r_y \pmod m$. For each $r_y$, the number of $r_x$ with $r_x \equiv r_y \pmod m$ is at most $\lceil p/m \rceil \le (p + m - 1)/m$. So $\Pr[h(x) = h(y)] \le \lceil p/m \rceil / p \le 1/m + 1/p < 2/m$. This gives $O(1/m)$; a tighter analysis shows $\Pr \le 1/m$ exactly.

</details>

</aside>

In a hash table of size $m = 100$ using a universal hash family, $n = 200$ keys are inserted. What is the expected number of collisions for a query key not in the table?

<details class="exercise__answer"><summary>Answer</summary>

By the collision bound theorem: expected collisions $\le n/m = 200/100 = 2$. So the expected chain length for a query is at most 2.

**Exercise 6 (medium, symbolic).**

In the FKS construction, if the top-level hash produces slot sizes $n_1 = 3, n_2 = 5, n_3 = 2, n_4 = 4$ for a set of $n = 14$ keys, what is the total space for the second-level tables?

<details class="exercise__answer"><summary>Answer</summary>

Each slot $i$ uses a second-level table of size $n_i^2$. Total space: $9 + 25 + 4 + 16 = 54 = O(n)$. This is less than $4n = 56$ as guaranteed by the Markov bound analysis.

</details>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 7 (hard, short answer).**

Prove that the expected number of trials to find a top-level hash function $h$ for FKS with total second-level space $O(n)$ is $O(1)$.

<details class="exercise__answer"><summary>Answer</summary>

We showed $\mathbb{E}[\sum n_i^2] \le 2n$. By Markov's inequality, $\Pr[\sum n_i^2 > 4n] < 1/2$. So each random choice of $h$ succeeds (total space $\le 4n$) with probability at least $1/2$. The number of trials until first success follows a geometric distribution with parameter $p \ge 1/2$, giving expected number of trials $\le 1/p \le 2 = O(1)$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 8 (hard, short answer).**

A $k$-universal hash family generalises universality from pairs to $k$-tuples. Show that a 2-universal family is also universal, but a universal family need not be 2-universal.

<details class="exercise__answer"><summary>Answer</summary>

2-universal implies universal: for $x \ne y$ and any $i$, $\Pr[h(x) = i \wedge h(y) = i] = 1/m^2$. Summing over $i$: $\Pr[h(x) = h(y)] = m \cdot (1/m^2) = 1/m$. So the collision probability is exactly $1/m \le 1/m$. Universal does not imply 2-universal: the family $h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$ is universal but not necessarily 2-universal. A universal family only constrains pairwise collisions, not the joint distribution of $h(x)$ and $h(y)$. For example, the family of all constant functions (every $h$ maps all keys to the same slot) has $\Pr[h(x) = h(y)] = 1 \le 1/m$ only when $m = 1$, but for $m > 1$ it is not universal. A more careful example: take $\mathcal{H} = \{h_0, h_1\}$ where $h_0$ maps all even keys to 0 and all odd to 1, and $h_1$ maps all keys to 0. Then $\Pr[h(x) = h(y)] = 1/2$ for $m = 2$, satisfying universality, but $\Pr[h(0) = 0 \wedge h(1) = 0] = 1/2 \ne 1/4 = 1/m^2$, violating 2-universality.

</details>

</aside>

## Advanced results [Master]

**Theorem 1 (cuckoo hashing).** For dynamic sets, cuckoo hashing (Pagh and Rodler, 2004) achieves $O(1)$ worst-case lookup using two hash functions $h_1, h_2$ and two tables. Each key $x$ is stored in either table 1 at position $h_1(x)$ or table 2 at position $h_2(x)$. Lookup checks both locations. Insertion may require relocating existing keys, and fails (requiring rehash) if a cycle forms. For load factor $\alpha < 1/2$, the expected insertion time is $O(1)$ and the failure probability is exponentially small.

**Theorem 2 (power of two choices).** If each key is hashed to $d \ge 2$ random locations and placed in the least loaded, the maximum load drops from $\Theta(\log n / \log \log n)$ (for $d = 1$) to $\Theta(\log \log n)$ [ref: TODO_REF CLRS §11.3]. This "power of two choices" phenomenon has far-reaching consequences in load balancing and parallel computing.

**Theorem 3 (dynamic perfect hashing).** Dietzfelbinger et al. (1994) extended FKS to dynamic sets, supporting insertions and deletions while maintaining $O(1)$ worst-case lookup and $O(n)$ space. The amortized insertion and deletion cost is $O(1)$ with high probability. The key idea is to rebuild the second-level tables incrementally when a slot grows beyond its allocated size.

**Synthesis.** Universal hashing is the foundational reason hash tables achieve robust $O(1)$ expected performance: the pairwise independence property bounds collision probability regardless of the input distribution, and the FKS construction extends this to $O(1)$ worst-case performance for static sets. The central insight is that random choice from a structured algebraic family (linear functions over finite fields) gives the same probabilistic guarantees as a truly random function, and putting these together with the two-level construction of FKS yields a data structure whose space and time are simultaneously optimal. This builds toward the randomised algorithms of [47.05.02] where random choices from structured families provide adversarial robustness in sorting and searching, and appears again in [47.03.01] where the BPP class captures the same principle at the complexity-theoretic level: random choices from polynomial-size families suffice for efficient computation.

## Full proof set [Master]

**Proposition 1 (collision bound).** *If $h$ is chosen from a universal family mapping to $\{0, \ldots, m-1\}$ and $n$ keys are stored, then for any key $x$, the expected number of collisions with $x$ is at most $n/m$.*

*Proof.* $\mathbb{E}[\sum_{y \in S, y \ne x} \mathbf{1}[h(x) = h(y)]] = \sum_{y \ne x} \Pr[h(x) = h(y)] \le (n-1)/m \le n/m$. $\square$

**Proposition 2 (FKS space bound).** *In the FKS construction with top-level table size $m = n$, the expected total second-level space is at most $2n$.*

*Proof.* $\mathbb{E}[\sum_i n_i^2] = \mathbb{E}[\sum_i n_i + 2\binom{n_i}{2}] = n + 2 \sum_{x \ne y} \Pr[h(x) = h(y)] \le n + 2\binom{n}{2}/m \le n + n = 2n$. $\square$

**Proposition 3 (second-level success probability).** *For a slot with $n_i$ keys and second-level table size $m_i = n_i^2$, a randomly chosen universal hash function is collision-free with probability at least $1/2$.*

*Proof.* The expected number of collisions is $\binom{n_i}{2}/n_i^2 = n_i(n_i - 1)/(2n_i^2) < 1/2$. By Markov's inequality, $\Pr[\text{at least 1 collision}] < 1/2$, so $\Pr[\text{collision-free}] \ge 1/2$. $\square$

## Historical & philosophical context [Master]

Universal hashing was introduced by Larry Carter and Mark Wegman in their 1979 paper "Universal Classes of Hash Functions" [ref: TODO_REF CLRS §11.3]. Their key insight was that a randomly chosen hash function from a carefully designed algebraic family provides the same probabilistic guarantees as a truly random function, while being efficiently computable. The $h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$ construction is their most famous example.

The FKS construction was introduced by Michael Fredman, Janos Komlos, and Endre Szemeredi in 1984, proving that static hash tables can achieve $O(1)$ worst-case lookup in $O(n)$ space — resolving a question that had been open since the invention of hash tables. The result was surprising because it showed that perfect hashing does not require quadratic space.

Cuckoo hashing was introduced by Rasmus Pagh and Flemming Rodler in 2004, extending the FKS ideas to dynamic sets with $O(1)$ worst-case lookups. The name comes from the cuckoo bird's habit of pushing eggs out of other birds' nests, mirroring the algorithm's eviction strategy.
