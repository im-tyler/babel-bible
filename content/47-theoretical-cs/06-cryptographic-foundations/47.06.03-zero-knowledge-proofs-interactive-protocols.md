## Intuition [Beginner]

You want to prove to a friend that you solved a Sudoku puzzle without revealing the solution. Zero-knowledge proofs make this possible. You convince the verifier that a statement is true while revealing nothing beyond its truth.

The classic example is the cave. Imagine a circular cave with a magic door in the middle that requires a secret password to open. You want to prove you know the password without revealing it. Your friend stands outside the cave and watches you enter from one side. They then shout "come out the left!" or "come out the right!" at random. If you know the password, you can always comply (opening the door if needed). If you do not know it, you can only comply half the time (when they ask for the side you entered from). Repeating this 20 times, the probability of fooling your friend drops to less than one in a million.

A zero-knowledge proof has three properties. Completeness: an honest prover with a valid proof convinces an honest verifier. Soundness: a cheating prover cannot convince the verifier of a false statement. Zero-knowledge: the verifier learns nothing beyond the truth of the statement, because everything they saw could have been simulated without the prover.

The deep result is that every NP statement has a zero-knowledge proof. You can prove you know a 3-colouring of a graph, a satisfying assignment for a formula, or a Hamiltonian cycle, without revealing any information about the witness.

## Visual [Beginner]

```
Zero-knowledge proof for Graph 3-Colouring:

  Prover has: valid 3-colouring (R, G, B) of the graph
  Verifier sees: the graph (but not the colouring)

  Round i:
    1. Prover: randomly permute colours (R->B, G->R, B->G)
    2. Prover: commit to each vertex's permuted colour
    3. Verifier: pick a random edge (u, v)
    4. Prover: open commitments for u and v
    5. Verifier: check u and v have different colours

  If graph IS 3-colourable: prover always passes (colours differ)
  If graph is NOT 3-colourable: >= 1 edge has same colour
    Verifier catches this with prob >= 1/|E| per round
```

| Property | Meaning | How it holds |
|---|---|---|
| Completeness | True statement: honest prover convinces verifier | Valid 3-colouring means every edge has different colours |
| Soundness | False statement: cheating prover caught with high prob | Invalid colouring has at least one bad edge, verifier finds it |
| Zero-knowledge | Verifier learns nothing beyond validity | Simulator picks random edge first, colours only that edge correctly |

## Worked example [Beginner]

We do a zero-knowledge proof that the graph $K_3$ (triangle with vertices $a, b, c$) is 3-colourable. The actual colouring is: $a = R, b = G, c = B$.

**Round 1.** Prover picks permutation $\pi$: swap R and G, keep B. New colours: $a = G, b = R, c = B$. Prover commits to each: $\text{com}(a, G), \text{com}(b, R), \text{com}(c, B)$. Sends commitments to verifier.

Verifier picks edge $(a, c)$. Prover opens $\text{com}(a, G)$ and $\text{com}(c, B)$. Verifier checks: $G \neq B$. Pass.

**Round 2.** Prover picks new permutation $\pi'$: rotate $(R, G, B) \to (B, R, G)$. New colours: $a = B, b = R, c = G$. Commits and sends.

Verifier picks edge $(b, a)$. Prover opens: $b = R, a = B$. Verifier checks: $R \neq B$. Pass.

After many rounds, the verifier is convinced. But the verifier never sees the original colouring, only random permutations of it. Each opened pair is consistent with many possible colourings, so no information about the original is revealed.

## Formal definition [Intermediate+]

**Definition (interactive proof system).** An interactive proof system for a language $L$ is a pair $(P, V)$ where $P$ is computationally unbounded and $V$ is PPT, satisfying:

- *(Completeness)* For every $x \in L$: $\Pr[\langle P, V \rangle(x) = 1] = 1$.
- *(Soundness)* For every $x \notin L$ and every $P^*$: $\Pr[\langle P^*, V \rangle(x) = 1] \leq 1/3$.

**Definition (computational zero-knowledge).** An interactive proof $(P, V)$ for $L$ is *computational zero-knowledge* if there exists a PPT simulator $S$ such that for every PPT $V^*$, the following distributions are computationally indistinguishable:

$$\{\text{View}_{V^*}(\langle P, V^* \rangle(x))\}_{x \in L} \approx_c \{S(x, V^*)\}_{x \in L}.$$

**Definition (commitment scheme).** A commitment scheme is a pair $(Commit, Open)$ where $Commit(m, r)$ produces a commitment $c$ to message $m$ using randomness $r$, satisfying:

- *(Binding)* It is computationally infeasible to find $(m_1, r_1, m_2, r_2)$ with $m_1 \neq m_2$ and $Commit(m_1, r_1) = Commit(m_2, r_2)$.
- *(Hiding)* $Commit(m_1, r_1)$ and $Commit(m_2, r_2)$ are computationally indistinguishable for any $m_1, m_2$.

### Counterexamples to common slips

- *"Zero-knowledge means the proof reveals no information."* The proof reveals that the statement is true. Zero-knowledge means the proof reveals nothing *beyond* the truth of the statement. The distinction matters.

- *"Any interactive proof is zero-knowledge."* No. A standard interactive proof may leak information about the witness. Zero-knowledge is an additional property requiring the existence of a simulator.

- *"Zero-knowledge proofs are always non-interactive."* The basic definition is interactive. Non-interactive zero-knowledge (NIZK) is a special case requiring a common reference string. The Fiat-Shamir heuristic converts some interactive ZK proofs to non-interactive ones in the random oracle model.

### Key theorem with proof [Intermediate+]

**Theorem (GMW: every NP language has a ZK proof).** *Assuming one-way functions exist, every language in NP has a computational zero-knowledge proof system.*

*Proof sketch.* By NP-completeness, it suffices to give a ZK proof for Graph 3-Colouring (3COL). The reduction from any NP language to 3COL is polynomial-time.

*Protocol.* Let $G = (V, E)$ be a graph and $\varphi : V \to \{1, 2, 3\}$ a valid 3-colouring (the witness). Repeat $|E|^2$ times:

1. Prover picks a random permutation $\pi$ of $\{1, 2, 3\}$ and commits to $\pi(\varphi(v))$ for each $v \in V$.
2. Verifier picks a random edge $(u, v) \in E$.
3. Prover opens the commitments for $u$ and $v$.
4. Verifier checks $\pi(\varphi(u)) \neq \pi(\varphi(v))$.

*Completeness.* If $\varphi$ is valid, $\varphi(u) \neq \varphi(v)$ for every edge, so $\pi(\varphi(u)) \neq \pi(\varphi(v))$. The verifier always accepts.

*Soundness.* If $G$ is not 3-colourable, every colouring has at least one edge with the same colour on both endpoints. The verifier finds this edge with probability at least $1/|E|$ per round. Over $|E|^2$ rounds, the probability the cheating prover escapes is at most $(1 - 1/|E|)^{|E|^2} \leq e^{-|E|}$, which is negligible.

*Zero-knowledge.* The simulator $S$ for a single round works as follows: pick a random edge $(u^*, v^*)$ and a random pair of distinct colours $c_u, c_v$. Commit to $c_u$ for $u^*$, $c_v$ for $v^*$, and random values for all other vertices. Run the verifier $V^*$ to get its query edge $(u, v)$. If $(u, v) = (u^*, v^*)$, output the transcript. Otherwise, rewind and try again.

The simulator succeeds with probability $1/|E|$ per attempt, so the expected number of attempts is $|E|$. The output distribution is identical to the real interaction because: (1) the permutation is random in both, (2) the opened colours are a random pair of distinct colours in both, and (3) the unopened commitments are computationally hiding in both.

The commitment scheme exists if one-way functions exist (by Naor's theorem, 1991). $\square$

**Bridge.** The GMW theorem is the foundational reason zero-knowledge is universal for NP: by reducing any NP statement to 3-colouring and applying the commitment-based protocol, every claim of the form "I know a witness for $x \in L$" can be proven without revealing the witness. This builds toward the broader theory of cryptographic protocols where ZK proofs are composed into secure multiparty computation, and this is exactly the pattern that appears again in [47.06.04] where the Fiat-Shamir heuristic converts interactive ZK protocols into digital signatures. The central insight is that commitment schemes (which exist iff OWFs exist) provide the hiding and binding properties that make zero-knowledge possible, and putting these together with the interactive proof framework of [47.03.04] shows that ZK proofs add a privacy layer on top of the IP framework: IP = PSPACE captures what can be *proved*, while ZK captures what can be proved *privately*.

**Exercise 1 (easy, true-false).**

True or false: the GMW protocol for 3-colouring requires that the commitment scheme is perfectly hiding.

<details class="exercise__answer"><summary>Answer</summary>

**False.** Computational hiding suffices. The zero-knowledge property holds because the simulator's output is computationally indistinguishable from the real protocol, which follows from computational hiding of the commitments. Perfect zero-knowledge would require perfectly hiding commitments, which exist but are less efficient.

</details>

<aside class="exercise" data-difficulty="easy" data-type="symbolic" markdown="1">

**Exercise 2 (easy, symbolic).**

In the GMW protocol for 3-colouring, the protocol is repeated $|E|^2$ times. Why is this number of repetitions chosen? What is the probability a cheating prover succeeds in all rounds?

<details class="exercise__answer"><summary>Answer</summary>

Each round catches a cheating prover with probability at least $1/|E|$ (since at least one edge is improperly coloured). The probability of escaping all $|E|^2$ rounds is at most $(1 - 1/|E|)^{|E|^2} \leq e^{-|E|}$, which is negligible in the graph size. The repetition count $|E|^2$ ensures soundness error drops below any desired threshold.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Construct a zero-knowledge proof for Graph Isomorphism: given two graphs $G_1$ and $G_2$, prove they are isomorphic without revealing the isomorphism.

<details class="exercise__answer"><summary>Answer</summary>

*Protocol.* Prover knows isomorphism $\sigma$ with $G_2 = \sigma(G_1)$.

1. Prover picks a random permutation $\pi$ and computes $H = \pi(G_1)$. Sends $H$.
2. Verifier sends a random challenge $i \in \{1, 2\}$.
3. If $i = 1$: prover sends $\pi$. Verifier checks $H = \pi(G_1)$.
4. If $i = 2$: prover sends $\pi \circ \sigma^{-1}$. Verifier checks $H = (\pi \circ \sigma^{-1})(G_2)$.

*Completeness:* If $G_1 \cong G_2$, the prover can answer both challenges correctly.

*Soundness:* If $G_1 \not\cong G_2$, no $H$ can be isomorphic to both. So the prover can prepare for at most one challenge, and fails with probability $1/2$ per round.

*Zero-knowledge:* Simulator picks random $i^*$ and $\psi$, computes $H = \psi(G_{i^*})$. If the verifier challenges $i = i^*$, output the transcript. Otherwise, rewind. The output is identical to the real protocol.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 4 (medium, short answer).**

Explain the difference between honest-verifier zero-knowledge (HVZK) and general zero-knowledge. Why does the GMW protocol achieve general ZK?

<details class="exercise__answer"><summary>Answer</summary>

HVZK means the simulator only needs to simulate the view of the honest verifier $V$ (who follows the protocol). General ZK requires simulation for *any* PPT verifier $V^*$ (who may deviate arbitrarily). The GMW protocol achieves general ZK because the simulator rewinds $V^*$: it runs $V^*$ to get its challenge, and if the challenge does not match the simulator's prepared response, it rewinds $V^*$ to the beginning of the round and tries again. This works because the commitment scheme hides the prepared response from $V^*$, so $V^*$'s challenge is independent of the simulator's trapdoor. The expected number of rewinding attempts is polynomial.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 5 (medium, symbolic).**

The Fiat-Shamir heuristic converts an interactive ZK protocol into a non-interactive one by replacing the verifier's challenge with a hash of the prover's first message. Show that if the hash function is modelled as a random oracle, this produces a secure NIZK.

<details class="exercise__answer"><summary>Answer</summary>

In the random oracle model, the hash function $H$ is a truly random function. The prover computes $H$ on their commitment message to get the challenge. Since $H$ is random, the challenge is uniformly distributed and independent of the commitment (from the prover's perspective, it is as if the verifier sent a random challenge).

Soundness in the random oracle model: a cheating prover must either (1) predict the output of $H$ on their commitment (impossible since $H$ is random), or (2) find two commitments that hash to challenges for which they can answer both (the "forking lemma" bounds this probability).

Zero-knowledge: the simulator programs the random oracle. It picks the challenge first, computes a commitment consistent with that challenge, and sets $H(\text{commitment}) = \text{challenge}$. The resulting transcript is identical to a real execution, and the random oracle programming is undetectable because the simulator's commitment has the same distribution as the real one.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Prove that commitment schemes exist if and only if one-way functions exist. (One direction: OWF implies commitment. For the other, show that a commitment scheme yields a OWF.)

<details class="exercise__answer"><summary>Answer</summary>

*OWF implies commitment (Naor 1991).* Given a PRG $G : \{0,1\}^n \to \{0,1\}^{3n}$ (from any OWF via HILL), define: to commit to bit 0, send $G(s)$ for random $s$; to commit to bit 1, send $G(s) \oplus r$ for random $r$ chosen by the receiver. Hiding follows from the PRG property (both $G(s)$ and $G(s) \oplus r$ look random). Binding follows from the fact that finding $s_0, s_1$ with $G(s_0) = G(s_1) \oplus r$ would break the PRG.

*Commitment implies OWF.* Given a commitment scheme $(Commit, Open)$, define $f(m, r) = Commit(m, r)$. This is polynomial-time computable. To invert: given $c$, find $(m, r)$ with $Commit(m, r) = c$. By the binding property, there is at most one valid opening (computationally), so inversion requires finding the committed message. If $f$ were easy to invert, the hiding property would be violated (an inverter could recover $m$ from $c$, distinguishing commitments to different messages). More formally, if $A$ inverts $f$ with probability $\varepsilon$, then $A(Commit(m, r))$ returns some $(m', r')$ with $Commit(m', r') = Commit(m, r)$. By binding, $m' = m$ with high probability, so $A$ recovers $m$ from the commitment, violating hiding. Rubric: full credit for both directions.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

A Sigma protocol is a 3-round interactive proof $(commit, challenge, response)$ with special honest-verifier zero-knowledge (SHVZK) and special soundness. Prove that any Sigma protocol can be compiled into a signature scheme using the Fiat-Shamir transform.

<details class="exercise__answer"><summary>Answer</summary>

Let $(P, V)$ be a Sigma protocol for relation $R$ with special soundness (given two accepting transcripts with the same commitment but different challenges, the witness can be extracted) and SHVZK (simulator produces valid transcript given just the challenge).

*Signature scheme.* Key generation: pick $(pk, sk) = (x, w)$ where $(x, w) \in R$. Sign message $m$: compute commitment $a = P_1(w, x)$, challenge $c = H(a, m)$, response $z = P_2(w, x, a, c)$. Signature is $(a, c, z)$.

Verify: check $V(x, a, c, z) = 1$ and $c = H(a, m)$.

*Unforgeability (sketch).* By the forking lemma: if a forger produces a valid signature $(a, c, z)$ on a new message $m$, rewind the forger with a different random oracle response $c' \neq c$ to the query $H(a, m)$. With probability $\varepsilon/q_H$ (where $q_H$ is the number of hash queries), the forger produces $(a, c', z')$ also valid. By special soundness, the two transcripts $(a, c, z)$ and $(a, c', z')$ allow extraction of the witness $w$, breaking the hardness of $R$.

*Zero-knowledge property of signatures.* The Fiat-Shamir transform produces a "proof of knowledge" in the random oracle model: anyone can verify the signature, but the signature reveals nothing about $w$ beyond knowledge of it, because the simulator can program $H$ to produce valid signatures without $w$. Rubric: full credit for the forking lemma argument and the extraction.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 8 (hard, short answer).**

Explain why perfect zero-knowledge for all of NP would imply that the polynomial hierarchy collapses. What does this say about the relationship between statistical and computational ZK?

<details class="exercise__answer"><summary>Answer</summary>

If every NP language had a perfect zero-knowledge proof (PZK), then the PZK class would contain all of NP. But it is known that PZK is contained in AM (the Arthur-Merlin class), and if NP is contained in AM, then the polynomial hierarchy collapses to its second level (PH = AM). This is because Boppora, Feldman and Fortnow showed that PZK is contained in SZE (statistical zero-knowledge), which is contained in AM.

In practice, the GMW protocol achieves *computational* zero-knowledge (CZK) for all of NP, not perfect. The gap between statistical and computational ZK is the same as the gap between information-theoretic and computational security: statistical ZK provides stronger guarantees but is more limited in scope (SZK does not contain all of NP unless PH collapses), while computational ZK relies on hardness assumptions but achieves universality.

</details>

</aside>

## Advanced results [Master]

Three directions extend the ZK framework.

**Result 1 (NIZK and the common reference string).** Blum, Feldman and Micali (1988) introduced non-interactive zero-knowledge proofs in the common reference string (CRS) model. The prover sends a single message, and the verifier checks it using a publicly known CRS. The CRS is generated by a trusted setup. Feige, Lapidot and Shamir (1990) showed that every NP language has a NIZK in the CRS model assuming one-way functions. NIZKs are essential for efficient cryptographic protocols because they eliminate the interaction rounds [ref: TODO_REF Arora-Barak Ch. 8].

**Result 2 (zk-SNARKs).** Succinct Non-interactive ARguments of Knowledge (SNARKs) are NIZKs with sublinear proof size and sublinear verification time. The construction of Bitansky, Canetti, Chiesa and Tromer (2012) uses arithmetic circuit satisfiability and elliptic curve pairings. A zk-SNARK for an NP statement of size $n$ has proof size $O(1)$ group elements and verification time $O(1)$ pairings plus input processing. zk-SNARKs are used in blockchain systems (Zcash, Ethereum) for privacy-preserving transactions [ref: TODO_REF Arora-Barak Ch. 8].

**Result 3 (ZK and secure multiparty computation).** The GMW theorem extends beyond proofs to computation: Goldreich, Micali and Wigderson (1987) proved that any polynomial-time function can be computed by $n$ parties such that each party learns only the output and nothing about other parties' inputs. The construction uses ZK proofs as a subroutine to enforce honest behaviour. This is the foundational result of secure multiparty computation (MPC), with applications to privacy-preserving machine learning, threshold cryptography, and distributed voting [ref: TODO_REF Arora-Barak Ch. 8].

**Synthesis.** Zero-knowledge proofs are the foundational reason privacy and verification can coexist: the simulator paradigm ensures that interaction reveals nothing beyond the statement's truth, and this builds toward the entire field of privacy-preserving cryptography where ZK proofs compose into MPC, NIZK, and SNARKs. The central insight is that the three properties (completeness, soundness, zero-knowledge) are not in tension: the same commitment scheme that enables soundness (binding) also enables zero-knowledge (hiding), and this is exactly the pattern that appears again in [47.06.01] where one-way functions provide both pseudorandomness and one-wayness. Putting these together with the interactive proof theory of [47.03.04] shows that ZK proofs are interactive proofs refined with a privacy guarantee, and the fact that every NP statement has a ZK proof means that privacy is free: any proof can be made zero-knowledge without changing what can be proved, only how it is proved.

## Full proof set [Master]

**Proposition 1 (GMW protocol for 3COL is zero-knowledge).** *The GMW protocol for Graph 3-Colouring is computational zero-knowledge assuming computationally hiding commitments.*

*Proof.* We construct a simulator $S$ for a single round. $S$ receives $G = (V, E)$ and has black-box access to verifier $V^*$.

$S$ picks a random edge $(u^*, v^*) \in E$ and random distinct colours $c_{u^*}, c_{v^*} \in \{1, 2, 3\}$. For all other vertices $w \notin \{u^*, v^*\}$, $S$ commits to a random colour. $S$ sends all commitments to $V^*$ and receives its challenge edge $(u, v)$.

If $(u, v) = (u^*, v^*)$, $S$ opens the commitments for $u$ and $v$ (revealing $c_{u^*}, c_{v^*}$) and outputs the transcript. If $(u, v) \neq (u^*, v^*)$, $S$ rewinds $V^*$ to the beginning and tries again.

*Success probability.* Each attempt succeeds with probability at least $1/|E|$ (the probability $V^*$ picks the edge $S$ prepared). Expected number of attempts: $|E|$.

*Indistinguishability.* The real transcript has: commitments to $\pi(\varphi(v))$ for random $\pi$, opened colours $\pi(\varphi(u)), \pi(\varphi(v))$ for the queried edge. The simulated transcript has: commitments to random values (except the prepared edge), opened colours $c_{u^*}, c_{v^*}$ (random distinct pair). By the hiding property of commitments, the unopened commitments are indistinguishable in both distributions. The opened colours are uniformly random distinct pairs in both. So the full transcripts are computationally indistinguishable. $\square$

**Proposition 2 (composition of ZK proofs).** *If $(P_1, V_1)$ is a ZK proof for $L_1$ and $(P_2, V_2)$ is a ZK proof for $L_2$, then the parallel composition (run both protocols simultaneously) is a ZK proof for $L_1 \wedge L_2$.*

*Proof.* The composed simulator $S$ runs $S_1$ and $S_2$ independently to simulate the transcripts for both protocols. Since $S_1$'s output is indistinguishable from the real $P_1$-$V_1^*$ interaction, and $S_2$'s output is indistinguishable from the real $P_2$-$V_2^*$ interaction, the joint output is indistinguishable from the real joint interaction by the hybrid argument (replace one protocol's transcript at a time). $\square$

## Connections [Master]

- [47.06.01] — One-way functions provide the commitment schemes needed for ZK proofs; the GMW theorem that every NP language has a ZK proof is conditional on OWF existence.

- [47.03.04] — IP = PSPACE establishes the power of interactive proofs; ZK proofs add the zero-knowledge property, showing that privacy is compatible with the full power of IP.

## Historical & philosophical context [Master]

Zero-knowledge proofs were introduced by Goldwasser, Micali and Rackoff (1985) in their paper "The Knowledge Complexity of Interactive Proof Systems." The concept was initially met with scepticism—the idea that one could prove knowledge without revealing it seemed paradoxical. The GMW theorem (1986, published 1991) established that ZK proofs exist for all of NP, transforming the concept from a curiosity into a fundamental tool.

The philosophical significance is that knowledge and privacy are not in conflict. Any proof can be restructured to reveal nothing beyond its conclusion. This has profound implications: in principle, every authentication, verification, and audit process can be performed without exposing private data. The gap between principle and practice is engineering, not theory [ref: TODO_REF Arora-Barak Ch. 8].

The development of zk-SNARKs (2012-2013) brought ZK proofs into practical use. Zcash (2016) was the first large-scale deployment, using zk-SNARKs to prove transaction validity without revealing sender, receiver, or amount. The technology has since spread to Ethereum, where zk-rollups use SNARKs for scalability and privacy.
