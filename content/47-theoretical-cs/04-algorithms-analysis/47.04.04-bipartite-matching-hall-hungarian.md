## Intuition [Beginner]

Imagine a group of workers and a set of jobs. Each worker is qualified for some subset of the jobs. You want to assign workers to jobs so that no two workers get the same job and no worker gets more than one job. This is a bipartite matching problem.

The word "bipartite" means the graph has two sides (workers and jobs), and all edges go between the two sides — never within the same side. A "matching" is a set of edges where no two edges share a vertex — each worker gets at most one job, and each job gets at most one worker.

The central question is: what is the largest possible matching? How many workers can you simultaneously assign?

Hall's theorem gives a beautiful answer. It says: every worker can be matched to a distinct job if and only if for every subset of workers, the total number of jobs they are collectively qualified for is at least as large as the number of workers in the subset. This makes intuitive sense: if three workers are collectively qualified for only two jobs, you cannot match all three.

The augmenting-path algorithm finds a maximum matching by starting with any matching and repeatedly improving it. An augmenting path is a path that alternates between unmatched and matched edges, starting and ending at unmatched vertices. Flipping the edges along this path (matched becomes unmatched and vice versa) increases the matching size by one.

When no augmenting path exists, the matching is maximum. This is a consequence of the max-flow min-cut theorem applied to the flow network constructed from the bipartite graph.

The Hungarian algorithm solves the weighted version: each worker-job pair has a cost or value, and you want the matching with minimum total cost (or maximum value). It runs in $O(n^3)$ time by maintaining a feasible "potential" function on the vertices and iteratively improving the matching.

## Visual [Beginner]

A bipartite graph with a matching:

```
Workers (U)          Jobs (V)
  w1 ──────── j1
  w2 ──────── j2
  │  ╲           │
  │    ╲         │
  w3 ──────── j3
              /
            /
  w4 ──────── j4
```

The bold edges form a matching of size 3: $w1\text{-}j1$, $w2\text{-}j2$, $w3\text{-}j3$. Worker $w4$ is unmatched.

An augmenting path: $w4 \to j3 \to w3 \to j4$. This path has 3 edges, alternating unmatched (w4-j3), matched (w3-j3), unmatched (w3-j4). Flipping:

```
Before: M = {w1-j1, w2-j2, w3-j3}, size 3
After:  M' = {w1-j1, w2-j2, w3-j4, w4-j3}, size 4 (perfect!)
```

| Concept | Definition | Condition |
|---|---|---|
| Matching | Set of disjoint edges | No shared vertices |
| Maximum matching | Largest possible | No augmenting path exists |
| Perfect matching | Covers all vertices | $|M| = |U| = |V|$ |
| Hall's condition | $|N(S)| \ge |S|$ for all $S \subseteq U$ | Necessary and sufficient for perfect matching |

## Worked example [Beginner]

Three students $s_1, s_2, s_3$ apply for three projects $p_1, p_2, p_3$. The compatibility edges are: $s_1$ can do $p_1, p_2$; $s_2$ can do $p_1, p_3$; $s_3$ can do $p_2, p_3$.

Start with an empty matching. Find an augmenting path from $s_1$ to $p_1$: just the edge $(s_1, p_1)$. Add it. Matching: $\{(s_1, p_1)\}$, size 1.

Find an augmenting path from $s_2$. $s_2$ is unmatched. Try $p_1$: matched to $s_1$. From $s_1$, try $p_2$: unmatched. Path: $s_2 \to p_1 \to s_1 \to p_2$. Flip: remove $(s_1, p_1)$, add $(s_2, p_1)$ and $(s_1, p_2)$. Matching: $\{(s_2, p_1), (s_1, p_2)\}$, size 2.

Find an augmenting path from $s_3$. $s_3$ is unmatched. Try $p_2$: matched to $s_1$. From $s_1$, all neighbours ($p_1, p_2$) are explored. $p_1$ is matched to $s_2$. From $s_2$, try $p_3$: unmatched. Path: $s_3 \to p_2 \to s_1 \to p_1 \to s_2 \to p_3$. Flip: remove $(s_1, p_2)$ and $(s_2, p_1)$, add $(s_3, p_2)$, $(s_1, p_1)$, and $(s_2, p_3)$. Matching: $\{(s_3, p_2), (s_1, p_1), (s_2, p_3)\}$, size 3. This is a perfect matching.

## Formal definition [Intermediate+]

**Definition (bipartite matching).** Let $G = (U \cup V, E)$ be a bipartite graph. A *matching* is a set $M \subseteq E$ such that no two edges in $M$ share a common endpoint. A *maximum matching* maximises $|M|$. A *perfect matching* satisfies $|M| = |U| = |V|$.

**Theorem (Hall's marriage theorem).** *Let $G = (U, V, E)$ be a bipartite graph with $|U| \le |V|$. There exists a matching that covers every vertex in $U$ if and only if for every $S \subseteq U$:*

$$|N(S)| \ge |S|.$$

*Proof.* The forward direction is immediate: if a matching covers $U$, then every $S \subseteq U$ is matched to distinct vertices in $V$, so $|N(S)| \ge |S|$.

For the backward direction, we prove the contrapositive. Suppose no matching covers $U$. Let $M$ be a maximum matching, and let $u \in U$ be unmatched. Consider the set $Z$ of all vertices reachable from $u$ by alternating paths (paths alternating between edges not in $M$ and edges in $M$). Let $S = Z \cap U$ and $T = Z \cap V$.

Every vertex in $T$ is matched (reached by a non-matching edge from $S$ and followed by a matching edge to $S$), so $|T| = |S| - 1$ (one unmatched vertex $u$ is in $S$). Every neighbour of $S$ is in $T$ (otherwise there would be an augmenting path, contradicting maximality). So $N(S) = T$ and $|N(S)| = |T| = |S| - 1 < |S|$. Hall's condition is violated. $\square$

**Theorem (König's theorem).** *In a bipartite graph, the size of a maximum matching equals the size of a minimum vertex cover.*

*Proof.* The max-flow reduction connects this to the max-flow min-cut theorem. Construct a flow network with source $s$ connected to $U$ (capacity 1), edges from $U$ to $V$ (capacity $\infty$), and $V$ connected to sink $t$ (capacity 1). The maximum flow equals the maximum matching. The minimum $s$-$t$ cut corresponds to a vertex cover: the $U$-side vertices in the cut plus the $V$-side vertices in the cut form a vertex cover of the same size as the max flow. $\square$

### Counterexamples to common slips

- *"Hall's theorem applies to any graph."* Hall's theorem specifically requires a bipartite graph. For general graphs, the Tutte-Berge formula replaces Hall's condition.

- *"The Hungarian algorithm finds maximum-cardinality matchings."* The Hungarian algorithm finds maximum-weight matchings in weighted bipartite graphs. For unweighted maximum matching, the augmenting-path or Hopcroft-Karp algorithm is used.

- *"A maximum matching is the same as a maximal matching."* A maximal matching cannot be extended by adding an edge. A maximum matching has the largest possible size. A maximal matching can be half the size of a maximum matching.

### Key theorem with proof [Intermediate+]

**Theorem (augmenting-path algorithm).** *A matching $M$ in a bipartite graph $G$ is maximum if and only if there is no $M$-augmenting path.*

*Proof.* If an $M$-augmenting path $P$ exists, flipping edges along $P$ produces a matching of size $|M| + 1$, so $M$ is not maximum.

Conversely, suppose $M$ is not maximum. Let $M^*$ be a maximum matching with $|M^*| > |M|$. Consider the symmetric difference $M \oplus M^* = (M \setminus M^*) \cup (M^* \setminus M)$. Each vertex is incident to at most one edge from $M$ and at most one from $M^*$, so every component of $M \oplus M^*$ is a path or an even cycle. Since $|M^*| > |M|$, there must be a component that is a path starting and ending with $M^*$-edges. This path starts and ends at $M$-unmatched vertices and alternates between $M^*$-edges (not in $M$) and $M$-edges (in $M$). This is an $M$-augmenting path. $\square$

**Theorem (Hungarian algorithm complexity).** *The Hungarian algorithm finds a maximum-weight perfect matching in a complete bipartite graph $K_{n,n}$ with edge weights $w_{ij}$ in $O(n^3)$ time.*

*Proof sketch.* The algorithm maintains a feasible potential $(u, v)$ where $u_i$ for $i \in U$ and $v_j$ for $j \in V$ satisfy $u_i + v_j \ge w_{ij}$ for all $i, j$. The equality subgraph $G_=$ has edge $(i, j)$ iff $u_i + v_j = w_{ij}$. The algorithm iteratively:

1. Finds a maximum matching in $G_=$ (using the augmenting-path method).
2. If the matching is perfect, it is optimal by LP duality.
3. If not, updates the potential to add edges to $G_=$ (tightening the dual) and repeats.

Each potential update increases the size of the matching or adds edges, and the total number of iterations is $O(n)$, each taking $O(n^2)$ time. Total: $O(n^3)$. $\square$

**Bridge.** The augmenting-path method for bipartite matching builds toward the entire theory of combinatorial optimisation as the interplay between primal solutions (matchings) and dual certificates (vertex covers, potentials): the central insight is that Hall's condition provides the structural reason a matching exists, and this is exactly what makes bipartite matching the model problem for the max-flow min-cut framework. This appears again in [47.04.03] where the Ford-Fulkerson algorithm generalises augmenting paths from matchings to arbitrary flows, and in [47.04.06] where minimum spanning trees use a dual cut-based optimality condition analogous to König's theorem. The foundational reason bipartite matching is tractable is that the bipartite structure prevents the "blossom" obstructions that Edmonds' general matching algorithm must handle. Putting these together with the Hungarian algorithm shows that the primal-dual framework — simultaneously maintaining a feasible matching and a feasible potential — is the generalises from the specific bipartite setting to all of linear programming.

**Exercise 3 (medium, symbolic).**

Let $G$ be a bipartite graph with parts $U = \{u_1, u_2, u_3, u_4\}$ and $V = \{v_1, v_2, v_3, v_4\}$ where $u_i$ is adjacent to $v_j$ iff $i \ne j$. Find a maximum matching and a minimum vertex cover, verifying König's theorem.

<details class="exercise__answer"><summary>Answer</summary>

A perfect matching: $M = \{(u_1, v_2), (u_2, v_3), (u_3, v_4), (u_4, v_1)\}$. Size 4. A minimum vertex cover: any set of 4 vertices that hits all edges. But since every $u_i$ is connected to 3 $v_j$s, and there are $4 \times 3 = 12$ edges, a vertex cover of size 3 would need to cover all 12 edges, but each vertex covers at most 4 edges (a $v_j$ covers $3$ edges from $u_i$ with $i \ne j$, plus... actually each $v_j$ covers exactly 3 edges). So a cover of size 3 covers at most 9 edges, which is less than 12. A cover of size 4: $\{v_1, v_2, v_3, v_4\}$ covers all 12 edges. So the minimum vertex cover has size 4, matching the maximum matching size.

</details>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Prove that the Hopcroft-Karp algorithm finds a maximum bipartite matching in $O(|E| \sqrt{|V|})$ time.

<details class="exercise__hint"><summary>Hint</summary>

The algorithm finds a maximal set of vertex-disjoint shortest augmenting paths in each phase, doubling the shortest path length every $O(\sqrt{|V|})$ phases.

</details>

<details class="exercise__answer"><summary>Answer</summary>

The Hopcroft-Karp algorithm works in phases. In each phase, it uses BFS to find the length $k$ of the shortest augmenting path, then uses DFS to find a maximal set of vertex-disjoint augmenting paths of length $k$. Each phase takes $O(|E|)$ time.

After $\sqrt{|V|}$ phases, the shortest augmenting path has length $> \sqrt{|V|}$. This means there are fewer than $\sqrt{|V|}$ unmatched vertices reachable by alternating paths. Since each augmenting path increases the matching by 1, at most $\sqrt{|V|}$ more phases are needed. Total phases: $O(\sqrt{|V|})$. Total time: $O(|E| \sqrt{|V|})$.

More precisely: let $M_k$ be the matching after $k$ phases and $d_k$ the shortest augmenting path length. The maximal set of disjoint paths in each phase ensures $d_{k+1} \ge d_k$. After $\sqrt{|V|}$ phases, $d_k > 2\sqrt{|V|}$, which means the matching is within $\sqrt{|V|}$ of optimal (by the Berge theorem and the fact that long augmenting paths require many unmatched vertices). So at most $\sqrt{|V|}$ additional phases suffice.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

Prove König's theorem: in a bipartite graph, the size of a maximum matching equals the size of a minimum vertex cover.

<details class="exercise__answer"><summary>Answer</summary>

Let $M$ be a maximum matching. Construct a minimum vertex cover as follows. Let $U_L$ be the set of unmatched vertices in $U$. Let $Z$ be the set of all vertices reachable from $U_L$ by alternating paths (using unmatched edges from $U$ to $V$ and matched edges from $V$ to $U$). Define the vertex cover $C = (U \setminus Z) \cup (V \cap Z)$.

$C$ is a vertex cover: every edge $(u, v)$ with $u \in U, v \in V$ is covered. If $u \in Z$ and $v \notin Z$, then $(u, v)$ is a matched edge (otherwise $v$ would be added to $Z$), so $v \in V \setminus Z$. But then $(u, v)$ is matched with $u \in Z \cap U$ and $v \notin Z$. Since $u$ is reachable by an alternating path, the matched edge from $u$ goes to $v$ (which is not in $Z$). But alternating paths use matched edges from $V$ to $U$, so $v$ should be in $Z$. Contradiction. If $u \notin Z$ and $v \in Z$: covered since $v \in V \cap Z \subseteq C$. If both in $Z$ or both not in $Z$: covered.

$|C| = |M|$: $U \setminus Z$ consists of matched vertices not reachable by alternating paths. $V \cap Z$ consists of matched vertices reachable by alternating paths. The mapping from $C$ to $M$ is a bijection.

Since any vertex cover has size $\ge |M|$ (each edge of $M$ needs a distinct cover vertex), $C$ is a minimum vertex cover with $|C| = |M|$.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 8 (hard, symbolic).**

A Latin rectangle is an $r \times n$ matrix with entries from $\{1, \ldots, n\}$ such that no symbol repeats in any row or column. Use Hall's theorem to prove that any $r \times n$ Latin rectangle with $r < n$ can be extended to an $(r+1) \times n$ Latin rectangle.

<details class="exercise__answer"><summary>Answer</summary>

Define a bipartite graph where $U$ is the set of $n$ columns and $V$ is the set of symbols $\{1, \ldots, n\}$. Place an edge $(j, s)$ if symbol $s$ does not appear in column $j$ of the existing rectangle. Each column has exactly $n - r$ unused symbols (since $r$ symbols are already used in that column). Each symbol $s$ appears exactly once in each row, so exactly $r$ times total, hence is unused in exactly $n - r$ columns.

For any subset $S \subseteq U$ of columns, the total number of edges from $S$ to $N(S)$ is $|S|(n - r)$ (each column has $n - r$ edges). Each symbol in $N(S)$ has at most $n - r$ edges. So $|S|(n - r) \le |N(S)|(n - r)$, giving $|N(S)| \ge |S|$. Hall's condition holds, so a perfect matching exists. This matching assigns to each column a distinct unused symbol, forming a new row. The result is an $(r+1) \times n$ Latin rectangle.

</details>

</aside>

## Advanced results [Master]

Three directions extend bipartite matching theory.

**Theorem 1 (Edmonds' blossom algorithm).** *Maximum matching in general (non-bipartite) graphs can be found in $O(|V|^2 |E|)$ time. The algorithm handles "blossoms" — odd cycles that arise in non-bipartite graphs and prevent the simple augmenting-path approach from working* [ref: TODO_REF CLRS §26.3].

In bipartite graphs, alternating paths never cycle back to a previously visited vertex on the same side. In general graphs, odd cycles ("blossoms") can trap the search. Edmonds' algorithm contracts blossoms into single vertices, finds augmenting paths in the contracted graph, and then expands the blossoms to recover the augmenting path in the original graph.

**Theorem 2 (LP relaxation integrality).** *The natural linear programming relaxation of bipartite matching (maximise $\sum_{e} x_e$ subject to $\sum_{e \ni v} x_e \le 1$ for all $v$, $x_e \ge 0$) has an integral optimum. This is dual to the LP relaxation of vertex cover, and both are integral for bipartite graphs.*

This integrality is the foundational reason bipartite matching can be solved in polynomial time via linear programming. The Hungarian algorithm is essentially a primal-dual method for this LP pair.

**Theorem 3 (online bipartite matching).** *In the online setting where vertices on one side arrive one at a time and must be matched immediately or never, the optimal competitive ratio is $1 - 1/e \approx 0.632$ (Karp, Vazirani, and Vazirani 1990). The RANKING algorithm achieves this ratio.*

Online matching has applications in ad allocation, job scheduling, and kidney exchange. The competitive ratio of $1 - 1/e$ matches the performance of the greedy algorithm for the secretary problem.

**Synthesis.** Bipartite matching is the foundational reason that the max-flow framework provides a unified theory of combinatorial optimisation: the central insight is that the reduction from matching to max-flow, combined with LP duality and Hall's theorem, creates a three-way equivalence between combinatorial, algebraic, and optimisation perspectives on the same problem. This builds toward the general matching theory of Edmonds' blossom algorithm in [47.04.06] where the bipartite restriction is removed, and appears again in [47.02.03] where the contrast between tractable matching and NP-complete Hamiltonian cycle illustrates the boundary of polynomial-time computation. The generalises from the specific augmenting-path method to the entire primal-dual framework, and putting these together with König's theorem shows that the matching-cover duality is the structural backbone of bipartite graph theory.

## Full proof set [Master]

**Proposition 1 (Hall's theorem).** *A bipartite graph $G = (U, V, E)$ has a matching covering $U$ iff $|N(S)| \ge |S|$ for all $S \subseteq U$.*

*Proof.* Forward: if matching $M$ covers $U$, every $S \subseteq U$ is matched to $|S|$ distinct vertices in $V$, all in $N(S)$, so $|N(S)| \ge |S|$.

Backward (contrapositive): suppose no matching covers $U$. Let $M$ be a maximum matching and $u_0 \in U$ unmatched. Build $Z$ = vertices reachable from $u_0$ via alternating paths. Set $S = Z \cap U$, $T = Z \cap V$. Every $v \in T$ is reached from some $u \in S$ via a non-matching edge, then followed by a matching edge back to $S$ (since $M$ is maximum, no alternating path reaches an unmatched vertex in $V$). So $M$ gives a bijection from $T$ to $S \setminus \{u_0\}$, giving $|T| = |S| - 1$. Every neighbour of $S$ is in $T$ (otherwise there is an augmenting path). So $|N(S)| = |T| = |S| - 1 < |S|$. $\square$

**Proposition 2 (König's theorem).** *In a bipartite graph, the maximum matching size equals the minimum vertex cover size.*

*Proof.* Let $M$ be a maximum matching. Construct $Z$ from unmatched $U$-vertices via alternating paths. The vertex cover is $C = (U \setminus Z) \cup (V \cap Z)$.

Cover: every edge $(u, v)$ with $u \in U, v \in V$ is covered. If $u \in Z \cap U$ and $v \notin Z$: the edge $(u, v)$ is in $M$ (otherwise $v$ would be in $Z$). But matched edges go from $Z \cap U$ to $Z \cap V$, so $v \in Z$. Contradiction. So either $u \notin Z$ (in $C$) or $v \in Z$ (in $C$).

Size: the matching $M$ pairs $U \setminus Z$ with $V \setminus Z$ and $U \cap Z$ with $V \cap Z$ (by the alternating-path structure). So $|M| = |U \setminus Z| + |V \cap Z| = |C|$. Since any vertex cover has size $\ge |M|$, $C$ is minimum. $\square$

## Historical & philosophical context [Master]

Hall's theorem was proved by Philip Hall in 1935, though equivalent results were known to König (1916) and Menger (1927) in related forms. Hall's formulation in terms of the marriage problem (matching men to women subject to compatibility constraints) gave the theorem its popular name and直观 appeal.

The Hungarian algorithm was published by Harold Kuhn in 1955, who attributed the underlying ideas to the work of Hungarian mathematicians König and Egerváry. Kuhn showed that the primal-dual method for the assignment problem runs in $O(n^3)$ time. James Munkres (1957) proved that Kuhn's algorithm always terminates in $O(n^3)$ steps, and the algorithm is often called the Kuhn-Munkres algorithm.

König's theorem (1916) was one of the earliest results connecting matchings to vertex covers, establishing the minimax relationship that later generalised to the max-flow min-cut theorem. The equivalence between maximum matching and minimum vertex cover in bipartite graphs was a precursor to LP duality theory [ref: TODO_REF CLRS §26.3].
