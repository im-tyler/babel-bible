## Intuition [Beginner]

You have a road map with cities as vertices and roads as edges, each with a travel time. You want to know the fastest route between every pair of cities, not just two specific ones. This is the all-pairs shortest-paths problem.

One approach: run Dijkstra's algorithm from every city. This works but can be slow if the map is dense. A different approach, Floyd-Warshall, asks a clever question: what is the shortest path from city A to city B if you are only allowed to pass through the first $k$ cities as intermediate stops?

As $k$ grows from 0 to $n$, the answer gets better and better. When $k = 0$, you can only use direct roads. When $k = 1$, you can also stop at city 1. When $k = n$, you can stop anywhere, and you have the true shortest paths.

Each step is simple: the shortest path from A to B using cities $\{1, \ldots, k\}$ either does not use city $k$ (so it equals the previous answer) or does use city $k$ (so it is the shortest A-to-$k$ plus shortest $k$-to-B). Take the minimum of these two options.

Johnson's algorithm takes a different approach. It handles negative edge weights by "reweighting" the graph so that all weights become non-negative, then runs Dijkstra from every vertex. The reweighting preserves shortest paths, so the answers are correct.

## Worked example [Beginner]

We run Floyd-Warshall on a 4-vertex graph with edge weights:

- 0→1: 3, 0→3: 8, 1→2: 1, 2→3: 2, 3→1: -4.

Initial distance matrix ($k = 0$, only direct edges):

```
     0    1    2    3
0 [  0    3   ∞    8  ]
1 [  ∞    0    1   ∞  ]
2 [  ∞   ∞    0    2  ]
3 [  ∞   -4   ∞    0  ]
```

Step $k = 1$ (can use vertex 1 as intermediate):
- 0→2: min(∞, 3+1) = 4. 0→3: min(8, 3+∞) = 8. 3→2: min(∞, -4+1) = -3.

```
     0    1    2    3
0 [  0    3    4    8  ]
1 [  ∞    0    1   ∞  ]
2 [  ∞   ∞    0    2  ]
3 [  ∞   -4   -3    0  ]
```

Step $k = 2$ (can use vertices 1, 2):
- 0→3: min(8, 4+2) = 6. 1→3: min(∞, 1+2) = 3. 3→3: min(0, -3+2) = -1.

Wait — a negative diagonal entry means a negative cycle. Let me recalculate. 3→2→3 = -3+2 = -1. And 3→3 = 0. So min(0, -1) = -1. This indicates a negative-weight cycle through vertex 3.

Actually, let me redo with a graph that has no negative cycles. Remove the edge 3→1. Then:

Initial ($k = 0$):
```
     0    1    2    3
0 [  0    3   ∞    8  ]
1 [  ∞    0    1   ∞  ]
2 [  ∞   ∞    0    2  ]
3 [  ∞   ∞    ∞    0  ]

$k = 1$: 0→2: min(∞, 3+1) = 4.
$k = 2$: 0→3: min(8, 4+2) = 6. 1→3: min(∞, 1+2) = 3.
$k = 3$: no improvements (3 has no outgoing edges except to itself).

```

0 [  0    3    4    6  ]

2 [  ∞   ∞    0    2  ]

## Formal definition [Intermediate+]

**Definition (all-pairs shortest-paths).** Given a weighted directed graph $G = (V, E, w)$ with $w : E \to \mathbb{R}$ and $|V| = n$, the *all-pairs shortest-paths problem* is to compute an $n \times n$ matrix $D$ where $D_{ij} = \delta(i, j)$, the weight of a shortest path from $i$ to $j$, and a predecessor matrix $\Pi$ for path reconstruction.

**Definition (Floyd-Warshall recurrence).** Define $d_{ij}^{(k)}$ as the weight of a shortest path from $i$ to $j$ using only intermediate vertices from $\{1, 2, \ldots, k\}$. The recurrence is:

$$d_{ij}^{(0)} = \begin{cases} 0 & \text{if } i = j \\ w(i, j) & \text{if } (i, j) \in E \\ \infty & \text{otherwise} \end{cases}$$

$$d_{ij}^{(k)} = \min(d_{ij}^{(k-1)}, \; d_{ik}^{(k-1)} + d_{kj}^{(k-1)})$$

The final answer is $D_{ij} = d_{ij}^{(n)}$.

**Theorem (Floyd-Warshall correctness).** *If $G$ has no negative-weight cycles, then $d_{ij}^{(n)} = \delta(i, j)$ for all $i, j$.*

**Definition (Johnson's reweighting).** Given $G = (V, E, w)$, add a new vertex $s$ with zero-weight edges to all vertices. Run Bellman-Ford from $s$ to compute shortest-path distances $h(v) = \delta(s, v)$. Define the reweighted graph $\hat{w}(u, v) = w(u, v) + h(u) - h(v)$.

**Lemma (reweighting preserves shortest paths).** *If $p = \langle v_0, v_1, \ldots, v_k \rangle$ is any path from $v_0$ to $v_k$, then $\hat{w}(p) = w(p) + h(v_0) - h(v_k)$. Therefore, $p$ is a shortest path under $w$ if and only if $p$ is a shortest path under $\hat{w}$.*

Furthermore, if $G$ has no negative-weight cycles, then $\hat{w}(u, v) \ge 0$ for all edges, enabling the use of Dijkstra's algorithm.

### Counterexamples to common slips

- *"Running Dijkstra from every vertex solves APSP in $O(VE + V^2 \log V)$."* This is correct only for non-negative edge weights. If the graph has negative edges (but no negative cycles), Dijkstra is incorrect and Johnson's reweighting is needed first.

- *"Floyd-Warshall is always slower than running Dijkstra $n$ times."* For dense graphs ($E = \Theta(V^2)$), Floyd-Warshall's $O(V^3)$ matches $n$ runs of Dijkstra at $O(V^2)$ each. For sparse graphs, $n$ Dijkstra runs take $O(V(V \log V + E))$ which is faster.

- *"Johnson's algorithm requires no negative edges."* Johnson's handles negative edges by reweighting them away. It requires no negative cycles, not no negative edges.

### Key theorem with proof [Intermediate+]

**Theorem (Floyd-Warshall correctness).** *For all $i, j$ and $k \in \{0, \ldots, n\}$, $d_{ij}^{(k)}$ equals the weight of a shortest path from $i$ to $j$ with all intermediate vertices in $\{1, \ldots, k\}$.*

*Proof.* By induction on $k$.

Base: $k = 0$. No intermediate vertices are allowed, so the only paths are direct edges or the empty path (when $i = j$). $d_{ij}^{(0)}$ is set to exactly these values.

Inductive step: assume $d_{ij}^{(k-1)}$ is correct. Consider a shortest path $p$ from $i$ to $j$ using intermediates in $\{1, \ldots, k\}$. Either $p$ does not use vertex $k$ as an intermediate, in which case $w(p) = d_{ij}^{(k-1)}$, or $p$ uses $k$. In the latter case, $p$ decomposes as $i \leadsto k \leadsto j$ where the $i \leadsto k$ subpath uses intermediates in $\{1, \ldots, k-1\}$ (it cannot pass through $k$ before reaching $k$) and similarly for $k \leadsto j$. So $w(p) = d_{ik}^{(k-1)} + d_{kj}^{(k-1)}$. Taking the minimum of these two options gives $d_{ij}^{(k)}$. $\square$

**Bridge.** The Floyd-Warshall recurrence is the foundational reason all-pairs shortest paths can be computed in $O(V^3)$ time: it builds up the solution by progressively allowing more intermediate vertices, and each step is a simple minimisation. This builds toward the transitive closure computation, where the same structure appears with OR replacing MIN and AND replacing PLUS, and it appears again in min-plus matrix multiplication, where the Floyd-Warshall algorithm is exactly the computation of the closure of the weight matrix over the min-plus semiring. The central insight is that shortest paths compose: the shortest $i$-to-$j$ path through $k$ is the concatenation of the shortest $i$-to-$k$ and $k$-to-$j$ paths. Putting these together with Johnson's reweighting yields a complete toolkit for APSP: Floyd-Warshall for dense or small graphs, Johnson for sparse graphs with negative edges.

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Prove the reweighting lemma: if $h(v)$ is the shortest distance from a source $s$ to $v$ (with no negative cycles), then $\hat{w}(u, v) = w(u, v) + h(u) - h(v) \ge 0$ for all edges $(u, v)$.

<details class="exercise__hint"><summary>Hint</summary>

Use the triangle inequality for shortest paths.

</details>

<details class="exercise__answer"><summary>Answer</summary>

By the triangle inequality (shortest-path property), for any edge $(u, v)$: $h(v) \le h(u) + w(u, v)$. Rearranging: $w(u, v) + h(u) - h(v) \ge 0$, i.e., $\hat{w}(u, v) \ge 0$. This holds because $h(v)$ is the shortest distance from $s$ to $v$, and going from $s$ to $u$ (cost $h(u)$) then taking the edge to $v$ (cost $w(u,v)$) gives a path of cost $h(u) + w(u,v)$, which must be at least $h(v)$. Rubric: full credit for the triangle inequality argument.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 4 (medium, symbolic).**

Show that reweighting preserves shortest paths: if $p = \langle v_0, v_1, \ldots, v_k \rangle$, then $\hat{w}(p) = w(p) + h(v_0) - h(v_k)$.

<details class="exercise__answer"><summary>Answer</summary>

$\hat{w}(p) = \sum_{i=0}^{k-1} \hat{w}(v_i, v_{i+1}) = \sum_{i=0}^{k-1} [w(v_i, v_{i+1}) + h(v_i) - h(v_{i+1})]$. The $h$ terms telescope: $\sum_{i=0}^{k-1} h(v_i) - h(v_{i+1}) = h(v_0) - h(v_k)$. So $\hat{w}(p) = w(p) + h(v_0) - h(v_k)$. Since $h(v_0) - h(v_k)$ depends only on the endpoints, the ordering of paths from $v_0$ to $v_k$ by weight is the same under $w$ and $\hat{w}$.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 5 (medium, short answer).**

What is the time complexity of Johnson's algorithm on a graph with $V$ vertices and $E$ edges? When is it faster than Floyd-Warshall?

<details class="exercise__answer"><summary>Answer</summary>

Johnson's algorithm runs Bellman-Ford once ($O(VE)$) and then Dijkstra from each vertex ($V$ times, each $O(E + V \log V)$ with a binary heap). Total: $O(VE + V(V \log V + E)) = O(VE + V^2 \log V)$. This is faster than Floyd-Warshall's $O(V^3)$ when $E = o(V^2)$ (sparse graphs), because $VE = V \cdot o(V^2) = o(V^3)$.

</details>

</aside>

**Exercise 6 (hard, symbolic).**

Show that Floyd-Warshall can be used to compute the transitive closure of a directed graph in $O(V^3)$ time. Define the appropriate recurrence.

<details class="exercise__hint"><summary>Hint</summary>

Replace "min" with "OR" and "+" with "AND". $t_{ij}^{(k)}$ asks: is there a path from $i$ to $j$ using intermediates in $\{1, \ldots, k\}$?

</details>

<details class="exercise__answer"><summary>Answer</summary>

Define $t_{ij}^{(0)} = \text{true}$ if $i = j$ or $(i,j) \in E$, false otherwise. Recurrence: $t_{ij}^{(k)} = t_{ij}^{(k-1)} \lor (t_{ik}^{(k-1)} \land t_{kj}^{(k-1)})$. This asks: is there a path from $i$ to $j$ using intermediates in $\{1, \ldots, k\}$? Either there is already a path without using $k$, or there is a path $i \leadsto k \leadsto j$. The transitive closure is $\{t_{ij}^{(n)}\}$. Time: $O(V^3)$ (same triple loop). Rubric: full credit for the recurrence and its interpretation.

</details>

</aside>

**Exercise 7 (hard, short answer).**

Prove that if the input graph has a negative-weight cycle, Floyd-Warshall will produce a negative diagonal entry in the distance matrix.

<details class="exercise__answer"><summary>Answer</summary>

Let $C = \langle v_1, v_2, \ldots, v_k, v_1 \rangle$ be a negative-weight cycle with $w(C) < 0$. In the final matrix, $d_{v_1, v_1}^{(n)}$ represents the shortest "path" from $v_1$ to $v_1$ using all vertices as intermediaries. Since $C$ is available, the path $v_1 \to v_2 \to \cdots \to v_k \to v_1$ has weight $w(C) < 0$. The Floyd-Warshall recurrence will discover this: when all vertices of $C$ are allowed as intermediaries, the shortest "path" from $v_1$ to $v_1$ includes the cycle $C$ and has weight $w(C) < 0$. More generally, any vertex on a negative cycle will have a negative diagonal entry.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 8 (hard, symbolic).**

The min-plus product of two $n \times n$ matrices $A$ and $B$ is defined as $(A \otimes B)_{ij} = \min_k (A_{ik} + B_{kj})$. Show that Floyd-Warshall computes the min-plus closure of the weight matrix $W$: the matrix $D = W \oplus W^2 \oplus \cdots \oplus W^n$ where $W^k$ is the $k$-fold min-plus product.

<details class="exercise__answer"><summary>Answer</summary>

$W^k$ (min-plus power) represents the shortest paths using exactly $k$ edges. $W^1 = W$ (direct edges). $W^2 = W \otimes W$ gives shortest 2-edge paths. The min-plus closure $D = W \oplus W^2 \oplus \cdots \oplus W^n$ takes the minimum over all path lengths from 1 to $n$ edges. Since a shortest path in an $n$-vertex graph uses at most $n-1$ edges (no negative cycles), this captures all shortest paths.

The Floyd-Warshall recurrence $d_{ij}^{(k)} = \min(d_{ij}^{(k-1)}, d_{ik}^{(k-1)} + d_{kj}^{(k-1)})$ can be viewed as one step of min-plus matrix multiplication: $D^{(k)} = D^{(k-1)} \oplus (D^{(k-1)} \otimes D^{(k-1)})$ restricted to paths through vertex $k$. This is a structured version of computing the min-plus closure. Rubric: full credit for connecting the recurrence to the semiring framework.

</details>

</aside>

## Advanced results [Master]

Three results extend the basic APSP theory.

**Theorem 1 (min-plus semiring).** The Floyd-Warshall algorithm is a computation in the min-plus semiring $(\mathbb{R} \cup \{\infty\}, \min, +)$. The distance matrix $D$ is the closure of the weight matrix $W$ in this semiring: $D = W^{*}$ where $W^{*} = I \oplus W \oplus W^2 \oplus \cdots$.

This algebraic perspective unifies shortest paths with other semiring computations. Replacing $(\min, +)$ with $(\lor, \land)$ gives transitive closure. Replacing with $(+, \times)$ gives matrix powers. The Floyd-Warshall algorithm is a specialisation of Kleene's algorithm for computing semiring closures [ref: TODO_REF Tarjan Ch. 7].

**Theorem 2 (APSP via matrix multiplication).** If $n \times n$ matrix multiplication over a ring can be done in time $O(n^{\omega})$, then APSP can be solved in $\tilde{O}(n^{2+\mu})$ time where $\mu$ satisfies $\omega(1, \mu, 1) = 1 + 2\mu$, using the relationship between min-plus products and standard matrix multiplication over appropriate rings.

The current best APSP algorithm for dense graphs is by Williams (2018), running in $n^3 / 2^{\Theta(\sqrt{\log n})}$ time, using fast matrix multiplication techniques applied to the min-plus product.

**Theorem 3 (negative cycle detection).** Floyd-Warshall detects a negative-weight cycle if and only if some diagonal entry $d_{ii}^{(n)} < 0$. The detection takes no additional time beyond the main algorithm.

**Synthesis.** The Floyd-Warshall and Johnson algorithms are the foundational reason APSP can be solved efficiently: Floyd-Warshall provides a simple $O(V^3)$ solution for all graphs, while Johnson adapts Dijkstra to handle negative weights via reweighting. The central insight is that shortest paths compose — the path $i \leadsto k \leadsto j$ has weight equal to the sum of its parts — and this is exactly the property exploited by both algorithms. This builds toward the semiring generalisation, where the same algorithmic structure computes transitive closure, matrix powers, and other algebraic closures. Putting these together with the single-source shortest-path algorithms (Bellman-Ford, Dijkstra) yields a complete shortest-path toolkit: choose the algorithm based on graph density and edge weight properties.

## Full proof set [Master]

**Proposition 1 (Floyd-Warshall correctness).** *For all $k \in \{0, \ldots, n\}$ and all $i, j$, $d_{ij}^{(k)}$ equals the weight of a shortest path from $i$ to $j$ using only vertices in $\{1, \ldots, k\}$ as intermediaries.*

*Proof.* By strong induction on $k$.

Base ($k = 0$): No intermediate vertices. The only paths are direct edges (weight $w(i,j)$) or the identity path $i$ to $i$ (weight 0). $d_{ij}^{(0)}$ is defined to match these values exactly.

Inductive step: Assume the claim for $k - 1$. Let $p$ be a shortest path from $i$ to $j$ using intermediates in $\{1, \ldots, k\}$.

Case 1: $p$ does not contain vertex $k$. Then $p$ uses only intermediates in $\{1, \ldots, k-1\}$, so $w(p) = d_{ij}^{(k-1)}$.

Case 2: $p$ contains vertex $k$. Decompose $p$ as $i \leadsto k \leadsto j$ where both subpaths use only intermediates in $\{1, \ldots, k-1\}$ (if $k$ appeared in a subpath, it would create a cycle that could be removed without increasing weight, contradicting minimality). Then $w(p) = d_{ik}^{(k-1)} + d_{kj}^{(k-1)}$.

In either case, $w(p) \ge \min(d_{ij}^{(k-1)}, d_{ik}^{(k-1)} + d_{kj}^{(k-1)}) = d_{ij}^{(k)}$. And $d_{ij}^{(k)}$ is achievable by one of the two cases, so $w(p) = d_{ij}^{(k)}$. $\square$

**Proposition 2 (Johnson's reweighting preserves shortest paths).** *Let $h : V \to \mathbb{R}$ be any function satisfying $h(v) \le h(u) + w(u,v)$ for all $(u,v) \in E$. Define $\hat{w}(u,v) = w(u,v) + h(u) - h(v)$. Then $p$ is a shortest $s$-to-$t$ path under $w$ iff $p$ is a shortest $s$-to-$t$ path under $\hat{w}$.*

*Proof.* For any path $p = \langle v_0, v_1, \ldots, v_k \rangle$ from $s = v_0$ to $t = v_k$:

$$\hat{w}(p) = \sum_{i=0}^{k-1} [w(v_i, v_{i+1}) + h(v_i) - h(v_{i+1})] = w(p) + h(s) - h(t).$$

The offset $h(s) - h(t)$ depends only on the endpoints, not on the path. So if $p$ minimises $w(p)$ among all $s$-to-$t$ paths, it also minimises $\hat{w}(p) = w(p) + h(s) - h(t)$, and vice versa. $\square$

**Proposition 3 (reweighting yields non-negative weights).** *If $G$ has no negative-weight cycles and $h(v) = \delta(s, v)$ for an added source $s$, then $\hat{w}(u, v) \ge 0$ for all $(u, v) \in E$.*

*Proof.* By the triangle inequality for shortest paths, $h(v) \le h(u) + w(u, v)$ for every edge $(u, v)$. Rearranging: $w(u, v) + h(u) - h(v) \ge 0$, i.e., $\hat{w}(u, v) \ge 0$. $\square$

## Connections [Master]

- [47.04.01] — Amortized analysis applies to sequences of shortest-path queries; the Floyd-Warshall algorithm precomputes all answers in one pass.

- [47.04.06] — MST algorithms find minimum-weight spanning subgraphs; APSP algorithms find minimum-weight paths. Both are fundamental graph optimisation primitives.

- [47.04.03] — Min-cost flow algorithms use shortest-path computations as a subroutine; Johnson's reweighting is used to handle negative reduced costs.

- [50.03.01] — Big-O analysis bounds the running times of both Floyd-Warshall ($\Theta(V^3)$) and Johnson's ($O(VE + V^2 \log V)$).

## Historical & philosophical context [Master]

The Floyd-Warshall algorithm was published by Robert Floyd in 1962, though the same idea was discovered independently by Bernard Roy in 1959 and Stephen Warshall in 1962 (for transitive closure). The algorithm is an instance of a more general technique: computing the closure of a matrix over a closed semiring, as formalised by Lehmann (1977) and later by Tarjan [ref: TODO_REF CLRS §25.2 notes].

Johnson's algorithm was published by Donald Johnson in 1977. The key insight — that shortest-path potentials can be used to reweight edges non-negatively — is a discrete analogue of the concept of gauge transformations in physics, where a transformation preserves the physical content while simplifying the equations. The potential function $h(v)$ plays the role of a "gauge" that adjusts edge weights without changing shortest paths.

The APSP problem has deep connections to matrix multiplication. Subramanian (1991) and later Williams (2018) showed that APSP can be solved faster than $O(n^3)$ using techniques from fast matrix multiplication, though the improvements are sub-polynomial. Whether APSP requires $\Omega(n^3)$ time in the worst case (without fast matrix multiplication) remains open and is one of the central questions in fine-grained complexity theory [ref: TODO_REF CLRS §25.2].
