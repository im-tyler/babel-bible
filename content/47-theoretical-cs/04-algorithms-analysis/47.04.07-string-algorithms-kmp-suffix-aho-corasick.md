## Intuition [Beginner]

You have a long document and want to find every occurrence of the word "abracadabra." The obvious way is to check every starting position: try position 1, then position 2, and so on. Each check compares up to 11 characters. For a document of length $n$, this takes up to $11n$ comparisons — slow if the pattern or text is long.

The problem is that this naive approach forgets what it has learned. Suppose you are matching "aab" against the text "aaab." You compare positions 1-3: "aaa" vs "aab" — mismatch at position 3. The naive approach starts over at position 2. But you already know the first two characters of the text are "aa," which is also a prefix of "aab." You should not recheck them.

The Knuth-Morris-Pratt (KMP) algorithm captures this insight. It precomputes a "failure function" that tells you, after a mismatch, how many characters you can skip without rechecking. The text is scanned left to right without ever backing up.

Suffix arrays solve a different problem: what if you want to search for many different patterns in the same long text? Instead of scanning the text each time, you sort all suffixes of the text alphabetically. Then searching for a pattern is just binary search among the sorted suffixes.

Aho-Corasick generalises KMP to multiple patterns simultaneously. You build a trie (prefix tree) of all patterns and add "failure links" that work like KMP's failure function. Scanning the text once finds all occurrences of all patterns.

## Visual [Beginner]

KMP failure function for pattern "aabbaab":

| Position q | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| P[q] | a | a | b | b | a | a | b |
| pi[q] | 0 | 1 | 0 | 0 | 1 | 2 | 3 |

The failure function $\pi[q]$ gives the length of the longest proper prefix of $P[1..q]$ that is also a suffix. For $q = 2$: "aa" has prefix "a" = suffix "a", so $\pi[2] = 1$. For $q = 6$: "aabbaa" has longest prefix-suffix "aa", so $\pi[6] = 2$.

A suffix array for text "banana$":

| 6 | a$ | 2 |
| 4 | ana$ | 3 |
| 2 | anana$ | 4 |
| 1 | banana$ | 5 |
| 5 | na$ | 6 |
| 3 | nana$ | 7 |

Search for "ana": binary search finds it at sorted positions 3-4 (indices 4, 2).

## Formal definition [Intermediate+]

**Definition (string-matching problem).** Given a pattern $P[1..m]$ and a text $T[1..n]$ over an alphabet $\Sigma$, find all shifts $s \in \{0, 1, \ldots, n - m\}$ such that $T[s+1..s+m] = P[1..m]$.

**Definition (KMP failure function).** The *failure function* (also called the prefix function) for pattern $P[1..m]$ is:

$$\pi[q] = \max\{k : 1 \le k < q \text{ and } P[1..k] \text{ is a suffix of } P[1..q]\}$$

for $q = 1, \ldots, m$. By convention, $\pi[q] = 0$ if no such $k$ exists.

**Theorem (KMP running time).** *The KMP algorithm computes the failure function in $O(m)$ time and finds all occurrences of $P$ in $T$ in $O(n)$ time, for a total of $O(n + m)$.*

*Proof sketch.* The failure function computation uses an amortised argument: the variable $k$ (current prefix length) starts at 0, increases by at most 1 per iteration (to at most $m$), and the total number of decreases across all iterations is bounded by the total increases, which is at most $m$. So the total work is $O(m)$.

For matching: the text index $i$ advances monotonically from 1 to $n$. The pattern index $q$ increases by at most 1 per character of text (total increases bounded by $n$) and decreases by at least 1 per failure-function step (total decreases bounded by total increases). So the total matching work is $O(n)$. $\square$

**Definition (suffix array).** The *suffix array* $\text{SA}$ of a string $T[1..n]$ is a permutation of $\{1, \ldots, n\}$ such that $T[\text{SA}[1]..n] < T[\text{SA}[2]..n] < \cdots < T[\text{SA}[n]..n]$ in lexicographic order.

**Definition (LCP array).** The *longest common prefix array* $\text{LCP}$ is defined by $\text{LCP}[i] = |\text{lcp}(T[\text{SA}[i-1]..n], T[\text{SA}[i]..n])|$ for $i = 2, \ldots, n$.

The LCP array accelerates pattern search in the suffix array: when binary searching for a pattern, the LCP with the current midpoint tells you how many characters have already been matched, avoiding redundant comparisons.

**Definition (Aho-Corasick automaton).** Given a set of patterns $\{P_1, \ldots, P_k\}$, the *Aho-Corasick automaton* is a finite automaton (a trie augmented with failure links) that recognises the language $\Sigma^* (P_1 \cup \cdots \cup P_k) \Sigma^*$.

- The *goto function* $g(q, a)$ follows trie edges.
- The *failure function* $f(q)$ gives the longest proper suffix of the path to $q$ that is also a trie node (analogous to KMP's failure function).
- The *output function* $\text{out}(q)$ lists all patterns that end at node $q$.

**Theorem (Aho-Corasick running time).** *The Aho-Corasick automaton can be built in $O(m_1 + \cdots + m_k)$ time and searches a text of length $n$ in $O(n + z)$ time where $z$ is the number of matches.*

### Counterexamples to common slips

- *"KMP is always faster than the naive algorithm."* For short patterns or random text, the naive algorithm is often faster in practice due to lower constant factors. KMP's advantage is worst-case efficiency on adversarial inputs.

- *"A suffix array is the same as a suffix tree."* A suffix array stores the same information as a suffix tree but in a more compact form ($n$ integers vs $O(n)$ tree nodes). Some operations are slower on suffix arrays but the space savings are significant.

- *"Aho-Corasick is just running KMP multiple times."* Aho-Corasick shares computation across patterns. The trie merges common prefixes, and failure links allow all patterns to be tracked simultaneously in a single pass over the text.

### Key theorem with proof [Intermediate+]

**Theorem (suffix array construction in $O(n \lg n)$).** *The suffix array of a string of length $n$ can be constructed in $O(n \lg n)$ time using a prefix-doubling approach.*

*Proof sketch.* Sort suffixes by their first character. Then sort by their first 2 characters (using the sort-by-1 as a tiebreaker). Then by 4, 8, etc. After $\lceil \lg n \rceil$ doubling steps, the sort is by the full suffix. Each step sorts $n$ items by a key from a bounded alphabet (pairs of ranks from the previous step), which can be done in $O(n)$ using radix sort. Total: $O(n \lg n)$.

More precisely, let $\text{rank}_k(i)$ be the rank of $T[i..i+k-1]$ among all length-$k$ substrings. Initially, $\text{rank}_1(i) = T[i]$. To compute $\text{rank}_{2k}$, sort the pairs $(\text{rank}_k(i), \text{rank}_k(i+k))$. Each sorting step takes $O(n)$ with radix sort, and we need $\lceil \lg n \rceil$ steps. $\square$

**Bridge.** String algorithms are the foundational reason pattern matching can be done in linear time: KMP avoids redundant comparisons by exploiting the prefix structure of the pattern, suffix arrays enable indexed search by sorting suffixes, and Aho-Corasick generalises KMP to multiple patterns using automata theory. This builds toward the regular expression engines of [47.01.03], where automata-based matching handles patterns described by regular expressions, and it appears again in [47.01.02], where the NFA-to-DFA conversion parallels the trie-to-automaton construction in Aho-Corasick. The central insight is that the prefix structure of strings — what overlaps exist between prefixes and suffixes — is exactly the information needed to avoid redundant work. Putting these together with the automata theory of [47.01.01] yields a complete picture: string matching is pattern recognition, and the right automaton makes it efficient.

**Exercise 6 (hard, symbolic).**

Prove that the KMP matching phase makes at most $2n$ character comparisons on a text of length $n$.

<details class="exercise__hint"><summary>Hint</summary>

Use a potential-function argument: the potential is $q$ (current pattern position). Each comparison either increases $q$ (matching) or decreases $q$ via the failure function.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Each iteration of the matching loop either: (a) matches and increments $q$, or (b) mismatches and follows the failure function, decreasing $q$. In case (b), if $q = 0$ and there is a mismatch, $i$ advances without changing $q$.

Consider the quantity $k = i + q$ (text position + pattern position). A matching step increases $k$ by 1 (both $i$ and $q$ may change, but $k$ increases). A failure-function step decreases $q$ by at least 1 while keeping $i$ fixed, so $k$ decreases by at least 1. When $q = 0$ and there is a mismatch, $i$ advances and $k$ increases by 1.

The total number of matching steps is at most $n$ (bounded by text length). The total number of failure-function steps is bounded by the total matching steps (since $q$ cannot go below 0). So the total number of iterations is at most $2n$. Each iteration makes at most 1 comparison, so total comparisons $\le 2n$. Rubric: full credit for the potential argument.

</details>

</aside>

**Exercise 7 (hard, symbolic).**

Describe the prefix-doubling algorithm for suffix array construction. Show that it runs in $O(n \log n)$ time.

<details class="exercise__answer"><summary>Answer</summary>

Phase 0: Assign each suffix a rank based on its first character. Phase $k$: each suffix has a rank pair $(\text{rank}(T[i..i+2^k-1]), \text{rank}(T[i+2^k..i+2^{k+1}-1]))$. Sort all suffixes by these rank pairs using radix sort. Update ranks based on the new sorted order. Repeat until all suffixes have distinct ranks (at most $\lceil \log n \rceil$ phases). Each phase: radix sort on pairs of integers in $\{1, \ldots, n\}$ takes $O(n)$ time. $\lceil \log n \rceil$ phases gives $O(n \log n)$ total. Rubric: full credit for the algorithm and time analysis.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 8 (hard, short answer).**

Prove that the Aho-Corasick automaton finds all occurrences of all patterns in $O(n + z)$ time where $z$ is the total number of matches.

<details class="exercise__answer"><summary>Answer</summary>

The automaton processes each character of the text in $O(1)$ amortised time: following a goto edge is $O(1)$, and following failure links decreases the depth of the current node (analogous to KMP). The total number of failure-link traversals across the entire text is $O(n)$ by the same amortised argument as KMP. Outputting matches takes $O(z)$ time (one step per match). Building the automaton takes $O(\sum m_i)$ time for the trie and $O(\sum m_i)$ for the failure links (BFS on the trie). Total: $O(n + \sum m_i + z)$. Rubric: full credit for the amortised argument.

</details>

</aside>

**True.** The suffix array stores $n$ integers (one per suffix), each in the range $\{1, \ldots, n\}$, requiring $O(n)$ space. The LCP array adds another $n$ integers, still $O(n)$ total.

## Advanced results [Master]

Three results extend the basic string algorithm toolkit.

**Theorem 1 (linear-time suffix array construction).** *The suffix array of a string of length $n$ can be constructed in $O(n)$ time using the SA-IS (Suffix Array Induced Sorting) algorithm of Nong, Zhang, and Chan (2011).*

SA-IS is based on the observation that suffixes can be classified into L-type and S-type, and that the S-type suffixes form a recursively smaller subproblem. The algorithm recursively sorts the S-type suffixes and then uses induced sorting to place the L-type suffixes. This achieves $O(n)$ worst-case time, matching the theoretical bound for suffix tree construction.

**Theorem 2 (LCP array from suffix array in linear time).** *Given the suffix array and the text, the LCP array can be computed in $O(n)$ time using Kasai's algorithm.*

Kasai's algorithm processes suffixes in text order (not SA order). The key insight is that the LCP of consecutive suffixes in text order differs by at most 1 from the LCP of the previous pair, allowing the computation to proceed in amortised $O(1)$ per position [ref: TODO_REF Gusfield Ch. 3].

**Theorem 3 (Aho-Corasick as automaton).** *The Aho-Corasick automaton is a DFA (in the sense of [47.01.01]) whose language is $\Sigma^*(P_1 \cup \cdots \cup P_k)\Sigma^*$. The number of states is $O(\sum m_i)$ and the construction time is $O(\sum m_i \cdot |\Sigma|)$.*

The Aho-Corasick automaton generalises the KMP automaton (which is the special case $k = 1$). The failure links play the same role as the KMP failure function: they allow the automaton to "fall back" to the longest matching prefix without reprocessing any text characters. This connects string matching directly to automata theory.

**Synthesis.** String algorithms are the foundational reason pattern matching can be done efficiently: KMP exploits prefix structure for linear-time matching, suffix arrays enable indexed search by sorting, and Aho-Corasick extends the approach to multiple patterns using automata. The central insight is that the prefix-suffix structure of strings — captured by the failure function, the suffix array ordering, or the trie with failure links — is exactly the information needed to avoid redundant comparisons. This builds toward the regular expression engines of [47.01.03] and the NFA constructions of [47.01.02], where the same automata-theoretic ideas handle more complex patterns. Putting these together yields a layered approach to string processing: exact matching (KMP), indexed search (suffix arrays), and multi-pattern matching (Aho-Corasick), each building on the same structural insights about how strings overlap.

## Full proof set [Master]

**Proposition 1 (KMP failure function correctness).** *For each $q \in \{1, \ldots, m\}$, $\pi[q]$ equals the length of the longest proper prefix of $P[1..q]$ that is also a suffix of $P[1..q]$.*

*Proof.* The failure function computation iterates $q$ from 2 to $m$. At each step, it maintains the invariant that $k$ is the length of the longest proper prefix of $P[1..q-1]$ that is also a suffix. If $P[k+1] = P[q]$, then $k+1$ is a prefix-suffix of $P[1..q]$. Otherwise, it follows the chain of decreasing $k$ values (via $\pi[k]$) until either a match is found or $k = 0$. By induction on $q$, the invariant holds and $\pi[q]$ is computed correctly. $\square$

**Proposition 2 (KMP matching finds all occurrences).** *If $P$ occurs in $T$ at shift $s$, then KMP reports shift $s$.*

*Proof.* When KMP processes text position $i = s + m$, the pattern pointer $q$ equals $m$ (all characters matched). At this point, the algorithm reports the match at shift $i - m = s$. The failure function ensures that $q$ never skips a valid partial match: if $P[1..q]$ matches $T[i-q+1..i]$ and the next character mismatches, then $q$ is set to $\pi[q]$, which is the longest proper prefix of $P[1..q]$ that could start a new match. This is exactly the right amount of backtracking — no valid shift is missed. $\square$

**Proposition 3 (suffix array search correctness).** *Binary search on the suffix array finds all occurrences of $P[1..m]$ in $T[1..n]$.*

*Proof.* The suffixes in SA order are lexicographically sorted. The pattern $P$ defines a lexicographic range: all suffixes starting with $P$ are contiguous in the sorted order. Binary search finds the leftmost and rightmost suffixes in this range. Each such suffix $\text{SA}[i]$ corresponds to an occurrence of $P$ at position $\text{SA}[i]$ in $T$. Since all suffixes starting with $P$ are found, all occurrences are found. $\square$

## Connections [Master]

- [47.01.01] — DFAs are the foundation for the KMP and Aho-Corasick automata; each string-matching algorithm builds a specific DFA that recognises the pattern language.

- [47.01.02] — The Aho-Corasick automaton uses failure links analogous to the NFA subset construction: both convert nondeterministic state tracking into deterministic transitions.

- [47.01.03] — Regular expressions generalise exact string matching; the Thompson construction builds NFAs from regex that are processed by techniques similar to KMP and Aho-Corasick.

- [47.04.01] — Amortized analysis justifies the $O(n + m)$ bound of KMP: the potential (pattern position) bounds total work across the entire matching phase.

## Historical & philosophical context [Master]

The Knuth-Morris-Pratt algorithm was discovered independently by Knuth and Pratt (1970) and by Morris (1970), and published jointly in 1977. Knuth credits the key insight to Morris, who was implementing a text editor and needed an efficient search algorithm. The failure function was independently discovered by several researchers, highlighting that the prefix-suffix structure is a natural and unavoidable concept in string matching [ref: TODO_REF CLRS §31.3 notes].

The Aho-Corasick algorithm was published by Alfred Aho and Margaret Corasick in 1975, originally developed for bibliographic search in the Bell Labs library system. The algorithm's elegance lies in converting the multi-pattern problem into an automaton construction, bridging string matching and automata theory. The failure links are the exact multi-pattern generalisation of the KMP failure function [ref: TODO_REF Gusfield Ch. 2].

Suffix arrays were introduced by Manber and Myers in 1993 as a space-efficient alternative to suffix trees. While suffix trees use $O(n)$ space with a constant factor of about 20 bytes per character, suffix arrays use only 4 bytes per character (one integer). The trade-off is slightly slower query times ($O(m \log n)$ vs $O(m)$ for suffix trees), but the space savings made suffix arrays practical for indexing large text corpora like genome sequences. The linear-time SA-IS construction algorithm (2011) closed the theoretical gap between suffix arrays and suffix trees [ref: TODO_REF Gusfield Ch. 3].
