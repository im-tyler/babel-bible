## Intuition [Beginner]

Imagine you are a robot walking through a city, trying to get from your hotel to the museum. You have no map, and you can only remember a constant number of street names at any time. Can you navigate? The answer depends on whether you are allowed to make lucky guesses.

If you must plan deterministically, with only a fixed-size memory, you are limited to the class L — problems solvable in logarithmic space. Logarithmic space means you can remember only a handful of numbers about the size of the input, not the input itself. Remarkably, some useful problems still fit in this budget.

If you are allowed to make nondeterministic guesses — "try this street, and if it works, great" — you enter the class NL. The gap between L and NL is the space analogue of the gap between P and NP, but it lives at a much tighter scale.

The biggest surprise in this area is that NL is closed under complement. If a problem can be solved with nondeterministic log-space guesses, then its negation can too. This is nothing like the NP situation, where whether NP equals coNP is a major open question. For logarithmic space, nondeterminism is weak enough that we can prove it adds no power when you flip the answer.

## Visual [Beginner]

The space complexity hierarchy at the logarithmic level and its relationship to the polynomial levels.

```
       PSPACE
      /      \
   P  -----  NP
   |         |
   L  -----  NL
```

The key relationships. L sits inside NL, just as P sits inside NP. But unlike the P-vs-NP question, we know NL = coNL (Immerman-Szelepcsenyi, 1987). The inclusion NL within P follows from the fact that a log-space nondeterministic machine has at most polynomially many configurations.

| class | resource | deterministic version |
|---|---|---|
| L | $\mathrm{SPACE}(\log n)$ | itself |
| NL | $\mathrm{NSPACE}(\log n)$ | inside $\mathrm{SPACE}(\log^2 n)$ by Savitch |
| P | $\mathrm{TIME}(n^k)$ | itself |
| NP | $\mathrm{NTIME}(n^k)$ | unknown |

## Worked example [Beginner]

Take the PATH problem: given a directed graph $G$ with nodes $s$ and $t$, does a directed path from $s$ to $t$ exist? This is the canonical NL-complete problem.

A nondeterministic log-space algorithm works as follows. Start at $s$. At each step, nondeterministically pick an outgoing edge and follow it. Store only the current node number, which takes $O(\log n)$ bits for a graph with $n$ nodes. After at most $n$ steps (since any simple path has at most $n$ nodes), either you reach $t$ and accept, or you give up and reject.

If a path exists, some sequence of lucky guesses follows it, so the machine accepts. If no path exists, no guessing sequence can reach $t$, so the machine always rejects. This shows PATH is in NL.

The harder direction is that every NL problem reduces to PATH. The idea is that any nondeterministic log-space computation on input $x$ has a configuration graph with polynomially many vertices, and the machine accepts if and only if there is a path from the start configuration to an accepting one. Since the configuration graph can be constructed on-the-fly in log space, this gives a log-space reduction.

## Formal definition [Intermediate+]

For a function $f : \mathbb{N} \to \mathbb{N}$, define $\mathrm{SPACE}(f(n))$ and $\mathrm{NSPACE}(f(n))$ as in [47.02.05]. The **logarithmic-space classes** are
$$
\mathrm{L} = \mathrm{SPACE}(\log n), \qquad \mathrm{NL} = \mathrm{NSPACE}(\log n).
$$
Equivalently, $\mathrm{L}$ is the class of languages decidable by a deterministic Turing machine that uses $O(\log n)$ cells on its work tape (the read-only input tape is not counted), and $\mathrm{NL}$ is the nondeterministic variant.

A **log-space reduction** from $A$ to $B$ is a function $f$ computable in $O(\log n)$ space such that $x \in A \iff f(x) \in B$. A language $B$ is **NL-complete** if $B \in \mathrm{NL}$ and every $A \in \mathrm{NL}$ log-space reduces to $B$.

The language
$$
\mathrm{PATH} = \{ \langle G, s, t \rangle : G \text{ is a directed graph with a path from } s \text{ to } t \}
$$
is the canonical NL-complete problem. Its complement $\overline{\mathrm{PATH}} = \{ \langle G, s, t \rangle : \text{no path from } s \text{ to } t \text{ in } G \}$ is $\mathrm{coNL}$-complete.

The key structural facts are: $\mathrm{L} \subseteq \mathrm{NL} \subseteq \mathrm{P}$, $\mathrm{NL} \subseteq \mathrm{SPACE}(\log^2 n)$ by Savitch's theorem [47.02.05], and $\mathrm{NL} = \mathrm{coNL}$ by the Immerman-Szelepcsenyi theorem. Whether $\mathrm{L} = \mathrm{NL}$ remains open [ref: TODO_REF Arora-Barak §4.3].

### Counterexamples to common slips [Intermediate+]

- *"NL = coNL implies L = NL."* It does not. Closure under complement says that the complement of every NL language is also in NL. This does not collapse NL to its deterministic version. The relationship between L and NL is analogous to P versus NP — closure under complement is a weaker statement than derandomisation of nondeterminism.

- *"Savitch gives the best known simulation of NL in deterministic space."* This is correct: $\mathrm{NL} \subseteq \mathrm{SPACE}(\log^2 n)$ is the best known upper bound. Whether $\mathrm{NL} \subseteq \mathrm{SPACE}(\log n) = \mathrm{L}$ is a major open question. No improvement on Savitch's bound is known even for the specific case of graph reachability.

- *"PATH is NL-complete, so it cannot be in L."* NL-completeness means PATH is among the hardest problems in NL, but we do not know whether L = NL. If L = NL, then PATH would be in L. The completeness result only tells us that PATH is in L if and only if every NL problem is in L.

## Key theorem with proof [Intermediate+]

**Theorem (Immerman-Szelepcsenyi).** *For space-constructible $f(n) \ge \log n$, $\mathrm{NSPACE}(f(n)) = \mathrm{coNSPACE}(f(n))$. In particular, $\mathrm{NL} = \mathrm{coNL}$.*

*Proof.* The proof uses an inductive counting technique. We show $\overline{\mathrm{PATH}} \in \mathrm{NL}$, which suffices since $\overline{\mathrm{PATH}}$ is $\mathrm{coNL}$-complete.

Let $G$ be a graph on $n$ vertices. For a vertex $s$ and integer $k \ge 0$, define $R_k(s) = \{ v : v \text{ is reachable from } s \text{ in at most } k \text{ steps} \}$ and $c_k = |R_k(s)|$. Note $c_0 = 1$ (only $s$ itself). The goal is to decide, given $s, t$, whether $t \notin R_{n-1}(s)$.

**Inductive step:** Suppose an NL machine knows $c_k$. It can compute $c_{k+1}$ in NL as follows. Initialize a counter $d = 0$. For each vertex $u$ (loop over all $n$ vertices): nondeterministically verify that $u \in R_k(s)$ by guessing a path of length $\le k$ from $s$ to $u$. If verification succeeds, check if any neighbour of $u$ is the current vertex $v$. If so, $v$ is reachable in $k+1$ steps.

To compute $c_{k+1}$: iterate over all vertices $v$. For each $v$, use the above procedure to check membership in $R_{k+1}(s)$. But there is a subtlety: the nondeterministic verification might reject a reachable vertex. The fix is to use the count $c_k$ as a certificate of completeness: after scanning all $u$, if we have found exactly $c_k$ vertices in $R_k(s)$, then we know we have found all of them. This ensures no reachable vertex is missed.

Starting from $c_0 = 1$, compute $c_1, c_2, \ldots, c_{n-1}$ inductively. After computing $c_{n-1}$, we know exactly how many vertices are reachable from $s$. Now check whether $t$ is among them: guess paths to $c_{n-1}$ distinct vertices, and if none equals $t$, then $t$ is unreachable — accept. If $t$ is found among them, reject.

The machine uses $O(\log n)$ space: it stores counters $k, d, c_k, c_{k+1}$ (each $O(\log n)$ bits), the current vertex (also $O(\log n)$), and the nondeterministic path guesses reuse the same space. $\square$

**Bridge.** This theorem is the foundational reason the space complexity landscape is more tractable at the logarithmic level than the time complexity landscape at the polynomial level: nondeterminism in log space is so constrained that even its complement fits within the same class, and this is exactly the kind of structural collapse that has no known time analogue (NP versus coNP remains wide open). The inductive counting technique builds toward the circuit complexity results in [47.03.02] where similar counting arguments appear, and it appears again in the derandomisation programme [47.05.05] where explicit constructions of pseudorandom generators for space-bounded computation rely on counting reachability sets. The central insight is that knowing $|R_k|$ certifies completeness of a nondeterministic search — putting these together, the inductive chain $c_0 \to c_1 \to \cdots \to c_{n-1}$ turns a "there exists" question into a sequence of verified counts, and the bridge is that this counting trick works in NL precisely because log-space machines can count to polynomial quantities while storing only logarithmically many bits.

Hence $\overline{\mathrm{2SAT}} \in \mathrm{NL}$, and by NL = coNL, $2\mathrm{SAT} \in \mathrm{NL}$. Rubric: full credit for the implication graph construction and the PATH reduction.

**Exercise 4 (medium, symbolic).**

Prove that log-space reductions are transitive: if $A \le_L B$ and $B \le_L C$, then $A \le_L C$.

<details class="exercise__hint"><summary>Hint</summary>

Compose the two reductions. The challenge is that the second reduction reads the output of the first, which cannot be stored in log space. Use the trick of recomputing bits of the intermediate output on demand.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Let $f$ reduce $A$ to $B$ in log space and $g$ reduce $B$ to $C$ in log space. The composed reduction $h(x) = g(f(x))$ cannot write $f(x)$ to a tape (it may be polynomially long). Instead, simulate $g$ on a virtual input tape: whenever $g$ needs the $i$-th symbol of $f(x)$, recompute $f(x)$ from scratch up to position $i$. Recomputing $f(x)$ uses $O(\log |x|)$ space. The simulation stores the position counter $i$ ($O(\log n)$ bits) and the work space of both machines ($O(\log n)$ each). Total space: $O(\log n)$. Hence $h$ is a log-space reduction. Rubric: full credit for the recomputation trick and the space analysis.

</details>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Give a full proof that the PATH problem is NL-hard: every language $L \in \mathrm{NL}$ log-space reduces to PATH.

<details class="exercise__hint"><summary>Hint</summary>

Encode the configuration graph of the NL machine. The reduction must be computable in log space, so the graph cannot be explicitly written out.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Let $M$ be an NL machine deciding $L$ using $c \log n$ space. A configuration of $M$ on input $x$ consists of the state, input head position ($O(\log n)$ bits), and work tape contents and head position ($O(\log n)$ bits), totaling $O(\log n)$ bits — so there are $n^{O(1)}$ distinct configurations. Define $G_{M,x}$ with configurations as vertices and an edge $(C_1, C_2)$ iff $M$ can transition from $C_1$ to $C_2$ in one step.

The reduction $f$ maps $x$ to $\langle G_{M,x}, C_s, C_a \rangle$ where $C_s$ is the start configuration and $C_a$ is the (unique) accepting configuration. Then $x \in L$ iff there is a path from $C_s$ to $C_a$ in $G_{M,x}$.

The graph $G_{M,x}$ is not written explicitly. Instead, the reduction outputs a circuit (or oracle) description that, given two configuration descriptions $C_1, C_2$, determines whether $(C_1, C_2)$ is an edge by simulating one step of $M$. This simulation uses $O(\log n)$ space. Since log-space reductions can use this on-the-fly construction, $f$ is a log-space reduction. Rubric: full credit for the configuration-graph encoding and the on-the-fly edge computation.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

Prove that $\mathrm{NL} \subseteq \mathrm{NC}^2$ (the class of problems solvable by polynomial-size, $O(\log^2 n)$-depth circuits). Conclude that NL problems have fast parallel algorithms.

<details class="exercise__hint"><summary>Hint</summary>

Use the Savitch recursion to build a circuit. Each recursive call becomes a sub-circuit, and the depth is $O(\log^2 n)$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

For an NL machine $M$ using $O(\log n)$ space, the configuration graph has $N = n^{O(1)}$ vertices. Build a circuit for the PATH predicate using the Savitch recursion: $\mathrm{REACH}(a, b, t)$ checks reachability in $2^t$ steps. The base case $t = 0$ is a single gate (edge check). For $t \ge 1$: $\mathrm{REACH}(a, b, t) = \bigvee_m (\mathrm{REACH}(a, m, t-1) \wedge \mathrm{REACH}(m, b, t-1))$. The OR over all midpoints has $N = n^{O(1)}$ fan-in (realised by a tree of OR gates of depth $O(\log N) = O(\log n)$). The recursion depth is $T = O(\log N) = O(\log n)$. Total circuit depth: $O(\log n) \cdot O(\log n) = O(\log^2 n)$. Circuit size: polynomial, since each level produces $N^2$ sub-problems (pairs of endpoints) and there are $O(\log n)$ levels. Hence $\mathrm{NL} \subseteq \mathrm{NC}^2$. Rubric: full credit for the circuit construction and the depth/size analysis.

</details>

</aside>

<details class="exercise__answer"><summary>Answer</summary>

Hence $\overline{L(M)} \in \mathrm{NSPACE}(O(f(n))) = \mathrm{NSPACE}(f(n))$, giving $\mathrm{coNSPACE}(f(n)) \subseteq \mathrm{NSPACE}(f(n))$. The reverse inclusion is by symmetry. ## Advanced results [Master]

The Immerman-Szelepcsenyi theorem opened the door to a detailed understanding of space-bounded computation. The results below explore the consequences and generalisations.

**Theorem 1 (NL = coNL, full detail).** The proof has two key components: (a) computing $c_k$ from $c_{k-1}$ in NL, and (b) using $c_{n-1}$ to decide $\overline{\mathrm{PATH}}$. For (a), the machine maintains two counters: $c_{k-1}$ (given) and $c_k$ (being computed). For each vertex $v$, it determines whether $v \in R_k(s)$ by scanning all vertices $u$ and guessing paths of length $k-1$ from $s$ to $u$. When a path to $u$ is found, check if $(u, v)$ is an edge. The count $c_{k-1}$ ensures completeness: after scanning all $u$, if exactly $c_{k-1}$ of them were verified as reachable, the scan was exhaustive. For (b), with $c_{n-1}$ known, guess $c_{n-1}$ distinct reachable vertices and verify that $t$ is not among them [ref: TODO_REF Arora-Barak §4.3].

**Theorem 2 (Savitch at the logarithmic level).** The bound $\mathrm{NL} \subseteq \mathrm{SPACE}(\log^2 n)$ from Savitch's theorem [47.02.05] is the best known deterministic upper bound for NL. No improvement is known, and the question $\mathrm{NL} \stackrel{?}{=} \mathrm{L}$ remains one of the fundamental open problems in complexity theory. The Reingold theorem (2005) showed that undirected graph reachability is in L, giving $\mathrm{SL} = \mathrm{L}$, but the directed case remains open.

**Theorem 3 (NL and circuit complexity).** The inclusion $\mathrm{NL} \subseteq \mathrm{NC}^2$ follows from the Savitch recursion translated into circuits. This places NL problems in the realm of efficient parallel computation. The converse relationship — whether $\mathrm{NC}^1 \subseteq \mathrm{NL}$ — involves the complexity of formula evaluation and is related to the question of whether Boolean formula value is complete for $\mathrm{NC}^1$ [ref: TODO_REF Arora-Barak §6.2].

**Theorem 4 (downward self-reducibility and NL).** Many NL-complete problems exhibit downward self-reducibility: the answer for input size $n$ can be obtained from answers for inputs of size $n-1$. For PATH, removing a vertex and asking about reachability in the smaller graph is a form of self-reducibility. This property connects to the randomised algorithms for space-bounded computation in [47.05.01], where random walks on graphs approximate reachability probabilities.

**Theorem 5 (space-bounded hierarchies).** The space hierarchy theorem gives $\mathrm{L} \subsetneq \mathrm{SPACE}(\log^2 n) \subsetneq \mathrm{PSPACE}$, so the logarithmic landscape has genuine structure. Whether the intermediate classes form a strict hierarchy depends on unresolved questions. The Immerman-Szelepcsenyi theorem implies that the nondeterministic space hierarchy at the log level has no "gap" at the complement: $\mathrm{NL} = \mathrm{coNL}$, $\mathrm{NSPACE}(\log^2 n) = \mathrm{coNSPACE}(\log^2 n)$, and so on [ref: TODO_REF Arora-Barak §4.3].

**Synthesis.** The foundational reason NL = coNL is that nondeterministic log-space machines are constrained enough that their computation graphs can be analysed inductively, and this is exactly the structural property that distinguishes space from time at every level of the hierarchy. The inductive counting technique builds toward the more general result that $\mathrm{NSPACE}(f) = \mathrm{coNSPACE}(f)$ for all space-constructible $f$, and it appears again in the derandomisation of space-bounded computation [47.05.05] where explicit constructions for pseudorandom generators rely on counting reachable configurations. The central insight is that a count serves as a compact certificate of exhaustive search — putting these together, knowing $|R_k|$ lets you verify that every reachable vertex has been found, and this certificate propagates through the inductive chain. The bridge is that this self-certifying property of space-bounded nondeterminism generalises to give $\mathrm{NSPACE}(f) = \mathrm{coNSPACE}(f)$ at every level, which is dual to the collapse $\mathrm{PSPACE} = \mathrm{NPSPACE}$ from [47.02.05], and the generalises connection from NL to NC circuits shows that space-bounded nondeterminism sits naturally within the parallel computation framework.

## Full proof set [Master]

**Proposition 1 (PATH is NL-complete).** *$\mathrm{PATH} \in \mathrm{NL}$ and every $L \in \mathrm{NL}$ log-space reduces to PATH.*

*Proof.* Membership: a nondeterministic log-space machine starts at $s$ and repeatedly guesses a successor vertex. Store only the current vertex ($O(\log n)$ bits) and a step counter ($O(\log n)$ bits). If $t$ is reached within $n$ steps, accept.

Hardness: let $M$ decide $L$ in $\mathrm{NL}$. The configuration graph $G_{M,x}$ has $N = n^{O(1)}$ vertices, each described in $O(\log n)$ bits. The reduction maps $x$ to $(G_{M,x}, C_s, C_a)$ where $C_s$ is the start configuration and $C_a$ is the accepting configuration. The graph is described implicitly: given two configuration descriptions, the edge relation is computed by simulating one step of $M$, using $O(\log n)$ space. Then $x \in L$ iff there is a path from $C_s$ to $C_a$ in $G_{M,x}$. $\square$

**Proposition 2 (Immerman-Szelepcsenyi).** *$\mathrm{NL} = \mathrm{coNL}$.*

*Proof.* We show $\overline{\mathrm{PATH}} \in \mathrm{NL}$. Let $G$ have vertex set $\{0, 1, \ldots, n-1\}$ and let $c_k = |R_k(s)|$. Compute $c_0 = 1$. Inductively compute $c_k$ from $c_{k-1}$: for each vertex $v$, determine whether $v \in R_k(s)$ by scanning all vertices $u$. For each $u$, nondeterministically guess a path of length $\le k-1$ from $s$ to $u$. If a path is found and $(u, v)$ is an edge, mark $v$ as reachable. Keep a count of how many $u$'s were verified. After scanning all $u$, if the count equals $c_{k-1}$, the search was complete (every reachable $u$ was found). Increment $c_k$ if $v$ was marked. This uses $O(\log n)$ space for counters and vertex labels.

After $n-1$ rounds, $c_{n-1}$ is known. To decide whether $t \notin R_{n-1}(s)$: nondeterministically guess paths to $c_{n-1}$ distinct vertices. If $t$ is not among them and exactly $c_{n-1}$ vertices are found, accept ($t$ is unreachable). Otherwise reject. $\square$

**Proposition 3 (NL within P).** *$\mathrm{NL} \subseteq \mathrm{P}$.*

*Proof.* An NL machine $M$ on input $x$ uses $O(\log n)$ space, giving $N = n^{O(1)}$ configurations. Construct the configuration graph $G_{M,x}$ with $N$ vertices. Reachability in a directed graph with $N$ vertices is decidable in $O(N^2)$ time by BFS/DFS. The accepting configuration is reachable from the start configuration iff $x \in L(M)$. Total time: $N^{O(1)} = n^{O(1)}$, hence $\mathrm{NL} \subseteq \mathrm{P}$. $\square$

**Proposition 4 (generalisation to all space bounds).** *For space-constructible $f(n) \ge \log n$, $\mathrm{NSPACE}(f(n)) = \mathrm{coNSPACE}(f(n))$.*

*Proof.* The identical inductive counting argument works. The $\mathrm{NSPACE}(f)$ machine has $N = 2^{O(f(n))}$ configurations. The machine stores counters of size $O(f(n))$ bits and vertex labels of $O(f(n))$ bits, using total space $O(f(n))$. The induction runs for $N$ rounds, each requiring $O(f(n))$ space. $\square$

## Connections [Master]

- Space complexity and PSPACE [47.02.05] provides the framework that this unit specialises to the logarithmic level. Savitch's theorem gives $\mathrm{NL} \subseteq \mathrm{SPACE}(\log^2 n)$, and the Immerman-Szelepcsenyi theorem complements this by showing that nondeterministic and co-nondeterministic log space coincide.

- Time complexity P and NP [47.02.01] is the polynomial-time analogue of this unit's topic. The relationship $\mathrm{L} \subseteq \mathrm{NL}$ mirrors $\mathrm{P} \subseteq \mathrm{NP}$, but the Immerman-Szelepcsenyi theorem shows that NL = coNL, whereas NP versus coNP remains open.

- Interactive proofs [47.03.04] use the counting technique from this unit in the arithmetisation of quantifier predicates. The NL = coNL proof is a miniature version of the IP = PSPACE argument, where counting certifiable sets replaces probabilistic verification.

- Circuit complexity [47.03.02] connects to NL through the inclusion $\mathrm{NL} \subseteq \mathrm{NC}^2$, which translates the Savitch recursion into a bounded-depth circuit family.

## Historical & philosophical context [Master]

The Immerman-Szelepcsenyi theorem was proved independently by Neil Immerman and R ostislav Szelepcsenyi in 1987, resolving a question that had been open since the early 1970s. Immerman's paper, "Nondeterministic Space is Closed Under Complementation" (1988), and Szelepcsenyi's paper, "The Method of Forced Estimation for Proving Nondeterministic Space is Closed Under Complementation" (1988), both appeared in the SIAM Journal on Computing. The result was surprising because the analogous question for time (NP versus coNP) remains wide open, and most researchers had expected the space version to be equally hard [ref: TODO_REF Arora-Barak §4.3].

The technique of inductive counting introduced in these proofs has become a standard tool in complexity theory. It demonstrates that counting and verification can be interleaved in a way that preserves space bounds, a principle that reappears in the IP = PSPACE proof and in the theory of pseudorandom generators for space-bounded computation.

The Reingold theorem of 2005, showing that undirected graph reachability is in L (and hence $\mathrm{SL} = \mathrm{L}$), extended the structural understanding of space-bounded classes. Directed reachability remains in NL but not known to be in L, making it the canonical candidate for separating L from NL.
