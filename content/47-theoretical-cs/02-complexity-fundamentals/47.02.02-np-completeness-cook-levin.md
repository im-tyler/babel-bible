## Intuition [Beginner]

Imagine you are given a massive jigsaw puzzle. Checking whether a proposed solution is correct is quick — you just verify that every piece fits. But finding the solution yourself might take a very long time.

This is the essence of NP. A problem is in NP if every "yes" answer comes with a short proof that can be verified quickly. SAT, the Boolean satisfiability problem, is the canonical example: given a formula like "(x or y) and (not x or z)", is there an assignment of true/false values to the variables that makes the whole formula true? Checking a proposed assignment is easy. Finding one could be very hard.

NP-completeness identifies the hardest problems in NP. A problem is NP-complete if (1) it is in NP, and (2) every other problem in NP can be converted into it efficiently (in polynomial time). This means if you could solve any single NP-complete problem quickly, you could solve every NP problem quickly, and P would equal NP.

The Cook-Levin theorem (1971) proves that SAT is NP-complete. The proof works by showing that the computation of any Turing machine (as defined in [42.04.01]) solving an NP problem can be encoded as a Boolean formula. The formula is satisfiable if and only if the machine accepts the input. This encoding converts the question "does this machine accept this input?" into "is this formula satisfiable?", establishing SAT as a universal hard problem.

The structure of NP-completeness: every problem in NP reduces to SAT.

```
   NP problems          Polynomial        SAT
  ┌───────────┐        reductions      ┌─────┐
  │  CLIQUE   │ ──────────────────────→ │     │
  │  TSP      │ ──────────────────────→ │     │

  │  SUBSET-SUM│ ──────────────────────→ │     │
  │  ...      │ ──────────────────────→ │     │
  └───────────┘                         └─────┘
```

A polynomial-time reduction from problem A to problem B is a function $f$ computable in polynomial time such that $x \in A$ if and only if $f(x) \in B$.

| Problem | In NP? | NP-complete? | Known poly algorithm? |
|---|---|---|---|
| SAT | Yes | Yes (Cook-Levin) | None known |
| 3-SAT | Yes | Yes (reduction from SAT) | None known |
| Sorting | Yes | No (in P) | Yes |
| Halting problem | No (undecidable) | No | No |

## Worked example [Beginner]

We reduce SAT to 3-SAT to show 3-SAT is NP-complete.

A SAT instance: $(x_1 \lor x_2) \land (\neg x_1 \lor x_3 \lor x_4 \lor \neg x_2)$.

The second clause has 4 literals, so this is not a 3-SAT instance. We convert it using auxiliary variables.

**Clause splitting.** Replace $(\neg x_1 \lor x_3 \lor x_4 \lor \neg x_2)$ with two 3-literal clauses using a new variable $z$:

$(\neg x_1 \lor x_3 \lor z) \land (\neg z \lor x_4 \lor \neg x_2)$

**Correctness.** If the original 4-literal clause is satisfiable, at least one literal is true. If $\neg x_1$ or $x_3$ is true, set $z$ to false: both new clauses are satisfied. If $x_4$ or $\neg x_2$ is true, set $z$ to true: both new clauses are satisfied.

Conversely, if the new clauses are both satisfied, consider $z$. If $z$ is false, then $\neg x_1 \lor x_3$ must be true. If $z$ is true, then $x_4 \lor \neg x_2$ must be true. Either way, at least one literal in the original clause is true.

This gadget construction splits any clause of length $k > 3$ into clauses of length 3 using $k - 3$ new variables. The formula grows polynomially, and the conversion runs in polynomial time.

## Formal definition [Intermediate+]

**Definition (NP-completeness).** A language $B$ is *NP-complete* if:

1. $B \in \mathsf{NP}$, and
2. For every $A \in \mathsf{NP}$, $A \le_p B$ (every language in NP is polynomial-time reducible to $B$).

Condition (2) is equivalent to: there exists an NP-complete language $C$ such that $C \le_p B$. This is the standard method for proving new problems NP-complete.

**Definition (computation tableau).** A Turing machine (as defined in [42.04.01]) $M$ running on input $x$ of length $n$ within time $t = n^k$ produces a *tableau* — a $t \times t$ table where row $i$ represents the configuration of $M$ at step $i$. Each cell contains a symbol from the tape alphabet, a state, or a combination (the cell under the tape head carries both the tape symbol and the current state).

**Definition (SAT).** The Boolean satisfiability problem is:

$$\text{SAT} = \{\varphi : \varphi \text{ is a satisfiable Boolean formula in conjunctive normal form (CNF)}\}.$$

A Boolean formula $\varphi$ in CNF is a conjunction of clauses, each clause a disjunction of literals (variables or their negations).

**Definition (3-SAT).** The restricted problem where every clause has exactly 3 literals:

$$\text{3-SAT} = \{\varphi : \varphi \in \text{SAT} \text{ and every clause of } \varphi \text{ has exactly 3 literals}\}.$$

### The tableau encoding

Given a language $L \in \mathsf{NP}$, let $M$ be a nondeterministic Turing machine (as defined in [42.04.01]) that decides $L$ in time $n^k$. For input $x$ of length $n$, the computation tableau is an $n^k \times n^k$ grid. The Cook-Levin construction builds a Boolean formula $\varphi_x$ that encodes this tableau.

**Variables.** For each cell $(i, j)$ and each possible content $s$ (tape symbol or state-symbol pair), introduce a variable $T_{i,j,s}$ meaning "cell $(i, j)$ contains symbol $s$."

**Clauses.** The formula $\varphi_x = \varphi_{\text{cell}} \land \varphi_{\text{start}} \land \varphi_{\text{accept}} \land \varphi_{\text{move}}$ where:

- $\varphi_{\text{cell}}$: each cell contains exactly one symbol (at-least-one and at-most-one constraints),
- $\varphi_{\text{start}}$: row 0 encodes the start configuration (input $x$ on the tape, head at position 0, start state),
- $\varphi_{\text{accept}}$: some row contains an accept state,
- $\varphi_{\text{move}}$: the contents of each $2 \times 3$ window of cells are consistent with the transition function of $M$.

The size of $\varphi_x$ is $O(n^{2k})$, polynomial in $|x|$.

### Counterexamples to common slips

- *"NP-complete means the problem has no polynomial algorithm."* We do not know whether P = NP. NP-completeness means the problem is as hard as any NP problem, but a polynomial algorithm might exist — finding one (or proving none exists) would resolve the P vs NP question.

- *"NP means 'not polynomial'."* NP stands for "nondeterministic polynomial time" — the class of problems solvable in polynomial time by a nondeterministic machine. Many NP problems (like sorting) are also in P.

- *"The Cook-Levin theorem proves P does not equal NP."* The theorem proves SAT is NP-complete. It does not resolve P vs NP. It shows that resolving P vs NP can be done by studying any single NP-complete problem.

### Key theorem with proof [Intermediate+]

**Theorem (Cook-Levin).** *SAT is NP-complete.*

*Proof.* We show (1) SAT is in NP and (2) every language in NP reduces to SAT.

(1) A certificate for $\varphi \in \text{SAT}$ is a satisfying assignment $a$. The verifier evaluates $\varphi$ on $a$ in polynomial time. So $\text{SAT} \in \mathsf{NP}$.

(2) Let $L \in \mathsf{NP}$. There exists a nondeterministic Turing machine (as defined in [42.04.01]) $M$ deciding $L$ in time $n^k$. For input $x$ of length $n$, construct $\varphi_x$ as described above. The construction runs in $O(n^{2k})$ time.

$\varphi_x$ is satisfiable iff $M$ has an accepting computation on $x$. If $M$ accepts $x$, the accepting tableau provides a satisfying assignment for $\varphi_x$ (set $T_{i,j,s}$ to true exactly when cell $(i,j)$ contains $s$ in the accepting computation). If $\varphi_x$ is satisfiable, the satisfying assignment describes a valid tableau: the cell, start, and move constraints guarantee it represents a legal computation of $M$, and the accept constraint guarantees the computation accepts. So $x \in L$ iff $\varphi_x \in \text{SAT}$.

The function $x \mapsto \varphi_x$ is computable in polynomial time, so $L \le_p \text{SAT}$. $\square$

**Bridge.** The Cook-Levin theorem is the foundational reason NP-completeness is a meaningful concept: there exists a single "universal" problem (SAT) to which all NP problems reduce. This builds toward the web of NP-complete problems in [47.02.03], where reductions from SAT establish hardness for combinatorial problems like CLIQUE and VERTEX-COVER. The central insight is that the tableau encoding is the bridge between computation (a machine processing symbols over time) and logic (a formula asserting the existence of a valid computation). This is exactly the connection that appears again in the PCP theorem of [47.03.06], where the tableau is probabilistically checked, and it generalises the encoding idea from the halting problem of [42.04.02]. Putting these together yields the modern understanding of intractability: NP-complete problems are not just hard individually but are polynomial-time interreducible, forming a single equivalence class of difficulty.

**Exercise 3 (medium, symbolic).**

Show that the reduction from SAT to 3-SAT produces a formula of size $O(m)$ where $m$ is the size of the original formula, preserving satisfiability.

<details class="exercise__hint"><summary>Hint</summary>

Each clause of length $k$ introduces $k - 3$ new variables and $k - 2$ new clauses. Sum over all clauses.

</details>

<details class="exercise__answer"><summary>Answer</summary>

For a clause of length $k$, the gadget introduces $k - 3$ new variables and $k - 2$ new clauses of length 3. The total number of new variables is $\sum_{i=1}^{p} (k_i - 3)$ where $p$ is the number of clauses and $k_i$ is the length of clause $i$. The total number of new literals is $\sum_{i=1}^{p} 3(k_i - 2) = 3\sum k_i - 6p$. Both are $O(m)$ where $m = \sum k_i$. The conversion preserves satisfiability by the gadget argument in the worked example. Rubric: full credit for the size analysis and the preservation argument.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="short-answer" markdown="1">

**Exercise 4 (medium, short answer).**

Prove that if $A \le_p B$ and $B \le_p C$, then $A \le_p C$ (transitivity of polynomial-time reductions).

<details class="exercise__answer"><summary>Answer</summary>

Let $f$ be a polynomial-time reduction from $A$ to $B$ running in time $p_1(n)$, and $g$ a polynomial-time reduction from $B$ to $C$ running in time $p_2(n)$. The composition $h(x) = g(f(x))$ is a polynomial-time reduction from $A$ to $C$: $f(x)$ has length at most $p_1(|x|)$, so $g(f(x))$ runs in time $p_2(p_1(|x|))$, which is polynomial in $|x|$. And $x \in A$ iff $f(x) \in B$ iff $g(f(x)) \in C$.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 5 (medium, symbolic).**

Describe the four components of the Cook-Levin formula $\varphi_x = \varphi_{\text{cell}} \land \varphi_{\text{start}} \land \varphi_{\text{accept}} \land \varphi_{\text{move}}$. For each, state what property of the tableau it encodes and how many clauses it uses.

<details class="exercise__answer"><summary>Answer</summary>

$\varphi_{\text{cell}}$: each cell $(i,j)$ contains at least one symbol ($\bigvee_s T_{i,j,s}$) and at most one symbol ($\neg T_{i,j,s} \lor \neg T_{i,j,s'}$ for $s \ne s'$). Uses $O(n^{2k} \cdot |\Gamma|^2)$ clauses where $|\Gamma|$ is the number of possible cell contents.

$\varphi_{\text{start}}$: row 0 encodes the initial configuration — $T_{0,0,q_{\text{start}}}$ (head at position 0 in start state), $T_{0,j,x_j}$ for $0 \le j < n$ (input symbols), $T_{0,j,\sqcup}$ for $n \le j < n^k$ (blank tape). Uses $O(n^k)$ clauses.

$\varphi_{\text{accept}}$: some cell in the tableau contains the accept state — $\bigvee_{i,j} T_{i,j,q_{\text{accept}}}$. Uses $O(n^{2k})$ clauses.

$\varphi_{\text{move}}$: for each $2 \times 3$ window centred on the head position, the contents are consistent with some transition of $M$. Each window generates $O(|\delta|)$ clauses where $\delta$ is the transition function. Uses $O(n^{2k} \cdot |\delta|)$ clauses.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Prove that the move constraint $\varphi_{\text{move}}$ correctly enforces that the tableau represents a valid computation. Specifically, show that any $2 \times 3$ window that is consistent with the transition function guarantees global consistency.

<details class="exercise__hint"><summary>Hint</summary>

The head position and state propagate locally: at most one cell in each row contains a state. A $2 \times 3$ window captures the entire effect of one transition.

</details>

<details class="exercise__answer"><summary>Answer</summary>

In each row, exactly one cell $(i, j)$ contains a state (since $M$ has one head). The transition function $\delta(q, s) = (q', s', D)$ means: in state $q$ reading symbol $s$, write $s'$, move to state $q'$, and shift the head by $D \in \{L, R\}$. A $2 \times 3$ window with the state cell at position $(i, j)$ in the centre captures cells $(i, j-1), (i, j), (i, j+1)$ in row $i$ and $(i+1, j-1), (i+1, j), (i+1, j+1)$ in row $i+1$. The window enforces: row $i+1$ has the same tape contents as row $i$ except at position $j$ (which changes from $s$ to $s'$), and the state in row $i+1$ is at position $j + D$. Since every cell not near the head is unchanged between rows, and the head's effect is captured by the window, global consistency follows from local consistency on all windows. Rubric: full credit for the argument that the $2 \times 3$ window is a complete local check.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 7 (hard, short answer).**

Prove that if $A$ is NP-complete and $A \le_p B$ and $B \in \mathsf{NP}$, then $B$ is NP-complete.

<details class="exercise__answer"><summary>Answer</summary>

For any $C \in \mathsf{NP}$, since $A$ is NP-complete, $C \le_p A$. By assumption $A \le_p B$. By transitivity of polynomial-time reductions, $C \le_p B$. Since this holds for all $C \in \mathsf{NP}$ and $B \in \mathsf{NP}$, $B$ is NP-complete. This is the standard reduction method for proving NP-completeness. Rubric: full credit for the transitivity argument.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 8 (medium, symbolic).**

Ladner's theorem states that if P does not equal NP, then there exist languages in NP that are neither in P nor NP-complete. Explain why this theorem is significant for the structure of NP.

<details class="exercise__answer"><summary>Answer</summary>

Ladner's theorem shows that the landscape of NP is not binary (P or NP-complete). If P $\ne$ NP, there must be NP-intermediate problems — problems in NP that are harder than any P problem but easier than any NP-complete problem, in the sense of polynomial-time reducibility. This means there is a rich structure of difficulty levels within NP, not just two categories. The proof constructs an artificial language by padding and delaying the computation in a way that makes it too hard for P but not hard enough for NP-completeness.

</details>

## Advanced results [Master]

Three results extend the Cook-Levin theorem.

**Theorem 1 (Cook's theorem for circuit-SAT).** *The circuit satisfiability problem (CIRCUIT-SAT) is NP-complete.*

The proof encodes the computation tableau directly as a Boolean circuit rather than a CNF formula. Each row of the tableau is computed from the previous row by a fixed circuit that checks the $2 \times 3$ windows. The overall circuit has polynomial size and is satisfiable iff the TM accepts. CIRCUIT-SAT is sometimes used as the starting point for NP-completeness proofs instead of SAT because the circuit model is closer to hardware [ref: TODO_REF Arora & Barak §2.3].

**Theorem 2 (Berman-Hartmanis conjecture).** *All known natural NP-complete problems are polynomial-time isomorphic (i.e., related by polynomial-time bijections with polynomial-time inverses).*

This conjecture (1977) states that all NP-complete sets are "essentially the same" up to polynomial-time relabelling. It has been verified for all natural NP-complete problems but remains unproven in general. A consequence is that if the conjecture holds and P $\ne$ NP, then no sparse set (a set with only polynomially many strings of each length) is NP-complete.

**Theorem 3 (NP-completeness of 3-SAT).** *3-SAT is NP-complete.*

The reduction from SAT to 3-SAT uses the gadget construction from the worked example. Each clause of length $k > 3$ is split into $k - 2$ clauses of length 3 using $k - 3$ auxiliary variables. The reduction preserves satisfiability and runs in polynomial time. Since SAT is NP-complete and 3-SAT is in NP, transitivity gives NP-completeness of 3-SAT [ref: TODO_REF Sipser §7.5].

**Synthesis.** The Cook-Levin theorem is the foundational reason NP-completeness is a coherent and useful concept: it provides a single universal bottleneck problem to which all NP problems reduce. The central insight is that computation can be encoded as logic — the tableau construction translates "does this machine accept?" into "is this formula satisfiable?" This builds toward the entire theory of NP-completeness reductions, where hundreds of practical problems are shown to be NP-complete by chains of reductions from SAT. The bridge is that the tableau encoding appears again in the PCP theorem, where it is made robust to probabilistic checking, and this is exactly what connects computational hardness to inapproximability results. Putting these together with the polynomial hierarchy of [47.02.04] yields the modern map of computational complexity: a landscape of difficulty levels connected by reductions.

## Full proof set [Master]

**Proposition 1 (Cook-Levin: SAT is NP-complete).** *$\text{SAT}$ is NP-complete.*

*Proof.* Membership: $\text{SAT} \in \mathsf{NP}$ because a satisfying assignment is a polynomial-length certificate verifiable in polynomial time by evaluating the formula.

Hardness: Let $L \in \mathsf{NP}$ via NDTM $M$ running in time $t = n^k$. Construct $\varphi_x$ encoding the computation tableau. The formula has four parts:

$\varphi_{\text{cell}} = \bigwedge_{i,j} \left[\left(\bigvee_s T_{i,j,s}\right) \land \bigwedge_{s \ne s'} (\neg T_{i,j,s} \lor \neg T_{i,j,s'})\right]$

$\varphi_{\text{start}} = T_{0,0,q_0 \cdot x_0} \land \bigwedge_{j=1}^{n-1} T_{0,j,x_j} \land \bigwedge_{j=n}^{t-1} T_{0,j,\sqcup}$

$\varphi_{\text{accept}} = \bigvee_{0 \le i < t} \bigvee_{0 \le j < t} T_{i,j,q_{\text{accept}}}$

$\varphi_{\text{move}} = \bigwedge_{i,j} \bigvee_{\text{legal windows}} \bigwedge_{\text{cells in window}} T_{i,j,s}$

The construction time is $O(n^{2k})$. If $M$ accepts $x$, the accepting tableau satisfies $\varphi_x$. If $\varphi_x$ is satisfiable, the satisfying assignment describes a valid accepting computation of $M$ on $x$. So $x \in L \iff \varphi_x \in \text{SAT}$. $\square$

**Proposition 2 (SAT reduces to 3-SAT).** *There exists a polynomial-time reduction from SAT to 3-SAT.*

*Proof.* Given a CNF formula $\varphi$ with clauses $C_1, \ldots, C_m$, for each clause $C_i$ of length $k_i$:

- If $k_i = 1$, say $C_i = (\ell)$, replace with $(\ell \lor p \lor q) \land (\ell \lor \neg p \lor q) \land (\ell \lor p \lor \neg q) \land (\ell \lor \neg p \lor \neg q)$ for new variables $p, q$.
- If $k_i = 2$, say $C_i = (\ell_1 \lor \ell_2)$, replace with $(\ell_1 \lor \ell_2 \lor p) \land (\ell_1 \lor \ell_2 \lor \neg p)$ for new variable $p$.
- If $k_i = 3$, keep as is.
- If $k_i > 3$, say $C_i = (\ell_1 \lor \cdots \lor \ell_k)$, replace with $(\ell_1 \lor \ell_2 \lor z_1) \land (\neg z_1 \lor \ell_3 \lor z_2) \land \cdots \land (\neg z_{k-3} \lor \ell_{k-1} \lor \ell_k)$ for new variables $z_1, \ldots, z_{k-3}$.

Each gadget introduces $O(k_i)$ new variables and clauses. Total size is $O(\sum k_i) = O(|\varphi|)$. The reduction preserves satisfiability by the gadget correctness argument. $\square$

## Historical & philosophical context [Master]

The Cook-Levin theorem was proved independently by Stephen Cook (1971) and Leonid Levin (1973). Cook's paper "The Complexity of Theorem-Proving Procedures" showed that SAT is NP-complete by encoding Turing machine computations as Boolean formulas. Levin's paper, published in Russian, proved the same result using a different but equivalent construction [ref: TODO_REF Arora & Barak §2.3].

Richard Karp's 1972 paper "Reducibility among Combinatorial Problems" showed that 21 fundamental combinatorial problems (including CLIQUE, VERTEX-COVER, HAM-CYCLE, and SUBSET-SUM) are all NP-complete by building chains of reductions from SAT. This transformed NP-completeness from a theoretical curiosity into a practical tool: any new problem could be shown intractable (assuming P $\ne$ NP) by reducing a known NP-complete problem to it.

The philosophical significance of NP-completeness is that it reveals a deep unity among seemingly unrelated problems. Satisfiability of Boolean formulas, optimal tour planning, graph colouring, and scheduling are all "the same problem" in the sense that they are polynomial-time interreducible. This suggests that the boundary between tractable and intractable is a fundamental feature of computation itself, not an artifact of any particular problem formulation [ref: TODO_REF Sipser §7.4].
