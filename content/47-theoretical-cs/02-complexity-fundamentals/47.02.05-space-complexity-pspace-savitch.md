## Intuition [Beginner]

Time and memory are the two fundamental resources a computer uses. So far the focus has been on time: how many steps does a program need? But you can also ask a different question: how much memory does it need? This shift in perspective opens a new landscape of complexity classes and reveals structure that time alone cannot see.

The class PSPACE collects every problem that can be solved using an amount of memory that grows polynomially in the input size. Because a program that runs for a polynomial number of steps can touch at most a polynomial number of memory cells, every problem in P or NP also lives in PSPACE. The reverse is not known — PSPACE could be much larger — and the containment PSPACE within EXPTIME is known to be strict by the space hierarchy theorem.

A surprise awaits when you add nondeterminism to space-bounded computation. For time, adding nondeterminism is conjectured to add enormous power (P versus NP). For space, it adds almost nothing: Savitch's theorem shows that any problem solvable by a nondeterministic machine using $f(n)$ cells of memory can also be solved by a deterministic machine using only $f(n)^2$ cells. The quadratic overhead is the price of determinism, and it is remarkably small.

The canonical hard problem for PSPACE is evaluating fully quantified Boolean formulas — formulas where every variable is bound by an "exists" or "for all" quantifier, and the quantifiers can alternate. This problem, called TQBF, is to PSPACE what SAT is to NP: every PSPACE problem reduces to it, so if TQBF were easy then all of PSPACE would be easy.

The relationship between the main complexity classes discussed in this unit. Every class is contained in the ones above it.

             \      |        /
              NP -- coNP    /
                \   |      /
                  P

The key inclusions. P is inside NP inside PSPACE inside EXPTIME. NPSPACE collapses to PSPACE by Savitch. The polynomial hierarchy PH sits between NP and PSPACE but is not known to equal either. The strict inclusions P within EXPTIME and PSPACE within EXPTIME follow from hierarchy theorems.

| inclusion | reason |
|---|---|
| P within NP | definition (deterministic is a special case of nondeterministic) |
| NP within PSPACE | a poly-time nondeterministic computation uses at most poly space |
| PSPACE = NPSPACE | Savitch's theorem: nondeterministic space squaring |
| PSPACE within EXPTIME | poly-space machine has at most exponentially many configurations |

## Worked example [Beginner]

Consider a robot navigating a grid. The grid has $n$ rows and $n$ columns, with walls in some cells. The robot starts at the top-left corner and wants to reach the bottom-right corner. At each step it can move up, down, left, or right into an adjacent cell that is not a wall.

The question "can the robot reach the goal?" is a reachability problem. A naive depth-first search uses a stack that could grow as deep as the number of cells, which is $n^2$. But you can also think about it differently: does there exist a path of length at most $n^2$ from start to goal?

Savitch's approach is recursive. To test whether there is a path of length at most $2^t$ from cell $a$ to cell $b$, pick every intermediate cell $m$ and recursively ask: is there a path of length at most $2^{t-1}$ from $a$ to $m$, and from $m$ to $b$? The recursion depth is $t$, and each level stores one intermediate cell (which takes $O(\log n)$ bits), so the total memory is $O(t \cdot \log n)$.

For a grid of size $n^2$, the longest possible path has length at most $n^2 = 2^{2 \log n}$, so $t = 2 \log n$ levels of recursion suffice. The total memory is $O(\log^2 n)$ — far less than storing the entire grid path. This is the essence of Savitch's theorem applied to a concrete problem.

## Formal definition [Intermediate+]

Let $M$ be a Turing machine with a read-only input tape and a read-write work tape. On input $x$ of length $n$, define $\mathrm{space}_M(x)$ as the maximum number of work-tape cells that $M$ visits during its computation on $x$. For a function $f : \mathbb{N} \to \mathbb{N}$, define
$$
\mathrm{SPACE}(f(n)) = \{ L : \text{there exists a deterministic TM } M \text{ deciding } L \text{ with } \mathrm{space}_M(x) \le f(|x|) \text{ for all } x \},
$$
and
$$
\mathrm{NSPACE}(f(n)) = \{ L : \text{there exists a nondeterministic TM } M \text{ deciding } L \text{ with } \mathrm{space}_M(x) \le f(|x|) \text{ for all } x \}.
$$
The classes $\mathrm{PSPACE}$ and $\mathrm{NPSPACE}$ are the polynomial-bounded unions:
$$
\mathrm{PSPACE} = \bigcup_{k \ge 0} \mathrm{SPACE}(n^k), \qquad \mathrm{NPSPACE} = \bigcup_{k \ge 0} \mathrm{NSPACE}(n^k).
$$

A **configuration** of a space-$S$ Turing machine on an input of length $n$ is a tuple $(q, h_i, h_w, c)$ where $q$ is the machine state, $h_i \in \{0, \ldots, n-1\}$ is the input head position, $h_w \in \{0, \ldots, S-1\}$ is the work-tape head position, and $c \in \Gamma^S$ is the work-tape contents. The number of distinct configurations is at most $|Q| \cdot n \cdot S \cdot |\Gamma|^S = 2^{O(S + \log n)}$. For $S \ge \log n$, this is $2^{O(S)}$.

The **configuration graph** $G_{M,x}$ is a directed graph whose vertices are the configurations of $M$ on input $x$, with an edge from $C_1$ to $C_2$ if $M$ can move from $C_1$ to $C_2$ in one step. Then $M$ accepts $x$ if and only if there is a path in $G_{M,x}$ from the start configuration to an accepting configuration.

A language $L$ is **PSPACE-complete** if $L \in \mathrm{PSPACE}$ and every language $A \in \mathrm{PSPACE}$ is polynomial-time reducible to $L$: there is a polynomial-time computable function $f$ with $x \in A \iff f(x) \in L$. The language $\mathrm{TQBF}$ (True Quantified Boolean Formulas) is the set of all fully quantified Boolean formulas that evaluate to true:
$$
\mathrm{TQBF} = \{ \langle \phi \rangle : \phi = Q_1 x_1 \cdots Q_m x_m \cdot \psi(x_1, \ldots, x_m) \text{ is true} \},
$$
where each $Q_i \in \{\exists, \forall\}$ and $\psi$ is a quantifier-free Boolean formula [ref: TODO_REF Sipser §8.3].

### Counterexamples to common slips [Intermediate+]

- *"Savitch's theorem implies P = NP because squaring space is cheap."* Savitch addresses space, not time. The deterministic simulation of Savitch's theorem may take time exponential in the space bound (the recursive $\mathrm{PATH}$ predicate has exponential time complexity because it enumerates all midpoints), so it gives no information about P versus NP.

- *"PSPACE-completeness is the same as NP-completeness but with more quantifiers."* The analogy is useful but the technical difference matters. A PSPACE-complete problem allows an unbounded alternation of existential and universal quantifiers, while NP-complete problems involve a single block of existential quantifiers. The reduction must be polynomial-time, which is the same requirement, but the quantifier structure is what makes TQBF harder than SAT under standard assumptions.

- *"NPSPACE is larger than PSPACE because nondeterminism helps."* This is precisely what Savitch's theorem disproves. The quadratic space overhead of simulating nondeterminism deterministically is modest enough that $\mathrm{NSPACE}(n^k) \subseteq \mathrm{SPACE}(n^{2k})$, so taking the union over all $k$ gives $\mathrm{NPSPACE} = \mathrm{PSPACE}$.

## Key theorem with proof [Intermediate+]

The central result of this unit is Savitch's theorem, which collapses nondeterministic and deterministic space up to a quadratic factor. Together with PSPACE-completeness of TQBF, it establishes the structural landscape of space-bounded computation.

**Theorem (Savitch).** *For any space-constructible $f : \mathbb{N} \to \mathbb{N}$ with $f(n) \ge \log n$,*
$$
\mathrm{NSPACE}(f(n)) \subseteq \mathrm{SPACE}(f(n)^2).
$$
*In particular, $\mathrm{PSPACE} = \mathrm{NPSPACE}$.*

*Proof.* Let $L \in \mathrm{NSPACE}(f(n))$, decided by a nondeterministic TM $M$ using $O(f(n))$ space. The configuration graph $G_{M,x}$ has at most $N = 2^{O(f(n))}$ vertices. Define a recursive predicate:
$$
\mathrm{PATH}(a, b, t) = \begin{cases} 1 & \text{if there is a path from } a \text{ to } b \text{ of length} \le 2^t \text{ in } G_{M,x}, \\ 0 & \text{otherwise.} \end{cases}
$$
For the base case $t = 0$: check whether $a = b$ or there is an edge from $a$ to $b$ (at most one step). For $t \ge 1$: there is a path of length at most $2^t$ from $a$ to $b$ if and only if there exists a midpoint $c$ such that both $\mathrm{PATH}(a, c, t-1)$ and $\mathrm{PATH}(c, b, t-1)$ hold.

The deterministic algorithm evaluates $\mathrm{PATH}(C_s, C_a, T)$ where $C_s$ is the start configuration, $C_a$ is any accepting configuration, and $2^T \ge N$. It iterates over all possible midpoints $c$ (each represented in $O(f(n))$ space), recursively checks both halves, and accepts if any midpoint works. The recursion depth is $T = O(f(n))$. At each level the algorithm stores one midpoint configuration using $O(f(n))$ space, and the two recursive calls reuse the same space. Total space: $O(T \cdot f(n)) = O(f(n)^2)$. $\square$

**Corollary.** $\mathrm{PSPACE} = \mathrm{NPSPACE}$.

*Proof.* $\mathrm{NSPACE}(n^k) \subseteq \mathrm{SPACE}(n^{2k})$ by Savitch. Taking the union over all $k$ gives $\mathrm{NPSPACE} \subseteq \mathrm{PSPACE}$. The reverse inclusion $\mathrm{PSPACE} \subseteq \mathrm{NPSPACE}$ is immediate since deterministic machines are a special case. $\square$

**Theorem (TQBF is PSPACE-complete).** *$\mathrm{TQBF}$ is PSPACE-complete.*

*Proof sketch.* Membership in PSPACE: evaluate a fully quantified Boolean formula $\phi = Q_1 x_1 \cdots Q_m x_m \cdot \psi$ recursively. Try both values of $x_1$; if $Q_1 = \exists$, accept if either recursive call accepts; if $Q_1 = \forall$, accept if both accept. The recursion depth is $m \le \mathrm{poly}(n)$ and each level stores one assignment to the already-processed variables, using polynomial space total.

Hardness: given a PSPACE machine $M$ using $S = n^c$ space and an input $x$, encode the computation as a quantified formula. The key idea is the same Savitch-style recursive predicate. Define $\mathrm{REACH}(C_1, C_2, t)$ to mean "configuration $C_2$ is reachable from $C_1$ in at most $2^t$ steps." Then $\mathrm{REACH}(C_1, C_2, t)$ holds if and only if there exists a midpoint $C_m$ such that for all intermediate configurations, both halves are reachable:
$$
\mathrm{REACH}(C_1, C_2, t) \iff \exists C_m \forall (C_a, C_b) \in \{(C_1, C_m), (C_m, C_2)\} : \mathrm{REACH}(C_a, C_b, t-1).
$$
The alternation of quantifiers mirrors the recursion. Unrolling $T = O(S)$ levels gives a fully quantified formula of polynomial size. $\square$

**Bridge.** This theorem is the foundational reason space-bounded complexity has a different character from time-bounded complexity: nondeterminism costs only a quadratic factor in space (Savitch) but is conjectured to cost an exponential factor in time (P versus NP), and this is exactly the structural asymmetry that makes PSPACE a natural home for alternating quantifiers and two-player games. The recursive PATH predicate builds toward the Immerman-Szelepcsenyi theorem [47.02.06], where the same configuration-graph reachability question is studied at the logarithmic level, and it appears again in the IP = PSPACE proof [47.03.04], where the arithmetisation of the quantifier alternations gives an interactive protocol. The central insight is that space can be reused — the two recursive calls in Savitch share the same memory — whereas time cannot, putting these together gives a deterministic simulation whose resource consumption is multiplicative in space but exponential in time, and the bridge is that TQBF packages this recursive reachability as a single formula whose alternating quantifiers are the logical fingerprint of PSPACE.

**Exercise 1 (easy, true-false).**

True or false: a Turing machine that uses $O(\log n)$ space on inputs of length $n$ must run in polynomial time.

<details class="exercise__hint"><summary>Hint</summary>

Count the number of distinct configurations.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**False.** A $O(\log n)$-space machine has at most $n^{O(1)}$ distinct configurations (polynomially many), so it must either halt within polynomial time or loop forever. A looping machine does not run in polynomial time. If the machine is guaranteed to halt, then yes, it runs in polynomial time.

</details>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Show that $\mathrm{PATH} = \{ \langle G, s, t \rangle : G \text{ is a directed graph with a path from } s \text{ to } t \}$ is in $\mathrm{NL} \subseteq \mathrm{PSPACE}$.

<details class="exercise__hint"><summary>Hint</summary>

A nondeterministic log-space machine can guess the path one vertex at a time, storing only the current vertex.

</details>

<details class="exercise__answer"><summary>Answer</summary>

A nondeterministic TM stores the current vertex (using $O(\log n)$ bits for a graph on $n$ vertices) and the remaining step counter. At each step it nondeterministically selects a neighbour and moves there. If it reaches $t$, accept. If the step counter reaches $n$ without finding $t$, reject (any simple path has at most $n$ vertices). The space used is $O(\log n)$ for the vertex label and counter, so $\mathrm{PATH} \in \mathrm{NL} \subseteq \mathrm{PSPACE}$. Rubric: full credit for the guessing procedure and the $O(\log n)$ space analysis.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 4 (medium, symbolic).**

Prove that $\mathrm{PSPACE} \subseteq \mathrm{EXPTIME}$ by bounding the running time of any polynomial-space Turing machine.

<details class="exercise__hint"><summary>Hint</summary>

Count distinct configurations. A machine that visits the same configuration twice is in a loop.

</details>

<details class="exercise__answer"><summary>Answer</summary>

A TM using $S = n^c$ space on inputs of length $n$ has at most $N = 2^{O(S)} = 2^{O(n^c)}$ distinct configurations. Any computation that visits more than $N$ configurations must repeat one, entering an infinite loop. A machine that halts therefore does so within $N = 2^{O(n^c)}$ steps, which is exponential time. Hence $\mathrm{PSPACE} \subseteq \mathrm{EXPTIME}$. Rubric: full credit for the configuration counting argument and the conclusion.

</details>

</aside>

**Exercise 5 (medium, multiple-choice).**

The formula $\forall x_1 \exists x_2 \forall x_3 \cdot (x_1 \lor x_2) \land (\neg x_2 \lor x_3)$ is a fully quantified Boolean formula with 3 variables. How many assignments must be checked in a direct evaluation?

<ol class="exercise__options" type="A">

<li>$2^{2^3} = 256$</li>
<li>$3! = 6$</li>
<li>$2 \times 2 \times 2 = 8$</li>
</ol>

<details class="exercise__hint"><summary>Hint</summary>

The outermost quantifier is $\forall x_1$, so both values of $x_1$ must lead to acceptance. For each, $\exists x_2$ requires finding one good value.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**A or D.** The formula has 3 variables, each taking 2 values. A brute-force evaluation tries all $2^3 = 8$ assignments to $(x_1, x_2, x_3)$. </details>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 6 (hard, symbolic).**

Let $\mathrm{GG} = \{ \langle G, s, t, k \rangle : G \text{ is a directed graph and the geography game starting from } s \text{ with } k \text{ moves reaches } t \}$ be the generalised geography problem. Show that $\mathrm{GG}$ is PSPACE-hard by reducing from TQBF.

<details class="exercise__hint"><summary>Hint</summary>

Model existential quantifiers as "Player 1 picks a value" and universal quantifiers as "Player 2 picks a value." Construct a graph where the two players alternate moves corresponding to quantifier assignments.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Given a QBF formula $Q_1 x_1 \cdots Q_m x_m \cdot \psi$, construct a graph for the geography game. Create a path of diamond gadgets: for each variable $x_i$, create a node where the current player chooses left ($x_i = 0$) or right ($x_i = 1$). If $Q_i = \exists$, it is Player 1's turn; if $Q_i = \forall$, it is Player 2's turn. After all variables are set, the game enters a subgraph encoding $\psi$: Player 1 wins iff $\psi$ is satisfied by the chosen assignment. Player 1 has a winning strategy in this game iff the QBF formula is true. The reduction is polynomial-time, so GG is PSPACE-hard. Membership in PSPACE follows from the standard game-tree search using polynomial space. Rubric: full credit for the quantifier-to-player construction and the polynomial-time reduction argument.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 7 (hard, symbolic).**

Prove the space hierarchy theorem for space-constructible functions: if $f(n) = o(g(n))$ and both are space-constructible with $g(n) \ge \log n$, then $\mathrm{SPACE}(f(n)) \subsetneq \mathrm{SPACE}(g(n))$.

<details class="exercise__hint"><summary>Hint</summary>

Diagonalise: define a language that disagrees with every $f(n)$-space machine on at least one input by simulating each machine for $g(n)$ steps and flipping the answer.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Define a TM $D$ that on input $\langle M \rangle$ (encoding of machine $M$) simulates $M$ on $\langle M \rangle$ using at most $g(n)$ space, where $n = |\langle M \rangle|$. If $M$ uses more than $g(n)$ space, reject. If $M$ halts, output the opposite of $M$'s answer. $D$ uses $O(g(n))$ space (the simulation overhead is a constant factor since $g$ is space-constructible and provides enough room), so $L(D) \in \mathrm{SPACE}(O(g(n))) = \mathrm{SPACE}(g(n))$ by the tape compression theorem.

Now suppose $L(D) \in \mathrm{SPACE}(f(n))$, decided by machine $M_0$. For large $n$, $M_0$ on input $x$ of length $n$ uses at most $f(n) < g(n)$ space. Run $D$ on input $\langle M_0 \rangle$: $D$ successfully simulates $M_0$ (since $M_0$ uses less than $g(|\langle M_0\rangle|)$ space) and outputs the opposite, contradicting $M_0 = D$. Hence $L(D) \notin \mathrm{SPACE}(f(n))$. Rubric: full credit for the diagonal machine construction, the space analysis, and the contradiction argument.

</details>

</aside>

**Exercise 8 (hard, symbolic).**

Show that the polynomial hierarchy $\mathrm{PH} \subseteq \mathrm{PSPACE}$. Conclude that if $\mathrm{P} = \mathrm{PSPACE}$ then $\mathrm{PH}$ collapses to $\mathrm{P}$.

<details class="exercise__hint"><summary>Hint</summary>

Each level $\Sigma_k$ of PH involves $k$ alternating quantifiers over polynomial-length certificates. Evaluate these quantifiers using polynomial space by recursion.

</details>

<details class="exercise__answer"><summary>Answer</summary>

A language $L \in \Sigma_k$ is decided by a polynomial-time predicate with $k$ alternating quantifier blocks: $x \in L \iff \exists y_1 \forall y_2 \cdots Q_k y_k : R(x, y_1, \ldots, y_k)$ where each $|y_i| \le \mathrm{poly}(|x|)$. A polynomial-space machine evaluates this by recursion on the quantifiers: for $\exists y_1$, iterate over all polynomial-length strings $y_1$ (there are $2^{\mathrm{poly}(n)}$ many) and accept if any recursive call accepts; for $\forall y_2$, accept if all recursive calls accept. Each recursive call adds one polynomial-length string to the stack, and the stack depth is $k = O(1)$. Total space: $O(k \cdot \mathrm{poly}(n)) = \mathrm{poly}(n)$. Hence $\Sigma_k \subseteq \mathrm{PSPACE}$ for every fixed $k$, and $\mathrm{PH} = \bigcup_k \Sigma_k \subseteq \mathrm{PSPACE}$. If $\mathrm{P} = \mathrm{PSPACE}$ then $\mathrm{PH} \subseteq \mathrm{P}$, so PH collapses to P. Rubric: full credit for the recursive quantifier evaluation argument and the collapse conclusion.

</details>

</aside>

## Advanced results [Master]

The theory of PSPACE extends well beyond the core theorems. The results below develop the space hierarchy, sharpen the relationship between time and space, and explore the structure of PSPACE-complete problems.

**Theorem 1 (Savitch's theorem, detailed analysis).** The deterministic simulation in Savitch's theorem uses time $2^{O(f(n))}$ even though space is only $f(n)^2$. This exponential time overhead is inherent in the recursive approach: the $\mathrm{PATH}(a, b, t)$ predicate explores all $2^{O(f(n))}$ midpoints at each level, and the recursion has $O(f(n))$ levels. Any improvement to polynomial time while maintaining polynomial space would resolve open questions in complexity theory, since $\mathrm{NL} \subseteq \mathrm{P}$ would follow from polynomial-time simulation of nondeterministic logarithmic space [ref: TODO_REF Arora-Barak §4.2].

**Theorem 2 (TQBF completeness, full proof).** The PSPACE-completeness of TQBF requires two directions. Membership was sketched above; the hardness direction encodes the PSPACE computation as a quantified formula using a technique analogous to the Cook-Levin construction [47.02.02]. Given $M$ using $S = n^c$ space, let $C_s$ and $C_a$ be start and accepting configurations. Define $\phi_t(C_1, C_2)$ to encode "$C_2$ is reachable from $C_1$ in at most $2^t$ steps." For $t = 0$, $\phi_0$ checks that $C_1 = C_2$ or there is a single-step transition, expressible as a constant-size Boolean formula. For $t \ge 1$, apply the Savitch recursion:
$$
\phi_t(C_1, C_2) = \exists C_m \forall i \in \{0, 1\} : \phi_{t-1}(C_{1}^{(i)}, C_{2}^{(i)})
$$
where $(C_1^{(0)}, C_2^{(0)}) = (C_1, C_m)$ and $(C_1^{(1)}, C_2^{(1)}) = (C_m, C_2)$. Unrolling $T = O(S)$ levels gives a quantified formula of size polynomial in $n$ (each configuration is $O(S)$ bits and $T = O(S)$, but the key observation is that the same variable names are reused across levels). The formula is true if and only if $M$ accepts $x$, so the reduction is correct [ref: TODO_REF Sipser §8.3].

**Theorem 3 (space hierarchy theorem).** For space-constructible $f, g$ with $f(n) = o(g(n))$ and $g(n) \ge \log n$, the inclusion $\mathrm{SPACE}(f(n)) \subsetneq \mathrm{SPACE}(g(n))$ is strict [ref: TODO_REF Arora-Barak §4.1]. The proof diagonalises against all $f(n)$-space machines: construct a language $L_D$ that on input $\langle M \rangle$ simulates $M$ on $\langle M \rangle$ for $g(n)$ space and flips the answer. As long as $g(n)$ exceeds the simulation overhead, $L_D$ separates the classes. Important corollaries: $\mathrm{L} \subsetneq \mathrm{PSPACE}$ and $\mathrm{PSPACE} \subsetneq \mathrm{EXPSPACE}$.

**Theorem 4 (PSPACE and games).** Many two-player games of polynomial length are PSPACE-complete. The generalised geography problem and the problem of deciding whether the first player has a winning strategy in generalised chess, Go, or checkers on an $n \times n$ board are PSPACE-complete [ref: TODO_REF Arora-Barak §4.3]. The correspondence between alternating quantifiers and alternating game moves is the structural reason: an existential quantifier corresponds to "there exists a move for Player 1" and a universal quantifier to "for every response by Player 2," so TQBF is equivalent to asking whether Player 1 has a winning strategy in a polynomial-depth game tree.

**Theorem 5 (PSPACE and the polynomial hierarchy).** The polynomial hierarchy $\mathrm{PH} = \bigcup_{k \ge 0} \Sigma_k^{\mathrm{P}}$ satisfies $\mathrm{PH} \subseteq \mathrm{PSPACE}$. Each level $\Sigma_k^{\mathrm{P}}$ involves $k$ alternating polynomial-length quantifiers over a polynomial-time predicate, and evaluating these quantifiers sequentially using polynomial space places $\Sigma_k^{\mathrm{P}} \subseteq \mathrm{PSPACE}$. Whether $\mathrm{PH} = \mathrm{PSPACE}$ is open; if they are equal, then the polynomial hierarchy collapses. This is dual to the P vs NP question: both ask whether additional quantifier alternation adds computational power [ref: TODO_REF Arora-Barak §5.2].

**Synthesis.** The foundational reason the space complexity landscape differs from the time landscape is that space can be reused: Savitch's theorem exploits this by having two recursive calls share the same memory, which builds toward the surprising equality PSPACE = NPSPACE and this is exactly the structural fact that makes space more tractable than time as a resource to analyse. The alternating quantifiers in TQBF are dual to the alternating moves in two-player games, and this is exactly the bridge to interactive proofs [47.03.04], where the IP = PSPACE theorem shows that every PSPACE computation can be verified by a polynomial-time verifier interacting with a powerful prover. The central insight unifying these results is that the configuration graph of a space-bounded machine has polynomial diameter when measured in the logarithm of the number of configurations, and putting these together the Savitch recursion, the TQBF encoding, and the game-tree characterisation all express the same polynomial-space reachability question in different languages. The space hierarchy theorem [47.02.01] provides the strict separations that make this structure non-vacuous, and the generalises connection from PSPACE to the polynomial hierarchy [47.02.03] shows that space complexity subsumes the entire quantifier-alternation hierarchy.

## Full proof set [Master]

**Proposition 1 (Savitch's theorem).** *For space-constructible $f(n) \ge \log n$, $\mathrm{NSPACE}(f(n)) \subseteq \mathrm{SPACE}(f(n)^2)$.*

*Proof.* Let $M$ be a nondeterministic TM deciding $L$ in space $O(f(n))$. The configuration graph $G_{M,x}$ has $N = 2^{O(f(n))}$ vertices. Define $\mathrm{PATH}(a, b, t)$ as above. The algorithm for $\mathrm{PATH}(a, b, t)$: if $t = 0$, return 1 iff $a = b$ or $(a, b)$ is an edge. If $t \ge 1$, for each vertex $c$ (enumerated in $O(f(n))$ space): compute $\mathrm{PATH}(a, c, t-1)$; if it returns 1, compute $\mathrm{PATH}(c, b, t-1)$; if both return 1, return 1. After all midpoints, return 0.

Space analysis: the recursion depth is $t$. At each level, the algorithm stores one midpoint $c$ using $O(f(n))$ space and one return value. The two recursive calls are sequential and reuse space. Total space: $O(t \cdot f(n))$. Setting $t = O(f(n))$ so that $2^t \ge N$ gives total space $O(f(n)^2)$. $\square$

**Proposition 2 (PSPACE = NPSPACE).** *$\mathrm{PSPACE} = \mathrm{NPSPACE}$.*

*Proof.* For any $k$, $\mathrm{NSPACE}(n^k) \subseteq \mathrm{SPACE}(n^{2k})$ by Proposition 1. Union over $k$: $\mathrm{NPSPACE} = \bigcup_k \mathrm{NSPACE}(n^k) \subseteq \bigcup_k \mathrm{SPACE}(n^{2k}) = \mathrm{PSPACE}$. The reverse inclusion is immediate. $\square$

**Proposition 3 (TQBF is in PSPACE).** *The language TQBF is decidable in polynomial space.*

*Proof.* Given a QBF $\phi = Q_1 x_1 \cdots Q_m x_m \cdot \psi$, evaluate recursively. The recursive procedure takes a partial assignment $\sigma$ to the first $i$ variables and evaluates the remaining quantifiers. For $Q_{i+1} = \exists$: try $\sigma[x_{i+1} := 0]$ and $\sigma[x_{i+1} := 1]$; accept if either recursive call accepts. For $Q_{i+1} = \forall$: accept if both recursive calls accept. When all variables are assigned, evaluate $\psi$ in polynomial time. Each recursive level stores one additional variable value, so the space is $O(m) \le O(|\phi|)$. $\square$

**Proposition 4 (PSPACE-completeness of TQBF).** *Every language $L \in \mathrm{PSPACE}$ is polynomial-time reducible to TQBF.*

*Proof.* Let $M$ decide $L$ in space $p(n)$ for polynomial $p$. For input $x$, construct the formula $\phi_T(C_s, C_a)$ using the recursive Savitch encoding with $T = O(p(n))$ levels. At each level, a configuration is encoded by $O(p(n))$ Boolean variables. The formula $\phi_0(C_1, C_2)$ checks equality or single-step reachability with a constant-size circuit. The recursion $\phi_t = \exists C_m \forall i : \phi_{t-1}(C_1^{(i)}, C_2^{(i)})$ introduces $O(p(n))$ new variables per level and $O(p(n))$ levels, giving a formula of total size $\mathrm{poly}(n)$. The formula is true iff $M$ accepts $x$. The construction is polynomial-time since each level's encoding is polynomial and there are polynomially many levels. $\square$

**Proposition 5 (space hierarchy).** *If $f(n)$ is space-constructible and $f(n) = o(g(n))$ with $g(n) \ge \log n$ space-constructible, then $\mathrm{SPACE}(f(n)) \subsetneq \mathrm{SPACE}(g(n))$.*

*Proof.* Define TM $D$: on input $x$ of length $n$, use $g(n)$ space to simulate $M_x$ on $x$ (where $M_x$ is the machine encoded by $x$, padded if needed). If $M_x$ attempts to use more than $g(n)$ space, reject. Otherwise, output the complement of $M_x$'s output. $D$ uses $O(g(n))$ space, so $L(D) \in \mathrm{SPACE}(O(g(n))) = \mathrm{SPACE}(g(n))$.

Suppose $L(D) \in \mathrm{SPACE}(f(n))$. Then some machine $M_0$ decides $L(D)$ in space $c \cdot f(n)$ for constant $c$. For large enough $n$, $c \cdot f(n) < g(n)$. Let $x_0$ encode $M_0$ with appropriate padding. Then $D$ on $x_0$ successfully simulates $M_0$ (which uses $< g(|x_0|)$ space) and outputs the opposite, so $x_0 \in L(D) \iff x_0 \notin L(M_0)$, contradicting $M_0$ deciding $L(D)$. $\square$

## Connections [Master]

- Time complexity and P/NP [47.02.01] provides the foundational complexity classes that PSPACE extends. The containment $\mathrm{P} \subseteq \mathrm{NP} \subseteq \mathrm{PSPACE}$ follows from P/NP theory, and the polynomial hierarchy [47.02.03] sits between NP and PSPACE. This unit owns the space-bounded analogue and the structural collapse PSPACE = NPSPACE.

- NP-completeness [47.02.02] introduces the reduction technique that PSPACE-completeness mirrors. The Cook-Levin reduction from any NP machine to SAT is the template for the reduction from any PSPACE machine to TQBF, with the key difference being the unbounded quantifier alternation.

- L, NL, and Immerman-Szelepcsenyi [47.02.06] studies space-bounded classes below PSPACE. Savitch's theorem applies at the logarithmic level too, giving $\mathrm{NL} \subseteq \mathrm{SPACE}(\log^2 n)$, but the Immerman-Szelepcsenyi theorem gives the tighter result $\mathrm{NL} = \mathrm{coNL}$, which has no known time analogue.

- Interactive proofs [47.03.04] establish that $\mathrm{IP} = \mathrm{PSPACE}$, giving a characterisation of PSPACE in terms of interactive protocols. The arithmetisation of the TQBF quantifier structure is the core of the LFKN-Shamir proof, and the PSPACE-completeness of TQBF developed here is the starting point for that construction.

- Models of computation [42.04.01] supply the Turing machine model and configuration formalism that underlies the space-bounded analysis. The configuration graph is a refinement of the universal machine construction from that unit.

## Historical & philosophical context [Master]

Walter Savitch proved his theorem in 1970 in his PhD thesis at the University of California, Berkeley, under the supervision of Manuel Blum. The result appeared in the *Journal of Computer and System Sciences* as "Relationships Between Nondeterministic and Deterministic Tape Complexities" (Savitch, 1970). At the time, the relationship between nondeterministic and deterministic space was wide open, and Savitch's quadratic bound was a strikingly strong result — it showed that nondeterminism is far less powerful in the space setting than was (and still is) conjectured for time [ref: TODO_REF Sipser §8.2].

The PSPACE-completeness of TQBF was established by Larry Stockmeyer and Albert Meyer in 1973 in their paper "Word Problems Requiring Exponential Time," which also introduced the polynomial hierarchy. The connection between PSPACE and two-player games was explored by Schaefer (1978), who proved PSPACE-completeness for a wide range of games. The geography game reduction became a standard textbook example.

The equality IP = PSPACE, proved by Lund, Fortnow, Karloff, and Nisan (1990) and Shamir (1990), gave a completely unexpected characterisation of PSPACE through interaction rather than quantifiers, connecting this unit to the interactive proof framework. The space hierarchy theorem, a consequence of earlier work by Hartmanis, Lewis, and Stearns (1965), provides the strict inclusions that give the theory nontrivial content.
