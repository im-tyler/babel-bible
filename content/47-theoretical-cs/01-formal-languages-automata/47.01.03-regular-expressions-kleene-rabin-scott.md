## Intuition [Beginner]

You have used wildcards in file names. The pattern `*.txt` matches every file ending in `.txt`. The `*` means "any sequence of characters." This idea — describing a set of strings by a pattern — is the essence of a regular expression.

A regular expression builds patterns from three operations. *Union* (written $R_1 \cup R_2$ or $R_1 | R_2$) means "strings matching $R_1$ or strings matching $R_2$." *Concatenation* (written $R_1 R_2$) means "a string matching $R_1$ followed by a string matching $R_2$." *Star* (written $R^*$) means "zero or more repetitions of strings matching $R$."

These three operations, applied to individual symbols and the empty string, generate every pattern that a finite automaton can recognise. This is Kleene's theorem, and it is one of the foundational results of computer science: the algebraic description (regular expressions) and the machine description (DFAs from [47.01.01]) define exactly the same class of languages.

Consider the language of strings over $\{0, 1\}$ that end in "01." The regular expression is $(0 \cup 1)^*01$. Read it as: any sequence of 0s and 1s (the $(0 \cup 1)^*$ part), followed by a 0, followed by a 1. The star handles the "any prefix" part; the final 01 nails down the suffix.

## Visual [Beginner]

The three regex operations drawn as NFA fragments (the Thompson construction):

```
Union R1|R2:         Concatenation R1R2:      Star R*:
   ┌──[R1]──┐        [R1] ──ε──> [R2]        ┌──[R]──┐
ε─>│        │─>ε     start    end             ε─>│   ↑   │
   └──[R2]──┘        (output of R1 feeds      └───┘──ε──┘
   ↑                 into input of R2)        ε-self-loop
   start                                   + ε-bypass
```

| Regex | Meaning | Example strings |
|---|---|---|
| $0^*$ | Zero or more 0s | $\varepsilon$, 0, 00, 000 |
| $01$ | The string "01" | 01 only |
| $(0 \cup 1)^*$ | Any string of 0s and 1s | $\varepsilon$, 0, 1, 01, 10, ... |
| $(0 \cup 1)^*01$ | Any string ending in 01 | 01, 001, 1101 |
| $\emptyset$ | The empty language | (no strings) |

## Worked example [Beginner]

Convert a DFA for "strings ending in 01" into a regular expression using the state elimination method.

Start with the 3-state DFA from [47.01.01]: states $q_0$, $q_1$, $q_2$, with $q_0$ start and $q_2$ accept.

**Step 1.** Set up language equations. Let $R_i$ denote the set of all strings that take the DFA from state $q_0$ to state $q_i$. We get a system:

- $R_0 = R_0 \cdot 0 \cup R_0 \cdot 1 \cup \{\varepsilon\}$ (from $q_0$ on 0 or 1, return to $q_0$; plus the empty string for starting here).

Wait — the equations follow the transitions. Let $L_{ij}$ denote the labels on edges from state $i$ to state $j$. Then for each state $q_i$:

- $R_0 = R_0 \cdot 0 \cup R_1 \cdot 0 \cup R_2 \cdot 1 \cup \{\varepsilon\}$

Actually, it is cleaner to use Arden's lemma directly. We write equations for the language of strings reaching each state from the start:

- $X_0 = X_0(0 \cup 1) \cup \{\varepsilon\}$ (since $q_0$ has self-loops on both 0 and 1).

By Arden's lemma ($X = XA \cup B$ has solution $X = BA^*$): $X_0 = \{\varepsilon\}(0 \cup 1)^* = (0 \cup 1)^*$.

**Step 2.** The accepted language is $X_2$, the set of strings reaching $q_2$. We need to account for paths through intermediate states. The state elimination procedure systematically removes states, replacing them with regular expression labels on the remaining edges. After eliminating $q_1$ and simplifying, the result is $(0 \cup 1)^*01$.

**Step 3.** Verify: any string matching $(0 \cup 1)^*01$ has an arbitrary prefix followed by 01, so it ends in 01. Conversely, any string ending in 01 can be decomposed as a prefix (matching $(0 \cup 1)^*$) followed by the suffix 01. The regex and the DFA describe the same language.

## Formal definition [Intermediate+]

**Definition (regular expression).** Let $\Sigma$ be a finite alphabet. The set of *regular expressions* over $\Sigma$ is defined inductively:

1. $\emptyset$ is a regular expression (denoting the empty language $\emptyset$).
2. $\varepsilon$ is a regular expression (denoting the singleton language $\{\varepsilon\}$).
3. For each $a \in \Sigma$, the symbol $a$ is a regular expression (denoting $\{a\}$).
4. If $R_1$ and $R_2$ are regular expressions, then $(R_1 \cup R_2)$ is a regular expression (denoting $L(R_1) \cup L(R_2)$).
5. If $R_1$ and $R_2$ are regular expressions, then $(R_1 \circ R_2)$ is a regular expression (denoting $L(R_1) \cdot L(R_2) = \{uv : u \in L(R_1), v \in L(R_2)\}$).
6. If $R$ is a regular expression, then $R^*$ is a regular expression (denoting $L(R)^* = \bigcup_{k=0}^{\infty} L(R)^k$).

The language $L(R)$ denoted by a regular expression $R$ is defined by the interpretation above. A language $L$ is *regular* iff there exists a regular expression $R$ with $L(R) = L$.

**Definition (Kleene star).** For a language $A \subseteq \Sigma^*$, the *Kleene star* is $A^* = \bigcup_{k=0}^{\infty} A^k$ where $A^0 = \{\varepsilon\}$ and $A^{k+1} = A^k \cdot A = \{uv : u \in A^k, v \in A\}$.

**Theorem (Arden's lemma).** Let $A, B \subseteq \Sigma^*$ with $\varepsilon \notin A$. Then the equation $X = XA \cup B$ has the unique solution $X = BA^*$ [ref: TODO_REF Hopcroft, Motwani & Ullman §3.4].

*Proof.* First, $BA^* \subseteq (BA^*)A \cup B$ since $BA^* = B \cup BA^+ = B \cup (BA^*)A$. Conversely, if $X = XA \cup B$, then by induction $X = XA^n \cup \bigcup_{k=0}^{n-1} BA^k$ for all $n$. Since $\varepsilon \notin A$, any $w \in X$ with $|w| = m$ must arise from $BA^k$ with $k \le m$, so $X \subseteq BA^*$. $\square$

### Counterexamples to common slips

- *"$R^*$ includes only non-empty strings."* No: $R^*$ always includes $\varepsilon$ (the $k = 0$ term). Even $\emptyset^* = \{\varepsilon\}$.

- *"Concatenation distributes over union in both directions."* $R(S \cup T) = RS \cup RT$ and $(R \cup S)T = RT \cup ST$ hold, but $R^* R^* = R^*$ does not mean the star distributes — $R^*$ is idempotent but $R^{**} = R^*$ is a consequence of $R^* = (R^*)^*$, not a distributive law.

- *"Regular expressions and regex libraries are the same thing."* Practical regex engines (Perl, Python's `re`) add backreferences, lookahead, and other features that go beyond the formal regular expressions defined here. These extensions recognise non-regular languages.

### Key theorem with proof [Intermediate+]

**Theorem (Kleene's theorem: DFA $\Leftrightarrow$ regex).** *A language $L \subseteq \Sigma^*$ is regular (recognised by some DFA) if and only if $L = L(R)$ for some regular expression $R$.*

*Proof sketch (DFA to regex).* Given a DFA $M = (Q, \Sigma, \delta, q_0, F)$ with $Q = \{q_1, \ldots, q_n\}$ and $q_1 = q_0$, define $R_{ij}^{(k)}$ to be the set of strings that take $M$ from state $q_i$ to state $q_j$ without passing through any state with index greater than $k$. Base case $k = 0$: $R_{ij}^{(0)}$ is the union of single symbols $a$ with $\delta(q_i, a) = q_j$, plus $\{\varepsilon\}$ if $i = j$. Inductive step:

$$R_{ij}^{(k)} = R_{ij}^{(k-1)} \cup R_{ik}^{(k-1)} (R_{kk}^{(k-1)})^* R_{kj}^{(k-1)}.$$

Each $R_{ij}^{(k)}$ is denoted by a regular expression built from the previous step. The language of $M$ is $\bigcup_{q_j \in F} R_{1j}^{(n)}$.

*Proof sketch (regex to NFA).* By structural induction on $R$. Each regex is converted to an NFA with a single start and single accept state using the Thompson construction: atomic regexes $a$ become two-state NFAs; union is handled by a new start state with $\varepsilon$-transitions to both sub-NFAs; concatenation chains the accept state of the first to the start of the second; star adds $\varepsilon$-transitions from the accept state back to the start and a bypass. The result is an NFA recognising $L(R)$, which by the subset construction of [47.01.02] converts to a DFA. $\square$

**Bridge.** Kleene's theorem is the foundational reason the regular languages have a dual characterisation: the operational description (DFAs reading strings symbol by symbol) and the algebraic description (regular expressions combining languages by union, concatenation, and star) define exactly the same class. This builds toward the Thompson construction used in every regex compiler, which appears again in [47.01.02] where the NFA-to-DFA conversion completes the chain from algebraic pattern to deterministic machine. The central insight is that the three regex operations correspond directly to NFA composition patterns, and putting these together with the subset construction yields a complete pipeline from human-readable pattern to executable automaton.

## Advanced results [Master]

The Thompson construction and the state elimination algorithm are the two pillars of Kleene's theorem. The first goes from algebra to machines, the second from machines to algebra. Several refinements sharpen the picture.

**Theorem 1 (Thompson construction state bound).** A regular expression $R$ of length $|R|$ (number of symbols and operators) can be converted to an NFA with at most $2|R|$ states, each with at most two outgoing transitions [ref: TODO_REF Hopcroft, Motwani & Ullman §3.2]. This linear bound makes the Thompson construction the basis of practical regex compilers: the NFA can be simulated in $O(|w| \cdot |R|)$ time by tracking the set of active states.

**Theorem 2 (Glushkov automaton).** For a regular expression $R$ over $\Sigma$ with $k$ symbol occurrences, the Glushkov (position) automaton has exactly $k + 1$ states, one for each position in $R$ plus a start state. The automaton is constructed by computing the *first*, *last*, and *follow* sets of positions, which are properties of the parse tree of $R$. The Glushkov automaton is always an NFA without $\varepsilon$-transitions, and it is often smaller than the Thompson NFA.

**Theorem 3 (Kleene algebra axioms).** The regular expressions, modulo the equations $R + R = R$ (idempotence), $R + \emptyset = R$, $R \circ \varepsilon = R$, $(R^*)^* = R^*$, and $R^* = \varepsilon + R \circ R^*$, form an idempotent semiring with star satisfying the *unfolding* and *induction* axioms of Kleene algebra. Kozen (1994) proved that these axioms are complete for the equational theory of regular events: $L(R) = L(S)$ iff $R = S$ is provable from the Kleene algebra axioms. This algebraic characterisation is used in program verification and static analysis.

**Synthesis.** Kleene's theorem is the foundational reason the regular languages admit three equivalent characterisations — automata-theoretic (DFAs), algebraic (regular expressions), and algebraic-structural (Kleene algebra) — and the conversion algorithms between them are the practical backbone of pattern matching, lexical analysis, and network packet filtering. The central insight is that the three regex operations (union, concatenation, star) mirror the three NFA composition patterns, and putting these together with the subset construction of [47.01.02] gives a complete equivalence proof. This equivalence builds toward the pumping lemma of [47.01.04], where the finiteness of the automaton imposes a structural limitation on regularity that no regex can overcome, and it appears again in [47.01.05] where context-free grammars extend the algebraic framework with recursion, breaking the regular barrier.

## Full proof set [Master]

**Proposition 1 (DFA to regex via state elimination).** *Every DFA $M$ has a regular expression $R$ with $L(R) = L(M)$.*

*Proof.* Number the states $q_1, \ldots, q_n$ with $q_1$ the start state. Add a new unique start state $q_s$ with an $\varepsilon$-transition to $q_1$, and a new unique accept state $q_f$ with $\varepsilon$-transitions from each old accept state. The augmented machine has states $\{q_s, q_1, \ldots, q_n, q_f\}$ and transitions labelled by regular expressions (initially, single symbols and $\varepsilon$).

Eliminate states $q_1, \ldots, q_n$ one at a time. To eliminate state $q_k$: for every pair $(q_i, q_j)$ with $i, j \neq k$, replace the current label $R_{ij}$ on the edge from $q_i$ to $q_j$ by $R_{ij} \cup R_{ik} R_{kk}^* R_{kj}$. This accounts for all paths from $q_i$ to $q_j$ that may pass through $q_k$ any number of times. After removing $q_k$ and its incident edges, the language of paths between the remaining states is preserved.

After all $n$ states are eliminated, only $q_s$ and $q_f$ remain, connected by a single edge labelled with the desired regular expression $R$. By construction, $L(R) = L(M)$. $\square$

**Proposition 2 (regex to NFA: Thompson construction).** *Every regular expression $R$ over $\Sigma$ has an NFA $N$ with $L(N) = L(R)$, a single start state, and a single accept state, with at most $2|R|$ states.*

*Proof.* By structural induction. For atomic regexes $\emptyset$, $\varepsilon$, and $a \in \Sigma$: construct small NFAs directly (0, 1, and 2 states respectively). For $R_1 \cup R_2$: create a new start state $s$ with $\varepsilon$-transitions to the start states of $N(R_1)$ and $N(R_2)$, and merge their accept states into a new accept state $f$ via $\varepsilon$-transitions. For $R_1 \circ R_2$: connect the accept state of $N(R_1)$ to the start state of $N(R_2)$ via an $\varepsilon$-transition; the start of $N(R_1)$ is the new start, the accept of $N(R_2)$ is the new accept. For $R^*$: add $\varepsilon$-transitions from the accept of $N(R)$ back to its start, from a new start $s$ to the start of $N(R)$, and from the accept of $N(R)$ to a new accept $f$; also add $\varepsilon$ from $s$ to $f$ for the zero-repetitions case. The state count is linear in the size of $R$ by induction. $\square$

## Connections [Master]

- [47.01.01] — DFAs are the machine model that Kleene's theorem connects to regular expressions; the product construction for Boolean closure reappears in the state elimination algorithm's handling of union paths.

- [47.01.02] — NFAs are the intermediate representation in the Thompson construction; the subset construction converts the Thompson NFA back to a DFA, completing the DFA-regex-DFA cycle.

- [47.01.04] — The pumping lemma exploits the finiteness of the DFA equivalent of any regular expression, proving that certain languages (like $\{0^n 1^n : n \ge 0\}$) have no regex description.

- [47.01.05] — Context-free grammars extend regular expressions by allowing recursive nonterminals, which enables the description of nested structures like balanced parentheses.

## Historical & philosophical context [Master]

Stephen Kleene proved the equivalence of finite automata and regular expressions in 1956 in his paper "Representation of Events in Nerve Nets and Finite Automata" [ref: TODO_REF Kleene 1956]. His proof used what are now called "regular events" — sets of strings closed under union, concatenation, and star — and showed they coincide with the sets recognised by McCulloch-Pitts nerve nets. The term "regular expression" was introduced later; Kleene used "regular event."

Ken Thompson, one of the creators of Unix, implemented the Thompson construction in the QED text editor (1968) and later in `ed` and `grep`. His construction produces NFAs with a linear number of states, making regex matching efficient in practice. The construction was described in his 1968 CACM paper "Regular Expression Search Algorithm" [ref: TODO_REF Hopcroft, Motwani & Ullman §3.2].

The algebraic theory of regular expressions was developed by John Conway in *Regular Algebra and Finite Machines* (1971) and systematised by Dexter Kozen in 1994, who proved the completeness of the Kleene algebra axioms for the equational theory of regular events. Kozen's result placed regular expressions on the same algebraic footing as rings and groups, with a finitely axiomatised equational theory.
