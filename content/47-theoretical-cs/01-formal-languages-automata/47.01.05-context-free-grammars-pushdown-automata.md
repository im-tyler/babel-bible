## Intuition [Beginner]

Think of a sentence in English. "The cat sat on the mat." A grammar for English would say: a sentence is a noun phrase followed by a verb phrase. A noun phrase is "the" followed by a noun. A noun is "cat" or "mat." And so on.

The key idea is that a grammar is a collection of replacement rules. You start with a single symbol (the "start symbol") and repeatedly replace symbols according to the rules until only words remain. Different choices of which rule to apply when produce different sentences.

A context-free grammar (CFG) works this way for formal languages. Each rule says "replace this single symbol with this sequence of symbols." The word "context-free" means the rule does not depend on what other symbols are nearby — the replacement applies regardless of context.

This is more powerful than a finite automaton because the rules can nest structures inside each other. A finite automaton cannot count an arbitrary number of open parentheses and match them with close parentheses, because it has only finitely many states. But a CFG handles this with a single rule: "a balanced string is an open parenthesis, a balanced string, and a close parenthesis."

A pushdown automaton (PDA) is the machine counterpart of a CFG. It is like an NFA with a stack. The stack provides unlimited memory, but with a restricted access pattern: you can only look at and change the top of the stack. This is exactly enough to match parentheses — push on open, pop on close.

## Visual [Beginner]

A parse tree for the string "a + b * c" using a simple arithmetic grammar:

```
        E
       /|\
      E  +  E
      |    /|\
      T   T  *  T
      |   |     |
      F   F     F
      |   |     |
      a   b     c
```

Rules: $E \to E + E \mid E * E \mid T$, $T \to F$, $F \to a \mid b \mid c$.

The parse tree shows that "a + b * c" is derived by first splitting into $E + E$, then the right $E$ splits into $T * T$. This captures the structure that multiplication binds tighter than addition.

| String | Derived? | Parse tree depth |
|---|---|---|
| a | $F \to a$ | 3 |
| a + b | $E \to E + E \to T + T \to F + F \to a + b$ | 4 |
| a + b * c | As above | 5 |

## Worked example [Beginner]

We build a CFG for the language of all strings of balanced parentheses over the alphabet `{ ( , ) }`.

**Rules.** The grammar has one variable $S$ and the rules:

- $S \to (S)$ (wrap a balanced string in parentheses)
- $S \to SS$ (concatenate two balanced strings)
- $S \to \varepsilon$ (the empty string is balanced)

**Derive the string "()()".** Start with $S$. Apply $S \to SS$: we get $SS$. Apply $S \to (S)$ to the first $S$: we get $(S)S$. Apply $S \to \varepsilon$ to the inner $S$: we get $()S$. Apply $S \to (S)$ to the remaining $S$: we get $()(S)$. Apply $S \to \varepsilon$: we get "()()." Done.

**Derive "(()".** Can we? We need a balanced string, but "(()" has two open and one close — it is not balanced. No derivation can produce it. The grammar correctly generates only balanced strings.

**Parse tree for "(())".** $S \to (S) \to ((S)) \to (())$. The tree is:

```
    S
   /|\
  ( S )
   /|\
  ( S )
    |
    ε
```

## Formal definition [Intermediate+]

**Definition (CFG).** A *context-free grammar* is a 4-tuple $G = (V, \Sigma, R, S)$ where:

- $V$ is a finite set of *variables* (nonterminals),
- $\Sigma$ is a finite set of *terminals* with $V \cap \Sigma = \emptyset$,
- $R \subseteq V \times (V \cup \Sigma)^*$ is a finite set of *production rules*, written $A \to \alpha$,
- $S \in V$ is the *start variable*.

**Definition (derivation).** For strings $\alpha, \beta \in (V \cup \Sigma)^*$, we write $\alpha \Rightarrow \beta$ (alpha derives beta in one step) if $\alpha = \gamma A \delta$ and $\beta = \gamma \omega \delta$ for some $A \to \omega \in R$. The reflexive transitive closure is $\Rightarrow^*$. The *language generated* by $G$ is $L(G) = \{w \in \Sigma^* : S \Rightarrow^* w\}$.

**Definition (parse tree).** A *parse tree* (derivation tree) for $G$ is a rooted tree where: the root is labelled $S$; each internal node labelled $A$ has children labelled $\alpha_1, \ldots, \alpha_k$ where $A \to \alpha_1 \cdots \alpha_k \in R$; each leaf is labelled with a terminal or $\varepsilon$; reading the leaves left-to-right yields the derived string.

**Definition (ambiguity).** A CFG $G$ is *ambiguous* if there exists a string $w \in L(G)$ that has two or more distinct parse trees (equivalently, two distinct leftmost derivations). A language $L$ is *inherently ambiguous* if every CFG for $L$ is ambiguous.

**Definition (Chomsky normal form).** A CFG is in *Chomsky normal form* (CNF) if every rule is of the form $A \to BC$ (two variables) or $A \to a$ (one terminal), and $S \to \varepsilon$ is allowed only if $\varepsilon \in L(G)$.

**Theorem (conversion to CNF).** *Every CFG $G$ can be converted into a CFG $G'$ in Chomsky normal form with $L(G') = L(G) \setminus \{\varepsilon\}$ (and $L(G') = L(G)$ if $S \to \varepsilon$ is added back).*

The conversion proceeds in stages: add a new start variable, eliminate $\varepsilon$-productions (rules of the form $A \to \varepsilon$ where $A \ne S$), eliminate unit productions ($A \to B$), and split long right-hand sides into binary branching rules.

**Definition (PDA).** A *pushdown automaton* is a 6-tuple $P = (Q, \Sigma, \Gamma, \delta, q_0, F)$ where:

- $Q$ is a finite set of states,
- $\Sigma$ is the input alphabet,
- $\Gamma$ is the *stack alphabet*,
- $\delta : Q \times (\Sigma \cup \{\varepsilon\}) \times (\Gamma \cup \{\varepsilon\}) \to \mathcal{P}(Q \times (\Gamma \cup \{\varepsilon\}))$ is the transition function,
- $q_0 \in Q$ is the start state,
- $F \subseteq Q$ is the set of accept states.

A transition $\delta(q, a, b) \ni (q', c)$ means: in state $q$, reading input $a$ (or $\varepsilon$), and popping $b$ from the stack (or $\varepsilon$ for no pop), go to state $q'$ and push $c$ onto the stack (or $\varepsilon$ for no push).

### Counterexamples to common slips

- *"A CFG with rules $A \to aB$ is context-free."* Yes, this is a right-linear grammar, which is a special case of CFG. Right-linear grammars generate exactly the regular languages.

- *"CNF restricts the grammar so much that some languages cannot be expressed."* CNF is a normal form: every context-free language has a CFG in CNF. The conversion may increase the number of variables but does not change the language.

- *"A PDA's stack can read any position."* The stack is last-in-first-out (LIFO). You can only read and modify the top element. This restriction is what makes the PDA weaker than a Turing machine.

### Key theorem with proof [Intermediate+]

**Theorem (CFG = PDA).** *A language $L$ is context-free (generated by some CFG) if and only if it is recognised by some PDA.*

*Proof sketch (CFG to PDA).* Given $G = (V, \Sigma, R, S)$, construct a PDA $P$ with two states $q_{\text{start}}$ and $q_{\text{loop}}$. The PDA operates as follows: push $S$ onto the stack. In $q_{\text{loop}}$, repeatedly do one of two things. If the top of the stack is a variable $A$, nondeterministically choose a rule $A \to \alpha$ and replace $A$ on the stack with $\alpha$ (pushing the symbols right-to-left so the leftmost symbol is on top). If the top of the stack is a terminal $a$, read the next input symbol and check that it matches $a$; if so, pop $a$. Accept when the input is exhausted and the stack is empty.

The PDA simulates a leftmost derivation of $G$: each variable replacement corresponds to a grammar rule, and each terminal match corresponds to reading the input. Every accepting computation of $P$ yields a valid derivation of the input string, and every derivation yields an accepting computation.

*Proof sketch (PDA to CFG).* Given $P = (Q, \Sigma, \Gamma, \delta, q_0, F)$, construct $G$ with variables $A_{pq}$ for each pair of states $p, q \in Q$. The variable $A_{pq}$ generates all strings that take $P$ from state $p$ with empty stack to state $q$ with empty stack. Rules: for each transition that pushes a symbol and a later transition that pops it, add a rule $A_{pq} \to a A_{rs} b$ encoding the matching push-pop pair. The start variable is $A_{q_0, q_f}$ for each $q_f \in F$.

**Bridge.** The CFG-PDA equivalence is the foundational reason the context-free languages admit two complementary descriptions: a generative model (grammar rules) and a recognitive model (machine with stack). This builds toward the CYK parsing algorithm of [47.01.06], which uses the CNF structure for dynamic programming, and it appears again in [47.02.01] where CFL membership is shown to be in P. The central insight is that the stack provides exactly the right amount of memory to handle one level of nesting — this is exactly the computational resource that separates context-free from regular languages, and it generalises the finite-state model of [47.01.01] by adding unbounded LIFO storage. Putting these together with the pumping lemma for CFLs yields a complete characterisation of what can and cannot be expressed with nested structure.

Prove that the language $L = \{a^i b^j c^k : i = j \text{ or } j = k\}$ is inherently ambiguous.

**Exercise 8 (hard, symbolic).**

Describe the construction that converts a PDA $P$ to a CFG $G$ with $L(G) = L(P)$. Define the variables and rules of $G$ in terms of $P$.

</details>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 9 (hard, short answer).**

Prove that the class of context-free languages is closed under union but not under intersection. Give the union construction and a counterexample for intersection.

<details class="exercise__answer"><summary>Answer</summary>

Union: given CFGs $G_1 = (V_1, \Sigma, R_1, S_1)$ and $G_2 = (V_2, \Sigma, R_2, S_2)$ with disjoint variable sets, construct $G = (V_1 \cup V_2 \cup \{S\}, \Sigma, R_1 \cup R_2 \cup \{S \to S_1 \mid S_2\}, S)$. Then $L(G) = L(G_1) \cup L(G_2)$.

Intersection counterexample: $L_1 = \{a^n b^n c^m\}$ and $L_2 = \{a^m b^n c^n\}$ are both context-free, but $L_1 \cap L_2 = \{a^n b^n c^n : n \ge 0\}$ is not context-free by the pumping lemma for CFLs. Rubric: full credit for the construction and the counterexample.

</details>

The foundational gap is the absence of a `PDA` structure with stack operations, the proof that `CFG.language G = PDA.language (CFG.toPDA G)`, and the Chomsky normal form normalisation algorithm with a correctness proof showing `CFG.language (CFG.toCNF G) = CFG.language G`. ## Advanced results [Master]

Three results deepen the theory of context-free languages.

**Theorem 1 (CYK algorithm).** *Given a CFG $G$ in Chomsky normal form and a string $w$ of length $n$, the membership question $w \in L(G)$ can be decided in $O(n^3 |G|)$ time.*

The algorithm uses dynamic programming. For each substring $w[i..j]$, compute the set of variables that can derive it. A substring of length 1 can be derived by variables $A$ with rule $A \to w[i]$. For longer substrings, split at every position $k$ and check if there exist $A \to BC$ with $B$ deriving $w[i..k]$ and $C$ deriving $w[k+1..j]$. This places CFL recognition in P.

**Theorem 2 (closure under substitution).** If $L$ is context-free over $\Sigma = \{a_1, \ldots, a_k\}$ and each $L_i$ is context-free, then the substitution $s(L) = \{w_1 \cdots w_m : a_{i_1} \cdots a_{i_m} \in L, w_j \in L_{i_j}\}$ is context-free.

The proof replaces each terminal $a_i$ in the grammar for $L$ with the start variable of the grammar for $L_i$, yielding a CFG for the substituted language. This closure property implies closure under union, concatenation, and star, but not intersection.

**Theorem 3 (Pumping lemma for CFLs).** If $L$ is context-free, then there exists $p$ (the pumping length) such that any $s \in L$ with $|s| \ge p$ can be written $s = uvxyz$ with $|vy| > 0$, $|vxy| \le p$, and $uv^ixy^iz \in L$ for all $i \ge 0$ [ref: TODO_REF Sipser §2.3].

The proof uses the self-embedding property of parse trees: a parse tree for a sufficiently long string must repeat a variable on some root-to-leaf path, and the subtree rooted at the lower occurrence can be pumped.

**Synthesis.** The CFG-PDA equivalence is the foundational reason the context-free languages form a robust class with both generative and recognitive characterisations. The central insight is that one level of unbounded nesting — captured by the stack in a PDA or by recursive rules in a CFG — is exactly the right amount of structure to go beyond regularity while remaining decidable and efficiently parsable. This builds toward the CYK algorithm, which exploits the CNF structure for polynomial-time parsing, and it appears again in the Chomsky hierarchy where context-free languages sit between regular and context-sensitive. Putting these together with the DFA-NFA equivalence of [47.01.01] and the subset construction of [47.01.02] reveals a pattern: each level of the hierarchy adds a specific computational resource (deterministic states, nondeterministic states, a stack, a tape) and this is exactly what defines the boundary between adjacent levels.

## Full proof set [Master]

**Proposition 1 (CFG to PDA construction correctness).** *Let $G = (V, \Sigma, R, S)$ be a CFG. The PDA $P$ constructed with states $\{q_1, q_2, q_3\}$, stack alphabet $V \cup \Sigma \cup \{\$\}$, and transitions that (i) push $\$$ then $S$ onto the stack in $q_1$, (ii) loop in $q_2$ replacing variables by rule right-hand sides and matching terminals, and (iii) accept in $q_3$ by popping $\$$, satisfies $L(P) = L(G)$.*

*Proof.* ($L(G) \subseteq L(P)$.) For any $w \in L(G)$, there is a leftmost derivation $S \Rightarrow \alpha_1 \Rightarrow \alpha_2 \Rightarrow \cdots \Rightarrow w$. The PDA simulates this derivation: when a variable $A$ is on top of the stack, it applies the same rule $A \to \gamma$ used in the derivation. When a terminal $a$ is on top, it matches the next input symbol. Since the derivation produces $w$, the PDA consumes $w$ and empties its stack.

($L(P) \subseteq L(G)$.) Any accepting computation of $P$ starts with $S$ on the stack and ends with an empty stack after reading the entire input. Each time $P$ replaces a variable $A$ by a rule right-hand side, this corresponds to one derivation step. The sequence of replacements yields a derivation $S \Rightarrow^* w$. $\square$

**Proposition 2 (conversion to Chomsky normal form preserves the language).** *Let $G$ be a CFG. The grammar $G'$ obtained by (i) adding a new start variable, (ii) eliminating $\varepsilon$-productions for non-start variables, (iii) eliminating unit productions, and (iv) splitting long right-hand sides into binary rules satisfies $L(G') = L(G)$.*

*Proof.* Each step preserves the language. Step (i) adds $S_0 \to S$ without changing derivable strings. Step (ii) removes $A \to \varepsilon$ for $A \ne S_0$ by adding, for each rule $B \to \alpha A \beta$, the rule $B \to \alpha \beta$ (simulating the nullable $A$). This preserves all non-empty derivations and handles $\varepsilon$ through $S_0$. Step (iii) removes $A \to B$ by transitively replacing with the rules of $B$. Step (iv) replaces $A \to A_1 A_2 \cdots A_k$ with a chain $A \to A_1 B_1, B_1 \to A_2 B_2, \ldots, B_{k-2} \to A_{k-1} A_k$ using fresh variables. Each step preserves the set of derivable terminal strings. $\square$

**Proposition 3 (PDA to CFG construction correctness).** *Let $P$ be a PDA. The grammar $G$ with variables $A_{pq}$ and rules as described above satisfies $L(G) = L(P)$.*

*Proof.* By double inclusion. For $w \in L(P)$, the accepting computation of $P$ decomposes into nested push-pop pairs on the stack. Each pair from state $p$ to state $q$ (pushing symbol $X$ at state $r$, popping $X$ at state $s$) corresponds to a rule $A_{pq} \to a A_{rs} b$ in $G$. The nesting structure of push-pop pairs mirrors the parse tree structure. Conversely, every derivation in $G$ encodes a valid PDA computation because each rule corresponds to a genuine push-pop matching. $\square$

## Connections [Master]

## Historical & philosophical context [Master]

Context-free grammars were introduced by Noam Chomsky in 1956 as part of his hierarchy of formal grammars, originally developed to model the syntax of natural languages [ref: TODO_REF Chomsky 1956]. Chomsky defined four levels of grammars (type 0 through type 3), with context-free grammars at type 2, and proved that each level generates a strictly larger class of languages than the one below.

The equivalence between CFGs and PDAs was established independently by several researchers in the early 1960s. The PDA model was formalised by A. G. Oettinger in 1961 and independently by M. P. Schutzenberger. The CFG-to-PDA construction, where the PDA simulates a leftmost derivation using its stack, is sometimes called the "top-down parser" construction because it mirrors recursive-descent parsing used in compiler design.

Chomsky normal form was introduced by Chomsky as part of his study of the algebraic properties of context-free grammars. The normal form is essential for the CYK algorithm (Cocke, Younger, and Kasami, independently discovered circa 1965-1967) and for proving the pumping lemma for context-free languages. The fact that any CFG can be put into CNF without changing its language is a structural result analogous to the subset construction for NFAs: it shows that the apparently complex generality of CFG rules can be reduced to a simple canonical form [ref: TODO_REF Sipser §2.1].
