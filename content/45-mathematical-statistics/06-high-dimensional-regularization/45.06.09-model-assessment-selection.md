## Intuition [Beginner]

You have built a model and it predicts your training data well. The dangerous question is whether it will predict new data well, and the honest answer is almost always: not as well. A model fitted to a dataset has secretly memorised some of that dataset's random quirks, so it flatters itself when graded on the same data it learned from. Model assessment is the discipline of grading a model fairly, on data it has not seen, so that you can pick the model that will actually serve you in the future rather than the one that merely looks best in the rear-view mirror.

There are two complementary ways to do this. The first way reasons about how much the training score overstates the truth, and adds a penalty to correct for it. The more flexible the model, the bigger the penalty, because more flexibility means more memorising. Mallows' Cp, the Akaike information criterion, and the Bayesian information criterion are three such penalties, differing in how harshly they punish complexity.

The second way is more direct and needs no theory of the penalty: just hold some data back. Fit on part of the data, test on the part you hid, and average the results. This is cross-validation. Repeated holding-out, or resampling the data with replacement as the bootstrap does, gives a usable estimate of future error from the data you already have. Both routes chase the same target: the error you would suffer on tomorrow's data, not yesterday's.

## Visual [Beginner]

Picture two curves drawn against model complexity rising from left to right. The training-error curve slides steadily downward: a more flexible model always fits the data it was trained on better. The test-error curve falls at first, bottoms out, then climbs back up as the model starts chasing noise. The gap between the two curves is the self-flattery, and it widens with complexity.

The takeaway: training error is an over-optimistic ruler. Every method here is a way to recover the honest ruler, the U-shaped test curve, and to stop at its lowest point.

## Worked example [Beginner]

You fit three models to $n = 100$ data points and record the average squared training error for each. You also know, from repeated measurements, that the noise in each response has variance about $\sigma^2 = 1$. Mallows' Cp corrects each training error by adding $2$ times the number of parameters $d$ divided by $n$, multiplied by the noise variance.

Model A uses $d = 2$ parameters and has training error $1.40$. The penalty is $2 \times 2 / 100 \times 1 = 0.04$, so its Cp score is $1.40 + 0.04 = 1.44$.

Model B uses $d = 6$ parameters and has training error $1.05$. The penalty is $2 \times 6 / 100 \times 1 = 0.12$, so its Cp score is $1.05 + 0.12 = 1.17$.

Model C uses $d = 20$ parameters and has training error $0.80$. The penalty is $2 \times 20 / 100 \times 1 = 0.40$, so its Cp score is $0.80 + 0.40 = 1.20$.

Now compare. The raw training errors say Model C is best, because $0.80$ is the smallest. But after the honesty penalty, Model B wins with $1.17$, just edging out Model C's $1.20$, and beating Model A's $1.44$ by a wide margin. The penalty caught Model C in the act of memorising: its training error was lowest only because twenty parameters let it trace the noise.

What this tells us: raw training error rewards complexity for the wrong reason. Adding a penalty that grows with the parameter count restores a fair contest, and here it tells you to prefer the six-parameter model over the twenty-parameter one.

**Exercise (easy, true-false).**

Training error is an unbiased estimate of how well a model will do on new data.

<details class="exercise__hint"><summary>Hint</summary>

The model was fitted to the training data, so it has already adapted to that data's random features.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**False.** Training error is biased downward: it understates the true error because the model has partly memorised the noise in the training set. The amount of understatement is called the optimism, and it grows with model complexity. Feedback-correct: training error is over-optimistic, which is why we correct it or use held-out data. Feedback-wrong: a fair estimate of new-data error must use data the model did not learn from, or a penalty that corrects the training score.

</details>

</aside>

<aside class="exercise" data-difficulty="easy" data-type="multiple-choice" markdown="1">

**Exercise (easy, multiple choice).**

In $5$-fold cross-validation, how is each data point used?

A. It is used only for training, never for testing
B. It is used only for testing, never for training
C. It is tested exactly once and used for training in the other folds
D. It is tested in all five folds

<details class="exercise__hint"><summary>Hint</summary>

The data is split into five blocks; each block takes a turn as the test set while the other four train the model.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**Option C.** Each point sits in exactly one fold. When that fold is the held-out test set, the point is predicted by a model trained on the other four folds; in the four rounds where the point's fold is a training fold, the point helps fit the model. So every point is tested once and trains four times. Feedback-correct: this is what makes cross-validation use all the data for both roles. Feedback-wrong: a point tested in every fold (D) would leak into its own training set, defeating the purpose.

</details>

</aside>

## Formal definition [Intermediate+]

Fix the regression setting of [45.06.02]: data $(X_i, Y_i)$ are i.i.d., $Y = f(X) + \varepsilon$ with $\mathbb{E}[\varepsilon \mid X] = 0$ and $\operatorname{Var}(\varepsilon \mid X = x) = \sigma^2$, and a fitting procedure returns $\hat f$ from a training sample $\mathcal{D}$. Write fitted values $\hat y_i = \hat f(X_i)$ and a loss $L$, taken to be squared error $L(y, \hat y) = (y - \hat y)^2$ unless stated.

**Definition (training, in-sample, and expected error).** The *training error* is $\overline{\operatorname{err}} = \frac{1}{n}\sum_{i=1}^n L(Y_i, \hat y_i)$. The *in-sample error* averages, at the same inputs $X_i$, over fresh responses $Y_i^{\text{new}}$: $\operatorname{Err}_{\text{in}} = \frac{1}{n}\sum_{i=1}^n \mathbb{E}_{Y^{\text{new}}}[L(Y_i^{\text{new}}, \hat y_i) \mid \mathcal{D}]$. The *expected (extra-sample) prediction error* is $\operatorname{Err} = \mathbb{E}[L(Y_0, \hat f(X_0))]$ at an independently drawn $(X_0, Y_0)$. The *optimism* is $\operatorname{op} = \mathbb{E}[\operatorname{Err}_{\text{in}}] - \mathbb{E}[\overline{\operatorname{err}}]$, the systematic gap by which training error understates error [from 45.06.02].

**Definition (effective degrees of freedom).** For fitted values at fixed inputs and homoscedastic noise, the *effective degrees of freedom* is $\mathrm{df} = \frac{1}{\sigma^2}\sum_{i=1}^n \operatorname{Cov}(\hat y_i, Y_i)$. For a linear smoother $\hat{\mathbf{y}} = \mathbf{S}\mathbf{y}$ this equals $\operatorname{tr}(\mathbf{S})$, and for ordinary least squares with $d$ predictors $\mathrm{df} = d$.

**Definition (in-sample criteria).** *Mallows' Cp* estimates in-sample error for squared-error loss with $d$ parameters as
$$
C_p = \overline{\operatorname{err}} + \frac{2 d}{n}\,\hat\sigma^2,
$$
where $\hat\sigma^2$ is a noise estimate from a low-bias model. The *Akaike information criterion* generalises this to log-likelihood loss: with maximized log-likelihood $\hat\ell$ and $d$ parameters,
$$
\mathrm{AIC} = -\frac{2}{n}\hat\ell + \frac{2 d}{n}, \qquad\text{equivalently}\qquad \mathrm{AIC} = -2\hat\ell + 2d
$$
on the deviance scale. The *Bayesian information criterion* (Schwarz) is
$$
\mathrm{BIC} = -2\hat\ell + (\log n)\, d.
$$
The log-likelihood $\hat\ell = \ell(\hat\theta)$ uses the maximum likelihood estimate $\hat\theta$ of [45.01.04].

**Definition (cross-validation).** Partition the index set into $K$ folds $\kappa: \{1, \dots, n\} \to \{1, \dots, K\}$. Let $\hat f^{-k}$ be the procedure fitted with fold $k$ removed. The *K-fold cross-validation estimate* is
$$
\mathrm{CV}(K) = \frac{1}{n}\sum_{i=1}^n L\big(Y_i, \hat f^{-\kappa(i)}(X_i)\big).
$$
$K = n$ is *leave-one-out cross-validation* (LOOCV). For a model-tuning parameter $\alpha$, write $\mathrm{CV}(\alpha)$; the *one-standard-error rule* selects the most parsimonious $\alpha$ whose $\mathrm{CV}(\alpha)$ is within one standard error of the minimum.

**Definition (bootstrap prediction error).** Draw $B$ bootstrap samples $\mathcal{D}^{*b}$ by resampling $\mathcal{D}$ with replacement. The *leave-one-out bootstrap* estimate is
$$
\widehat{\operatorname{Err}}^{(1)} = \frac{1}{n}\sum_{i=1}^n \frac{1}{|C^{-i}|}\sum_{b \in C^{-i}} L\big(Y_i, \hat f^{*b}(X_i)\big),
$$
where $C^{-i}$ indexes bootstrap samples not containing point $i$. The *.632 estimator* is $\widehat{\operatorname{Err}}^{(.632)} = 0.368\,\overline{\operatorname{err}} + 0.632\,\widehat{\operatorname{Err}}^{(1)}$; the *.632+ estimator* replaces the fixed weight by one driven by the relative overfitting rate (defined in Advanced results).

### Counterexamples to common slips [Intermediate+]

- *AIC and Cp are not interchangeable in general.* For Gaussian models with known $\sigma^2$ they agree, since $-2\hat\ell$ reduces to $\overline{\operatorname{err}}/\sigma^2$ up to constants; for non-Gaussian likelihoods AIC is the primary object and Cp is the squared-error special case.
- *Choosing $K = n$ is not always best.* LOOCV is nearly unbiased for $\operatorname{Err}$ but has high variance because its $n$ training sets are almost identical; $K = 5$ or $10$ trades a little bias for a large variance reduction.
- *The model-selection step must live inside the cross-validation loop.* Screening predictors on the full dataset and then cross-validating only the final fit leaks the held-out responses into the selection, producing error estimates biased far below the truth.
- *BIC's penalty is not "AIC with a bigger constant" in spirit.* AIC estimates predictive Kullback-Leibler discrepancy; BIC approximates a posterior model probability. They answer different questions, which is why their large-$n$ behaviour diverges.

## Key theorem with proof [Intermediate+]

The signature result links the three threads: the optimism of the training error is exactly twice the effective degrees of freedom scaled by noise, and this single identity is what Cp and AIC estimate, what BIC modifies, and what cross-validation approximates without ever naming.

**Theorem (optimism, AIC, and the Cp identity).** *For a fitting procedure producing fitted values $\hat y_i$ at fixed inputs $X_i$ with homoscedastic noise of variance $\sigma^2$,*
$$
\operatorname{op} = \mathbb{E}[\operatorname{Err}_{\text{in}}] - \mathbb{E}[\overline{\operatorname{err}}] = \frac{2}{n}\sum_{i=1}^n \operatorname{Cov}(\hat y_i, Y_i) = \frac{2\sigma^2}{n}\,\mathrm{df}.
$$
*Consequently $\mathbb{E}[\operatorname{Err}_{\text{in}}] = \mathbb{E}[\overline{\operatorname{err}}] + \frac{2\sigma^2}{n}\mathrm{df}$, and $C_p = \overline{\operatorname{err}} + \frac{2d}{n}\hat\sigma^2$ is an unbiased estimate of $\mathbb{E}[\operatorname{Err}_{\text{in}}]$ when $\mathrm{df} = d$ and $\hat\sigma^2$ is unbiased. For a likelihood model with $d$ free parameters, the maximized log-likelihood overstates the expected log-likelihood by $d$ asymptotically, so $\mathrm{AIC} = -2\hat\ell + 2d$ is the bias-corrected estimate of the expected Kullback-Leibler discrepancy.*

*Proof.* The optimism identity is established in [45.06.02], Exercise 6: for each $i$, comparing the fresh-response in-sample term with the reused-response training term,
$$
\mathbb{E}[(Y_i^{\text{new}} - \hat y_i)^2] - \mathbb{E}[(Y_i - \hat y_i)^2] = 2\operatorname{Cov}(Y_i, \hat y_i),
$$
because the $\hat y_i^2$ and $\mathbb{E}[Y^2]$ terms cancel (the two responses share mean and variance) and $Y_i^{\text{new}}$ is independent of $\hat y_i$ while $Y_i$ is not. Averaging over $i$ gives $\operatorname{op} = \frac{2}{n}\sum_i \operatorname{Cov}(\hat y_i, Y_i)$, and substituting the definition $\sum_i \operatorname{Cov}(\hat y_i, Y_i) = \sigma^2\,\mathrm{df}$ yields $\operatorname{op} = \frac{2\sigma^2}{n}\mathrm{df}$. For $C_p$, set $\mathrm{df} = d$ and replace $\sigma^2$ by an unbiased $\hat\sigma^2$ independent of the fit; taking expectations recovers $\mathbb{E}[C_p] = \mathbb{E}[\overline{\operatorname{err}}] + \frac{2d}{n}\sigma^2 = \mathbb{E}[\operatorname{Err}_{\text{in}}]$.

For AIC, let the candidate model have density $g(\cdot \mid \theta)$ and let $\theta_0$ minimise the Kullback-Leibler divergence $D(p \,\|\, g_\theta) = \mathbb{E}_p[\log p(Y)] - \mathbb{E}_p[\log g(Y \mid \theta)]$ from the true density $p$ [from 45.01.04]. The quantity worth knowing is the expected log-likelihood at a fresh observation, $\mathbb{E}_{Y_0}[\log g(Y_0 \mid \hat\theta)]$, whose negative is the predictive part of the divergence. A second-order Taylor expansion of $\frac{1}{n}\hat\ell = \frac{1}{n}\sum_i \log g(Y_i \mid \hat\theta)$ around $\theta_0$, together with the same expansion of the population objective around $\theta_0$, gives two correction terms. Writing $J = -\mathbb{E}[\nabla^2 \log g]$ and $K = \operatorname{Var}(\nabla \log g)$ at $\theta_0$, the maximized in-sample log-likelihood exceeds $n$ times the expected log-likelihood by $\operatorname{tr}(J^{-1}K)$ in expectation, and the expected log-likelihood evaluated at $\hat\theta$ falls short of its value at $\theta_0$ by the same $\operatorname{tr}(J^{-1}K)$. The two corrections add, so $\mathbb{E}[\hat\ell] - n\,\mathbb{E}_{Y_0}[\log g(Y_0 \mid \hat\theta)] \approx \operatorname{tr}(J^{-1}K)$. Under correct specification the information matrix identity $J = K$ collapses $\operatorname{tr}(J^{-1}K) = \operatorname{tr}(I_d) = d$, the parameter count. Bias-correcting by subtracting $d$ from $\hat\ell$ and scaling by $-2$ gives $\mathrm{AIC} = -2\hat\ell + 2d$ as an estimate of $-2n\,\mathbb{E}_{Y_0}[\log g(Y_0 \mid \hat\theta)]$, the expected predictive deviance. $\square$

**Bridge.** This identity is the foundational reason that complexity penalties exist at all: the training error is biased low by exactly $2\sigma^2\mathrm{df}/n$, so a fair score must add that amount back, and Cp does this literally while AIC does it on the likelihood scale through the Kullback-Leibler accounting. This is exactly the optimism object of [45.06.02], now read as a model-selection penalty rather than a bias-variance bookkeeping term, and the effective-degrees-of-freedom quantity that governed shrinkage there reappears here as the unit of complexity charged. The same penalty builds toward cross-validation, which estimates $\operatorname{Err}$ directly and, for linear smoothers, reproduces the $\mathrm{df}$-based correction through the leave-one-out shortcut; and it generalises to BIC, where the constant $2$ becomes $\log n$ once the question shifts from prediction to posterior model probability. Putting these together, every method in this unit is one estimator of the same U-shaped curve, differing only in whether it corrects the training score analytically, holds data out, or resamples it.

**Exercise 1 (easy, numeric).**

Two models are fitted to the same data and graded by deviance. Model 1 has $-2\hat\ell = 120$ with $d = 3$ parameters; Model 2 has $-2\hat\ell = 110$ with $d = 8$. Compute the AIC of each and state which is preferred.

$\mathrm{AIC} = -2\hat\ell + 2d$. Smaller is better.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Model 1: $120 + 2\times 3 = 126$. Model 2: $110 + 2\times 8 = 126$. They tie at $126$: Model 2's $10$-point gain in deviance is exactly cancelled by its $5$ extra parameters costing $10$. AIC is indifferent; one would break the tie toward the simpler Model 1.

</details>

</aside>

<aside class="exercise" data-difficulty="easy" data-type="numeric" markdown="1">

**Exercise 2 (easy, numeric).**

For the two models of Exercise 1 with $n = 100$, compute the BIC of each ($\mathrm{BIC} = -2\hat\ell + (\log n) d$, natural log) and state which BIC prefers.

<details class="exercise__hint"><summary>Hint</summary>

$\log 100 \approx 4.605$. The penalty per parameter is now $4.605$, not $2$.

<details class="exercise__answer"><summary>Answer</summary>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 3 (medium, symbolic).**

Show that for a linear smoother $\hat{\mathbf{y}} = \mathbf{S}\mathbf{y}$, the in-sample optimism is $\frac{2\sigma^2}{n}\operatorname{tr}(\mathbf{S})$, so that effective degrees of freedom equals $\operatorname{tr}(\mathbf{S})$.

<details class="exercise__hint"><summary>Hint</summary>

Optimism is $\frac{2}{n}\sum_i \operatorname{Cov}(\hat y_i, Y_i)$; write $\hat y_i = \sum_j S_{ij}Y_j$ and use $\operatorname{Cov}(Y_j, Y_i) = \sigma^2\delta_{ij}$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

By the optimism identity, $\operatorname{op} = \frac{2}{n}\sum_i \operatorname{Cov}(\hat y_i, Y_i)$. With $\hat y_i = \sum_j S_{ij}Y_j$ and independent homoscedastic errors, $\operatorname{Cov}(\hat y_i, Y_i) = \sum_j S_{ij}\operatorname{Cov}(Y_j, Y_i) = S_{ii}\sigma^2$. Summing over $i$, $\operatorname{op} = \frac{2}{n}\sigma^2\sum_i S_{ii} = \frac{2\sigma^2}{n}\operatorname{tr}(\mathbf{S})$. Comparing to $\operatorname{op} = \frac{2\sigma^2}{n}\mathrm{df}$ gives $\mathrm{df} = \operatorname{tr}(\mathbf{S})$. For OLS, $\mathbf{S}$ is the hat matrix with $\operatorname{tr} = d$, recovering the $\frac{2d\sigma^2}{n}$ penalty of Cp.

</details>

</aside>

Derive the leave-one-out cross-validation shortcut for a linear smoother: $\mathrm{CV} = \frac{1}{n}\sum_i \left(\frac{Y_i - \hat y_i}{1 - S_{ii}}\right)^2$, where $\hat y_i = (\mathbf{S}\mathbf{y})_i$ is the full-data fit.

<details class="exercise__hint"><summary>Hint</summary>

For a linear smoother that reproduces constants, the leave-one-out fit $\hat y_i^{-i}$ satisfies $Y_i - \hat y_i^{-i} = (Y_i - \hat y_i)/(1 - S_{ii})$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Let $\hat y_i^{-i}$ be the prediction at $X_i$ from the smoother refitted without point $i$. For a linear smoother the weights renormalise after deletion, and the standard leave-one-out lemma gives the residual update $Y_i - \hat y_i^{-i} = \frac{Y_i - \hat y_i}{1 - S_{ii}}$. The derivation: replacing $Y_i$ by the held-out fit $\hat y_i^{-i}$ leaves that fit unchanged (the point now sits on its own prediction), so $\hat y_i = S_{ii}Y_i + \sum_{j\ne i}S_{ij}Y_j$ and $\hat y_i^{-i} = S_{ii}\hat y_i^{-i} + \sum_{j\ne i}S_{ij}Y_j$; subtracting, $\hat y_i - \hat y_i^{-i} = S_{ii}(Y_i - \hat y_i^{-i})$, and rearranging the implied $Y_i - \hat y_i^{-i}$ relation yields the stated ratio. Squaring and averaging gives $\mathrm{CV} = \frac{1}{n}\sum_i \big(\frac{Y_i - \hat y_i}{1 - S_{ii}}\big)^2$, computable from a single full-data fit. Replacing $S_{ii}$ by the average $\operatorname{tr}(\mathbf{S})/n$ gives generalized cross-validation, $\mathrm{GCV} = \frac{1}{n}\sum_i \big(\frac{Y_i - \hat y_i}{1 - \operatorname{tr}(\mathbf{S})/n}\big)^2$.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="numeric" markdown="1">

**Exercise 5 (medium, numeric).**

A bootstrap sample of size $n$ is drawn with replacement from $n$ points. As $n \to \infty$, what fraction of the original points appears at least once in the bootstrap sample? Give two decimal places.

<details class="exercise__hint"><summary>Hint</summary>

The probability a given point is missed in all $n$ draws is $(1 - 1/n)^n \to e^{-1}$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

**$0.63$ (more precisely $0.632$).** A fixed point is missed on every one of the $n$ independent draws with probability $(1 - 1/n)^n \to e^{-1} \approx 0.368$. So the expected fraction included at least once is $1 - e^{-1} \approx 0.632$. This is the $0.632$ that names the $.632$ bootstrap estimator: each bootstrap training set contains about $63.2\%$ of the distinct original points, leaving about $36.8\%$ genuinely out-of-sample.

</details>

</aside>

<aside class="exercise" data-difficulty="medium" data-type="symbolic" markdown="1">

**Exercise 6 (medium, symbolic).**

For Gaussian regression with known $\sigma^2$, show that AIC reduces to Mallows' Cp up to a constant and a factor, so that minimising one minimises the other.

<details class="exercise__hint"><summary>Hint</summary>

For $Y_i \sim N(\hat y_i, \sigma^2)$, the maximized log-likelihood is $\hat\ell = -\frac{n}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_i (Y_i - \hat y_i)^2$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

With known $\sigma^2$, $-2\hat\ell = n\log(2\pi\sigma^2) + \frac{1}{\sigma^2}\sum_i (Y_i - \hat y_i)^2 = \text{const} + \frac{n}{\sigma^2}\overline{\operatorname{err}}$. Then $\mathrm{AIC} = -2\hat\ell + 2d = \text{const} + \frac{n}{\sigma^2}\overline{\operatorname{err}} + 2d$. Multiplying $C_p = \overline{\operatorname{err}} + \frac{2d}{n}\sigma^2$ by $n/\sigma^2$ gives $\frac{n}{\sigma^2}\overline{\operatorname{err}} + 2d$, which is $\mathrm{AIC}$ minus the constant. The two criteria are affine reparametrisations of each other, so they rank models identically. The equivalence breaks when $\sigma^2$ is unknown and must be estimated, or when the likelihood is non-Gaussian.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="short-answer" markdown="1">

**Exercise 7 (hard, short-answer).**

Explain the selection-bias trap in cross-validation: why does screening predictors on the full dataset before cross-validating the final model produce a badly optimistic error estimate, and what is the correct procedure?

<details class="exercise__hint"><summary>Hint</summary>

If the held-out fold helped choose the predictors, it is no longer genuinely held out.

</details>

<details class="exercise__answer"><summary>Answer</summary>

If predictors are screened using all $n$ points — for instance, keeping the features most correlated with $Y$ across the whole dataset — then every observation, including those later held out, has already influenced which features enter the model. The held-out responses leaked into the selection step, so the cross-validated error of the final model understates the truth, sometimes dramatically: with many noise predictors one can drive cross-validation error near zero for a model with no real signal. The correct procedure wraps the entire model-building pipeline — screening, tuning, and fitting — inside the cross-validation loop, so that for each fold the predictors are re-screened using only that fold's training data, and the held-out fold sees a model built in complete ignorance of it. The rule is that any data-dependent decision that uses the response must be repeated afresh within every fold.

</details>

</aside>

<aside class="exercise" data-difficulty="hard" data-type="symbolic" markdown="1">

**Exercise 8 (hard, symbolic).**

Sketch the Laplace-approximation derivation of BIC: starting from the log marginal likelihood $\log \int L(\theta)\pi(\theta)\,d\theta$ of a $d$-parameter model, obtain $\log L(\hat\theta) - \frac{d}{2}\log n + O(1)$, hence $\mathrm{BIC} = -2\hat\ell + d\log n$.

<details class="exercise__hint"><summary>Hint</summary>

Expand $\log L(\theta)$ to second order around the MLE $\hat\theta$ and integrate the resulting Gaussian; the observed information scales like $n$.

</details>

<details class="exercise__answer"><summary>Answer</summary>

Let $\ell(\theta) = \log L(\theta)$ with maximiser $\hat\theta$ and observed information $\hat J = -\nabla^2\ell(\hat\theta)$, which grows like $n\bar J$ for a per-observation matrix $\bar J$. Taylor-expanding, $\ell(\theta) \approx \ell(\hat\theta) - \frac{1}{2}(\theta - \hat\theta)^\top \hat J (\theta - \hat\theta)$. Then
$$
\int L(\theta)\pi(\theta)\,d\theta \approx L(\hat\theta)\pi(\hat\theta)\int e^{-\frac{1}{2}(\theta-\hat\theta)^\top \hat J(\theta-\hat\theta)}\,d\theta = L(\hat\theta)\pi(\hat\theta)\,(2\pi)^{d/2}|\hat J|^{-1/2}.
$$
Taking logs, $\log\int L\pi\,d\theta \approx \ell(\hat\theta) + \frac{d}{2}\log(2\pi) + \log\pi(\hat\theta) - \frac{1}{2}\log|\hat J|$. Since $|\hat J| \approx n^d |\bar J|$, the dominant term is $\log|\hat J| \approx d\log n + \log|\bar J|$, so $\log\int L\pi\,d\theta = \ell(\hat\theta) - \frac{d}{2}\log n + O(1)$, the $O(1)$ collecting the prior, the $(2\pi)^{d/2}$, and $|\bar J|$ terms that stay bounded as $n\to\infty$. Multiplying by $-2$ and discarding the $O(1)$ remainder gives $\mathrm{BIC} = -2\ell(\hat\theta) + d\log n$, an approximation to $-2$ times the log model evidence; differences of BIC across models approximate $-2$ times log Bayes factors.

</details>

</aside>

## Advanced results [Master]

The criteria, cross-validation, and the bootstrap are three estimators of one quantity, and their differences are best read off from what each is consistent for and what it is efficient for.

The AIC-versus-BIC contrast is the sharpest such difference, and it is not a matter of tuning a constant. BIC is *selection-consistent*: if the data-generating model is among the candidates and the candidate set is fixed, the probability that BIC selects exactly the true model tends to one as $n \to \infty$. The $\log n$ penalty grows fast enough to extinguish the constant probability with which AIC's lighter $2d$ penalty over-selects, retaining superfluous parameters whose deviance gain is order $1$ while their cost is only $2$ per parameter. AIC, by contrast, is *asymptotically efficient* for prediction in the nonparametric or no-true-model regime: when the best finite-dimensional approximation improves steadily with dimension and no fixed model is correct, AIC's risk approaches that of the infeasible oracle that knows the true expected prediction error, a property BIC's over-penalisation forfeits by stopping too early. The tension is genuine — Yang's theorem shows no single criterion can be simultaneously consistent and minimax-rate efficient — so the choice encodes a belief about whether a true finite model exists.

Cross-validation occupies a middle position whose statistical character depends on $K$. Leave-one-out is approximately unbiased for $\operatorname{Err}$ and is asymptotically equivalent to AIC for smooth parametric models, by Stone's theorem; its weakness is variance, since the $n$ deletion fits are nearly identical and their errors are strongly positively correlated. Five- or ten-fold cross-validation introduces a small upward bias — each fold trains on only a fraction of the data, so the procedure is graded as if handicapped — in exchange for a large variance reduction from the more diverse, less correlated fold fits. The bias-variance of $K$ is therefore the practical reason the field standardised on $K \in \{5, 10\}$ rather than $K = n$. The one-standard-error rule formalises a preference for parsimony within this noise: among models whose cross-validation curve is statistically indistinguishable from the minimum, choose the simplest, because the apparent minimiser is itself a noisy selection and the flat region around it is real.

The bootstrap estimators correct a different bias. The naive bootstrap that trains and tests on the same resample reproduces training-error optimism, so the leave-one-out bootstrap $\widehat{\operatorname{Err}}^{(1)}$ tests each point only on resamples excluding it. But those resamples contain only the $\approx 63.2\%$ distinct points of the original, so each is trained on an effectively smaller, less rich dataset, biasing $\widehat{\operatorname{Err}}^{(1)}$ upward. The $.632$ estimator $0.368\,\overline{\operatorname{err}} + 0.632\,\widehat{\operatorname{Err}}^{(1)}$ blends the downward-biased training error with the upward-biased leave-one-out bootstrap in the proportions dictated by the expected inclusion fraction. This fixed blend overcorrects under heavy overfitting, where $\overline{\operatorname{err}}$ can reach zero. The $.632+$ refinement of Efron and Tibshirani makes the weight adaptive: define the *no-information error rate* $\gamma$ as the loss of the rule when inputs and responses are independent (estimated by averaging the loss over all $n^2$ input-response pairings), the *relative overfitting rate* $R = (\widehat{\operatorname{Err}}^{(1)} - \overline{\operatorname{err}})/(\gamma - \overline{\operatorname{err}})$, and the weight $w = 0.632/(1 - 0.368\,R)$; then $\widehat{\operatorname{Err}}^{(.632+)} = (1 - w)\overline{\operatorname{err}} + w\,\widehat{\operatorname{Err}}^{(1)}$. When overfitting is severe ($R \to 1$) the weight rises toward the leave-one-out bootstrap, discounting the now-meaningless training error.

The minimum description length principle gives BIC a coding interpretation. Encoding the data using a model costs the negative log-likelihood $-\hat\ell$ bits (the cost of the residuals given the fitted parameters) plus the cost of transmitting the $d$ parameters, which to the precision worth sending scales as $\frac{d}{2}\log n$ bits. Minimising total description length $-\hat\ell + \frac{d}{2}\log n$ is, up to the factor of $-2$, minimising BIC. The two derivations — Schwarz's Laplace approximation to the posterior model probability and Rissanen's shortest-code accounting — arrive at the same penalty from Bayesian and information-theoretic premises, which is why BIC is read both as an evidence approximation and as a complexity-control device.

**Synthesis.** The foundational reason these methods cohere is the optimism identity $\operatorname{op} = \frac{2\sigma^2}{n}\mathrm{df}$: the in-sample criteria estimate it analytically, cross-validation and the bootstrap estimate the underlying $\operatorname{Err}$ empirically, and effective degrees of freedom is the single complexity unit they all charge. This is exactly the bias-variance trade-off of [45.06.02] viewed as a selection problem, and the central insight is that no estimator of the U-shaped curve is free: Cp and AIC need a noise estimate and assume the optimism formula, cross-validation pays in variance for its assumption-freedom, and the bootstrap pays in the inclusion-fraction bias that the $.632$ family corrects. Putting these together, the AIC-versus-BIC divide is dual to the consistency-versus-efficiency dichotomy — BIC's $\log n$ penalty is the price of selecting the true model when one exists, AIC's $2d$ penalty is the price of predicting well when none does — and Yang's impossibility result shows the duality is structural, not a defect to be engineered away. The MDL reading generalises BIC beyond the Bayesian frame, identifying the $\frac{d}{2}\log n$ penalty as the cost of describing parameters to the precision the data can support, so that model selection and data compression are the same optimisation seen from two disciplines.

## Full proof set [Master]

The optimism identity, the AIC bias correction, the linear-smoother CV shortcut, and the Laplace derivation of BIC are proved in the Key theorem and Exercises 3, 4, and 8. The remaining Master claims are recorded here.

**Proposition (the expected bootstrap inclusion fraction is $1 - e^{-1}$).** *In a size-$n$ resample drawn with replacement from $n$ points, the expected fraction of distinct original points included tends to $1 - e^{-1} \approx 0.632$ as $n \to \infty$.*

*Proof.* For a fixed original point $i$, each of the $n$ independent draws selects $i$ with probability $1/n$, so $i$ is omitted from the entire resample with probability $(1 - 1/n)^n$. The expected number of included distinct points is $\sum_i \mathbb{P}(i \text{ included}) = n[1 - (1 - 1/n)^n]$, and the expected fraction is $1 - (1 - 1/n)^n$. Since $(1 - 1/n)^n \to e^{-1}$, the fraction tends to $1 - e^{-1}$. The convergence is monotone increasing in $n$ from $1 - 1/4 = 0.75$ at $n=2$ down through the limit; for $n \ge 20$ the fraction is within $0.01$ of $0.632$. $\square$

**Proposition (Mallows' Cp is unbiased for in-sample error in the Gaussian linear model).** *For $\mathbf{y} = \mathbf{X}\boldsymbol\beta + \boldsymbol\varepsilon$ with $\boldsymbol\varepsilon \sim (0, \sigma^2 \mathbf{I})$, OLS fit, and an unbiased $\hat\sigma^2$ independent of the fit, $\mathbb{E}[C_p] = \mathbb{E}[\operatorname{Err}_{\text{in}}]$.*

*Proof.* The OLS hat matrix $\mathbf{H} = \mathbf{X}(\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top$ is the linear smoother with $\operatorname{tr}(\mathbf{H}) = d$, so by Exercise 3 the optimism is $\frac{2\sigma^2}{n}d$. The Key theorem gives $\mathbb{E}[\operatorname{Err}_{\text{in}}] = \mathbb{E}[\overline{\operatorname{err}}] + \frac{2\sigma^2}{n}d$. Taking expectations of $C_p = \overline{\operatorname{err}} + \frac{2d}{n}\hat\sigma^2$ and using $\mathbb{E}[\hat\sigma^2] = \sigma^2$ with independence yields $\mathbb{E}[C_p] = \mathbb{E}[\overline{\operatorname{err}}] + \frac{2d}{n}\sigma^2 = \mathbb{E}[\operatorname{Err}_{\text{in}}]$. The estimate is exactly unbiased, not merely asymptotically, in this Gaussian linear case. $\square$

**Proposition (BIC selection consistency for nested Gaussian models).** *Among a fixed finite set of candidate models containing the true model, with i.i.d. data and the regularity conditions under which the MLE is consistent and asymptotically normal, the probability that BIC selects the true model tends to one as $n \to \infty$.*

*Proof.* Compare the true model $M_0$ of dimension $d_0$ with any competitor $M$. Two cases. If $M$ omits a true parameter (under-fitting), the likelihood-ratio statistic $2(\hat\ell_{M_0} - \hat\ell_{M})$ grows linearly in $n$ (the omitted effect contributes a positive Kullback-Leibler gap per observation), so $\mathrm{BIC}_M - \mathrm{BIC}_{M_0} = 2(\hat\ell_{M_0} - \hat\ell_M) + (d_M - d_0)\log n \to +\infty$ regardless of the bounded dimension term; $M$ is rejected with probability tending to one. If $M$ strictly contains $M_0$ (over-fitting, $d_M > d_0$), the likelihood-ratio statistic $2(\hat\ell_M - \hat\ell_{M_0})$ converges in distribution to $\chi^2_{d_M - d_0}$, hence is $O_p(1)$, while the penalty difference $(d_M - d_0)\log n \to +\infty$; thus $\mathrm{BIC}_M - \mathrm{BIC}_{M_0} = -O_p(1) + (d_M - d_0)\log n \to +\infty$ and $M$ is again rejected. Since the candidate set is finite, with probability tending to one $M_0$ has the smallest BIC. The same argument with the fixed penalty $2$ in place of $\log n$ fails in the over-fitting case: there $\mathrm{AIC}_M - \mathrm{AIC}_{M_0} = -\chi^2_{d_M-d_0} + 2(d_M - d_0)$, which is negative with constant positive probability, so AIC over-selects with non-vanishing probability. $\square$

**Proposition (leave-one-out cross-validation is asymptotically AIC for smooth models).** *For a regular parametric family with the MLE plug-in predictor, $n\,\mathrm{CV}_{\text{LOO}}$ and $\mathrm{AIC}$ (on the deviance scale) agree to first order: their difference is $o_p(1)$ per observation.*

*Proof.* Write the leave-one-out log-likelihood loss $-\log g(Y_i \mid \hat\theta^{-i})$ and Taylor-expand $\hat\theta^{-i}$ around the full-data $\hat\theta$. The one-step deletion update is $\hat\theta^{-i} - \hat\theta = -\hat J^{-1}\nabla\log g(Y_i \mid \hat\theta) + o_p(n^{-1})$ with observed information $\hat J = O(n)$, by the same leave-one-out lemma used in Exercise 4 specialised to the score. Substituting into the loss and summing,
$$
\sum_i -\log g(Y_i \mid \hat\theta^{-i}) = \sum_i -\log g(Y_i \mid \hat\theta) + \sum_i \nabla\log g(Y_i \mid \hat\theta)^\top \hat J^{-1}\nabla\log g(Y_i \mid \hat\theta) + o_p(1).
$$
The quadratic correction is $\operatorname{tr}(\hat J^{-1}\sum_i \nabla\log g\,\nabla\log g^\top) = \operatorname{tr}(\hat J^{-1}\hat K)$, the empirical version of the $\operatorname{tr}(J^{-1}K)$ term in the AIC derivation, which converges to $d$ under correct specification. So $-2n\,\mathrm{CV}_{\text{LOO}} = -2\hat\ell + 2d + o_p(1) = \mathrm{AIC} + o_p(1)$, Stone's asymptotic equivalence. $\square$

## Connections [Master]

The bias-variance decomposition and the optimism of training error [45.06.02] is the direct parent of this unit: the identity $\operatorname{op} = \frac{2\sigma^2}{n}\mathrm{df}$ proved there is what Cp and AIC estimate analytically and what cross-validation reproduces empirically, and the effective-degrees-of-freedom quantity that indexed shrinkage there is the complexity unit charged by every criterion here.

Point estimation by maximum likelihood [45.01.04] supplies the log-likelihood, the maximised value $\hat\ell$, and the Fisher-information machinery that the AIC bias correction and the BIC Laplace approximation both run on; the Kullback-Leibler divergence minimised in the AIC derivation is the population objective whose sample analogue the MLE maximises, so AIC is a bias-corrected estimate of the very discrepancy that defines the MLE's target under misspecification.

The lasso and its sparsity oracle [45.06.06] is where these selection criteria meet penalised estimation: the lasso's degrees of freedom equal the expected size of its active set (a Stein identity), so plugging that $\mathrm{df}$ into the Cp/AIC penalty yields a per-$\lambda$ criterion, while cross-validation along the lasso path with the one-standard-error rule is the standard route to choosing $\lambda$ in practice; the selection-bias warning of this unit is exactly why the entire path must be cross-validated rather than tuned on the full data.

Conjugate priors and exponential-family Bayes [45.03.02] underwrites the BIC derivation: the marginal likelihood $\int L(\theta)\pi(\theta)\,d\theta$ that BIC approximates by Laplace is the model evidence whose closed form the conjugate theory computes exactly, and differences of BIC approximate log Bayes factors, the conjugate evidence ratios that compare models. The smooth prior with positive density at the truth required by the Laplace step is the regularity condition that conjugate analysis makes explicit.

Bayesian point estimation [45.03.01] frames the BIC route as posterior model probability: where that unit minimises posterior expected loss to choose an action, BIC selects the model of highest approximate posterior probability, and the MDL coding interpretation reframes the same penalty as shortest description length, tying model selection to data compression.

## Historical & philosophical context [Master]

The complexity-penalty idea entered regression through **Colin Mallows**, who circulated the $C_p$ statistic in the 1960s and published it as a diagnostic for subset selection, plotting $C_p$ against subset size to read off models that neither under- nor over-fit [ref: TODO_REF Mallows 1973]. Independently and more generally, **Hirotugu Akaike** derived his information criterion in 1973-1974 from the principle that a fitted model should minimise the expected Kullback-Leibler divergence from the truth, showing that the maximised log-likelihood is an upward-biased estimate of the expected log-likelihood and that the bias is asymptotically the parameter count, which yields the $2k$ penalty [ref: TODO_REF Akaike 1974]. Akaike's reframing of model selection as the estimation of a predictive information loss was a conceptual break from hypothesis-testing approaches to choosing model order.

**Gideon Schwarz** gave the Bayesian alternative in 1978, deriving BIC as the leading terms of a Laplace approximation to the marginal likelihood of a candidate model and proving that, unlike AIC, it consistently identifies the true model dimension when one exists [ref: TODO_REF Schwarz 1978]. The divergence between the two criteria — Schwarz's consistency against Akaike's predictive efficiency — was clarified over the following decades and shown by **Yuhong Yang** to be a genuine impossibility: no model-selection rule can be both consistent and minimax-rate optimal for prediction [ref: TODO_REF Yang 2005]. **Jorma Rissanen** supplied a third lineage, the minimum description length principle, identifying the BIC penalty as the cost of encoding parameters to the precision the data support and casting model selection as data compression [ref: TODO_REF Rissanen 1978].

The resampling tradition runs parallel. **Mervyn Stone** and, in the same year, **Seymour Geisser** formalised cross-validation as a predictive assessment, with Stone proving the asymptotic equivalence of leave-one-out cross-validation and AIC [ref: TODO_REF Stone 1974]. **Bradley Efron**'s bootstrap, introduced in 1979, was adapted to prediction-error estimation through the leave-one-out bootstrap and the $.632$ and $.632+$ estimators developed by Efron and **Robert Tibshirani**, which correct the inclusion-fraction bias inherent in resampling-based error estimates [ref: TODO_REF Efron 1997].
