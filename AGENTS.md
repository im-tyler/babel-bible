# Babel Bible — Agent Entry Point

Single front door for any agent (or human) working on this project. Read the
docs in the order below; everything else is reference or generated state.

## What this project is

A self-study curriculum taking a learner from high-school algebra to
graduate-level mastery across math, physics, chemistry, biology, philosophy,
language, and the social sciences. Every unit is a markdown file with YAML
frontmatter under `content/` (~2,400 units, 50 sections). Each unit exists at
three depths (Beginner / Intermediate / Master) in one source document. A
companion site (built with Neutron, deployed via Teploy) lives at babelbible.org.

## Read in this order

1. **`OVERVIEW.md`** — the load-bearing vision doc: what Babel Bible is, who it's
   for, the non-negotiable invariants, and production strategy. Read first, read
   when in doubt.
2. **`docs/plans/AGENT_PRODUCTION_PLAYBOOK.md`** — self-contained recipe for
   producing one unit to 27/27 first-pass. Read top-to-bottom before writing any
   content.
3. **`SESSION_NOTES.md`** — current state and next actions. Where things stand
   right now.

## Reference layer (consult as needed, don't read cover-to-cover)

- **`docs/specs/`** — `UNIT_SPEC.md` (canonical content format), `QUALITY_RUBRIC.md`
  (the 27-check validator gate), orchestration/scaffold protocols.
- **`docs/briefs/`** — dispatch briefs handed to producing/auditing agents
  (`PRODUCER_BRIEF.md` is the main one; also audit, deepen-replace, exercise-pack).
- **`docs/catalogs/`** — `CONCEPT_CATALOG.md` (every named concept),
  `DEPENDENCY_MAP.md` (the prereq DAG), `FASTTRACK_BOOKLIST.md`, `MATHLIB_GAPS.md`,
  `NEED_TO_SOURCE.md`.
- **`_meta/NOTATION.md`**, **`style/editorial-voice.md`** — notation and voice.
- **`scripts/validate_unit.py`** — the validator that enforces the 27 checks.
